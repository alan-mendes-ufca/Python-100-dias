## Buscando dados de rede com Python

Coletar dados de rede é uma área na qual Python é muito bom. Na lição anterior, falamos sobre como os programas que coletam dados em rede são geralmente chamados de web crawlers ou programas spider. Mesmo na era do big data, os dados ainda são um ponto fraco e uma deficiência para as pequenas e médias empresas. Alguns dados podem ser obtidos por meio de APIs de dados abertos ou pagos; outros só devem ser coletados quando os termos de uso e a autorização da fonte permitirem. Não importa o caminho usado para obter recursos de dados de rede, Python é uma escolha muito boa, porque a biblioteca padrão do Python e as bibliotecas de terceiros fornecem um bom suporte para coleta de dados da web.

### A Biblioteca `requests`

Para usar Python para obter dados de rede, recomendamos a biblioteca de terceiros chamada `requests`. Na verdade, usamos essa biblioteca nas lições anteriores. De acordo com seu site oficial, `requests` é um wrapper construído na biblioteca padrão do Python e simplifica a operação de acesso a recursos de rede através de HTTP ou HTTPS. Mencionamos em aula que HTTP é um protocolo de solicitação-resposta. Quando inserimos o URL correto em um navegador, geralmente também chamado de endereço de site, e pressionamos Enter, enviamos uma solicitação HTTP a um servidor Web na rede. Depois que o servidor recebe a solicitação, ele nos fornece uma resposta HTTP. No Chrome, abra as Ferramentas do desenvolvedor e mude para a guia Rede, então você poderá ver como realmente são as solicitações e respostas HTTP, conforme mostrado abaixo.

<img src="res/20210822093434.png" style="zoom:40%;">

Por meio da biblioteca `requests`, podemos permitir que um programa Python envie solicitações a um servidor Web, assim como um navegador, e receba a resposta retornada pelo servidor. Da resposta, podemos extrair os dados que desejamos. As páginas da web que o navegador nos mostra são escritas em HTML, e o navegador é como um ambiente interpretador de HTML. O conteúdo que vemos em uma página da web está todo contido em tags HTML. Após obter o código HTML, podemos extrair o conteúdo dos atributos das tags ou do texto dentro das tags. O exemplo abaixo obtém o HTML da página institucional do Python.

```python
import requests

resp = requests.get('https://www.python.org/', timeout=10)
resp.raise_for_status()
print(resp.text)
```

> **Observação**: A variável `resp` no código acima é um objeto `Response`, um tipo encapsulado pela biblioteca `requests`. Através do atributo `status_code` deste objeto, podemos obter o código de status da resposta, e através do atributo `text` deste objeto, podemos obter o código HTML da página.

Como o `text` do objeto `Response` é uma string, podemos processá-lo com expressões regulares, embora essa técnica seja frágil para HTML real. A próxima aula apresenta analisadores próprios para HTML; use expressões regulares apenas quando o formato de entrada for simples e controlado.

```python
import re

html = '<a href="/curso" title="Introdução a Python">Curso</a>'
pattern = re.compile(r'<a.*?href="(.*?)".*?title="(.*?)".*?>')
for href, title in pattern.findall(html):
    print(href, title)
```

Além do conteúdo de texto, também podemos usar a biblioteca `requests` para obter recursos binários através de uma URL. O exemplo abaixo baixa o logotipo publicado pelo próprio projeto Python e o salva localmente.

```python
import requests

resp = requests.get('https://www.python.org/static/community_logos/python-logo.png', timeout=10)
resp.raise_for_status()
with open('python-logo.png', 'wb') as file:
    file.write(resp.content)
```

> **Observação**: O atributo `content` do objeto `Response` pode obter os dados binários retornados pelo servidor.

A biblioteca `requests` é muito fácil de usar e suas funções também são fortes e completas. Analisaremos mais sobre isso aos poucos, durante o uso. Se você deseja saber mais sobre a biblioteca `requests`, pode ler a [documentação oficial](https://docs.python-requests.org/en/latest/).

### Escrevendo código de coleta responsável

Antes de coletar dados, verifique os termos de uso, o arquivo `robots.txt` e a documentação do site. Colete apenas conteúdo autorizado, identifique o seu cliente quando necessário e limite a frequência das solicitações. A função abaixo trabalha sobre HTML já obtido de uma fonte autorizada e ilustra a extração de título e avaliação sem depender da estrutura de um site de terceiros.

```python
import re


def extract_movies(html):
    pattern = re.compile(
        r'<article class="movie">\s*<h2>([^<]+)</h2>\s*<span class="rating">([^<]+)</span>'
    )
    return pattern.findall(html)


html = '''
<article class="movie">
  <h2>Filme de exemplo</h2>
  <span class="rating">9.0</span>
</article>
'''

for title, rating in extract_movies(html):
    print(title, rating)
```

### Usando proxies de forma legítima

Proxies são comuns em redes corporativas, de teste e de privacidade, mas não devem ser usados para contornar limites de acesso ou ocultar uma coleta não autorizada. Configure o proxy fornecido pela sua organização por variáveis de ambiente, sem expor credenciais no código.

```python
import os

import requests

proxies = {
    scheme: address
    for scheme, address in {
        'http': os.environ.get('HTTP_PROXY'),
        'https': os.environ.get('HTTPS_PROXY'),
    }.items()
    if address
}

resp = requests.get(
    'https://www.python.org/',
    proxies=proxies or None,
    timeout=10,
)
resp.raise_for_status()
```

Consulte a [documentação do Requests sobre proxies](https://requests.readthedocs.io/en/latest/user/advanced/#proxies) e o guia do [MDN sobre servidores proxy e tunelamento](https://developer.mozilla.org/en-US/docs/Web/HTTP/Guides/Proxy_servers_and_tunneling) para conhecer os usos e limites dessa configuração.
