## DML em detalhes

Continuando com o banco de dados do sistema de seleção de cursos escolares criado na lição anterior, explicaremos como usar o DML. DML pode nos ajudar a inserir dados em tabelas bidimensionais (operação `insert`), excluir dados de tabelas bidimensionais (operação `delete`) e atualizar dados em tabelas bidimensionais (operação `update`). Antes de executar o DML, vamos primeiro mudar para o banco de dados `school` com o comando `use` abaixo.

```sql
USE `school`;
```

### Operação `insert`

Como o nome sugere, `insert` é usado para inserir linhas em uma tabela bidimensional. As formas de inserção incluem a inserção de uma linha completa, a inserção de parte de uma linha, a inserção de várias linhas e a inserção de resultados de consulta. Usamos o SQL abaixo para adicionar uma faculdade à tabela de faculdades.

```sql
INSERT INTO `tb_college` 
VALUES
    (DEFAULT, 'College of Computer Science', 'A place to study computer science and technology');
```

Como a chave primária da tabela college é um campo de incremento automático, `default` no SQL acima significa que esta coluna usa o valor padrão. Também podemos usar a seguinte maneira para fazer a mesma operação.

```sql
INSERT INTO `tb_college` (`col_name`, `col_intro`) 
VALUES 
    ('College of Computer Science', 'A place to study computer science and technology');
```

Recomendamos o seguinte: especifique quais campos receberão valores. Desta forma, não é necessário atribuir valores de acordo com a ordem dos campos definida quando a tabela foi criada. Você pode atribuir valores de acordo com a ordem dos campos fornecida na tupla antes de `values`. Mas você precisa observar que, exceto os campos que permitem `null` e os campos que possuem valores padrão, todos os outros campos devem ser listados um por um e devem receber valores na tupla após `values`. Se você deseja inserir vários registros de uma vez, podemos colocar várias tuplas após `values` para fazer a inserção em lote, conforme mostrado abaixo.

```sql
INSERT INTO `tb_college` 
    (`col_name`, `col_intro`) 
VALUES 
    ('College of Foreign Languages', 'A college for studying the languages of foreigners'),
    ('College of Economics and Management', 'Manage the world and benefit the people, govern the country; management science, the way to make the country prosperous'),
    ('College of Physical Education', 'Develop sports and improve physical fitness');
```

Ao inserir dados, observe que a chave primária não pode ser repetida. Se os dados inseridos tiverem a mesma chave primária de um registro existente na tabela, a operação `insert` produzirá um erro de entrada duplicada. Deixe-me lembrar a todos novamente: se uma operação `insert` deixar algumas colunas de fora, essas colunas deverão ter valores padrão ou permitir `null`, caso contrário, um erro também ocorrerá. Em sistemas empresariais, para fazer com que a operação `insert` não afete o desempenho de outras operações, principalmente a operação `select` da qual falaremos mais tarde, você pode adicionar `low_priority` entre `insert` e `into` para diminuir a prioridade do Operação `insert`. Esta forma também se aplica às operações `delete` e `update` apresentadas a seguir.

Suponha que haja uma tabela chamada `tb_temp` e tenha duas colunas, `a` e `b`, que salvam o nome da faculdade e a introdução da faculdade. Também podemos obter os dados da tabela `tb_temp` através de uma operação de consulta e inseri-los na tabela college, conforme mostrado abaixo. O `select` aqui é o DQL que mencionamos antes e será explicado em detalhes na próxima lição.

```sql
INSERT INTO `tb_college`
    (`col_name`, `col_intro`)
SELECT `a`, `b` 
  FROM `tb_temp`;
```

### Operação `delete`

Se precisar excluir dados de uma tabela, você pode usar a operação `delete`. Isso pode nos ajudar a excluir linhas especificadas ou todas as linhas. Por exemplo, se quisermos deletar a faculdade com ID `1`, podemos utilizar o SQL mostrado abaixo.

```sql
DELETE
  FROM `tb_college`
 WHERE col_id=1;
```

Observe que a cláusula `where` na operação `delete` acima é usada para especificar a condição. Somente as linhas que satisfaçam a condição serão excluídas. Se escrevermos acidentalmente o SQL abaixo, todos os registros da tabela college serão excluídos. Isto é muito perigoso e, no trabalho real, normalmente também não faríamos isso.

```sql
DELETE
  FROM `tb_college`;
```

Deve-se explicar que mesmo que todos os dados sejam excluídos, a operação `delete` não excluirá a tabela em si e não fará com que o valor do campo `AUTO_INCREMENT` volte ao valor inicial. Se precisar excluir todos os dados e também fazer com que o campo `AUTO_INCREMENT` retorne ao valor inicial, você pode usar `truncate table` para fazer uma operação de truncamento de tabela. A natureza de `truncate` é excluir a tabela original e criar uma tabela novamente. Sua velocidade é realmente mais rápida, pois não precisa excluir dados linha por linha. Mas lembre-se de uma coisa: usar `truncate table` para excluir dados é muito perigoso, porque excluirá todos os dados e, como a tabela original já foi excluída, a recuperação de dados excluídos incorretamente torna-se extremamente difícil.

### Operação `update`

Se quiser modificar os dados de uma tabela, você pode usar a operação `update`. Pode ser usado para modificar linhas especificadas ou todas as linhas. Por exemplo, alteramos "Yang Guo" na tabela do aluno para "Yang Xiao". Aqui assumimos que a carteira de estudante de "Yang Guo" é `1001`. O código é mostrado abaixo.

```sql
UPDATE `tb_student`
   SET `stu_name`='Yang Xiao'
 WHERE `stu_id`=1001;
```

Observe a cláusula `where` no SQL acima. Usamos a carteira de aluno como condição para encontrar o aluno correspondente e, em seguida, usamos a operação de atribuição anterior para alterar o nome para “Yang Xiao”. Por que não usamos diretamente o nome como condição de filtro aqui? Isso ocorre porque pode haver mais de um aluno chamado “Yang Guo” na tabela de alunos. Se `stu_name` for usado como condição de filtro, nossa operação `update` poderá atualizar várias linhas de dados ao mesmo tempo. Obviamente não é isso que queremos ver. Outro ponto a ser observado é a palavra-chave `set` na operação `update`, pois no SQL `=` não significa atribuição, mas sim o operador para testar a igualdade. Somente quando aparece após a palavra-chave `set` é que `=` tem a capacidade de atribuir um valor.

Caso queira modificar o nome e a data de nascimento do aluno ao mesmo tempo, você pode fazer uma pequena alteração no extrato `update` acima, conforme mostrado abaixo.

```sql
UPDATE `tb_student`
   SET `stu_name`='Yang Xiao'
     , `stu_birth`='1975-12-29'
 WHERE `stu_id`=1001;
```

A instrução `update` também pode usar uma consulta para obter dados e, em seguida, usá-la para atualizar os dados da tabela especificada. Os leitores interessados ​​podem estudar isso sozinhos. Ao escrever uma instrução `update`, geralmente há uma cláusula `where`, porque no trabalho real, as operações que atualizam a tabela inteira quase nunca são usadas. Todos devem prestar atenção a este ponto.
