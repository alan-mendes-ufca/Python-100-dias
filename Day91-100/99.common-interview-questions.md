## Common Questions in Interviews

### Computer Fundamentals

1. Questions related to the TCP/IP model.

   Suggested reading: Ruanyifeng's "Introduction to Internet Protocols (1)" and "Introduction to Internet Protocols (2)".

2. Questions related to HTTP and HTTPS.

   Suggested reading: Ruanyifeng's "Introduction to the HTTP Protocol" and "Overview of How the SSL/TLS Protocol Works".

3. Common Linux commands and services.

4. The relationship between processes and threads. When should multi-threading be used? When should multi-processing be used?

5. Relational database questions such as ACID, transaction isolation levels, locks, and SQL optimization.

6. Non-relational database questions such as CAP, BASE, and application scenarios.

### Python Fundamentals

1. Which standard libraries and third-party libraries have been used in development?

   Standard library examples: `sys`, `os`, `re`, `math`, `random`, `logging`, `json`, `pickle`, `shelve`, `socket`, `datetime`, `hashlib`, `configparser`, `urllib`, `itertools`, `collections`, `functools`, `threading`, `multiprocess`, `timeit`, `atexit`, `abc`, `asyncio`, `base64`, `concurrent.futures`, `copy`, `csv`, `operator`, `enum`, `heapq`, `http`, `profile`, `pstats`, `ssl`, `unittest`, `uuid`.

   Third-party library examples: `openpyxl`, `xlrd`, `xlwt`, `PyPDF2`, `ReportLab`, `PyYAML`, `jieba`, `pillow`, `requests`, `urllib3`, `responses`, `aiohttp`, `BeautifulSoup4`, `lxml`, `pyquery`, `PyMySQL`, `psycopg2`, `redis`, `PyMongo`, `Peewee`, `SQLAlchemy`, `alipay`, `PyJWT`, `itsdangerous`, `celery`, `flower`, `elasticsearch-dsl-py`, `PyCrypto`, `Paramiko`, `logbook`, `nose`, `pytest`, `coverage`, `Selenium`, `lineprofiler`, `memoryprofiler`, `matplotlib`, `pygal`, `OpenCV`.

2. The function, principle, and implementation of decorators.

3. Which magic methods have been used?

   Suggested reading: "A Guide to Python's Magic Methods".

4. Writing comprehensions, generators, and iterators.

5. The underlying implementation of lists, sets, and dictionaries.

6. Questions related to garbage collection.

7. Questions related to concurrent programming.

8. Knowledge related to coroutines and asynchronous I/O.

### Django and Flask

1. What problem is solved by the MVC architecture, or MTV?

2. The execution flow of middleware, and how to customize middleware.

3. How should a REST data API be designed, including URL, domain name, version, filtering, status code, and security?

4. Questions related to implementing CRUD operations with an ORM framework.

   - How to implement combined queries with multiple conditions.
   - How to execute raw SQL.
   - How to avoid the N+1 query problem.

5. How to execute asynchronous tasks and scheduled tasks.

6. How to implement page cache and query cache, and how to warm up the cache.

### Crawler-Related Topics

1. Components and data-processing flow of the Scrapy framework.
2. The purpose of crawling. In the project, where is crawler data used?
3. The tools used for packet capture, download, cleaning, storage, analysis, and visualization.
4. The source of the data. Be able to list ten websites easily.
5. The structure of the data. What is a crawled field used for in the project?
6. Anti-anti-crawling measures such as speed limiting, request headers, cookie pools, proxy pools, Selenium WebDriver, RoboBrowser, TOR, and OCR.
7. The amount of data. How much data was finally crawled, for example tens of thousands of records or several gigabytes of data?
8. Later data processing such as persistence, data completion, normalization, formatting, transfer, and classification.

### Data Analysis

1. Scientific computing libraries, such as common operations in SciPy and NumPy.
2. Data analysis libraries, such as common algorithms wrapped in pandas.
3. Common models and the matching scenarios, such as classification, regression, and clustering.
4. Which specific indicators were extracted.
5. How to evaluate whether a model is good or bad.
6. The actual steps for each kind of model, and how to evaluate the result.

### Project-Related Topics

1. The structure of the project team and the role you played in the team.
2. The business architecture of the project, including modules and submodules, and the technical architecture, including mobile side, PC side, and backend stack.
3. Tools related to software management and control, such as version control, issue management, and continuous integration.
4. Core business entities and their attributes, and the relationships between entities.
5. Which dependency libraries were used, and what kind of problems they mainly solved.
6. How the project was deployed and launched, and what the physical architecture of the project looked like, such as Nginx, Gunicorn or uWSGI, Redis, MongoDB, MySQL, and Supervisor.
7. How the project was tested, and whether performance tuning was done.
8. What difficulties appeared in the project, and how they were solved.
