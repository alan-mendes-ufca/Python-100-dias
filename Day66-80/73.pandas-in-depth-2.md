## pandas in Depth - 2

If we use pandas for data analysis, then `DataFrame` is definitely the type used most widely. It can be used to store and process heterogeneous two-dimensional data. The so-called "heterogeneous" here means that the data types of each column in a `DataFrame` do not need to be the same. This is also what makes it different from a two-dimensional NumPy array. `DataFrame` provides very rich properties and methods to help us reshape, clean, preprocess, pivot, and present data.

### Creating `DataFrame` Objects

#### Create a `DataFrame` from a two-dimensional array

```python
scores = np.random.randint(60, 101, (5, 3))
courses = ['Chinese', 'Math', 'English']
stu_ids = np.arange(1001, 1006)
df1 = pd.DataFrame(data=scores, columns=courses, index=stu_ids)
df1
```

```

1002       71    60      100
1003       94    81       93
1004       88    88       67
```


#### Read a CSV file and create a `DataFrame`

We can use the `read_csv` function in the `pandas` module to read a CSV file. The `read_csv` function has many parameters. Below are several important ones.

- `sep` / `delimiter`: separator, default is `,`
- `header`: position of the header row, default is `infer`
- `index_col`: which column is used as the row index
- `usecols`: columns that need to be loaded, which can be specified by position or by name
- `true_values` / `false_values`: which values are treated as boolean `True` / `False`
- `skiprows`: rows to skip, specified by row number, index, or function
- `skipfooter`: number of rows to skip at the bottom
- `nrows`: number of rows to read
- `na_values`: which values are treated as null values
- `iterator`: if set to `True`, the function returns an iterator object
- `chunksize`: used together with the previous parameter to set the amount of data fetched each time

```python
df3 = pd.read_csv('data/2018-beijing-residence-points-data.csv', index_col='id')
df3
```

> **Tip**: The CSV file in the code above is loaded through a relative path. In other words, there is a folder named `data` under the current working path, and `2018-beijing-residence-points-data.csv` is inside that folder. If you use Windows, it is also recommended to use `/` instead of `\` for path separators. If you want to use `\`, it is recommended to add `r` before the string and use a raw string to avoid escape characters.

#### Read a relational database table and create a `DataFrame`

The `read_sql` function in the `pandas` module can read data from a database through SQL and create a `DataFrame`. The second parameter of this function represents the database to connect to. For MySQL, we can create a database connection through `pymysql` or `mysqlclient` and then use that connection as the second parameter of `read_sql`.

```python

df5 = pd.read_sql('select * from tb_emp', conn, index_col='eno')
```

Important properties of a `DataFrame` are shown below.

| Property       | Description |
| -------------- | ----------- |
| `at` / `iat`   | Get a single value in a `DataFrame` through labels |
| `columns`      | The column index of a `DataFrame` |
| `dtypes`       | The data type of each column in a `DataFrame` |
| `empty`        | Whether the `DataFrame` is empty |
| `loc` / `iloc` | Get a group of values in a `DataFrame` through labels |
| `ndim`         | The number of dimensions of the `DataFrame` |
| `shape`        | The shape of the `DataFrame` |
| `size`         | The number of elements in the `DataFrame` |
| `values`       | The two-dimensional array corresponding to the data in the `DataFrame` |

### Operating on Data

#### Indexing and slicing

If we want to get one column of a `DataFrame`, for example the `ename` column from `emp_df`, we can use either of the following ways:

```python
emp_df.ename
```

or

```python
emp_df['ename']
```

By running the code above, we can see that what we get is a `Series` object. In fact, a `DataFrame` is the result of putting multiple `Series` objects together.

If we want to get one row of a `DataFrame`, we can use integer indexing or the index we set ourselves. For example, to take out the employee whose number is `2056`, the code is as follows:

```python
emp_df.iloc[1]
```

or

```python
emp_df.loc[2056]
```

If we take a single row or a single column from a `DataFrame`, what we get is a `Series` object. Of course, we can also use fancy indexing to get multiple rows or multiple columns, and the result will still be a `DataFrame`.

Get multiple columns:

```python
emp_df[['ename', 'job']]
```

Get multiple rows:

```python
emp_df.loc[[2056, 7800, 3344]]
```

If we want to get or modify the data in one cell of a `DataFrame`, we need to specify both the row index and the column index. For example, if we want to get the job information of employee `2056`, we can write:

```python
emp_df['job'][2056]
```

or

```python
emp_df.loc[2056]['job']
```

or

```python
emp_df.loc[2056, 'job']
```

We recommend the third approach because it performs indexing only once. If we want to modify this employee's job to `Architect`, we can write:

```python
emp_df.loc[2056, 'job'] = 'Architect'
```

Of course, we can also get multiple rows and columns by slicing.

```python
emp_df.loc[2056:3344]
```

#### Data filtering

As with `ndarray` and `Series`, we can use boolean indexing to filter data in a `DataFrame`. For example, if we want to filter employees whose monthly salary is greater than `3500`, we can do the following:

```python
emp_df[emp_df.sal > 3500]
```

Output:

```
          ename         job     mgr   sal    comm  dno
eno                                                   
2056   Qiao Feng     Analyst  7800.0  5000  1500.0   20
3251   Zhang Cuishan Programmer 2056.0 4000    NaN   20
5566   Song Yuanqiao Accountant 7800.0 4000 1000.0 10
7800   Zhang Sanfeng President    NaN  9000 1200.0 20
```

Of course, we can also combine multiple conditions. For example, to filter employees whose monthly salary is greater than `3500` and whose department number is `20`, we can write:

```python
emp_df[(emp_df.sal > 3500) & (emp_df.dno == 20)]
```

Output:

```
          ename         job     mgr   sal    comm  dno
eno                                                   
2056   Qiao Feng     Analyst  7800.0  5000  1500.0   20
3251   Zhang Cuishan Programmer 2056.0 4000    NaN   20
7800   Zhang Sanfeng President    NaN  9000 1200.0 20
```

Besides boolean indexing, the `query` method of a `DataFrame` can also do data filtering. The argument of `query` is a string, representing the expression used to filter the data, and this is often more in line with what Python programmers like.

```python
emp_df.query('sal > 3500 and dno == 20')
```
