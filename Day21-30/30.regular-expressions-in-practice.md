## Aplicações de Expressões Regulares

### Noções básicas de expressões regulares

Ao escrever programas para processamento de strings, muitas vezes nos deparamos com a necessidade de encontrar strings que correspondam a algumas regras em um trecho de texto. A expressão regular é a ferramenta usada para descrever essas regras. Em outras palavras, podemos usar expressões regulares para definir o padrão de correspondência de strings, ou seja, como verificar se uma string tem uma parte que corresponde a algum padrão, ou como extrair a parte que corresponde ao padrão de uma string, ou substituí-la.

Para dar um exemplo simples, se você já usou curingas como `*` e `?` ao pesquisar arquivos no Windows, as expressões regulares são um tipo semelhante de ferramenta de correspondência de texto. A diferença é que as expressões regulares são muito mais poderosas e podem descrever suas necessidades com muito mais precisão. Claro, a desvantagem é que escrever uma expressão regular é mais complicado do que usar curingas.

Como outro exemplo, suponha que obtemos uma string de algum lugar, talvez um arquivo de texto ou um artigo de notícias na web, e queremos encontrar nela números de telefone celular e fixo. É claro que poderíamos decidir que os números de telefone celular têm 11 dígitos, tendo em mente que nem todo número aleatório de 11 dígitos é um número de celular válido e que os telefones fixos seguem um padrão como `area-code-number`. Mas completar tal tarefa sem expressões regulares seria bastante problemático. Os computadores foram originalmente criados para realizar cálculos matemáticos, portanto as informações que processavam eram principalmente numéricas. Hoje, porém, grande parte da informação com a qual lidamos no trabalho diário são dados de texto, e queremos que os computadores reconheçam e processem textos que sigam determinados padrões. É exatamente por isso que as expressões regulares são tão importantes. Quase todas as linguagens de programação hoje os suportam, e Python fornece suporte por meio do módulo `re` da biblioteca padrão.

Para obter conhecimentos relacionados sobre expressões regulares, todos podem ler uma postagem de blog muito famosa chamada [Tutorial de 30 minutos sobre expressões regulares](https://deerchao.net/tutorials/regex/regex.htm). Depois de ler este artigo, você poderá entender a tabela abaixo. É o nosso breve resumo de alguns símbolos básicos em expressões regulares.

| Símbolo | Significado | Exemplo | Observação |
| -------------- | -------------------------------- | ------------------ | ------------------------------------------------------------ |
| `.` | Combine qualquer personagem | `b.t` | Corresponde a `bat`, `but`, `b#t`, `b1t` e assim por diante |
| `\w` | Combine letras, dígitos ou sublinhado | `b\wt` | Corresponde a `bat`, `b1t`, `b_t`, mas não a `b#t` |
| `\s` | Combine espaços em branco, incluindo `\r`, `\n`, `\t` e assim por diante | `love\syou` | Pode corresponder a `love you` |
| `\d` | Corresponder dígitos | `\d\d` | Pode corresponder a `01`, `23`, `99` e assim por diante |
| `\b` | Corresponder a um limite de palavra | `\bThe\b` |   |
| `^` | Corresponder ao início de uma string | `^The` | Corresponde a strings que começam com `The` |
| `$` | Combine o final de uma string | `.exe$` | Corresponde a strings que terminam com `.exe` |
| `\W` | Corresponder a um caractere que não seja uma palavra | `b\Wt` | Corresponde a `b#t`, `b@t`, mas não a `but`, `b1t`, `b_t` |
| `\S` | Corresponder a um caractere que não seja espaço em branco | `love\Syou` | Corresponde a `love#you`, mas não a `love you` |
| `\D` | Corresponder a um não dígito | `\d\D` | Corresponde a `9a`, `3#`, `0F` e assim por diante |
| `\B` | Corresponder a um limite que não seja de palavra | `\Bio\B` |   |
| `[]` | Combine qualquer caractere de um conjunto de caracteres | `[aeiou]` | Corresponde a qualquer vogal |
| `[^]` | Combine qualquer caractere que não esteja no conjunto de caracteres | `[^aeiou]` | Corresponde a qualquer não vogal |
| `*` | Combine zero ou mais vezes | `\w*` |   |
| `+` | Combine uma ou mais vezes | `\w+` |   |
| `?` | Combine zero ou uma vez | `\w?` |   |
| `{N}` | Corresponder exatamente aos tempos `N` | `\w{3}` |   |
| `{M,}` | Combine pelo menos `M` vezes | `\w{3,}` |   |
| `{M,N}` | Combine pelo menos `M` e no máximo `N` vezes | `\w{3,6}` |   |
| `\ | ` | Alternância | `foo\ | barra` | Pode corresponder a `foo` ou `bar` |
| `(?#)` | Comentário |   |   |
| `(exp)` | Combine `exp` e capture-o em um grupo nomeado automaticamente |   |   |
| `(?<name>exp)` | Combine `exp` e capture-o em um grupo chamado `name` |   |   |
| `(?:exp)` | Combine `exp` sem capturá-lo |   |   |
| `(?=exp)` | Combine a posição antes de `exp` | `\b\w+(?=ing)` | Corresponde `danc` em `I'm dancing` |
| `(?<=exp)` | Combine a posição após `exp` | `(?<=\bdanc)\w+\b` | Corresponde ao primeiro `ing` em `I love dancing and reading` |
| `(?!exp)` | Corresponder a uma posição não seguida por `exp` |   |   |
| `(?<!exp)` | Corresponder a uma posição não precedida por `exp` |   |   |
| `*?` | Repita quantas vezes quiser, mas o mínimo possível | `a.*b` / `a.*?b` | Em `aabab`, o primeiro corresponde à string inteira, o último corresponde a `aab` e `ab` |
| `+?` | Repita uma ou mais vezes, mas o mínimo possível |   |   |
| `??` | Repita zero ou uma vez, mas o mínimo possível |   |   |
| `{M,N}?` | Repita `M` a `N` vezes, mas o menor número possível |   |   |
| `{M,}?` | Repita pelo menos `M` vezes, mas o menor número possível |   |   |

> **Observação**: Se o caractere que você deseja corresponder for um caractere especial em expressões regulares, você poderá escapar dele com `\`. Por exemplo, se quiser corresponder um período literal, escreva `\.`. Escrever `.` diretamente corresponderia a qualquer caractere. Da mesma forma, para combinar parênteses literalmente, você deve escrever `\(` e `\)`, caso contrário eles serão tratados como sintaxe de agrupamento.

### Suporte Python para expressões regulares

Python fornece o módulo `re` para suportar operações relacionadas a expressões regulares. As funções principais em `re` estão listadas abaixo.

| Função | Descrição |
| ---------------------------------------------- | ------------------------------------------------------------ |
| `compile(pattern, flags=0)` | Compilar uma expressão regular e retornar um objeto padrão |
| `match(pattern, string, flags=0)` | Combine uma string com uma expressão regular; retorne um objeto de correspondência em caso de sucesso, caso contrário `None` |
| `search(pattern, string, flags=0)` | Procure a primeira ocorrência que corresponda à expressão regular; retorne um objeto de correspondência em caso de sucesso, caso contrário `None` |
| `split(pattern, string, maxsplit=0, flags=0)` | Divida uma string usando o regex como separador e retorne uma lista |
| `sub(pattern, repl, string, count=0, flags=0)` | Substitua o conteúdo correspondente ao regex pela string especificada; `count` controla o número de substituições |
| `fullmatch(pattern, string, flags=0)` | Uma versão de string completa de `match`, exigindo que toda a string corresponda |
| `findall(pattern, string, flags=0)` | Encontre todas as correspondências e retorne-as como uma lista de strings |
| `finditer(pattern, string, flags=0)` | Encontre todas as correspondências e retorne um iterador |
| `purge()` | Limpe o cache de expressões regulares compiladas implicitamente |
| `re.I` / `re.IGNORECASE` | Ignorar sinalizador de correspondência de maiúsculas e minúsculas |
| `re.M` / `re.MULTILINE` | Sinalizador de correspondência multilinha |

> **Observação**: As funções do módulo `re` listadas acima também podem ser substituídas no desenvolvimento real por métodos em um objeto de expressão regular compilado, um objeto `Pattern`. Se uma expressão regular precisar ser reutilizada, compilá-la primeiro com `compile` e depois usar o objeto padrão resultante é claramente a escolha mais sábia.

Os exemplos abaixo mostram como usar expressões regulares em Python.

#### Exemplo 1: valide um nome de usuário e um número QQ e forneça os prompts correspondentes

```python
"""
Requirement: the username must consist of letters, digits, or underscores and be 6 to 20 characters long.
The QQ number must be 5 to 12 digits long and cannot start with 0.
"""
import re

username = input('Please enter a username: ')
qq = input('Please enter the QQ number: ')
# The first argument to match is the regex string or regex object.
# The second argument is the string to be matched.
m1 = re.match(r'^[0-9a-zA-Z_]{6,20}$', username)
if not m1:
    print('Please enter a valid username.')
# fullmatch requires the entire string to match the regex,
# so start and end anchors are unnecessary here.
m2 = re.fullmatch(r'[1-9]\d{4,11}', qq)
if not m2:
    print('Please enter a valid QQ number.')
if m1 and m2:
    print('The information you entered is valid!')
```

> **Dica**: as expressões regulares acima são escritas como strings brutas, ou seja, uma string literal prefixada com `r`. Uma string bruta significa que cada caractere da string mantém seu significado original, sem processamento de escape. Como as expressões regulares contêm muitos metacaracteres e muitas barras invertidas, o uso de strings brutas evita a necessidade de escrever `\\` em todos os lugares. Por exemplo, `\d` teria que ser escrito como `\\d`, o que é inconveniente de escrever e difícil de ler.

#### Exemplo 2: Extraia números de telefones celulares domésticos de um trecho de texto

A imagem abaixo mostra os segmentos de números de telefonia móvel lançados pelas três operadoras nacionais até o final de 2017.

<img src="res/20210803203134.png" style="zoom:100%;">

```python
import re

# Create a regex object. Lookbehind and lookahead are used to make sure digits do not continue on either side.
pattern = re.compile(r'(?<=\D)1[34578]\d{9}(?=\D)')
sentence = '''The important thing is to say 8130123456789 times, my phone number is 13512346789,
Not 15600998765, not 110 or 119, Wang Dahuang's mobile number is 15600998765.'''
# Method 1: find all matches and store them in a list
tels_list = re.findall(pattern, sentence)
for tel in tels_list:
    print(tel)
print('--------Gorgeous dividers --------')

# Method 2: get match objects through an iterator
for temp in pattern.finditer(sentence):
    print(temp.group())
print('--------Gorgeous dividers --------')

# Method 3: repeatedly call search with a starting position
m = pattern.search(sentence)
while m:
    print(m.group())
    m = pattern.search(sentence, m.end())
```

> **Observação**: a expressão regular acima para números de celular chineses domésticos não é ideal, porque os números que começam com `14` são válidos apenas para determinados prefixos, como `145` ou `147`, e isso não é considerado acima. Um regex melhor para números móveis chineses domésticos seria `(?<=\D)(1[38]\d{9}|14[57]\d{8}|15[0-35-9]\d{8}|17[678]\d{8})(?=\D)`. A China também parece ter números começando com `19` e `16` agora, mas não os consideraremos aqui no momento.

#### Exemplo 3: Substitua conteúdo indesejável em uma string

```python
import re

sentence = 'Oh, shit! Are you an idiot?? Fuck you.'
purified = re.sub('fuck|shit|[Silly Shasha][Compared to the pen fork, the lack of a suspended carving]',
                  '*', sentence, flags=re.IGNORECASE)
print(purified)  # Oh, *! you are*?? * you.
```

> **Observação**: Todas as funções relacionadas a regex no módulo `re` têm um parâmetro `flags`. Ele representa sinalizadores de correspondência de regex, que podem ser usados para controlar se a correspondência ignora maiúsculas e minúsculas, se a correspondência multilinha está habilitada, se as informações de depuração são mostradas e assim por diante. Se precisar especificar vários valores para `flags`, você poderá combiná-los com o operador OR bit a bit, como `flags=re.I | re.M`.

#### Exemplo 4: Dividir uma string longa

```python
import re

poem = 'The moonlight in front of the window was suspected to be frost on the ground. Looking up at the bright moon, he bowed his head and thought about his hometown.'
sentences_list = re.split(r'[，。]', poem)
sentences_list = [sentence for sentence in sentences_list if sentence]
for sentence in sentences_list:
    print(sentence)
```
