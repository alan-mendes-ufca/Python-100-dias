## Uso de função avançada

Vamos continuar explorando o conhecimento relacionado à definição e uso de funções. Na lição anterior, já sabemos que as funções possuem parâmetros e valores de retorno, e os parâmetros podem ser de qualquer tipo de dados e os valores de retorno também podem ser de qualquer tipo de dados. Então há uma pequena questão aqui: podemos usar uma função como parâmetro de outra função e podemos usar uma função como valor de retorno de outra função? Vamos primeiro concluir: **funções em Python são funções de primeira classe**. As chamadas funções de primeira classe significam que as funções podem ser atribuídas a variáveis, as funções podem ser usadas como parâmetros de outras funções e as funções também podem ser usadas como valores de retorno de outras funções. O uso de tomar uma função como parâmetro ou valor de retorno de outra função é geralmente chamado de **função de ordem superior**.

### Funções de ordem superior

Voltemos a um exemplo que falamos antes: projete uma função que receba qualquer número de argumentos e some os elementos cujo tipo é `int` ou `float`. Fazemos uma pequena alteração no código anterior para torná-lo mais compacto, conforme mostrado abaixo.

```python
def calc(*args, **kwargs):
    items = list(args) + list(kwargs.values())
    result = 0
    for item in items:
        if type(item) in (int, float):
            result += item
    return result
```

Se quisermos que a função `calc` acima não apenas faça adição para muitos argumentos, mas também faça mais operações binárias ou mesmo operações binárias personalizadas, o que devemos fazer? O código acima só pode fazer adição porque a função utiliza o operador `+=`, e isso torna a função acoplada à adição. Se pudermos remover esse acoplamento, a universalidade e a flexibilidade da função serão melhores. A maneira de remover o acoplamento é transformar o operador `+` em uma chamada de função e, em seguida, projetá-lo como um parâmetro da função.

```python
def calc(init_value, op_func, *args, **kwargs):
    items = list(args) + list(kwargs.values())
    result = init_value
    for item in items:
        if type(item) in (int, float):
            result = op_func(result, item)
    return result
```

Observe que a função acima adiciona dois parâmetros. Aqui, `init_value` significa o valor inicial da operação e `op_func` significa uma função de operação binária. Para chamar a função alterada, vamos primeiro definir funções que fazem adição e multiplicação, conforme mostrado abaixo.

```python
def add(x, y):
    return x + y


def mul(x, y):
    return x * y
```

Se quisermos fazer uma adição, podemos chamar a função `calc` da seguinte maneira.

```python
print(calc(0, add, 1, 2, 3, 4, 5))  # 15
```

Se quisermos fazer uma multiplicação, podemos chamar a função `calc` da seguinte maneira.

```python
print(calc(1, mul, 1, 2, 3, 4, 5))  # 120
```

A função `calc` acima transforma o operador em um parâmetro de função, e desta forma remove o acoplamento com adição. Esta é uma habilidade de programação muito inteligente e prática, mas os iniciantes podem achar que é difícil de entender, por isso vale a pena ler com atenção. Uma coisa a notar é que há uma diferença clara entre passar uma função como parâmetro para outra função e chamar uma função diretamente. Para chamar uma função, precisamos de parênteses após o nome da função. Mas ao passar uma função como parâmetro, precisamos apenas do próprio nome da função.

Se não definirmos `add` e `mul` antecipadamente, também podemos usar as funções `add` e `mul` fornecidas pelo módulo `operator` na biblioteca padrão do Python. Eles representam operações binárias de adição e multiplicação, portanto podemos usá-los diretamente, conforme mostrado abaixo.

```python
import operator

print(calc(0, operator.add, 1, 2, 3, 4, 5))  # 15
print(calc(1, operator.mul, 1, 2, 3, 4, 5))  # 120
```

Python também inclui uma série de funções integradas de ordem superior. As funções `filter` e `map` que mencionamos anteriormente são funções de ordem superior. O primeiro pode filtrar elementos em uma sequência e o último pode mapear elementos em uma sequência. Por exemplo, se quisermos remover os números ímpares de uma lista de inteiros e elevar ao quadrado todos os números pares para obter uma nova lista, podemos usar diretamente essas duas funções para fazer isso.

```python
def is_even(num):
    """Check whether num is even"""
    return num % 2 == 0


def square(num):
    """Calculate the square"""
    return num ** 2


old_nums = [35, 12, 8, 99, 60, 52]
new_nums = list(map(square, filter(is_even, old_nums)))
print(new_nums)  # [144, 64, 3600, 2704]
```

Claro, também podemos fazer a mesma coisa com a compreensão de lista, e a forma de compreensão de lista é mais simples e elegante.

```python
old_nums = [35, 12, 8, 99, 60, 52]
new_nums = [num ** 2 for num in old_nums if num % 2 == 0]
print(new_nums)  # [144, 64, 3600, 2704]
```

Vamos discutir também uma função integrada, `sorted`. Ele pode classificar os elementos dos tipos de dados do contêiner, como listas e dicionários. Já falamos sobre o método `sort` do tipo `list`. Do lado da função, `sorted` não é diferente do método `sort` de uma lista, mas retorna um objeto de lista classificado em vez de modificar diretamente a lista original. Chamamos isso de **design de função sem efeitos colaterais**, o que significa que chamar a função não afetará o estado do programa ou do ambiente externo, exceto a produção de um valor de retorno. Ao usar a função `sorted` para classificar, podemos personalizar a regra de classificação na forma de uma função de ordem superior.

```python
old_strings = ['in', 'apple', 'zoo', 'waxberry', 'pear']
new_strings = sorted(old_strings)
print(new_strings)
```

Mas se quisermos classificar os elementos da lista pelo comprimento das strings em vez da ordem do dicionário, podemos passar um parâmetro chamado `key` para a função `sorted` e definir o parâmetro `key` para a função `len`, que obtém o comprimento de uma string. Já falamos sobre essa função antes. O código é mostrado abaixo.

```python
old_strings = ['in', 'apple', 'zoo', 'waxberry', 'pear']
new_strings = sorted(old_strings, key=len)
print(new_strings)  # ['in', 'zoo', 'pear', 'apple', 'waxberry']
```

> **Nota**: O método `sort` do tipo lista possui o mesmo parâmetro `key`. Os leitores interessados podem experimentar sozinhos.

### Funções Lambda

Ao usar funções de ordem superior, se a função usada como parâmetro ou valor de retorno for muito simples, uma linha de código é suficiente para finalizá-la e não há necessidade de pensar em reutilizar a função, então podemos usar uma função lambda. Funções lambda em Python são funções sem nomes, por isso muitas pessoas também as chamam de **funções anônimas**. Uma função lambda pode ter apenas uma linha de código e o resultado da expressão nessa linha é o valor de retorno da função anônima.

No código anterior, as funções `is_even` e `square` que escrevemos têm apenas uma linha de código, portanto, podemos considerar substituí-las por funções lambda, conforme mostrado abaixo.

```python
old_nums = [35, 12, 8, 99, 60, 52]
new_nums = list(map(lambda x: x ** 2, filter(lambda x: x % 2 == 0, old_nums)))
print(new_nums)  # [144, 64, 3600, 2704]
```

No código acima, podemos ver que a palavra-chave para definir uma função lambda é `lambda`. Depois vêm os parâmetros da função. Se houver muitos parâmetros, eles serão separados por vírgulas. A parte após os dois pontos é o corpo da função e geralmente é uma expressão. O resultado da expressão é o valor de retorno da função lambda, portanto não há necessidade de escrever a palavra-chave `return`.

Como dissemos antes, as funções em Python são funções de primeira classe, portanto, uma função pode ser atribuída diretamente a uma variável. Depois de aprender as funções lambda, algumas funções que escrevemos antes agora podem ser escritas em uma linha. Observe a função fatorial de uma linha e a função de verificação primária abaixo e veja se você consegue entendê-las.

```python
import functools
import operator

fac = lambda n: functools.reduce(operator.mul, range(2, n + 1), 1)
is_prime = lambda x: all(map(lambda f: x % f, range(2, int(x ** 0.5) + 1)))

print(fac(6))        # 720
print(is_prime(37))  # True
```

> **Dica 1**: `functools.reduce` realiza uma redução em uma sequência. Junto com `filter` e `map`, ele forma um trio poderoso para processamento de dados: **filtragem**, **mapeamento** e **redução**.
>
> **Dica 2**: a função lambda acima que verifica se um número é primo usa a função `range` para construir o intervalo de 2 a $\small{\sqrt{x}}$ e verifica se esse intervalo tem um fator de `x`. A função `all` também é uma função Python integrada. Se todos os valores booleanos na sequência passada para ele forem `True`, `all` retornará `True`; caso contrário, retornará `False`.

### Funções Parciais

Uma função parcial significa fixar alguns parâmetros de uma função e gerar uma nova função, para que não precisemos passar os mesmos parâmetros toda vez que chamarmos a função. Em Python, podemos usar a função `partial` no módulo `functools` para criar funções parciais.

Por exemplo, por padrão, a função `int` pode converter uma string como um número inteiro decimal. Se alterarmos seu parâmetro `base`, podemos definir três novas funções para converter strings binárias, octais e hexadecimais em inteiros, conforme mostrado abaixo.

```python
import functools

int2 = functools.partial(int, base=2)
int8 = functools.partial(int, base=8)
int16 = functools.partial(int, base=16)

print(int('1001'))    # 1001

print(int2('1001'))   # 9
print(int8('1001'))   # 513
print(int16('1001'))  # 4097
```

Você percebeu que tanto o primeiro parâmetro quanto o valor de retorno da função `partial` são funções? Ele processa a função passada e retorna uma nova função. Ao construir funções parciais, podemos combinar cenas reais de uso e transformar a função original em uma nova função mais fácil de usar. Isto é bastante interessante.

### Resumo

Funções em Python são funções de primeira classe. Eles podem ser atribuídos a variáveis e também podem ser usados como parâmetros e valores de retorno de outras funções. Isso significa que podemos usar funções de ordem superior em Python. O conceito de funções de ordem superior não é muito amigável para iniciantes, mas traz flexibilidade ao design de funções. Se a função que queremos definir for muito simples, tiver apenas uma linha de código e não precisar de um nome de função para reutilização, podemos usar uma função lambda.
