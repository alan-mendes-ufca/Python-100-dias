## O algoritmo k-vizinhos mais próximos

k-vizinhos mais próximos, ou kNN, é um método estatístico não paramétrico usado para classificação e regressão. Foi proposto em 1951 pelos estatísticos americanos Evelyn Fix e Joseph Hodges Jr. A ideia do algoritmo kNN é encontrar as instâncias $\small{k}$ em dados históricos que estão mais próximas da nova instância de entrada e, em seguida, classificar a nova instância de acordo com a classe à qual pertence a maioria desses vizinhos, ou gerar o valor alvo da nova instância de acordo com esses vizinhos. Já mostramos um exemplo simples desse algoritmo na lição anterior.

Diferente dos algoritmos convencionais de aprendizado de máquina, o kNN não possui um processo explícito de aprendizado ou treinamento. Ele usa uma ideia simples e clara: "fique perto do vermelhão, fique manchado de vermelho; fique perto da tinta, fique manchado de preto". Existem duas questões principais no algoritmo kNN. A primeira é como escolher o valor de $\small{k}$, ou seja, quantos vizinhos mais próximos usar para julgar a classe de uma nova instância ou decidir seu valor alvo. A segunda é como julgar se duas instâncias estão próximas ou distantes, e isso leva ao problema de medir a distância.

### Medição de distância

Podemos usar a distância para medir a semelhança entre duas instâncias no espaço de recursos. As medidas de distância comuns incluem distância de Minkowski, distância de Mahalanobis, distância de cosseno, distância de edição e assim por diante. O nome completo da distância de Minkowski é Distância de Minkowski. Para dois vetores $\small{n}$-dimensionais $\small{\mathbf{x}=(x_{1}, x_{2}, \cdots, x_{n})}$ e $\small{\mathbf{y}=(y_{1}, y_{2}, \cdots, y_{n})}$, sua distância pode ser definida como:

$$ d(\mathbf{x}, \mathbf{y}) = (\sum_{i=1}^{n}{\vert x_{i} - y_{i} \rvert}^{p})^{\frac{1}{p}} $$

Aqui, $\small{p \ge 1}$. Embora ainda possamos calcular a fórmula quando $\small{p \lt 1}$, ela não satisfaz mais estritamente a definição de distância, portanto geralmente não é tratada como uma distância real.

Quando $\small{p = 1}$, a distância de Minkowski se torna **distância de Manhattan**:

$$ d(\mathbf{x}, \mathbf{y}) = \sum_{i=1}^{n} \lvert x_{i} - y_{i} \rvert $$

Quando $\small{p = 2}$, a distância de Minkowski se torna **distância euclidiana**:

$$ d(\mathbf{x}, \mathbf{y}) = \sqrt{\sum_{i=1}^{n}(x_{i} - y_{i})^{2}} $$

Quando $\small{p \to \infty}$, a distância de Minkowski se torna **distância de Chebyshev**:

$$ d(\mathbf{x}, \mathbf{y}) = \underset{i}{max}(\lvert x_{i} - y_{i} \rvert) $$

Introduziremos outras medidas de distanciamento quando realmente precisarmos delas. Ao usar kNN para fazer classificação, nosso conjunto de dados geralmente consiste em dados numéricos e, nesse caso, a distância euclidiana é uma boa escolha.

#### Implementação baseada em NumPy

kNN precisa de cálculo de distância. Então, primeiro, vamos escrever uma função para a distância euclidiana:

```python
import numpy as np


def euclidean_distance(u, v):
    """Calculate the Euclidean distance between two n-dimensional vectors."""
    return np.sqrt(np.sum(np.abs(u - v) ** 2))
```

A seguir, vamos projetar uma função que gera um rótulo para novos dados de acordo com os rótulos vizinhos.

```python
from scipy import stats


def make_label(X_train, y_train, X_one, k):
    """
    Generate a label for new data according to the k nearest neighbors.
    :param X_train: features in the training set
    :param y_train: labels in the training set
    :param X_one: features of the sample to predict
    :param k: number of neighbors
    :return: generated label for the sample to predict
    """
    # calculate the distance from x to each training sample
    distes = [euclidean_distance(X_one, X_i) for X_i in X_train]
    # use one partition to find the indexes of the k smallest distances
    labels = y_train[np.argpartition(distes, k - 1)[:k]]
    # get the mode of the labels
    return stats.mode(labels).mode
```

> **Observação**: A função `np.partition` pode particionar um array uma vez, colocando os elementos menores `k` no lado esquerdo do array e os elementos maiores `n - k` no lado direito. Isso funciona como uma etapa de partição no quicksort. A função `np.argpartition` no código acima coloca os índices dos elementos menores `k` no lado esquerdo da matriz. Desta forma, após a fatia `[:k]` e a indexação sofisticada em `y_train`, obtemos os rótulos das amostras cuja distância dos novos dados é menor. Finalmente, usamos a função `mode` no módulo `stats` do SciPy para obter o modo dos rótulos e usá-lo como o rótulo de classe previsto para os novos dados.

Então a função de previsão torna-se simples:

```python
def predict_by_knn(X_train, y_train, X_new, k=5):
    """
    kNN algorithm.
    :param X_train: features in the training set
    :param y_train: labels in the training set
    :param X_new: array of samples to predict
    :param k: number of neighbors, default value is 5
    :return: array that saves predicted labels
    """
    return np.array([make_label(X_train, y_train, X, k) for X in X_new])
```

### Avaliação do modelo

É claro que, para julgar se o efeito de previsão de um classificador é bom, não podemos simplesmente olhar para a precisão, porque quando as classes estão desequilibradas, a precisão pode facilmente enganar o seu julgamento. Por exemplo, se o número de amostras de Iris virginica em nossos dados de teste for muito pequeno, mesmo que nosso modelo não consiga julgar Iris virginica, o modelo ainda poderá mostrar alta precisão. Portanto, também precisamos observar a precisão, o recall, a pontuação F1 e outras métricas. A **matriz de confusão** é uma ferramenta usada para mostrar detalhadamente o desempenho de um modelo de classificação e é adequada tanto para tarefas de classificação binária quanto para tarefas de multiclassificação.

Para um problema de classificação binária, a matriz de confusão é:

|        | Previsto Positivo | Negativo previsto |
| ---- | ------------------ | ------------------ |
| Positivo Real | Verdadeiro Positivo (TP) | Falso Negativo (FN) |
| Negativo real | Falso Positivo (FP) | Verdadeiro Negativo (TN) |

As métricas comuns são:

1. Precisão
2. Precisão
3. Lembrar
4. Pontuação F1
5. Especificidade e taxa de falso positivo
6. ROC e AUC

Vamos usar um exemplo para mostrar como usar uma matriz de confusão e calcular as métricas de avaliação de um modelo de classificação. Suponha que um sistema de exames médicos preveja se uma pessoa tem uma determinada doença. A “classe positiva” significa “tem a doença”, e a “classe negativa” significa “não tem a doença”. Os resultados do teste de 1000 amostras são mostrados abaixo.

|        | Previsto Doente | Previsto não doente |
| ---- | -------------- | ------------------ |
| Doente real | 80 (TP) | 20 (FN) |
| Real não doente | 30 (PF) | 870 (TN) |

1. **Precisão**.

$$ \text{Precisão} = \frac{\text{TP} + \text{TN}}{\text{TP} + \text{FP} + \text{FN} + \text{TN}} $$

No exemplo acima, a precisão do modelo é: $\frac{80 + 870}{80 + 30 + 20 + 870} = \frac{950}{1000} = 0,95$ .

2. **Precisão**. A precisão mede a proporção de amostras que são realmente positivas entre todas as amostras previstas como positivas. Também é frequentemente chamado de valor preditivo positivo.

$$ \text{Precisão} = \frac{\text{TP}}{\text{TP} + \text{FP}} $$

No exemplo acima, a precisão é: $\frac{80}{80 + 30} = \frac{80}{110} = 0,73$ .

3. **Recall**. O recall mede a proporção de amostras corretamente previstas como positivas entre todas as amostras realmente positivas. Também é frequentemente chamado de taxa de verdadeiros positivos.

$$ \text{Recall} = \frac{\text{TP}}{\text{TP} + \text{FN}} $$

No exemplo acima, o recall é: $\frac{80}{80 + 20} = \frac{80}{100} = 0,8$ .

4. **Pontuação F1**. A pontuação F1 é a média harmônica de precisão e recall. Tenta encontrar um equilíbrio entre precisão e recall, especialmente quando há uma compensação entre eles.

$$ \text{Pontuação F1} = \frac{2}{\frac{1}{\text{Precisão}} + \frac{1}{\text{Recall}}} = 2 \times \frac{\text{Precisão} \times \text{Recall}}{\text{Precisão} + \text{Recall}} $$

No exemplo acima, a pontuação F1 é cerca de `0.76`.

5. **Especificidade** e **Taxa de falsos positivos**.

$$ \text{Especificidade} = \frac{\text{TN}}{\text{TN} + \text{FP}} \\\\ \text{FPR} = 1 - \text{Especificidade} $$

No exemplo acima, a especificidade é: $\frac{870}{870 + 30} = \frac{870}{900} = 0,97$ .

6. **ROC** e **AUC**.

ROC traça a relação entre recall e taxa de falsos positivos, conforme mostrado abaixo.

AUC é a área sob a curva ROC. Mede a capacidade do modelo de distinguir classes positivas e negativas.

### Ajuste de parâmetros

Dissemos antes que o algoritmo kNN tem dois problemas principais. Uma é a medição da distância e a outra é a escolha do valor de `k`. Quando usamos `KNeighborsClassifier` do scikit-learn para criar um modelo classificador, podemos definir os hiperparâmetros do modelo. Aqui estão vários parâmetros importantes:

1. `n_neighbors`: o número de vizinhos, ou seja, o valor de `k` no algoritmo kNN.
2. `weights`: pode ser `uniform` ou `distance`. O primeiro significa que todas as amostras têm o mesmo peso e o segundo significa que quanto mais próxima a distância, maior o peso. O valor padrão é `uniform`. Claro, também podemos passar uma função personalizada para decidir o peso de cada amostra.
3. `algorithm`: tem quatro opções, `auto`, `ball_tree`, `kd_tree` e `brute`. O valor padrão é `auto`. `ball_tree` é uma estrutura de árvore que usa um método de partição baseado em esfera para colocar pontos de dados em uma estrutura de árvore hierárquica e tem um bom desempenho em dados de alta dimensão e dados esparsos. `kd_tree` também é uma estrutura em árvore. Ele escolhe uma dimensão para dividir o espaço em várias sub-regiões e depois pesquisa, evitando comparar com todos os vizinhos. Funciona bem para dados de baixa dimensão e dados com distribuição espacial uniforme, mas atende à maldição da dimensionalidade em espaço de alta dimensão. A opção `auto` escolhe automaticamente `ball_tree` ou `kd_tree` de acordo com a dimensão dos dados de entrada. A opção `brute` usa o algoritmo de pesquisa de força bruta. Para pequenos conjuntos de dados, é uma escolha simples e eficaz.
4. `leaf_size`: ao utilizar o algoritmo `ball_tree` ou `kd_tree`, este parâmetro é utilizado para limitar o maior número de amostras em nós folha da estrutura em árvore. O valor padrão é `30`. Este parâmetro afeta o desempenho da construção da árvore e da pesquisa do nó.
5. `p`: o `p` na fórmula de distância de Minkowski. O valor padrão é `2`, que significa distância euclidiana.

Podemos usar **pesquisa em grade** e **validação cruzada** para ajustar hiperparâmetros do modelo, avaliar a capacidade de generalização do modelo e melhorar o desempenho da previsão. Pesquisa em grade significa usar uma pesquisa exaustiva para percorrer o espaço de hiperparâmetros fornecido e encontrar a melhor combinação de hiperparâmetros. Validação cruzada significa dividir o conjunto de treinamento em vários subconjuntos e, em seguida, treinar e avaliar o modelo muitas vezes em diferentes conjuntos de treinamento e conjuntos de validação, para que possamos fazer um julgamento completo sobre o desempenho da previsão. A validação cruzada K-Fold é o método de validação cruzada mais comum. Ele divide o conjunto de dados em subconjuntos `K`. Cada vez, um subconjunto é escolhido como conjunto de validação e os outros subconjuntos `K-1` são usados ​​como conjunto de treinamento. Este processo é repetido vezes `K`, e o valor médio é utilizado como avaliação final de desempenho do modelo, conforme mostrado abaixo.

![](res/02_k-fold_cross_validation.png)

O código abaixo usa `GridSearchCV` no scikit-learn para fazer pesquisa em grade e validação cruzada. Desta forma, podemos encontrar os melhores parâmetros para usar o algoritmo kNN no conjunto de dados da íris.

```python

    estimator=KNeighborsClassifier(),
    param_grid={
        'n_neighbors': [1, 3, 5, 7, 9, 11, 13, 15],
        'weights': ['uniform', 'distance'],
        'p': [1, 2]
    },
    cv=5
)
gs.fit(X_train, y_train)
```

Podemos obter os melhores parâmetros e a pontuação com o código a seguir.


### Regressão kNN

kNN geralmente é usado para resolver problemas de classificação, mas é claro que também pode ser usado para resolver problemas de regressão. Sua idéia básica ainda é encontrar os vizinhos `k` mais próximos da nova instância e, em seguida, prever o valor alvo da nova instância de acordo com os rótulos desses vizinhos `k`. Em problemas de regressão, geralmente os chamamos de valores alvo em vez de rótulos de classe. Como o que precisamos prever não é um rótulo de classe, mas um número, geralmente usamos a média ou a média ponderada para lidar com os valores alvo dos vizinhos `k` e, em seguida, obter o valor alvo previsto da nova instância. Vamos voltar ao exemplo da lição anterior que usou a renda mensal para prever despesas com compras online e, em seguida, usar `KNeighborsRegressor` no scikit-learn para construir um modelo de regressão. Primeiro, fazemos alguns preparativos, conforme mostrado abaixo.

```python
# monthly income
```

### Resumo

kNN é um algoritmo de aprendizado de máquina simples, mas poderoso. É adequado para tarefas de classificação e regressão em pequenos conjuntos de dados. Suas vantagens são que é simples de entender, não possui processo de treinamento explícito, não depende de suposições sobre a distribuição de dados e pode se adaptar a padrões de dados complexos. Claro, as desvantagens do kNN também são muito claras. O maior problema é a eficiência computacional, portanto, quando o conjunto de dados é grande, pode não ser a melhor escolha. Além disso, kNN é muito sensível ao ruído, e o resultado da predição depende do valor de `k` e se as amostras estão balanceadas. Se o valor de `k` for muito pequeno, poderá causar ajuste excessivo. Se o valor de `k` for muito grande, poderá causar ajuste insuficiente. Amostras desequilibradas também podem levar a viés de desequilíbrio de classe. Em outras palavras, se a maioria das amostras de treinamento pertence à classe A e há poucas amostras da classe B, então, durante a classificação, é mais provável que kNN preveja a amostra de teste como classe A.
