## pandas in Depth - 6

Let us now look at the `Index` type. It provides indexing services for `Series` and `DataFrame` objects. With indexes, we can sort data with the `sort_index` method, align data, which is very important in data operations and merging, and achieve fast retrieval of data through indexing operations. Since `DataFrame` represents two-dimensional data, both its rows and its columns have indexes, namely `index` and `columns`.

Creating an `Index` object is relatively simple. Usually we only need to provide three parameters: `data`, `dtype`, and `name`, which represent the data used as the index, the data type of the index, and the name of the index. Since `Index` itself is also one-dimensional data, its methods and properties are very similar to those of `Series`. You can try creating an `Index` object and then see whether the properties and methods we learned before also work on `Index`. Next, we mainly look at several subtypes of `Index`.

### Range Index

A range index is an index made of monotonic integers. We can create it through the `RangeIndex` constructor, or through the class method `from_range` of `RangeIndex`.

```python
sales_data = np.random.randint(400, 1000, 12)
index = pd.RangeIndex(1, 13, name='Month')
ser = pd.Series(data=sales_data, index=index)
ser
```

Output:

```
Month
1     703
2     705
3     557
4     943
5     961
6     615
7     788
8     985
9     921
10    951
11    874
12    609
dtype: int64
```

### Categorical Index

A categorical index is an index built from nominal-scale data. If we need to group data by index and then do aggregation, a categorical index can be very useful. A categorical index also has a method named `reorder_categories`, which can specify an order for the index. The result of grouped aggregation will then be presented in that specified order.

```python
sales_data = [6, 6, 7, 6, 8, 6]
index = pd.CategoricalIndex(
    data=['Apple', 'Banana', 'Apple', 'Apple', 'Peach', 'Banana'],
    categories=['Apple', 'Banana', 'Peach'],
    ordered=True
)
ser = pd.Series(data=sales_data, index=index)
ser
```

Output:

```
Apple     6
Banana    6
Apple     7
Apple     6
Peach     8
Banana    6
dtype: int64
```

Group data by the index and then sum it with `sum`.

```python
ser.groupby(level=0).sum()
```

Output:

```
Apple     19
Banana    12
Peach      8
dtype: int64
```

Specify the order of the index:

```python
ser.index = index.reorder_categories(['Banana', 'Peach', 'Apple'])
ser.groupby(level=0).sum()
```

Output:

```
Banana    12
Peach      8
Apple     19
dtype: int64
```

### MultiIndex

The `MultiIndex` type in pandas is used to represent hierarchical or multi-level indexes. We can use class methods such as `from_arrays`, `from_product`, and `from_tuples` of `MultiIndex` to create multi-level indexes. Let us look at a few examples.

```python
tuples = [(1, 'red'), (1, 'blue'), (2, 'red'), (2, 'blue')]
index = pd.MultiIndex.from_tuples(tuples, names=['no', 'color'])
index
```

Output:

```
MultiIndex([(1,  'red'),
            (1, 'blue'),
            (2,  'red'),
            (2, 'blue')],
           names=['no', 'color'])
```

```python
arrays = [[1, 1, 2, 2], ['red', 'blue', 'red', 'blue']]
index = pd.MultiIndex.from_arrays(arrays, names=['no', 'color'])
index
```

Output:

```
MultiIndex([(1,  'red'),
            (1, 'blue'),
            (2,  'red'),
            (2, 'blue')],
           names=['no', 'color'])
```

```python
sales_data = np.random.randint(1, 100, 4)
ser = pd.Series(data=sales_data, index=index)
ser
```

Output:

```
no  color
1   red      43
    blue     31
2   red      55
    blue     75
dtype: int64
```

```python
ser.groupby('no').sum()
```

Output:

```
no
1     74
2    130
dtype: int64
```

```python
ser.groupby(level=1).sum()
```

Output:

```
color
blue    106
red      98
dtype: int64
```

```python
stu_ids = np.arange(1001, 1006)
semesters = ['Midterm', 'Final']
index = pd.MultiIndex.from_product((stu_ids, semesters), names=['Student ID', 'Term'])
courses = ['Chinese', 'Math', 'English']
scores = np.random.randint(60, 101, (10, 3))
df = pd.DataFrame(data=scores, columns=courses, index=index)
df
```

Output:

```

           Final        93    98       84
```

Group data according to the first-level index, and calculate each student's score in each subject by weighting the midterm as `25%` and the final as `75%`.

### Interval Index

As the name suggests, an interval index uses fixed interval ranges as the index. We usually use the `interval_range` function to create an interval index.

```python
index = pd.interval_range(start=0, end=5)
index
```

Output:

```
IntervalIndex([(0, 1], (1, 2], (2, 3], (3, 4], (4, 5]], dtype='interval[int64, right]')
```

`IntervalIndex` has a method named `contains`, which can check whether a certain element is inside the interval range.

```python
index.contains(1.5)
```

Output:

```
array([False,  True, False, False, False])
```

`IntervalIndex` also has a method named `overlaps`, which can check whether one range overlaps with the others.

```python
index.overlaps(pd.Interval(1.5, 3.5))
```

Output:

```
array([False,  True,  True,  True, False])
```

If we want the intervals to be left-closed and right-open, we can do that when creating the interval index through `closed='left'`. If we want both sides to be closed, we can set the `closed` parameter to `both`.

```python
index = pd.interval_range(start=0, end=5, closed='left')
index
```

Output:

```
IntervalIndex([[0, 1), [1, 2), [2, 3), [3, 4), [4, 5)], dtype='interval[int64, left]')
```

```python
index = pd.interval_range(start=pd.Timestamp('2022-01-01'), end=pd.Timestamp('2022-01-04'), closed='both')
index
```

Output:

```
IntervalIndex([[2022-01-01, 2022-01-02], [2022-01-02, 2022-01-03], [2022-01-03, 2022-01-04]], dtype='interval[datetime64[ns], both]')
```

### DateTime Index

`DatetimeIndex` is probably one of the most complex and most important index types. We usually use the `date_range()` function to create it. This function has several very important parameters: `start`, `end`, `periods`, `freq`, and `tz`, which represent start date-time, end date-time, number of periods, sampling frequency, and time zone.

Let us first see how to create a `DatetimeIndex` object, and then talk about related operations.

```python
pd.date_range('2021-1-1', '2021-6-30', periods=10)
```

Output:

```
DatetimeIndex(['2021-01-01', '2021-01-21', '2021-02-10', '2021-03-02',
               '2021-03-22', '2021-04-11', '2021-05-01', '2021-05-21',
               '2021-06-10', '2021-06-30'],
              dtype='datetime64[ns]', freq=None)
```

```python
pd.date_range('2021-1-1', '2021-6-30', freq='W')
```

> **Note**: `freq='W'` means the sampling frequency is one week, and by default Sunday is treated as the start of a week. If you want Monday to be the start of a week, you can change it to `freq='W-MON'`. You can also try values like `12H`, `M`, or `Q` and see what happens.

Output:

```
DatetimeIndex(['2021-01-03', '2021-01-10', '2021-01-17', '2021-01-24',
               '2021-01-31', '2021-02-07', '2021-02-14', '2021-02-21',
               '2021-02-28', '2021-03-07', '2021-03-14', '2021-03-21',
               '2021-03-28', '2021-04-04', '2021-04-11', '2021-04-18',
               '2021-04-25', '2021-05-02', '2021-05-09', '2021-05-16',
               '2021-05-23', '2021-05-30', '2021-06-06', '2021-06-13',
               '2021-06-20', '2021-06-27'],
              dtype='datetime64[ns]', freq='W-SUN')
```

`DatetimeIndex` can operate with the `DateOffset` type. This is easy to understand, because we can set a time difference and move the time forward or backward.

```python
index = pd.date_range('2021-1-1', '2021-6-30', freq='W')
index - pd.DateOffset(days=2)
```

Output:

If a `Series` object or `DataFrame` object uses `DatetimeIndex` as its index, then we can use the `asfreq()` method to specify a time frequency and sample the data. We still use the Baidu stock data from earlier as an example.

```python
baidu_df = pd.read_excel('data/2022-stock-data.xlsx', sheet_name='BIDU', index_col='Date')
baidu_df.sort_index(inplace=True)
baidu_df.asfreq('5D')
```

Output:

<img src="res/baidu_stock_asfreq.png" style="zoom:50%;">

You may have noticed that sampling one day every 5 days may hit non-trading days, and then the corresponding columns become null values. To solve this problem, when using the `asfreq` method, we can use the `method` parameter to specify a way to fill null values, so that data from nearby trading days is filled in.

```python
baidu_df.asfreq('5D', method='ffill')
```

Output:

<img src="res/baidu_stock_asfreq_ffill.png" style="zoom:50%;">

When using `DatetimeIndex`, we can also use the `resample()` method to resample data based on time. This is like grouping data by time periods, and after grouping we can still do aggregation.

```python
baidu_df.resample('1M').mean()
```
