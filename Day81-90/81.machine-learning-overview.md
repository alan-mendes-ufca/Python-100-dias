### What Machine Learning Is

Humans learn through memory and induction. Through memory, we can collect single facts. Through induction, we can derive new facts from old facts. Machine learning gives machines the ability to learn knowledge from data. This process does not need human help in the form of giving clear rules. In other words, machine learning focuses on learning a pattern from data, even if the data itself has problems such as noise. This is also the most basic difference between machine learning algorithms and traditional algorithms. Traditional algorithms need to be told how to find answers in a complex system, and they use the computing power of the computer to find the best result. The biggest weakness of traditional algorithms is that humans must first know what the best solution is. Machine learning algorithms do not need humans to tell the model the best solution. Instead, we provide sample data related to the problem.

We can divide the problems that computers need to solve into four kinds. The division depends on whether the input is exact or fuzzy, and whether the output is optimal or satisfactory. From that view, traditional algorithms are mainly good at the first kind of problem, while machine learning is better at the third and fourth kinds. The second kind is usually NP problems, such as the traveling salesman problem, graph coloring, and set cover. We often solve those with heuristic algorithms or approximation algorithms, although machine learning can also give satisfactory solutions there.

<img src="res/01_four_kinds_problems.png" style="zoom:45%;">

Of course, machine learning algorithms are not perfect. When we train a model with data, we need to use data that has been preprocessed and cleaned. If the quality of the data itself is very bad, it is also hard for us to train a good model. This is what we often call GIGO, garbage in garbage out. If we want to use machine learning, we also need to make sure whether there is some kind of relationship between variables. Machine learning cannot handle data that has no relationship at all. Most of the time, a machine learning model outputs a series of numbers and metrics, and humans need to read those numbers and metrics and make decisions, judge whether the model is good or bad, and decide how the model should be used in a real business scenario. Usually, the data we use to train a model contains noisy data, and many machine learning models are very sensitive to noise. If we cannot handle noisy data well, then it is not easy for us to get a good model.

### Categories of Machine Learning

Machine learning models can be classified by different standards. The main ones are below.

#### By learning style

<img src="res/01_machine_learning_category.png" style="zoom:32%;">

1. Supervised learning
   - regression: models used to predict continuous values, such as linear regression, Ridge regression, and Lasso regression.
   - classification: models used to predict discrete classes, such as logistic regression, support vector machines, decision trees, random forests, k-nearest neighbors, and naive Bayes.
2. Unsupervised learning
   - clustering: models used to group data, such as K-means clustering, hierarchical clustering, and DBSCAN.
   - dimensionality reduction: models used to reduce the number of features, such as principal component analysis (PCA), linear discriminant analysis (LDA), and t-distributed stochastic neighbor embedding (t-SNE).
   - association rule learning: models used to discover relationships between items in a dataset, such as the Apriori algorithm and the Eclat algorithm.
3. Semi-supervised learning
   - methods that combine supervised learning and unsupervised learning and use a large amount of unlabeled data and a small amount of labeled data to build models.
4. Reinforcement learning
   - learning methods based on reward mechanisms, such as Q-Learning, deep Q networks (DQN), and policy gradient methods.

#### By model complexity

1. Linear models: such as linear regression and logistic regression.
2. Non-linear models: such as support vector machines with kernel functions and neural networks.

#### By model structure

1. Generative models: models that can generate new data points, such as naive Bayes and hidden Markov models (HMM).
2. Discriminative models: models used only for classification or regression, such as logistic regression and support vector machines.

### Steps of Machine Learning

The steps of machine learning usually include several stages, from defining the problem, to preparing data, to deploying and maintaining the model. Every step is very important.

1. **Define the problem**. First, we need to understand the business and make clear what kind of problem it is. This directly affects later data collection, feature engineering, algorithm choice, and evaluation metrics.
2. **Collect data**. Training a machine learning model needs a large amount of data. The data may include structured data, unstructured data, and other types of datasets.
3. **Clean data**. Data cleaning should make sure the data quality is high and suitable for model training. This includes handling missing values and outliers, data standardization and normalization, special encoding, feature engineering, and so on.
4. **Split data**. To evaluate the generalization ability of a machine learning model, we need to split data into a training set and a test set. Besides that, we may also use cross-validation to split the data into several subsets and let each subset take turns being the validation set, so we can tune the hyperparameters of the model.
5. **Select a model**. The machine learning algorithms or models we choose are different for classification problems, regression problems, clustering problems, and deep learning.
6. **Train the model**. Use the training set to train the model so it can learn the relationship between input features and targets. Also, every machine learning algorithm has hyperparameters, and these need to be tuned according to the data and task.
7. **Evaluate the model**. The main goal of model evaluation is to make sure the model works well on new data, that is, the test set, and to make sure the model does not have overfitting or underfitting, as shown in the figure below. Underfitting makes the model predict badly, while overfitting makes the model lack generalization ability, which means it works badly on the test set and new data. Of course, to improve the model's performance or adaptability, we may also need to tune it through regularization, ensemble learning, algorithm adjustment, and other methods.

<img src="res/01_overfitting_vs_underfitting.png" style="zoom:85%;">

8. **Deploy the model**. When you are satisfied with the model's performance, you can deploy it to the production environment for real-time prediction or batch prediction. By monitoring the model's performance in real use, we make sure it keeps a good prediction effect. If the model performance drops, we may need to retrain or adjust it.
9. **Maintain the model**. A machine learning model does not stay unchanged. As time goes on, the model may need to keep its performance through retraining, incremental learning, and other methods.

> **Note**: If some of the concepts above are still hard to understand, you can skip them first. We will explain them when we meet these problems later.

#### The kNN idea

To predict monthly online-shopping expense from monthly income, one very simple idea is to make the historical data into a dictionary, using monthly income as the key in the dictionary and monthly online-shopping expense as the corresponding value. Then we can find expense by looking up income in the dictionary, as shown below.

```python
sample_data = {key: value for key, value in zip(x, y)}
```

However, the input monthly income may not match a key in the dictionary. If there is no key, we cannot get the value directly. At this time, we can find the `k` keys that are closest to the input monthly income, take the average of the values corresponding to these `k` keys, and use this average as the prediction of monthly online-shopping expense. This method is the simplest k-nearest neighbors algorithm, or kNN. It is a non-parametric statistical method used for classification and regression. Below, we use plain Python code to implement the kNN algorithm and do not use third-party libraries such as NumPy, SciPy, or Scikit-Learn for now. The main purpose is to help everyone understand the principle of the algorithm.

```python
import heapq
import statistics


def predict_by_knn(history_data, param_in, k=5):
    """Use kNN to make a prediction.
    :param history_data: historical data
    :param param_in: model input
    :param k: number of neighbors, default value is 5
    :return: model output, that is, the predicted value
    """
    neighbors = heapq.nsmallest(k, history_data, key=lambda x: (x - param_in) ** 2)
    return statistics.mean([history_data[neighbor] for neighbor in neighbors])
```

In the code above, we use `nsmallest` in Python's `heapq` module to find the smallest `k` elements in the historical data. Through the `key` parameter of this function, we define "smallest" to mean the smallest error from the input parameter `param_in`. Because error can be positive or negative, we usually need to square it or take the absolute value. For the arithmetic mean, we use the `mean` function in Python's `statistics` module. If you are not familiar with `heapq` and `statistics`, you can read the Python official documentation. We will not go into details here. Next, we use the function above to predict monthly online-shopping expense.

```python
incomes = [1800, 3500, 5200, 6600, 13400, 17800, 20000, 30000]
for income in incomes:
    print(f'Monthly income: {income:>5d} yuan, Monthly online-shopping expense: {predict_by_knn(sample_data, income):>6.1f} yuan')
```

From the output above, we can see one weakness of the kNN algorithm. Missing sample data or unbalanced sample data can make the prediction result very bad. The predicted online-shopping expenses for monthly incomes of `17800`, `20000`, and `30000` yuan are all `6044.4` yuan. This is because the largest monthly income in the historical data is only `19880`, so the nearest `k` neighbors of those three inputs are exactly the same, and therefore the predicted expenses are also the same. In the same way, the predicted expenses for monthly incomes of `1800` yuan and `3500` yuan are also the same, because the nearest `k` neighbors of `1800` and `3500` are also exactly the same. Of course, the kNN implementation we gave above has other problems as well. It is not hard to see that every time the `predict_by_knn` function looks for the nearest `k` neighbors, it must go through the whole historical dataset once. If the dataset is very large, this part will create a big cost.

#### Regression model

Let us change to another idea to predict online-shopping expense. We use monthly income and monthly online-shopping expense as the horizontal and vertical axes and draw a scatter chart. Since there is a strong positive correlation between online-shopping expense and monthly income, this means we can find a straight line to fit these historical data points. We call the equation of this straight line, $\small{y = aX + b}$, a regression equation or regression model, as shown below.

<img src="res/01_regression_model.png" style="zoom:50%;">

Now, our problem becomes how to calculate the slope $\small{a}$ and intercept $\small{b}$ of the regression model from the historical data we collected. To evaluate whether the regression model is good, that is, whether the slope and intercept we calculated are ideal, we can first define an evaluation standard. A simple and clear standard is this: put monthly income $\small{X}$ into the regression model, calculate the predicted monthly online-shopping expense $\small{\hat{y}}$, and compare it with the true value $\small{y}$. The smaller the error between the predicted value $\small{\hat{y}}$ and the true value $\small{y}$, the better the regression model is. As we said before, when we calculate error, we usually need to take the absolute value or square it. Here we can use the mean of squared errors as the standard, which is usually called mean squared error (MSE), as shown below.

$$
MSE = \frac{1}{n}{\sum_{i=1}^{n}(y_{i} - \hat{y_{i}})^{2}}
$$

The values of $\small{a}$ and $\small{b}$ that minimize the MSE are called the least-squares solution of the regression equation.

According to the formula above, we can write a function to compute mean squared error. It is usually called the loss function.

```python
import statistics


def get_loss(X_, y_, a_, b_):
    """Loss function.
    :param X_: independent variable of regression model
    :param y_: dependent variable of regression model
    :param a_: slope of regression model
    :param b_: intercept of regression model
    :return: MSE
    """
    y_hat = [a_ * x + b_ for x in X_]
    return statistics.mean([(v1 - v2) ** 2 for v1, v2 in zip(y_, y_hat)])
```

The values of $\small{a}$ and $\small{b}$ that make MSE the smallest are called the least-squares solution of the regression equation. The next work is to find this least-squares solution. Simply speaking, we put possible values of $\small{a}$ and $\small{b}$ into the loss function and see what values of $\small{a}$ and $\small{b}$ can make the loss function smallest. If we know nothing about the values of $\small{a}$ and $\small{b}$, we can find them by generating random numbers again and again. This method is called Monte Carlo simulation. To say it in a simple way, it means making random guesses, as shown below.

```python
import random

# define the minimum loss as a very large value first
min_loss, a, b = 1e12, 0, 0

for _ in range(100000):
    # get slope and intercept by generating random numbers
    _a, _b = random.random(), random.uniform(-2000, 2000)
    # put them into the loss function to compute the MSE
    curr_loss = get_loss(x, y, _a, _b)
    if curr_loss < min_loss:
        # record the smaller MSE as the minimum loss
        min_loss = curr_loss
        # record the corresponding a and b
        a, b = _a, _b

print(f'MSE = {min_loss}')
print(f'{a = }, {b = }')
```

Output:

```
MSE = 270690.1419424315
a = 0.4824040159203802, b = -1515.0162977046068
```

The code above runs the simulation `100000` times. The value of $\small{a}$ is chosen randomly from $\small[0, 1)$, and the value of $\small{b}$ is chosen randomly from $\small{[-2000, 2000)}$. The numbers below are the result on my computer. If you have stronger math knowledge, then we can put the regression model into the loss function. Since $\small{X}$ and $\small{y}$ are known historical data, the loss function is actually a binary function of $\small{a}$ and $\small{b}$, as shown below.

$$
MSE = f(a, b) = \frac{1}{n}{\sum_{i=1}^{n}(y_{i} - (ax_{i} + b))^{2}}
$$

According to the extremum theorem in calculus, we can take partial derivatives of $\small{f(a, b)}$ and set them equal to `0`. Then we can compute the values of `a` and `b` that make $\small{f(a, b)}$ smallest.

$$
\frac{\partial{f(a, b)}}{\partial{a}} = -\frac{2}{n}\sum_{i=1}^{n}x_{i}(y_{i} - x_{i}a - b) = 0
$$

$$
\frac{\partial{f(a, b)}}{\partial{b}} = -\frac{2}{n}\sum_{i=1}^{n}(y_i - x_{i}a - b) = 0
$$

Solving the equations above gives:

$$
a = \frac{\sum(x_{i} - \bar{x})(y_{i} - \bar{y})}{\sum(x_{i} - \bar{x})^{2}}
$$

$$
b = \bar{y} - a\bar{x}
$$

It should be explained that if the regression model itself is more complex and not as simple as the linear model $\small{y = aX + b}$, then we may not be able to solve the equations directly like above. Instead, we need to find the extreme point in other ways. We will explain this in later lessons. Back to the problem above, we can calculate the values of $\small{a}$ and $\small{b}$ with the formulas above. For convenience, we directly use functions in NumPy below. Because operations in NumPy are vectorized, we usually do not need to write loops by ourselves. If you are not familiar with NumPy, you can read my [Data Analysis with Python](https://www.zhihu.com/column/c_1217746527315496960) column.

```python
import numpy as np

x_bar, y_bar = np.mean(x), np.mean(y)
a = np.dot((x - x_bar), (y - y_bar)) / np.sum((x - x_bar) ** 2)
b = y_bar - a * x_bar
print(f'{a = }, {b = }')
```

Output:

```
a = 0.482084452824066, b = -1515.2028590756745
```

In fact, NumPy also has ready-made functions that can help us fit parameters. You can use the `lstsq` function in the `linalg` module to calculate the least-squares solution, or you can use the `polyfit` function or the `fit` method of the `Polynomial` type to get the least-squares solution. You can check whether the result is the same as the result obtained above by taking partial derivatives and finding the extreme point.

```python
a, b = np.polyfit(x, y, deg=1)
print(f'{a = }, {b = }')
```

or

```python
from numpy.polynomial import Polynomial

b, a = Polynomial.fit(x, y, deg=1).convert().coef
print(f'{a = }, {b = }')
```

> **Note**: The `deg` parameter in the code above means the highest-order term of the polynomial. `deg=1` means we are using a linear regression model.
