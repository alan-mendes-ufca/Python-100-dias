## Integração de plataformas de terceiros

Durante o desenvolvimento de uma aplicação Web, existem algumas tarefas que não podemos realizar sozinhos. Por exemplo, se o nosso projeto Web precisar de verificação do nome real de uma pessoa ou de uma empresa, claramente não teremos a capacidade de julgar se as informações de verificação fornecidas pelo usuário são verdadeiras. Neste momento, precisamos utilizar serviços fornecidos por uma plataforma de terceiros para concluir esta operação. Outro exemplo é que se o nosso projeto necessita de pagamento on-line, esse tipo de negócio geralmente também é finalizado através de um gateway de pagamento, em vez de ser implementado por nós mesmos. Precisamos apenas nos conectar a plataformas de terceiros, como WeChat Pay, Alipay ou UnionPay.

Existem basicamente duas maneiras de conectar uma plataforma de terceiros em um projeto: integração de API e integração de SDK.

1. Integração de API significa concluir uma operação ou obter dados visitando um URL fornecido por terceiros. Na China, muitas plataformas como esta fornecem muitos serviços comumente usados. Por exemplo, [Sopas de tâmaras](https://www.juhe.cn/) fornece muitos tipos de APIs, como serviços de vida, fintech, tráfego e geografia, recarga e pagamento e assim por diante. Podemos iniciar solicitações de rede por meio de um programa Python e obter dados visitando URLs. Essas interfaces API são iguais às interfaces de dados fornecidas em nosso próprio projeto. A diferença é que as APIs do nosso projeto são para uso próprio, enquanto as APIs fornecidas por plataformas de terceiros são abertas. É claro que aberto não significa gratuito. A maioria das APIs que podem fornecer dados comercialmente valiosos precisam de pagamento antes do uso.
2. A integração do SDK significa instalar uma biblioteca de terceiros e usar as classes e funções incluídas nessa biblioteca para usar os serviços fornecidos pela plataforma de terceiros. Por exemplo, se quisermos conectar o Alipay, precisamos primeiro instalar o SDK do Alipay e, em seguida, concluir a chamada do serviço de pagamento por meio das classes e métodos fornecidos pelo Alipay.

A seguir explicamos como integrar plataformas de terceiros através de exemplos concretos.

### Conectando um gateway SMS

Um serviço de SMS pode ser usado em muitos lugares em um projeto da Web, por exemplo, login com código de verificação móvel, lembretes de mensagens importantes, SMS de marketing de produto e assim por diante. Para enviar mensagens SMS, podemos conectar um gateway SMS. Na China, os gateways SMS conhecidos incluem Yunpian SMS, NetEase Yunxin, Luosimao, SendCloud e assim por diante. Esses gateways SMS geralmente oferecem funções de teste gratuitas. Abaixo, usamos [Luosimão](https://luosimao.com/) como exemplo para explicar como conectar um gateway SMS em um projeto. O funcionamento das demais plataformas é basicamente semelhante.

1. Registre uma conta. Novos usuários podem experimentá-lo gratuitamente.

2. Faça login no back-end de gerenciamento e entre na seção SMS.

3. Clique em “Trigger Send” para encontrar sua própria chave de API, que é sua marca de identidade.

    ![](res/luosimao-sms-apikey.png)

4. Clique em “Gerenciamento de assinatura” para adicionar uma assinatura SMS. As mensagens SMS devem conter uma assinatura. Para teste gratuito de SMS, você precisa adicionar a assinatura `【Tieke Test】` na mensagem SMS, caso contrário o SMS não poderá ser enviado.

    ![](res/luosimao-sms-signature.png)

5. Clique em “IP Whitelist” e preencha o endereço do servidor, ou seja, o endereço IP público do servidor que executa o projeto Django, na lista de permissões. Se você estiver executando localmente, poderá abrir um site de verificação de IP para ver o endereço IP público da sua máquina. Caso contrário, o SMS não poderá ser enviado.

    ![](res/luosimao-sms-whitelist.png)

6. Se não houver mais créditos de SMS, vá até a página “Recarregar” e escolha “Serviço SMS” para recarregar.

    ![](res/luosimao-pay-onlinebuy.png)

A seguir, podemos ligar para o gateway Luosimao SMS para implementar a função de envio de códigos de verificação SMS. O código é mostrado abaixo.

```python
def send_mobile_code(tel, code):
    """Send an SMS verification code"""
    resp = requests.post(
        url='http://sms-api.luosimao.com/v1/send.json',
        auth=('api', 'key-your-own-api-key'),
        data={
            'mobile': tel,
            'message': f'Your SMS verification code is {code}. Do not tell it to anyone no matter what. [Python Mini Course]'
        },
        verify=False
    )
    return resp.json()
```

Para executar o código acima, você precisa primeiro instalar a biblioteca de terceiros `requests`. Esta biblioteca envolve funções de solicitação de rede HTTP e é muito fácil de usar. Também falamos sobre essa biblioteca antes. A função `send_mobile_code` possui dois parâmetros. O primeiro parâmetro é o número do celular e o segundo parâmetro é o conteúdo do código de verificação SMS. A linha 5 do código precisa de sua própria chave de API, que é a chave de API que você viu na etapa 2 acima. A solicitação do gateway SMS da Luosimao retorna dados no formato JSON. Se o código acima retornar `{'err': 0, 'msg': 'ok'}`, significa que o SMS foi enviado com sucesso. Se o valor do campo `err` não for `0`, mas outro valor, significa que houve falha no envio do SMS. Você pode ver o que os diferentes valores significam na [documentação oficial para desenvolvedores](https://luosimao.com/docs/api/) da Luosimao. Por exemplo, `-20` significa que o saldo não é suficiente e `-32` significa que falta a assinatura SMS.

Podemos chamar a função acima em uma função de visualização para concluir o envio de um código de verificação por SMS. Posteriormente, podemos combinar esta função com o registro do usuário.

Funções para gerar um código de verificação aleatório e validar um número de celular.

```python
import random
import re

TEL_PATTERN = re.compile(r'1[3-9]\d{9}')


def check_tel(tel):
    """Check a mobile number"""
    return TEL_PATTERN.fullmatch(tel) is not None


def random_code(length=6):
    """Generate a random SMS verification code"""
    return ''.join(random.choices('0123456789', k=length))
```

Ver função para envio de um código de verificação por SMS.

```python
@api_view(('GET', ))
def get_mobilecode(request, tel):
    """Get an SMS verification code"""
    if check_tel(tel):
        redis_cli = get_redis_connection()
        if redis_cli.exists(f'vote:block-mobile:{tel}'):
            data = {'code': 30001, 'message': 'Please do not send another SMS verification code within 60 seconds'}
        else:
            code = random_code()
            send_mobile_code(tel, code)
            # Use Redis to stop repeated sending of SMS verification codes within 60 seconds
            redis_cli.set(f'vote:block-mobile:{tel}', 'x', ex=60)
            # Keep the verification code in Redis for 10 minutes, with a valid period of 10 minutes
            redis_cli.set(f'vote2:valid-mobile:{tel}', code, ex=600)
            data = {'code': 30000, 'message': 'The SMS verification code has been sent, please check it'}
    else:
        data = {'code': 30002, 'message': 'Please enter a valid mobile number'}
    return Response(data)
```

> **Observação**: O código acima usa Redis para implementar duas funções extras. Uma delas é impedir que o usuário envie outro código de verificação por SMS em 60 segundos. A outra é manter o código de verificação SMS do usuário por 10 minutos, o que significa que o período válido desse código de verificação é de apenas 10 minutos. Podemos pedir ao usuário que forneça este código durante o registro para verificar se o número do celular do usuário é real.

### Conectando um serviço de armazenamento em nuvem

Quando mencionamos o termo **armazenamento em nuvem**, geralmente nos referimos ao armazenamento de dados em um ambiente de servidor virtual fornecido por terceiros. Simplificando, alguns dados ou recursos são hospedados em uma plataforma de terceiros. Normalmente, as empresas que fornecem serviços de armazenamento em nuvem administram grandes data centers. Indivíduos ou organizações que precisam de armazenamento em nuvem compram ou alugam espaço de armazenamento para atender às necessidades de armazenamento de dados. Ao desenvolver um aplicativo Web, os recursos estáticos, especialmente os recursos estáticos carregados pelo usuário, podem ser colocados diretamente nos serviços de armazenamento em nuvem. O armazenamento em nuvem geralmente fornece URLs correspondentes para que os usuários possam acessar esses recursos estáticos. Serviços de armazenamento em nuvem bem conhecidos na China e fora da China, como Amazon S3 e Alibaba OSS, geralmente são bons e de baixo custo. Em comparação com a construção de um servidor de recursos estáticos por conta própria, o armazenamento em nuvem custa menos. Além disso, plataformas gerais de armazenamento em nuvem fornecem serviço CDN para acelerar o acesso a recursos estáticos. Portanto, de quase todos os ângulos, usar o armazenamento em nuvem para gerenciar os dados e recursos estáticos de um aplicativo Web é uma escolha muito boa, a menos que esses recursos envolvam privacidade pessoal ou comercial. Caso contrário, eles podem ser hospedados em armazenamento em nuvem.

Abaixo, usamos [Nuvem Qiniu](https://www.qiniu.com/) como exemplo para explicar como salvar arquivos enviados pelo usuário no armazenamento em nuvem Qiniu. Qiniu Cloud é um conhecido provedor de serviços de dados e computação em nuvem na China. Qiniu Cloud tem seus próprios produtos em armazenamento de arquivos em massa, CDN, vídeo sob demanda, interação ao vivo e análise e processamento inteligente de dados heterogêneos em grande escala. Mesmo os usuários que não pagam podem se conectar gratuitamente e utilizar os serviços que ele oferece. Abaixo está o processo de conexão do Qiniu Cloud:

A seguir, desde que sigamos o exemplo da documentação oficial, podemos conectar o Qiniu Cloud e usar o armazenamento em nuvem e outros serviços que ele oferece. Primeiro, podemos instalar a biblioteca de terceiros Qiniu através do comando abaixo.

```bash
pip install qiniu
```

A seguir, podemos usar as funções `put_file` e `put_stream` no módulo `qiniu` para implementar o upload de arquivos. O primeiro carrega um arquivo em um caminho especificado e o último carrega dados binários na memória para Qiniu Cloud. O código é mostrado abaixo.

```python
import qiniu

AUTH = qiniu.Auth('AccessKey from key management', 'SecretKey from key management')
BUCKET_NAME = 'myvote'


def upload_file_to_qiniu(key, file_path):
    """Upload the file at a specified path to Qiniu Cloud"""
    token = AUTH.upload_token(BUCKET_NAME, key)
    return qiniu.put_file(token, key, file_path)


def upload_stream_to_qiniu(key, stream, size):
    """Upload a binary data stream to Qiniu Cloud"""
    token = AUTH.upload_token(BUCKET_NAME, key)
    return qiniu.put_stream(token, key, stream, None, size)
```

> **Observação**: Se o frontend usar um formulário para fazer upload de um arquivo, o atributo `method` do formulário deve ser definido como `post`, o atributo `enctype` precisa ser definido como `multipart/form-data` e a tag `input` cuja O atributo `type` é `file` é o seletor de arquivo para fazer upload de um arquivo.

A função de visualização que implementa a função de upload é mostrada abaixo.

```python
from django.views.decorators.csrf import csrf_exempt


@csrf_exempt
def upload(request):
    # If the uploaded file is smaller than 2.5M, the type of the photo object is InMemoryUploadedFile, and the file is in memory
    # If the uploaded file is larger than 2.5M, the type of the photo object is TemporaryUploadedFile, and the file is under a temporary path
    photo = request.FILES.get('photo')
    _, ext = os.path.splitext(photo.name)
    # Use UUID and the original file extension to generate a unique new file name
    filename = f'{uuid.uuid1().hex}{ext}'
    # For files in memory, we can use the wrapped function upload_stream_to_qiniu above to upload the file to Qiniu Cloud
    # If the file is saved under a temporary path, we can use upload_file_to_qiniu to upload the file
    upload_stream_to_qiniu(filename, photo.file, photo.size)
    return redirect('/static/html/upload.html')
```

> **Aviso**: A função de visualização acima usa o decorador `csrf_exempt`. Este decorador permite que o formulário fique isento da exigência de fornecer um token CSRF. Além disso, a linha 11 do código usa a função `uuid1` no módulo `uuid` para gerar um identificador globalmente exclusivo.
