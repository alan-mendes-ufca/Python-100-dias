## Applying Concurrent Programming in Crawlers

In earlier lessons, we already introduced multithreading, multiprocessing, and asynchronous programming in Python. Through these three means, we can implement concurrent or parallel programming. On one hand, this can speed up code execution, and on the other hand, it can also bring a better user experience. Crawler programs are typical I/O-intensive tasks. For I/O-intensive tasks, both multithreading and asynchronous I/O are good choices, because when one part of the program is blocked by an I/O operation, other parts of the program can still keep running, so we do not need to waste a lot of time in waiting and blocking. Below, using crawling images from the [360 Image](https://image.so.com/) website and saving them locally as an example, we show the differences between a crawler program written with one thread, multiple threads, and asynchronous I/O, and we also make a simple comparison of their execution efficiency.

The page of the 360 Image website uses Ajax technology. This is a technology that many websites use for asynchronously loading data and partially refreshing pages. Simply speaking, the images on the page are obtained asynchronously by JavaScript code from JSON data and then dynamically rendered, and the whole page also uses waterfall loading, meaning as you scroll downward, more images are loaded. In the browser's Developer Tools, we can find the data interface that provides the dynamic content. As shown in the picture below, the image information we need is inside the JSON data returned by the server.

<img src="res/20211205221352.png" style="zoom:40%;">

For example, to get images from the "beauty" channel, we can request the URL shown below. In it, the parameter `ch` means the requested channel, and the parameter value `beauty` after `=` means the beauty channel. The parameter `sn` is equal to the page number. `0` means the first page, with `30` images in total. `30` means the second page, `60` means the third page, and so on.

```text
https://image.so.com/zjl?ch=beauty&sn=0
```

### Single-Threaded Version

Use the URL above to download `90` images in total from the beauty channel.

```python
"""
example04.py - single-threaded crawler
"""
import os

import requests


def download_picture(url):
    filename = url[url.rfind('/') + 1:]
    resp = requests.get(url)
    if resp.status_code == 200:
        with open(f'images/beauty/{filename}', 'wb') as file:
            file.write(resp.content)


def main():
    if not os.path.exists('images/beauty'):
        os.makedirs('images/beauty')
    for page in range(3):
        resp = requests.get(f'https://image.so.com/zjl?ch=beauty&sn={page * 30}')
        if resp.status_code == 200:
            pic_dict_list = resp.json()['list']
            for pic_dict in pic_dict_list:
                download_picture(pic_dict['qhimg_url'])


if __name__ == '__main__':
    main()
```

On macOS or Linux, we can use the `time` command to know the execution time of the code above and the CPU utilization, as shown below.

```bash
time python3 example04.py
```

Below is the result of running the single-threaded crawler code on my computer.

```text
python3 example04.py  2.36s user 0.39s system 12% cpu 21.578 total
```

Here, we only need to care that the total time of the code is `21.578` seconds, and CPU utilization is `12%`.

### Multithreaded Version

We use the thread pool technique talked about before, and change the code above into a multithreaded version.

```python
"""
example05.py - multithreaded crawler
"""
import os
from concurrent.futures import ThreadPoolExecutor

import requests


def download_picture(url):
    filename = url[url.rfind('/') + 1:]
    resp = requests.get(url)
    if resp.status_code == 200:
        with open(f'images/beauty/{filename}', 'wb') as file:
            file.write(resp.content)


def main():
    if not os.path.exists('images/beauty'):
        os.makedirs('images/beauty')
    with ThreadPoolExecutor(max_workers=16) as pool:
        for page in range(3):
            resp = requests.get(f'https://image.so.com/zjl?ch=beauty&sn={page * 30}')
            if resp.status_code == 200:
                pic_dict_list = resp.json()['list']
                for pic_dict in pic_dict_list:
                    pool.submit(download_picture, pic_dict['qhimg_url'])


if __name__ == '__main__':
    main()
```

Run the command shown below.

```bash
time python3 example05.py
```

The run result of the code is shown below.

```text
python3 example05.py  2.65s user 0.40s system 95% cpu 3.193 total
```

### Asynchronous I/O Version

We use `aiohttp` to change the code above into an asynchronous I/O version. To implement getting network resources and writing files in an asynchronous I/O way, we first need to install the third-party libraries `aiohttp` and `aiofile`. The command is shown below.

```bash
pip install aiohttp aiofile
```

The usage of `aiohttp` was briefly introduced in the earlier lesson. The `async_open` function in the `aiofile` module is roughly used in the same way as Python's built-in `open` function, but it supports asynchronous operations. Below is the crawler code of the asynchronous I/O version.

```python
"""
example06.py - asynchronous I/O crawler
"""
import asyncio
import json
import os

import aiofile
import aiohttp


async def download_picture(session, url):
    filename = url[url.rfind('/') + 1:]
    async with session.get(url, ssl=False) as resp:
        if resp.status == 200:
            data = await resp.read()
            async with aiofile.async_open(f'images/beauty/{filename}', 'wb') as file:
                await file.write(data)


async def fetch_json():
    async with aiohttp.ClientSession() as session:
        for page in range(3):
            async with session.get(
                url=f'https://image.so.com/zjl?ch=beauty&sn={page * 30}',
                ssl=False
            ) as resp:
                if resp.status == 200:
                    json_str = await resp.text()
                    result = json.loads(json_str)
                    for pic_dict in result['list']:
                        await download_picture(session, pic_dict['qhimg_url'])


def main():
    if not os.path.exists('images/beauty'):
        os.makedirs('images/beauty')
    loop = asyncio.get_event_loop()
    loop.run_until_complete(fetch_json())
    loop.close()


if __name__ == '__main__':
    main()
```

Run the command shown below.

```bash
time python3 example06.py
```

The run result of the code is shown below.

```text
python3 example06.py  0.82s user 0.21s system 27% cpu 3.782 total
```

### Summary

By comparing the results of executing the three pieces of code above, we can get one conclusion. Using multithreading and asynchronous I/O can both improve the performance of crawler programs, because we do not need to waste time on waiting and blocking caused by I/O operations. And the execution result of the `time` command also tells us that the CPU utilization of the single-threaded code is only `12%`, while the CPU utilization of the multithreaded version is as high as `95%`. The single-threaded crawler takes about `21` seconds to run, while the multithreaded and asynchronous I/O versions only take about `3` seconds. Also, when the running-time difference is not large, multithreaded code uses more CPU resources than asynchronous I/O code, because scheduling and switching multithreading also costs CPU time. At this point, the strengths and weaknesses of the three ways on I/O-intensive tasks are already very clear. Of course, this is only the result run on my computer. If the network condition is not ideal, or the target website responds slowly, then the advantage of multithreading and asynchronous I/O will be even more obvious. Interested readers can try it themselves.
