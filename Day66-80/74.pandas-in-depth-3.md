### Remodelação de dados

Às vezes, os dados brutos necessários para análise não vêm de um único lugar. No capítulo anterior, lemos três tabelas de um banco de dados relacional e obtivemos três objetos `DataFrame`, mas no trabalho real, podemos precisar combiná-los. Por exemplo, `emp_df` e `emp2_df` contêm dados de funcionários e têm exatamente a mesma estrutura, portanto, podemos usar a função `concat` fornecida pelo pandas para concatenar dois ou mais objetos `DataFrame`.

```python
all_emp_df = pd.concat([emp_df, emp2_df])
```

Saída:

```
Concatenated employee table with 19 rows.
```

O código acima concatena os dois objetos `DataFrame` que representam os dados dos funcionários. A seguir, usamos a função `merge` para combinar a tabela de funcionários e a tabela de departamentos em uma tabela.

Primeiro, use `reset_index` para redefinir o índice de `all_emp_df`, de modo que `eno` não seja mais o índice, mas uma coluna normal. O parâmetro `inplace` de `reset_index` é definido como `True`, o que significa que a operação é executada diretamente em `all_emp_df`.

```python
all_emp_df.reset_index(inplace=True)
```

Agora mescle os dados com a função `merge`. Claro, também podemos chamar o método `merge` em um objeto `DataFrame` para obter o mesmo efeito.

```python
pd.merge(all_emp_df, dept_df, how='inner', on='dno')
```

Saída:

```
Merged employee and department table with matched rows from both tables.
```

O primeiro parâmetro de `merge` é a tabela da esquerda e o segundo parâmetro é a tabela da direita. Se você tem experiência em SQL, essas palavras devem parecer muito familiares. Como você deve ter adivinhado, mesclar objetos `DataFrame` é muito semelhante a junções de tabelas em um banco de dados. Portanto, o parâmetro `how` no código acima representa a forma como as duas tabelas são mescladas e possui quatro opções: `left`, `right`, `inner` e `outer`. O parâmetro `on` representa qual coluna é usada para mesclar as tabelas, que é como a condição de junção no SQL. Se os nomes das colunas correspondentes nas tabelas esquerda e direita forem diferentes, podemos usar `left_on` e `right_on` em vez de `on`.

Se fizermos uma pequena alteração e definirmos `how='right'`, o resultado será uma junção correta. Isso significa que todas as linhas da tabela correta `dept_df` serão retornadas. Como `all_emp_df` não possui nenhum funcionário no departamento `40`, os cargos correspondentes serão preenchidos com valores nulos.

A linha extra fica assim:

```
eno  ename  job  mgr  sal  comm  dno  dname       dloc
NaN  NaN    NaN  NaN  NaN  NaN    40  Operations  Shenzhen
```

### Limpeza de dados

Normalmente, os dados que obtemos do Excel, CSV ou de um banco de dados não são perfeitos. Por motivos humanos ou do sistema, valores duplicados ou discrepantes podem ser misturados. Também pode haver valores ausentes em alguns campos. Além disso, os dados em um `DataFrame` podem ter formatos inconsistentes, unidades inconsistentes e vários outros problemas. Portanto, antes de iniciar a análise de dados, a limpeza dos dados torna-se especialmente importante.

#### Valores ausentes

Podemos usar o método `isnull` ou `isna` de um objeto `DataFrame` para encontrar valores ausentes na tabela.

```python
emp_df.isnull()
```

ou

```python
emp_df.isna()
```

Da mesma forma, os métodos `notnull` e `notna` podem marcar valores não nulos como `True`. Se quisermos excluir linhas ou colunas com valores ausentes, podemos usar o método `dropna`. O parâmetro `axis` controla se é necessário excluir ao longo do eixo 0 ou do eixo 1, o que significa se as linhas ou colunas serão excluídas quando valores nulos forem encontrados. Por padrão, a exclusão ocorre ao longo do eixo 0.

```python
emp_df.dropna()
```

Saída:

```
Only rows without any null values are kept.
```

Se quisermos excluir ao longo do eixo 1, podemos usar:

```python
emp_df.dropna(axis=1)
```

Saída:

```
Only columns without any null values are kept.
```

> **Aviso**: Muitos métodos de objetos `DataFrame` possuem um parâmetro `inplace`. Seu valor padrão é `False`, o que significa que a operação não modifica o `DataFrame` original, mas retorna um novo. Se o valor for definido como `True`, a operação será realizada diretamente no `DataFrame` original e o valor de retorno se tornará `None`.

Em alguns cenários específicos, podemos preencher os valores ausentes em vez de excluí-los. O método correspondente é `fillna`. Ao preencher valores nulos, podemos usar um valor especificado através do parâmetro `value`, ou usar a célula anterior através de `method='ffill'`, ou a próxima célula através de `method='bfill'`.

```python
emp_df.fillna(value=0)
```

> **Aviso**: A escolha do valor do preenchimento também é um tema que vale a pena discutir. No trabalho real, algum valor estatístico, como a média ou a moda, pode ser usado, ou um método de interpolação, como a interpolação aleatória ou a interpolação de Lagrange, pode ser usado. É ainda possível utilizar modelos de regressão ou modelos Bayesianos para preencher dados faltantes.

Saída:

```
All null values are replaced with 0.
```

#### Valores duplicados

A seguir, vamos primeiro adicionar duas linhas à tabela de departamentos que usamos anteriormente, de modo que agora haja dois departamentos denominados `R&D Department` e dois departamentos denominados `Sales Department`.

```python
dept_df.loc[50] = {'dname': 'R&D Department', 'dloc': 'Shanghai'}
dept_df.loc[60] = {'dname': 'Sales Department', 'dloc': 'Changsha'}
dept_df
```

Saída:

```
Department table now has duplicated department names for R&D and Sales.
```

Agora há dados duplicados em nossa tabela. Podemos usar o método `duplicated` de um `DataFrame` para julgar se existem valores duplicados. Se nenhum parâmetro for fornecido, ele verifica se os índices de linha estão duplicados. Também podemos especificar que a duplicação deve ser julgada de acordo com o nome do departamento `dname`.

```python
dept_df.duplicated('dname')
```

Saída:

```
Rows 50 and 60 are marked as duplicated by department name.
```

Pelo resultado, podemos ver que os departamentos `50` e `60` são duplicados em termos de nome de departamento. Se quisermos excluir duplicatas, podemos usar o método `drop_duplicates`. O parâmetro `keep` controla se a primeira, a última ou nenhuma das linhas duplicadas será mantida.

```python
dept_df.drop_duplicates('dname')
```

Saída:

```
The first duplicate is kept.
```

Altere o valor de `keep` para `last`:

```python
dept_df.drop_duplicates('dname', keep='last')
```

Saída:

```
The last duplicate is kept.
```

Da mesma forma, podemos limpar dados duplicados em `all_emp_df`. Por exemplo, se decidirmos que as linhas com os mesmos `ename` e `job` estão duplicadas, podemos removê-las assim:

```python
all_emp_df.drop_duplicates(['ename', 'job'], inplace=True)
```

#### Valores discrepantes

O nome estatístico completo de um valor discrepante é um valor suspeito anormal, também chamado de ponto periférico. A análise de outliers também é chamada de detecção de outliers. Um outlier é um “valor extremo” que aparece em uma amostra. Parece anormalmente grande ou anormalmente pequeno e sua distribuição é obviamente diferente dos outros valores observados. No trabalho real, alguns valores discrepantes podem ser causados ​​por erros humanos ou de sistemas, mas outros não. Eles podem aparecer repetidamente e de forma constante e são, na verdade, valores extremos normais. Por exemplo, em muitos produtos de jogos, os dados dos melhores jogadores costumam ser um valor extremamente atípico. Portanto, não devemos ignorar os valores discrepantes nem simplesmente removê-los da análise. Prestar atenção aos valores discrepantes e analisar as razões por trás deles muitas vezes se torna uma oportunidade para descobrir problemas e melhorar decisões.

Os métodos comuns para detectar valores discrepantes incluem o método Z-score, o método IQR, clustering DBScan e floresta de isolamento. Aqui apresentamos os dois primeiros de forma muito simples.

<img src="res/3sigma.png" style="zoom:50%;">

Se os dados seguem uma distribuição normal, de acordo com a regra 3σ, os valores cujo desvio da média excede três desvios padrão são definidos como outliers. Se os dados não seguirem uma distribuição normal, ainda podemos descrever a distância da média em termos de quantos desvios padrão ela está, e esse múltiplo é o escore Z.

$$ z = \frac {X - \mu} {\sigma} $$

$$ \lvert z \rvert > 3 $$

O limite do escore Z deve ser decidido de acordo com a experiência e as condições reais. Normalmente, os pontos de dados superiores a 3 desvios padrão são considerados valores discrepantes. O código abaixo mostra como detectar valores discrepantes usando o método Z-score.

```python
def detect_outliers_zscore(data, threshold=3):
    avg_value = np.mean(data)
    std_value = np.std(data)
    z_score = np.abs((data - avg_value) / std_value)
    return data[z_score > threshold]
```

No método IQR, IQR significa Inter-Quartile Range, que é a diferença entre o quartil superior Q3 e o quartil inferior Q1. Normalmente, valores menores que $\small{Q1 - 1,5 \times IQR}$ ou maiores que $\small{Q3 + 1,5 \times IQR}$ são considerados outliers. Este também é o método padrão usado pelos box plots.

```python
def detect_outliers_iqr(data, whis=1.5):
    q1, q3 = np.quantile(data, [0.25, 0.75])
    iqr = q3 - q1
    lower, upper = q1 - whis * iqr, q3 + whis * iqr
    return data[(data < lower) | (data > upper)]
```

Se quisermos excluir valores discrepantes, podemos usar o método `drop` de `DataFrame` para excluir linhas ou colunas especificadas de acordo com o índice de linha ou índice de coluna. Por exemplo, se considerarmos que salários inferiores a `2000` ou superiores a `8000` são discrepantes na tabela de funcionários, podemos remover esses registros da seguinte forma:

```python
emp_df.drop(emp_df[(emp_df.sal > 8000) | (emp_df.sal < 2000)].index)
```

Saída:

```
Rows with salary lower than 2000 or higher than 8000 are removed.
```

Se quisermos substituir valores discrepantes, podemos atribuir valores diretamente às células ou usar o método `replace`. Por exemplo, se quisermos substituir os salários de `1800` e `9000` pelo salário médio, e substituir um valor de subsídio de `800` por `1000`, podemos escrever:

```python
avg_sal = np.mean(emp_df.sal).astype(int)
emp_df.replace({'sal': [1800, 9000], 'comm': 800}, {'sal': avg_sal, 'comm': 1000})
```

Saída:

```
The specified outlier salary and subsidy values are replaced.
```

#### Pré-processamento

O pré-processamento de dados também é um grande tópico. Inclui divisão, transformação, redução, discretização e outras operações. Vejamos primeiro a divisão de dados. Se uma coluna contém dados de data e hora, geralmente precisamos dividi-la em dimensões como ano, trimestre, mês, dia, dia da semana, hora e minuto. Se a data e hora for representada como uma string, podemos primeiro usar a função `to_datetime` do pandas para convertê-la em um tipo de data e hora.

No exemplo abaixo, primeiro lemos um arquivo Excel e obtemos um grupo de dados de vendas. A primeira coluna é a data de venda. Nós o dividimos em `month`, `quarter` e `weekday`.

```python
sales_df = pd.read_excel(
    'data/2020-sales-data.xlsx',
    usecols=['Sales Date', 'Sales Area', 'Sales Channel', 'Brand', 'Sales Amount']
)
sales_df.info()
```

```python
sales_df['Month'] = sales_df['Sales Date'].dt.month
sales_df['Quarter'] = sales_df['Sales Date'].dt.quarter
sales_df['Weekday'] = sales_df['Sales Date'].dt.weekday
sales_df
```

Através do atributo `dt` de um `Series` de data e hora, obtemos um objeto para acessar partes de data e hora. Então podemos usar suas propriedades como `year`, `month`, `quarter` e `hour` para obter informações de tempo. O resultado ainda é um `Series`, portanto, esse atributo `dt` também é chamado de vetor de data e hora.

Agora vamos falar sobre o processamento de dados do tipo string. Primeiro, leia os dados do trabalho de um arquivo CSV especificado.

```python
jobs_df = pd.read_csv(
    'data/job-site-data.csv',
    usecols=['city', 'companyFullName', 'positionName', 'salary']
)
jobs_df.info()
```

Veja as primeiras 5 linhas:

```python
jobs_df.head()
```

Nem todos os empregos na tabela são cargos de análise de dados. Se quisermos filtrar as posições que contêm a palavra-chave `data analysis`, podemos verificar se o campo `positionName` contém essa frase. Aqui precisamos de correspondência difusa. Como a coluna `positionName` é uma string `Series`, podemos usar o atributo `str` para obter o vetor de string correspondente e, em seguida, usar métodos de string familiares nele.

```python
jobs_df = jobs_df[jobs_df.positionName.str.contains('data analysis')]
jobs_df.shape
```

Então também precisamos processar o campo `salary`. Se quisermos contar o salário médio de todos os cargos ou de cada cidade, primeiro precisamos transformar a faixa salarial em seu ponto médio.

```python
jobs_df.salary.str.extract(r'(\d+)[kK]?-(\d+)[kK]?')
```

> **Observação**: O código acima usa grupos de captura regex para extrair dois grupos de dígitos da string, correspondentes aos limites inferior e superior do salário.

As duas colunas extraídas ainda são strings, então precisamos convertê-las para `int` antes de calcular a média. O método `applymap` de um `DataFrame` pode fazer isso, pois seu argumento é uma função que é aplicada a cada elemento do `DataFrame`. Depois disso, podemos usar o método `apply` para transformar as duas colunas salariais em valores médios.

```python
temp_df = jobs_df.salary.str.extract(r'(\d+)[kK]?-(\d+)[kK]?').applymap(int)
temp_df.apply(np.mean, axis=1)
```

Agora podemos usar o resultado acima para substituir a coluna `salary` original ou adicionar uma nova coluna representando o salário do cargo.

```python
temp_df = jobs_df.salary.str.extract(r'(\d+)[kK]?-(\d+)[kK]?').applymap(int)
jobs_df['salary'] = temp_df.apply(np.mean, axis=1)
jobs_df.head()
```

Os métodos `applymap` e `apply` são frequentemente usados ​​no pré-processamento. Um objeto `Series` também possui um método `apply`. Um `DataFrame` possui adicionalmente um método denominado `transform`, que também realiza transformações por meio de uma função passada, semelhante ao método `map` de um `Series`. Uma coisa a enfatizar é que `apply` pode ter um efeito de redução, o que significa que pode reduzir mais dados em menos dados ou em um resultado, enquanto `transform` não tem efeito de redução. Ele apenas transforma os dados e o número de linhas permanece o mesmo.

Se quisermos analisar ou extrair os dados mais profundamente, tipos não numéricos, como strings e valores de data e hora, geralmente precisam ser convertidos em valores numéricos, porque tipos não numéricos não podem ser usados ​​para calcular correlação, não podem ser usados ​​em testes $\small{\chi^{2}}$, e assim por diante. Para dados de string, geralmente podemos dividi-los nos três tipos a seguir e processá-los de acordo:

1. Variáveis ​​ordinais: os dados da string possuem uma relação de ordem, portanto podem ser transformados em números ordenados.
2. Variáveis ​​​​categóricas ou nominais: os dados da string não têm relacionamento de magnitude ou nível, portanto, podemos usar a codificação one-hot para transformá-los em uma matriz de variável fictícia.
3. Variáveis ​​de escala: a string corresponde essencialmente a dados com magnitude e suporta aritmética, portanto só precisamos convertê-la no valor numérico correspondente.

Para o primeiro e terceiro tipos, podemos usar os métodos `apply` ou `transform` mencionados acima, ou posteriormente usar ferramentas como `OrdinalEncoder` de `scikit-learn`. Para o segundo tipo, podemos usar a função `get_dummies()` do pandas para gerar uma matriz de variável fictícia.

```python
persons_df = pd.DataFrame(
    data={
        'Name': ['Guan Yu', 'Zhang Fei', 'Zhao Yun', 'Ma Chao', 'Huang Zhong'],
        'Occupation': ['Doctor', 'Doctor', 'Programmer', 'Painter', 'Teacher'],
        'Education': ['Graduate', 'College', 'Graduate', 'High School', 'Bachelor']
    }
)
persons_df
```

Transforme a ocupação em uma matriz de variáveis ​​fictícias:

```python
pd.get_dummies(persons_df['Occupation'])
```

Transforme a educação em valores com diferentes níveis:

```python
def handle_education(x):
    edu_dict = {'High School': 1, 'College': 3, 'Bachelor': 5, 'Graduate': 10}
    return edu_dict.get(x, 0)


persons_df['Education'].apply(handle_education)
```

Finalmente, vamos falar sobre discretização. A discretização também é chamada de binning. Se uma variável assume valores contínuos, ela pode ter inúmeros valores possíveis, o que torna o agrupamento inconveniente. Neste momento, discretizar a variável contínua torna-se importante. A razão pela qual é chamado de categorização é que podemos preparar algumas caixas com antecedência, e cada caixa representa um intervalo de valores, de modo que valores contínuos podem ser atribuídos a caixas diferentes.

O exemplo abaixo mostra os dados de pontos de residência de Pequim de 2018. Podemos agrupar os dados de acordo com os pontos de residência da seguinte forma.

```python
luohu_df = pd.read_csv('data/2018-beijing-residence-points-data.csv', index_col='id')
luohu_df.score.describe()
```

Podemos observar que a pontuação máxima é `122.59` e a pontuação mínima é `90.75`. Portanto, podemos construir 7 compartimentos de `90` a `125`, um compartimento para cada ponto `5`. A função `cut` em pandas pode nos ajudar a armazenar os dados.

```python
bins = np.arange(90, 126, 5)
pd.cut(luohu_df.score, bins, right=False)
```

> **Observação**: O valor padrão do parâmetro `right` em `cut` é `True`, o que significa que as caixas ficam abertas à esquerda e fechadas à direita. Alterá-lo para `False` fecha o limite esquerdo e abre o limite direito.

Podemos agrupar os dados de acordo com o resultado da categorização e então usar funções de agregação para contar cada grupo. Esta é uma operação muito comum na análise de dados. Além de `cut`, o pandas também fornece uma função `qcut`, que pode agrupar dados de acordo com quantis. Os leitores interessados ​​​​podem explorá-lo por conta própria.
