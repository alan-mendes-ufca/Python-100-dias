## Tecnologia Docker Container em detalhes

Docker é um mecanismo de contêiner de aplicativo de código aberto desenvolvido em Go e lançado sob a licença Apache 2.0. Ele permite que os desenvolvedores empacotem um aplicativo e seus pacotes de dependências em um contêiner portátil e depois o publiquem em muitas distribuições Linux.

### Uma introdução ao Docker

Uma das partes mais problemáticas do desenvolvimento de software costuma ser a configuração do ambiente. Os usuários trabalham em muitos sistemas operacionais e, mesmo que uma linguagem seja multiplataforma, isso ainda não garante que o código será executado normalmente em todos os ambientes. Os pacotes de software necessários em diferentes sistemas também podem variar muito.

Isto levanta uma questão natural: podemos empacotar o ambiente de execução junto com o software? Podemos copiar exatamente o ambiente original?

As máquinas virtuais são uma solução. Uma máquina virtual pode executar um sistema operacional dentro de outro, como Linux no Windows ou Windows no macOS. Os aplicativos dentro da máquina virtual parecem estar sendo executados em uma máquina real. Mas as máquinas virtuais consomem muitos recursos do sistema e iniciá-las ou interrompê-las geralmente é lento.

Docker é um pacote de tecnologia de contêiner Linux, LXC. Ele usa mecanismos Linux, como namespaces e cgroups, mas fornece uma interface simples e fácil de usar para uso de contêineres. O Docker empacota o aplicativo e suas dependências em um pacote. A execução desse pacote produz um contêiner virtual. O programa é executado dentro do contêiner virtual como se estivesse sendo executado em uma máquina física real.

![](./res/docker_vs_vm.png)

Hoje, o Docker é usado principalmente em vários cenários:

1. Fornecimento de ambientes descartáveis.
2. Fornecer serviços de nuvem elásticos que podem ser ampliados ou reduzidos facilmente.
3. Praticar arquitetura de microsserviços executando vários serviços isolados em contêineres.

### Instalando o Docker

Usando o CentOS como exemplo, o processo de instalação é assim. Os usuários do Ubuntu, macOS e Windows podem verificar a documentação oficial da plataforma correspondente.

1. Confirme a versão do kernel.

   ```bash
   uname -r
   ```

2. Atualize as bibliotecas do sistema.

   ```bash
   yum update
   ```

3. Remova versões antigas do Docker que já possam existir.

   ```bash
   yum list installed | grep docker
   yum erase -y docker docker-common docker-engine
   ```

4. Instale as ferramentas e dependências `yum` necessárias.

   ```bash
   yum install -y yum-utils device-mapper-persistent-data lvm2
   ```

5. Adicione o repositório Docker CE.

   ```bash
   yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
   ```

6. Instale o Docker CE e inicie-o.

   ```bash
   yum -y install docker-ce
   systemctl start docker
   ```

7. Verifique a versão e o status.

   ```shell
   docker version
   docker info
   ```

Depois disso, podemos baixar uma imagem e criar um contêiner para ver se o Docker pode funcionar normalmente.

```shell
docker pull hello-world
docker images
docker container run --name mycontainer hello-world
docker container rm mycontainer
docker rmi hello-world
```

Saída de `docker images`:

```shell
REPOSITORY               TAG        IMAGE ID            CREATED             SIZE
docker.io/hello-world    latest     fce289e99eb9        7 months ago        1.84 kB
```

Resultado da execução do contêiner:

```shell
Hello from Docker!
This message shows that your installation appears to be working correctly.
```

Para muitos usuários, extrair imagens do Docker Hub pode ser lento, por isso geralmente é uma boa ideia configurar espelhos de registro em `/etc/docker/daemon.json`.

### Usando Docker

A maneira mais fácil de aprender o Docker é criar contêineres úteis diretamente.

#### Executando Nginx

Nginx é um servidor web de alto desempenho e também um excelente proxy reverso. Com o Docker, criar um contêiner Nginx é muito fácil:

```shell
docker container run -d -p 80:80 --rm --name mynginx nginx
```

Aqui:

- `-d` executa o contêiner em segundo plano;
- `-p` mapeia uma porta de host para uma porta de contêiner;
- `--rm` remove o contêiner automaticamente após ele parar;
- `--name` atribui um nome de contêiner personalizado.

Se quiser implantar seu próprio projeto web no Nginx, você pode copiar os arquivos para o contêiner:

```shell
docker container cp /root/web/index.html mynginx:/usr/share/nginx/html
```

ou monte um diretório host como um volume:

```shell
docker container stop mynginx
docker container run -d -p 80:80 --rm --name mynginx --volume /root/docker/nginx/html:/usr/share/nginx/html nginx
```

A saída de `docker ps` é assim:

```shell
CONTAINER ID    IMAGE    COMMAND                  CREATED            STATUS             PORTS                 NAMES
3c38d2476384    nginx    "nginx -g 'daemon ..."   4 seconds ago      Up 4 seconds       0.0.0.0:80->80/tcp    mynginx
```

Você pode inspecionar contêineres em execução com:

```shell
docker ps
```

E inicie, pare ou remova-os com:

```shell
docker start mynginx
docker stop mynginx
docker rm mynginx
docker rm -f mynginx
```

#### Executando MySQL

Também podemos usar o Docker para instalar um servidor MySQL.

```shell
docker search mysql
docker pull mysql:5.7
docker images
docker run -d -p 3306:3306 --name mysql57 \
  -v /root/docker/mysql/conf:/etc/mysql/mysql.conf.d \
  -v /root/docker/mysql/data:/var/lib/mysql \
  -e MYSQL_ROOT_PASSWORD=123456 mysql:5.7
```

Exemplo de saída de `docker search mysql`:

```shell
INDEX        NAME            DESCRIPTION        STARS        OFFICIAL        AUTOMATED
docker.io    docker.io/mysql MySQL is a ...     8486         [OK]
...
```

Exemplo de saída de `docker images` após baixar o MySQL:

```shell
REPOSITORY          TAG                 IMAGE ID            CREATED             SIZE
docker.io/nginx     latest              e445ab08b2be        2 weeks ago         126 MB
docker.io/mysql     5.7                 f6509bac4980        3 weeks ago         373 MB
```

Os mapeamentos de volume são especialmente importantes aqui, porque os contêineres são descartáveis ​​enquanto os dados do banco de dados precisam ser mantidos.

Se você usar o MySQL 8.x, algumas ferramentas cliente poderão falhar com problemas de autenticação `caching_sha2_password`. Uma solução alternativa é entrar no contêiner:

```shell
docker exec -it mysql8x /bin/bash
mysql -u root -p
```

e altere o modo de autenticação:

```sql
alter user 'root'@'%' identified with mysql_native_password by '123456' password expire never;
```

#### Executando Redis

A lição também demonstra a execução de vários contêineres e a permissão para que eles se comuniquem entre si pela rede. Por exemplo, quatro contêineres Redis podem ser usados ​​para construir uma configuração de um mestre e três réplicas:

```shell
docker run -d -p 6379:6379 --name redis-master redis
docker run -d -p 6380:6379 --name redis-slave-1 --link redis-master:redis-master redis redis-server --replicaof redis-master 6379
docker run -d -p 6381:6379 --name redis-slave-2 --link redis-master:redis-master redis redis-server --replicaof redis-master 6379
docker run -d -p 6382:6379 --name redis-slave-3 --link redis-master:redis-master redis redis-server --replicaof redis-master 6379
```

A opção `--link` é usada para criar um alias de rede para que os contêineres de réplica possam chegar ao mestre sem depender de um IP do contêiner que pode mudar posteriormente.

#### Executando o GitLab

GitLab é uma ferramenta de gerenciamento de repositório Git que também oferece recursos como wiki, rastreamento de problemas e integração contínua. A lição mostra como preparar diretórios de host para volumes montados:

```shell
mkdir -p /root/gitlab/{config,logs,data}
```

e execute o contêiner GitLab CE:

```shell
docker run -d -p 80:80 -p 22:22 --name gitlab \
  -v /root/gitlab/config:/etc/gitlab \
  -v /root/gitlab/logs:/var/log/gitlab \
  -v /root/gitlab/data:/var/opt/gitlab \
  gitlab/gitlab-ce
```

O GitLab inicia lentamente, por isso pode demorar um pouco até que a interface do navegador fique disponível.

### Construindo Imagens

Até agora criamos containers a partir de imagens oficiais, mas também podemos construir nossas próprias imagens. Em termos simples, **uma imagem Docker é construída a partir de sistemas de arquivos empilhados**. A camada inferior é `bootfs`, acima é `rootfs`, e o Docker usa montagem de união para combinar camadas na imagem final.

Existem duas maneiras principais de construir imagens:

1. Use `docker commit` em um contêiner configurado. Geralmente isso não é recomendado.
2. Use `docker build` junto com um `Dockerfile`. Esta é a abordagem recomendada.

#### Construindo uma imagem com `commit`

Por exemplo, podemos iniciar um contêiner Ubuntu:

```shell
docker run --name myubuntu -it ubuntu /bin/bash
```

instale o Apache dentro dele:

```shell
apt -y upgrade
apt -y install apache2
exit
```

em seguida, salve o contêiner configurado como uma imagem:

```shell
docker container ls -a
docker commit 014bdb321612 jackfrued/mywebserver
docker images
```

#### Construindo uma imagem com `Dockerfile`

A melhor maneira é usar um `Dockerfile`. A `Dockerfile` é uma pequena linguagem específica de domínio para construção de imagens. Depois que o arquivo for gravado, `docker build` pode criar a imagem de forma repetível. A lição demonstra isso com um pequeno projeto Flask contendo:

- código do aplicativo;
- um arquivo `requirements.txt`;
- um script de shell de inicialização;
- e um `Dockerfile` de nível superior.

Essa abordagem é muito mais fácil de manter do que usar `commit`, porque todo o processo de construção é explícito e pode ser controlado por versão.

### Resumo

O Docker torna os ambientes de aplicativos portáteis e repetíveis. É útil para ambientes temporários, implantação de serviços, configuração de banco de dados e middleware e arquitetura estilo microsserviço. Mais importante ainda, dá à equipe uma maneira de reduzir o clássico problema “funciona na minha máquina”, tornando os ambientes locais, de teste e de produção muito mais parecidos.
