## Visão geral da análise de dados

No mundo de hoje, todas as indústrias dependem cada vez mais da tecnologia da informação e uma enorme quantidade de dados é produzida todos os dias. Muitas vezes sentimos que há cada vez mais dados, mas fica cada vez mais difícil encontrar informações valiosas a partir deles. As informações aqui mencionadas podem ser entendidas como o resultado do processamento de um conjunto de dados. É algo extraído do conjunto de dados que pode ser utilizado para apoiar e orientar a tomada de decisões. O processo de **extração de informações valiosas de dados brutos** é o que chamamos de **análise de dados** e é uma parte importante da ciência de dados.

<img src="res/data_science.png" style="zoom:50%;">

> **Definição 1**: Análise de dados é a ciência de coletar, processar, organizar dados propositalmente e usar estatísticas, mineração e outras técnicas para explorar, analisar, apresentar e explicar dados.
>
> **Definição 2**: A análise de dados é a atividade de coletar, organizar e analisar dados, extrair deles informações e insights valiosos e usá-los para apoiar decisões e otimizar processos. (GPT-4o)
>
> **Definição 3**: Análise de dados é o processo de coletar, organizar, processar, examinar e explicar dados sistematicamente, extraindo deles informações valiosas, tirando conclusões e apoiando decisões. Seu núcleo é usar estatísticas, algoritmos e métodos lógicos para revelar os padrões, tendências ou relacionamentos por trás dos dados. (DeepSeek)

Para as pessoas que desejam realizar trabalhos de análise de dados, há duas partes de habilidades que precisam ser dominadas. Um é “pensamento de dados” e o outro são “ferramentas de análise”, conforme mostrado na imagem abaixo.

![](res/contents_of_data_analysis.png)

Na imagem acima, a parte das ferramentas de análise é realmente mais fácil de dominar. Linguagens de programação como SQL ou Python podem ser manipuladas pela maioria das pessoas, desde que passem por um estudo sistemático e uma quantidade adequada de prática. Ferramentas de business intelligence, como Power BI e Tableau, facilitam ainda mais o início, porque podemos concluir a visualização de dados por meio de operações de arrastar e soltar e, em seguida, produzir insights de negócios com base nisso. Pelo contrário, a parte do pensamento de dados não é fácil de dominar para a maioria dos iniciantes. Por exemplo, muitas pessoas estudaram cursos como teoria das probabilidades e estatística na escola, mas quando se deparam com cenários reais de negócios, ainda acham difícil mapear esse conhecimento para o cenário empresarial e resolver problemas reais. Além disso, se não dominamos os métodos básicos de análise, não entendemos os modelos de análise comuns e não temos conhecimento de negócios relacionado, mesmo que obtenhamos muitos dados úteis, ainda podemos sentir que não sabemos por onde começar, para não mencionar a produção de insights de negócios e a descoberta de valor de negócios. Portanto, para a parte de reflexão sobre dados, além do estudo sistemático de conhecimentos e habilidades relacionados, também precisamos continuar acumulando e praticando em cenários reais de negócios.

### Responsabilidades de um analista de dados

Quando o RH publica os requisitos do trabalho, cargos como engenharia de dados, análise de dados e mineração de dados são geralmente chamados de cargos de análise de dados. Mas de acordo com as diferenças na natureza do trabalho, eles ainda podem ser divididos em **governança de dados**, que é mais orientada para a engenharia, **análise de negócios**, que é mais orientada para os negócios, **mineração de dados**, que é mais orientada para algoritmos, **desenvolvimento de dados**, que é mais orientado para aplicações, e **gerente de produtos de dados**, que é mais orientado para produtos. Quando costumamos dizer analista de dados, queremos dizer principalmente **analista de dados de negócios**. Muitos analistas de dados iniciam sua carreira nessa função, e essa função também é a que tem o maior número de vagas. Algumas empresas colocam analistas de dados de negócios em departamentos de negócios específicos, como marketing, operações ou produtos. Algumas empresas possuem departamentos de dados especiais, como uma equipe de análise de dados ou uma equipe de ciência de dados. Algumas empresas permitem que os analistas de dados atuem diretamente na tomada de decisões de alto nível, portanto, pertencem ao departamento de estratégia corporativa. Por causa disso, ao ver cargos como **operações de dados**, **analista de negócios** ou **engenheiro de BI** em sites de empregos, você não deve se surpreender. Normalmente, a descrição do cargo de analista de dados de negócios que vemos em sites de empregos é mostrada abaixo:

1. Ser responsável pela produção de relatórios relacionados.
2. Construa e otimize o sistema métrico.
3. Monitore flutuações e anomalias de dados e encontre problemas.
4. Otimize e impulsione os negócios e promova operações digitais.
5. Encontre o espaço possível para o crescimento do mercado e do produto.

Pela descrição acima, podemos perceber que, como analista de dados de negócios, nosso trabalho não é dar uma conclusão simples e superficial. É combinar os negócios da empresa e concluir trabalhos como **monitorar dados**, **encontrar anomalias**, **encontrar motivos** e **explorar tendências**. Não importa se você usa Python, Excel, Tableau, SPSS ou alguma outra ferramenta de business intelligence, a ferramenta é apenas um meio para atingir o objetivo. **Pensar em dados é a habilidade principal**, e começar com problemas reais de negócios e, finalmente, **encontrar o valor comercial nos dados** é o objetivo final. Em muitas empresas, o analista de dados é apenas uma posição básica. Um analista de dados que seja muito bom em entender o negócio pode evoluir para cargos de gerenciamento como **gerente de análise de dados** ou **diretor de operações de dados**. Para um analista de dados familiarizado com algoritmos de aprendizado de máquina, ele pode evoluir para **engenheiro de mineração de dados** ou **especialista em algoritmos**. Além de necessitarem de conhecimentos matemáticos e estatísticos correspondentes, estas posições também têm requisitos mais elevados de capacidade de programação do que os analistas de dados, e também podem exigir experiência relacionada em armazenamento e processamento de big data.

Deixe-me também falar brevemente sobre várias outras direções. Uma posição de governança de dados ajuda principalmente uma empresa a construir um data warehouse ou data lake, para que os dados possam ser movidos de sistemas de negócios, sistemas de rastreamento e sistemas de registro para o data warehouse ou data lake, fornecendo infraestrutura para posterior análise e mineração de dados. Os cargos de governança de dados têm altos requisitos para SQL e HiveSQL, precisam do uso qualificado de ferramentas ETL e também de um bom entendimento do ecossistema Hadoop. Quanto a um gerente de produto de dados, além da tradicional pilha de habilidades do gerente de produto, também é necessária uma forte habilidade técnica. Por exemplo, eles precisam compreender algoritmos de recomendação comuns e modelos de aprendizado de máquina, ser capazes de fornecer uma base para melhorar algoritmos e ser capazes de definir padrões e definições para pontos de rastreamento. Embora não precisem ser especialistas em todos os algoritmos, eles ainda precisam pensar do ponto de vista do produto sobre a implementação de modelos de dados, métricas, algoritmos e assim por diante.

### Pilha de habilidades de um analista de dados

A pilha de habilidades de um analista de dados também inclui habilidades básicas e sociais. O que se segue é apenas o meu entendimento desta posição, apenas para referência.

1. Ciência da computação, como ferramentas de análise de dados, linguagens de programação e bancos de dados
2. Matemática e estatística, como pensamento de dados e pensamento estatístico
3. Inteligência artificial, como algoritmos de aprendizado de máquina e aprendizado profundo
4. Capacidade de entender o negócio, como comunicação, expressão e experiência
5. Capacidade de resumir e expressar, como resumo, relatórios e PPT de negócios

É claro que, para um iniciante, é impossível dominar o conteúdo de toda a pilha de habilidades em pouco tempo. Mas à medida que este trabalho se aprofunda, você tocará mais ou menos em todas as coisas mencionadas acima. Todos podem aprofundar uma ou algumas dessas competências de acordo com as reais necessidades do trabalho.

### Processo Geral de Análise de Dados

Quando mencionamos análise de dados, muitas vezes podemos realmente querer dizer **análise de dados em sentido estrito**. O principal objetivo desse tipo de análise de dados é gerar relatórios visuais e descobrir problemas de negócio por meio desses relatórios. Esse tipo de trabalho geralmente está atrasado. **A análise de dados em sentido amplo** também inclui a parte de mineração de dados. Ele não apenas usa dados para monitorar e analisar o negócio, mas também usa algoritmos de aprendizado de máquina para encontrar o conhecimento oculto por trás dos dados e, em seguida, usa esse conhecimento para apoiar decisões futuras, por isso tem uma natureza voltada para o futuro.

O trabalho básico de análise de dados geralmente inclui as seguintes partes, embora haja pequenas diferenças devido aos diferentes setores e conteúdos de trabalho.

1. Determine o objetivo, insira: entenda o negócio e determine a definição das métricas.
2. Obtenha dados: data warehouse, planilhas, APIs de terceiros, rastreadores da web, conjuntos de dados abertos e assim por diante.
3. Dados limpos: lide com valores ausentes, valores duplicados, valores discrepantes e outras tarefas de pré-processamento, como formatação, discretização e binarização.
4. Dados dinâmicos: classificação, estatísticas, agregação de grupo, tabelas cruzadas, tabelas dinâmicas e assim por diante.
5. Apresentar dados, saída: visualização de dados e publicação de resultados de trabalho, como um relatório de análise de dados.
6. Analise e obtenha insights, acompanhe: explique alterações nos dados e proponha planos correspondentes.

O trabalho mais aprofundado de mineração de dados geralmente inclui as seguintes partes, embora também haja pequenas diferenças devido aos diferentes setores e conteúdos de trabalho.

1. Determine o objetivo, insira: entenda o negócio e deixe claro o objetivo da mineração.
2. Preparação de dados: coleta de dados, descrição de dados, exploração de dados, julgamento de qualidade e assim por diante.
3. Processamento de dados: extrair dados, limpar dados, transformar dados, codificação especial, redução de dimensionalidade, seleção de recursos e assim por diante.
4. Modelagem de dados: comparação de modelos, seleção de modelos e aplicação de algoritmos.
5. Avaliação de modelos: validação cruzada, ajuste de parâmetros e avaliação de resultados.
6. Implantação do modelo, saída: colocar o modelo em uso, melhorar o negócio, monitorar operações e escrever relatórios.

### Bibliotecas relacionadas à análise de dados

Usar Python para fazer trabalhos relacionados à análise de dados é uma escolha muito boa. Primeiro, Python é muito fácil de começar. Além disso, em todo o ecossistema Python, existem muitos pacotes de software maduros e bibliotecas de ferramentas para ciência de dados. Diferente de algumas outras linguagens de programação para ciência de dados, como Julia e R, Python pode ser usado não apenas para ciência de dados, mas também para muitas outras coisas.

#### As três ótimas ferramentas clássicas

1. [NumPy](https://numpy.org/): suporta operações comuns de array e matriz. Por meio da classe `ndarray`, ele agrupa arrays multidimensionais e fornece métodos e funções para operar nesses arrays. Como o NumPy possui capacidade de computação paralela integrada, ao usar uma CPU multi-core, o NumPy realiza automaticamente a computação paralela.
2. [Pandas](https://pandas.pydata.org/): o núcleo dos pandas são suas estruturas de dados especiais `DataFrame` e `Series`, e isso torna os pandas capazes de lidar com tabelas e séries temporais que contêm diferentes tipos de dados, algo que o `ndarray` do NumPy não pode fazer. Ao usar pandas, podemos carregar muitas formas de dados de maneira fácil e suave e, em seguida, realizar operações como fatiar, fragmentar, remodelar, limpar, agregar e apresentar.
3. [Matplotlib](https://matplotlib.org/): matplotlib é uma biblioteca que contém muitos módulos de plotagem e pode criar gráficos de alta qualidade de acordo com os dados que fornecemos. Além disso, matplotlib também fornece o módulo `pylab`, e este módulo contém muitos componentes de plotagem como [MATLAB](https://www.mathworks.com/products/matlab.html).

#### Outras bibliotecas relacionadas

1. [SciPy](https://scipy.org/): melhora o NumPy e envolve um grande número de algoritmos de computação científica, incluindo álgebra linear, testes estatísticos, matrizes esparsas, processamento de sinais e imagens, problemas de otimização, transformada rápida de Fourier e assim por diante.
2. [Polars](https://pola.rs/): uma biblioteca de análise de dados de alto desempenho destinada a fornecer operações de dados mais rápidas do que pandas. Ele suporta processamento de dados em grande escala e pode usar CPUs multi-core para acelerar a computação. Ele pode ser usado para substituir pandas ao lidar com conjuntos de dados em grande escala.
3. [Seaborn](https://seaborn.pydata.org/): Seaborn é uma ferramenta de visualização de gráficos baseada em matplotlib. Embora belos gráficos estatísticos possam ser feitos diretamente com o matplotlib, no geral ainda não é simples e conveniente o suficiente. Seaborn é igual a um wrapper do matplotlib, permitindo aos usuários criar todos os tipos de gráficos estatísticos atraentes de uma forma mais concisa e eficaz.
4. [Scikit-learn](https://scikit-learn.org/): originalmente parte do SciPy. Ele fornece um grande número de ferramentas que podem ser usadas em aprendizado de máquina, incluindo pré-processamento de dados, aprendizado supervisionado, classificação e regressão de significado, aprendizado não supervisionado, agrupamento de significado, seleção de modelo, validação cruzada e assim por diante.
5. [Statsmodels](https://www.statsmodels.org/stable/index.html): uma biblioteca que contém algoritmos clássicos de estatística e econometria, ajudando os usuários a concluir tarefas como exploração de dados, análise de regressão e teste de hipóteses.
6. [PySpark](https://spark.apache.org/): a versão Python do Apache Spark, um mecanismo de processamento de big data, usado para processamento de dados em larga escala e computação distribuída. Ele pode limpar, transformar e analisar dados com eficiência em um ambiente distribuído.
7. [TensorFlow](https://www.tensorflow.org/): uma estrutura de aprendizado profundo de código aberto desenvolvida pelo Google, principalmente para tarefas de aprendizado profundo, e frequentemente usada para construir e treinar modelos de aprendizado de máquina, especialmente modelos complexos de redes neurais.
8. [Keras](https://keras.io/): uma API de rede neural de alto nível usada principalmente para construir e treinar modelos de aprendizado profundo. Keras é adequado para iniciantes e pesquisadores em aprendizado profundo, pois torna a construção e o treinamento de redes neurais muito mais simples.
9. [PyTorch](https://pytorch.org/): uma estrutura de aprendizagem profunda de código aberto desenvolvida pelo Facebook e amplamente utilizada em ambientes de pesquisa e produção. PyTorch é uma estrutura popular em pesquisas de aprendizagem profunda e é amplamente utilizada em áreas como visão computacional e processamento de linguagem natural.
10. [NLTK](https://www.nltk.org/) / [SpaCy](https://spacy.io/): bibliotecas de processamento de linguagem natural.
