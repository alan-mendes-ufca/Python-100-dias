## Agrupamento K-Means

Clustering é uma técnica importante em mineração de dados e aprendizado de máquina. É usado para dividir as amostras de um conjunto de dados em vários grupos ou categorias semelhantes. Este método é amplamente utilizado em muitos campos. Por exemplo:

- **Comércio eletrônico**: de acordo com os dados de comportamento do usuário, as empresas podem dividir os consumidores em diferentes grupos, como usuários que gastam muito, usuários altamente ativos, usuários com risco de rotatividade e usuários sensíveis a preço, para que possam criar estratégias operacionais direcionadas e fazer publicidade mais precisa. As empresas representativas incluem Alibaba e Amazon.
- **Finanças**: os bancos podem usar algoritmos de agrupamento para analisar registros de crédito dos usuários, nível de renda, comportamento de gastos e outros dados, além de dividir os usuários em diferentes grupos de risco. Os grupos de alto risco necessitam de uma análise de crédito mais rigorosa, enquanto os grupos de baixo risco podem beneficiar de melhores taxas de empréstimo ou limites de crédito.
- **Saúde**: Ao analisar dados de saúde dos pacientes, como histórico médico, dados genéticos e hábitos de vida, podemos dividir os pacientes em diferentes grupos de risco à saúde, melhorar a precisão da previsão de doenças e impulsionar a medicina de precisão. As empresas representativas incluem Johnson & Johnson e Pfizer.
- **Mídia social**: Ao analisar as relações de amizade, os interesses e as interações sociais dos usuários, podemos dividir os usuários em diferentes círculos sociais e usar isso para recomendação de amigos, conteúdo personalizado e otimização do sistema de recomendação de fluxo de informações na plataforma.

Clustering é um tipo de **aprendizado não supervisionado**, porque não precisa de rótulos predefinidos. Ele aprende apenas de acordo com as características dos dados, mede a semelhança ou distância das características e, em seguida, divide o conjunto de dados conhecido em várias classes diferentes. Diferente da classificação, o objetivo do clustering é descobrir a estrutura interna dos dados. Em termos gerais, os algoritmos de clustering podem ser divididos em **clustering baseado em distância**, **clustering baseado em densidade**, **clustering hierárquico**, **clustering espectral** e assim por diante.

### Como funciona o algoritmo

Nesta lição, nos concentraremos no algoritmo de agrupamento K-Means. K-Means é um método de particionamento baseado em protótipo. Seu objetivo é dividir um conjunto de dados em clusters `K` e tornar os pontos de dados dentro de cada cluster tão semelhantes quanto possível. O algoritmo funciona assim:

1. **Inicialize os centros do cluster.** Escolha aleatoriamente amostras `K` como centros iniciais do cluster, que também são chamados de centróides.
2. **Atribua amostras ao centróide mais próximo.** Calcule a distância de cada amostra a cada centróide e atribua a amostra ao cluster mais próximo.
3. **Atualize os centróides.** Calcule o valor médio de todas as amostras em cada cluster e use-o como o novo centróide.
4. **Repita as etapas 2 e 3** até que os centróides convirjam ou o número máximo de iterações seja alcançado.

### Descrição Matemática

Podemos descrever a mesma ideia de forma mais formal. Dado um conjunto de dados, o objetivo do K-Means é minimizar a função objetivo, que é a soma total dos erros quadráticos:

$$ J = \sum_{i=1}^{K} \sum_{x \in C_{i}} {\lVert x - \mu_{i} \rVert}^{2} $$

Aqui, `K` é o número de clusters, `C_i` é o conjunto de amostras no cluster `i`-ésimo, `\mu_i` é o centro do cluster `i`-ésimo e `x` é um dado ponto. Como este é um problema de otimização combinacional NP-difícil, na prática o resolvemos com um procedimento iterativo.

Primeiro, escolhemos aleatoriamente os pontos `K` como os centróides iniciais `\mu_1, \mu_2, \cdots, \mu_K`. Para cada amostra `x_j`, calculamos sua distância a cada centróide e atribuímos ao mais próximo:

$$ C_{i} = \lbrace {x_{j} \ \vert \ {\lVert x_{j} - \mu_{i} \rVert}^{2} \le {\lVert x_{j} - \mu_{k} \rVert}^{2} \ \text{para todos} \ k \ne i} \rbrace $$

Em seguida, atualizamos cada centróide com a média de todos os pontos do cluster:

$$ \mu_{i} = \frac{1}{\lvert C_{i} \rvert}\sum_{x \in C_{i}} x $$

Repetimos essas duas ações até que os centróides parem de mudar ou a mudança se torne menor que um determinado limite. É isso que dá ao algoritmo seu comportamento de convergência.

### Uma implementação simples em Python

Abaixo, usamos o código Python para implementar o clustering K-Means. Por enquanto, não usamos o scikit-learn. O objetivo principal é ajudar todos a entender como funciona o algoritmo.

```python
import numpy as np


def distance(u, v, p=2):
    """Calculate the distance between two vectors."""
    return np.sum(np.abs(u - v) ** p) ** (1 / p)


def init_centroids(X, k):
    """Randomly choose k centroids."""
    index = np.random.choice(np.arange(len(X)), k, replace=False)
    return X[index]


def closest_centroid(sample, centroids):
    """Find the centroid closest to a sample."""
    distances = [distance(sample, centroid) for i, centroid in enumerate(centroids)]
    return np.argmin(distances)


def build_clusters(X, centroids):
    """Divide the data into clusters according to the centroids."""
    clusters = [[] for _ in range(len(centroids))]
    for i, sample in enumerate(X):
        centroid_index = closest_centroid(sample, centroids)
        clusters[centroid_index].append(i)
    return clusters


def update_centroids(X, clusters):
    """Update centroid positions."""
    return np.array([np.mean(X[cluster], axis=0) for cluster in clusters])


def make_label(X, clusters):
    """Generate labels from the clusters."""
    labels = np.zeros(len(X))
    for i, cluster in enumerate(clusters):
        for j in cluster:
            labels[j] = i
    return labels


def kmeans(X, *, k, max_iter=1000, tol=1e-4):
    """K-Means clustering."""
    # randomly choose k centroids
    centroids = init_centroids(X, k)
    for _ in range(max_iter):
        # divide the data into different clusters by the centroids
        clusters = build_clusters(X, centroids)
        # calculate the new centroid positions again
        new_centroids = update_centroids(X, clusters)
        # stop early if the centroids almost do not change
        if np.allclose(new_centroids, centroids, rtol=tol):
            break
        # record the new centroid positions
        centroids = new_centroids
    # generate labels for the data
    return make_label(X, clusters), centroids
```

Ainda podemos usar o conjunto de dados da íris como exemplo e ver se nossa própria função `kmeans` pode dividir as três espécies em três grupos diferentes. Como se trata de um aprendizado não supervisionado, alimentamos todo o conjunto de dados diretamente no `kmeans`.

```python
from sklearn.datasets import load_iris

iris = load_iris()
X, y = iris.data, iris.target
labels, centers = kmeans(X, k=3)
```

Aqui, não compare diretamente `labels` com `y`. Dissemos antes que um algoritmo de agrupamento não conhece os rótulos dos dados. Ele apenas divide os dados em diferentes classes de acordo com os recursos. Portanto, as saídas `0`, `1` e `2` não correspondem diretamente a Iris setosa, Iris versicolor e Iris virginica. Usar diretamente a classe `KMeans` no módulo `cluster` do scikit-learn para implementar o clustering K-Means é uma escolha melhor, conforme mostrado abaixo.

```python
from sklearn.cluster import KMeans

km_cluster = KMeans(
    n_clusters=3,       # value of k, number of clusters
    max_iter=30,        # maximum number of iterations
    n_init=10,          # number of tries for choosing initial centroids
    init='k-means++',   # algorithm for choosing initial centroids
    algorithm='elkan',  # whether to use triangle-inequality optimization
    tol=1e-4,           # tolerance of centroid change
    random_state=3      # random seed
)
km_cluster.fit(X)
print(km_cluster.labels_)           # cluster labels
print(km_cluster.cluster_centers_)  # centroid positions
print(km_cluster.inertia_)          # sum of squared distances from samples to centroids
```

Abaixo, explicamos vários hiperparâmetros da classe `KMeans`:

1. `n_clusters`: o número de clusters, que é o valor de `K`. O padrão é `8`.
2. `max_iter`: o número máximo de iterações. O padrão é `300`.
3. `init`: o método para inicializar centróides. O valor padrão é `'k-means++'`, o que significa escolher K centróides dos dados por seleção aleatória muitas vezes, calculando as distâncias entre os pontos centrais escolhidos a cada vez e, em seguida, tomando o grupo com a maior distância como os centróides iniciais. Recomendamos usar esse valor. Se você definir como `'random'`, ele escolherá os centróides iniciais aleatoriamente.
4. `n_init`: o número de tentativas de inicialização utilizadas em conjunto com `init`.
5. `algorithm`: a implementação do K-Means. O padrão é `'lloyd'`. Outra opção é `'elkan'`, que usa otimização de desigualdade triangular e geralmente é mais eficiente quando `K` é maior.
6. `tol`: a tolerância de convergência. Se o conjunto de dados for grande, aumentar um pouco esse valor pode acelerar a convergência.

### Resumo

K-Means é um algoritmo de agrupamento clássico. Suas vantagens incluem implementação simples e convergência rápida. Suas desvantagens são que o resultado não é estável, porque depende da configuração inicial, não pode resolver problemas de desequilíbrio de amostra, converge facilmente para um ótimo local e é fortemente afetado por dados ruidosos.
