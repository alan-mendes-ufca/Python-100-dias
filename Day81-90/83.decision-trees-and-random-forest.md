## Decision Trees and Random Forests

A **decision tree** is a supervised learning algorithm based on a tree structure. It can be used for **classification** and **regression** tasks. It reaches a prediction target by splitting a dataset step by step into different subsets until some stopping conditions are met. We also use similar methods when making decisions in life. For example, the decision method used by some girl when meeting a blind date can be drawn as the decision tree shown below.

<img src="res/03_decision_tree_in_life.png" style="zoom:38%;">

If you know some programming, you will notice that using a decision tree to make a prediction is like executing a series of `if...else...` structures. If you know probability better, then building a decision tree can also be seen as the process of calculating conditional probabilities with feature space as the condition. The nodes in a decision tree can be divided into two kinds: internal nodes and leaf nodes. Internal nodes correspond to sample feature tests, and leaf nodes represent the result of the decision, that is, a class label or regression target value. Decision trees can be used to solve classification problems and regression problems. In this lesson, we still focus on classification problems.

### Building a Decision Tree

There are three core steps in training a decision-tree model:

1. feature selection
2. tree construction
3. tree pruning

#### Feature selection

Feature selection decides which features should be used to make splits. If a feature can make the resulting child nodes belong as much as possible to the same class, that means the node has high **purity**, and that feature has strong classification ability.

In decision-tree models, three common standards are used to choose strong features:

1. information gain
2. information gain ratio
3. Gini index

To explain information gain clearly, we first need to introduce **information entropy**. In 1948, Claude Shannon proposed the idea of information entropy in his famous paper *A Mathematical Theory of Communication*. It solved the problem of measuring information and gave a way to quantify the value of information. "Entropy" was first a concept in thermodynamics, where it reflects how chaotic a system is. The greater the entropy, the more chaotic the system is. In information theory, entropy can be seen as a measure of the uncertainty of a random variable, or a dataset. The greater the entropy, the greater the uncertainty of the variable or dataset, and the more information we need to determine it. The lower the entropy, the higher the purity of the data and the lower the uncertainty.

For example, suppose person A and person B join a shooting competition. If, from past results, person A's win rate is 100%, then we can very easily accept the result that A will win. If, from past results, person A's win rate is 50%, then it is hard for us to know who will win. Claude Shannon proposed the formula below to describe this uncertainty.

The entropy formula is:

$$
H(D) = -\sum_{i = 1}^{k} P(x_i)log_2P(x_i)
$$

In the formula above, $\small{D}$ means the dataset, $\small{k}$ means the total number of classes, and $\small{P(x_i)}$ means the proportion of class $\small{i}$ samples in the dataset. If we use $\small{x_1}$ and $\small{x_2}$ to represent "A wins" and "B wins", then clearly when $\small{P(x_1)=0.5}$ and $\small{P(x_2)=0.5}$, $\small{H=1}$ and the uncertainty is the greatest. When $\small{P(x_1)=1}$ and $\small{P(x_2)=0}$, $\small{H=0}$ and the uncertainty is the smallest. When $\small{P(x_1)=0.8}$ and $\small{P(x_2)=0.2}$, $\small{H \approx 0.72}$.

Clearly, the more information we know, the smaller the uncertainty of the random variable or dataset becomes. This information can be information directly about the random event we want to study, or it can be information about other events related to the random event we care about. In mathematics, it can be strictly proved that this related information can also reduce or remove uncertainty. So we define **conditional entropy**, which means the uncertainty of dataset $\small{D}$ when feature $\small{A}$ is given. The formula of conditional entropy is shown below.

$$
H(D \vert A) = \sum_{v \in A}\frac{\lvert D_{v} \rvert}{\lvert D \rvert}H(D_{v})
$$

In the formula above, $\small{v}$ takes all possible values of feature $\small{A}$. $\small{D_v}$ means the subset of samples where feature $\small{A}$ has value $\small{v}$, and $\small{\frac{\lvert D_{v} \rvert}{\lvert D \rvert}}$ means the weight, that is, the proportion of samples where the feature value is $\small{v}$. It can be proved that $\small{H(D) \ge H(D \vert A)}$. In other words, after adding the information of feature $\small{A}$, the uncertainty of dataset $\small{D}$ becomes lower. Of course, we should also notice the case where the equal sign holds. That means we added the information of feature $\small{A}$, but the uncertainty of $\small{D}$ did not become lower. In other words, the information we got is not related to the thing we want to study.

With the explanation above, we can now define **information gain**. It is the amount by which the uncertainty of dataset $\small{D}$ becomes smaller after getting the information of feature $\small{A}$. In other words, information gain describes how much the certainty of the dataset increases. The bigger the information gain of a feature is, the stronger the classification ability of the feature is, and the greater the certainty of the dataset becomes after this feature is given. Information gain can be described directly by the formula below.

$$
g(D, A) = E(D) - E(D \vert A)
$$

The larger the information gain, the stronger the feature's classification ability.

The functions for entropy and information gain can be written as follows:

```python
import numpy as np


def entropy(y):
    """
    Calculate information entropy.
    :param y: target values of the dataset
    :return: information entropy
    """
    _, counts = np.unique(y, return_counts=True)
    prob = counts / y.size
    return -np.sum(prob * np.log2(prob))


def info_gain(x, y):
    """
    Calculate information gain.
    :param x: given feature
    :param y: target values of the dataset
    :return: information gain
    """
    values, counts = np.unique(x, return_counts=True)
    new_entropy = 0
    for i, value in enumerate(values):
        prob = counts[i] / x.size
        new_entropy += prob * entropy(y[x == value])
    return entropy(y) - new_entropy
```

The classic ID3 algorithm selects features according to information gain.

We should note that when a feature has many different values, its information gain is often larger. So when we use information gain to choose a feature, it tends to prefer features with many values. To solve this problem, we can calculate **information gain ratio**, whose definition is shown below.

$$
R(D, A) = \frac{g(D, A)}{E_{A}(D)}
$$

Here, $\small{E_{A}(D) = -\sum_{i=1}^{n}{\frac{\lvert D_{i} \rvert}{\lvert D \rvert}log_{2}\frac{\lvert D_{i} \rvert}{\lvert D \rvert}}}$, and $\small{n}$ means the number of values of feature $\small{A}$. To say it simply, $\small{E_{A}(D)}$ is the information entropy of feature $\small{A}$. Information gain ratio is the ratio between the information gain of feature $\small{A}$ and the information entropy of feature $\small{A}$. We can use the following function to calculate information gain ratio.

```python
def info_gain_ratio(x, y):
    """
    Calculate information gain ratio.
    :param x: given feature
    :param y: target values of the dataset
    :return: information gain ratio
    """
    return info_gain(x, y) / entropy(x)


print(f'R(D,A0) = {info_gain_ratio(X_train[:, 0], y_train)}')
print(f'R(D,A1) = {info_gain_ratio(X_train[:, 1], y_train)}')
print(f'R(D,A2) = {info_gain_ratio(X_train[:, 2], y_train)}')
print(f'R(D,A3) = {info_gain_ratio(X_train[:, 3], y_train)}')
```

This means that this feature can complete a good split of the data once, so it can be used as the root node when we build the decision tree. Of course, building a decision tree is not only one split. We can continue calculating the information gain ratio on the split dataset to choose a new feature for splitting, and repeat this process until certain conditions are met. Then we can build a complete decision-tree model. In classic decision-tree algorithms, the C4.5 algorithm chooses features based on information gain ratio.

Besides information gain and information gain ratio, the **Gini index** is also a very good way to choose features. It can be used to evaluate the purity of a dataset. The Gini index is also called **Gini impurity**. Its value is between 0 and 1. The higher the purity of the dataset, the closer the Gini index is to 0. The lower the purity of the dataset, the closer the Gini index is to 1. If a dataset has $\small{n}$ classes, and the probability that a sample belongs to class $\small{k}$ is $\small{p_k}$, then the Gini index of the dataset can be calculated by the formula below.

$$
G(D) = 1 - \sum_{k=1}^{n}{p_{k}}^{2}
$$

For example, in the iris dataset, the three kinds of iris flowers each have 50 samples, so the Gini index of the whole dataset is:

$$
G(D) = 1 - [(\frac{1}{3})^{2} + (\frac{1}{3})^{2} + (\frac{1}{3})^{2}] = \frac{2}{3}
$$

If the numbers of the three kinds of iris flowers are 100, 25, and 25, then the Gini index of the whole dataset is:

$$
G(D) = 1 - [(\frac{2}{3})^{2} + (\frac{1}{6})^{2} + (\frac{1}{6})^{2}] = \frac{1}{2}
$$

If the numbers of the three kinds of iris flowers are 140, 5, and 5, then the Gini index of the whole dataset is:

$$
G(D) = 1 - [(\frac{14}{15})^{2} + (\frac{1}{30})^{2} + (\frac{1}{30})^{2}] = \frac{19}{150}
$$

We can see that as the purity of the dataset becomes higher and higher, the value of the Gini index becomes smaller and smaller. If dataset $\small{D}$ is split into $\small{k}$ parts according to feature $\small{A}$, then under the condition of given feature $\small{A}$, the Gini index of the dataset can be defined as:

$$
G(D, A) = \sum_{i=1}^{k}\frac{\lvert D_{i} \rvert}{\lvert D \rvert}G(D_{i})
$$

According to the formulas above, we can design the following functions to calculate the Gini index. You can compare them with the formulas above to see whether you can understand the code, or call the functions below to see which feature of the iris dataset can make the best split of the original dataset or the training set. In classic decision-tree algorithms, the CART algorithm chooses features according to the Gini index.

```python
def gini_index(y):
    """
    Calculate Gini index.
    :param y: target values of the dataset
    :return: Gini index
    """
    _, counts = np.unique(y, return_counts=True)
    return 1 - np.sum((counts / y.size) ** 2)


def gini_with_feature(x, y):
    """
    Calculate the Gini index after giving a feature.
    :param x: given feature
    :param y: target values of the dataset
    :return: Gini index after giving the feature
    """
    values, counts = np.unique(x, return_counts=True)
    gini = 0
    for value in values:
        prob = x[x == value].size / x.size
        gini += prob * gini_index(y[x == value])
    return gini


print(f'G(D)    = {gini_index(y_train)}')
print(f'G(D,A0) = {gini_with_feature(X_train[:, 0], y_train)}')
print(f'G(D,A1) = {gini_with_feature(X_train[:, 1], y_train)}')
print(f'G(D,A2) = {gini_with_feature(X_train[:, 2], y_train)}')
print(f'G(D,A3) = {gini_with_feature(X_train[:, 3], y_train)}')
```

#### Data splitting

The process of building a decision tree is a recursive process. After we choose a feature with the methods introduced above, we need to split the data. Simply speaking, we split the dataset into two or more subsets according to that feature. Two subsets correspond to a binary tree, and more subsets correspond to a multi-way tree. Each subset corresponds to a different value of the feature. Next, we repeat feature selection and data splitting on each subset until we meet a stopping condition. Stopping conditions are very important in the recursive process, and they can also help us avoid generating a tree structure that is too complex.

Common stopping conditions include:

1. the tree reaches a preset depth
2. the number of samples at the current node is smaller than a threshold
3. all samples at the node belong to the same class
4. the change in information gain or Gini index is smaller than a threshold

> **Note**: If you have experience writing tree structures, the process above should be easy to understand. Of course, if you are not familiar with recursion, binary trees, and multi-way trees, you can also read any data-structure book. The related knowledge is not complicated, so we will not explain it in detail here.

In the process of data splitting, another problem worth noticing is how to handle continuous feature values and missing values. For continuous values, we can go through all possible values of the feature and find a split point $\small{x}$ that makes the split subsets best in terms of information gain ratio or Gini index. During data splitting, we use $\small{x}$ to split the data into two subsets, $\small{D_1}$ and $\small{D_2}$, where $\small{D_1}$ contains samples whose feature value is less than or equal to $\small{x}$, and $\small{D_2}$ contains samples whose feature value is greater than $\small{x}$. For missing values, the C4.5 algorithm uses weighted assignment. When a feature is chosen for splitting, samples with missing values in that feature are assigned to every subset, but the weight of the sample in different subsets is different. This weight is calculated according to the proportion of the feature in each class. When the CART algorithm handles missing values, it usually creates a default branch for each feature. For samples with missing values, CART sends them to the default branch. When calculating the Gini index, CART can choose whether to include the samples with missing values.

#### Tree pruning

Pruning a more complex decision tree is very necessary. Pruning can reduce the complexity of the tree structure and also avoid the risk of overfitting.

Common pruning strategies include:

1. **Post-pruning**. As the name suggests, post-pruning is done after the decision tree is built. It simplifies the tree structure by evaluating and removing some unnecessary branches. The post-pruning process usually starts from leaf nodes and evaluates each node from bottom to top to see whether replacing it with a leaf node, that is, cutting away the node and its subtree, can improve model performance on the validation set. This method reduces the risk of overfitting and also keeps the ability to fit the data relatively well, but the amount of computation is large, and if there is no suitable validation set, the pruning effect is affected.
2. **Pre-pruning**. As the name suggests, pre-pruning means dynamically deciding during the process of building the decision tree whether to stop splitting a node. By evaluating whether the current node should continue to split before splitting, we can avoid generating a tree that is too complex. We already mentioned several common stopping conditions above. The pre-pruning strategy reduces unnecessary splits in the stage of building the decision tree, so the model complexity becomes lower, but there may be a risk of underfitting because stopping too early may miss potentially important decision rules.

### Overview of Random Forest

A random forest is an ensemble-learning algorithm based on decision trees. Ensemble learning means combining the prediction results of multiple models to improve overall performance. The core idea is that a combination of models is often better than a single model. Random forest builds many decision trees and then combines their predictions through voting for classification or averaging for regression. This improves the accuracy and robustness of the model.

<img src="res/03_random_forest.png" style="zoom:45%;">

The basic workflow of random forest is as follows:

1. Bootstrap sampling: randomly draw samples from the original training set with replacement to form different subsets.
2. Build decision trees: when each tree splits a node, it does not consider all features, but randomly chooses part of the features.
3. Ensemble learning: for classification, random forest decides the final class by voting; for regression, it usually takes the average of the predictions of each tree.

The main advantages of random forest are:

1. By integrating multiple decision trees, it can effectively improve model accuracy.
2. Because randomness is introduced, random forest is usually less likely to overfit than a single decision tree.
3. Random forest can provide feature-importance scores.
4. It can handle large-scale datasets and high-dimensional data.

Of course, because it needs to build many decision trees, a random forest model is usually more complex, and the cost of computing resources, memory resources, and time is also higher.

We can use `RandomForestClassifier` from the `ensemble` module of scikit-learn to build a random-forest model. Here we still use grid-search cross-validation to tune hyperparameters.

```python
from sklearn.ensemble import RandomForestClassifier

gs = GridSearchCV(
    estimator=RandomForestClassifier(n_jobs=-1),
    param_grid={
        'n_estimators': [50, 100, 150],
        'criterion': ['gini', 'entropy'],
        'max_depth': np.arange(5, 10),
        'max_features': ['sqrt', 'log2'],
        'min_samples_leaf': np.arange(1, 11),
        'max_leaf_nodes': np.arange(5, 15)
    },
    cv=5
)
gs.fit(X_train, y_train)
```

> **Tip**: The code above may run for a very long time.

Some important hyperparameters of random forest are:

1. `n_estimators`: the number of trees in the forest.
2. `bootstrap`: whether to use Bootstrap sampling. Its default value is `True`.
3. `n_jobs`: the number of jobs used for parallel running. Setting it to `-1` means using all available processor cores.

### Summary

Decision trees are simple, effective, and easy-to-understand prediction models. They are suitable for classification and regression tasks, but they are easy to overfit and sensitive to noisy data. Random forest improves generalization ability by integrating multiple decision trees and is less sensitive to noisy data, so it is suitable for more complex problems. In actual work, we can choose the proper model according to the specific scenario and tune the related hyperparameters to get the best effect. The comparison is shown below.

| **Property**         | **Decision Tree**          | **Random Forest**     |
| -------------------- | -------------------------- | --------------------- |
| **Model complexity** | Simple                     | More complex          |
| **Resistance to overfitting** | Weak            | Strong                |
| **Computing efficiency** | High                  | Lower                 |
| **Result stability** | Easily affected by single-data changes | Stable |
| **Suitable scenario** | Small data, simple problems | More data, complex problems |
