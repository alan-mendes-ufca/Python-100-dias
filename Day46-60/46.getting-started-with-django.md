## Introdução ao Django

No estágio inicial do desenvolvimento web, os desenvolvedores precisavam escrever cada página manualmente. Por exemplo, para um site de portal de notícias, suas páginas HTML precisavam ser alteradas todos os dias. À medida que a escala e o tamanho do site cresciam, esse caminho era definitivamente muito ruim. Para resolver esse problema, os desenvolvedores pensaram em utilizar programas para gerar conteúdo dinâmico para servidores Web. Em outras palavras, o conteúdo dinâmico nas páginas web não era mais escrito à mão, mas gerado automaticamente por programas.

No início, essa tecnologia era chamada de CGI. É claro que, com o passar do tempo, o CGI mostrou cada vez mais problemas, como muitos códigos clichê repetidos e desempenho geralmente baixo. Então PHP, ASP e JSP apareceram um após o outro no final da década de 1990. Normalmente, quando dizemos uma aplicação Web, queremos dizer uma aplicação que acessa recursos da rede através de um navegador. Como os navegadores são comuns e fáceis de usar, os aplicativos da Web são convenientes e simples de usar e evitam o problema de instalação e atualização de aplicativos. Do ponto de vista do desenvolvedor, eles também não precisam se preocupar com qual sistema operacional o usuário usa, e nem mesmo precisam distinguir entre PC e celular.

#### Protocolo HTTP

Aqui primeiro passamos algum tempo falando sobre o protocolo HTTP. HTTP é um protocolo de nível de aplicativo construído sobre TCP. Ele usa o serviço de transmissão confiável fornecido pelo TCP para trocar dados em aplicações Web. Segundo a Wikipedia, o HTTP foi originalmente projetado para fornecer uma forma de publicar e receber páginas HTML, ou seja, este protocolo é o portador dos dados enviados entre o navegador e o servidor Web.

Uma solicitação HTTP normalmente contém:

- linha de solicitação
- cabeçalhos
- linha em branco
- corpo opcional

Uma resposta HTTP normalmente contém:

- linha de status
- cabeçalhos
- linha em branco
- corpo

### Visão geral do Django

Python possui muitos frameworks Web, mais do que suas palavras-chave. A chamada estrutura da Web é uma infraestrutura usada para desenvolver aplicativos da Web do lado do servidor. Para simplificar, é uma série de módulos e ferramentas empacotados. Na verdade, mesmo sem uma estrutura Web, ainda podemos desenvolver aplicações Web do lado do servidor através de soquete ou CGI, mas em projetos comerciais o custo e o preço de fazer isso são geralmente inaceitáveis. Através de um framework Web, podemos transformar complexidade em simplicidade e reduzir o trabalho de criação, atualização e expansão de aplicações. As estruturas incluem Django, Flask, Tornado, Sanic, Pyramid, Bottle, Web2py, web.py e assim por diante.

Django é sem dúvida o peso pesado mais representativo entre os frameworks Python Web acima. Os desenvolvedores podem desenvolver rapidamente aplicativos Web confiáveis baseados em Django, porque ele reduz custos desnecessários no desenvolvimento Web, empacota padrões comuns de design e desenvolvimento e suporta a arquitetura MVC, que Django chama de arquitetura MTV.

Django usa o padrão MTV:

- `M`: Modelo, representando dados
- `T`: Template, responsável pela apresentação
- `V`: View, a lógica que conecta templates e modelos

MVC é uma arquitetura que funciona em qualquer lugar no desenvolvimento de sistemas de software. Ele divide os componentes de um sistema em Modelo, Visualização e Controlador e, dessa forma, separa modelo (dados) e visualização (exibição). Como o modelo e a visualização são separados, é necessário um intermediário para conectá-los, e essa função é desempenhada pelo controlador. Qualquer sistema de software com pouca escala usa arquitetura MVC, ou alguma outra arquitetura desenvolvida a partir do MVC. No Django, chamamos isso de MTV. O `M` no MTV é igual ao `M` no MVC e representa o modelo de dados. `T` significa modelo da web, a visualização que exibe dados. `V` significa a função de visualização. No Django, a função de visualização e a própria estrutura do Django desempenham juntas o papel de `C` no MVC.

![](./res/mvc.png)

Django nasceu em 2003. Ele nasceu de um projeto real e foi desenvolvido pela equipe CMS de um site de notícias online do Lawrence Publishing Group. Foi nomeado em homenagem ao guitarrista de jazz cigano belga Django Reinhardt. Django foi lançado como uma estrutura de código aberto no verão de 2005. Usando Django, podemos construir sites totalmente funcionais em um curto espaço de tempo, porque ele executa essas tarefas repetitivas e chatas em vez do programador, deixando o negócio principal verdadeiramente significativo para o programador. Esta é a melhor prática da ideia DRY.

### Seu primeiro projeto Django

1. Verifique sua versão do Python.

Diferentes versões do Django requerem diferentes versões do Python. Por exemplo:

- Django 1.11 requer Python 2.7 ou Python 3.4+
- Django 2.2 requer Python 3.5+
- Django 3.0 requer Python 3.6+

Verifique seu intérprete:

```bash
python3 --version
```

Você também pode inspecioná-lo no próprio Python:

```python
import sys
sys.version
sys.version_info
```

2. Instale o Django.

```bash
pip3 install -U pip
pip3 install django==2.2.13
```

3. Crie um projeto:

```bash
django-admin --version
django-admin startproject hellodjango
```

4. Abra o projeto no PyCharm ou outro IDE e crie um ambiente virtual se necessário.

Arquivos de projeto importantes:

- `hellodjango/__init__.py`: marca o diretório como um pacote Python
- `hellodjango/settings.py`: configuração do projeto
- `hellodjango/urls.py`: roteamento de URL
- `hellodjango/wsgi.py`: ponto de entrada WSGI
- `manage.py`: script de gerenciamento

5. Instale as dependências do projeto dentro do ambiente virtual, se necessário:

```bash
pip install django==2.2.13
```

6. Execute o servidor de desenvolvimento:

```bash
python manage.py runserver
```

Visite `http://127.0.0.1:8000` em um navegador.

Notas:

- O servidor integrado do Django serve apenas para desenvolvimento e teste.
- As alterações de código geralmente são recarregadas automaticamente.
- Você pode inspecionar os comandos de gerenciamento disponíveis com `python manage.py help`.
- O servidor padrão é executado na porta `8000`, mas você pode especificar outro IP ou porta.

### Configurar idioma e fuso horário

Django suporta internacionalização e localização. Em `settings.py`, altere:

```python
LANGUAGE_CODE = 'en-us'
TIME_ZONE = 'UTC'
```

para, por exemplo:

```python
LANGUAGE_CODE = 'pt-br'
TIME_ZONE = 'America/Sao_Paulo'
```

Atualizar o navegador mostrará a página de boas-vindas localizada do Django. Ajuste `LANGUAGE_CODE` e `TIME_ZONE` de acordo com o idioma e a região do seu projeto.

### Crie seu próprio aplicativo

Se você deseja construir sua própria aplicação web, primeiro você precisa criar um “app” dentro do projeto Django. Um projeto Django pode conter um ou mais aplicativos.

1. Execute o comando abaixo no terminal PyCharm para criar um aplicativo chamado `first`.

```bash
python manage.py startapp first
```

A execução desse comando cria um diretório `first` com a seguinte estrutura:

- `__init__.py`: um arquivo vazio que informa ao Python que este diretório deve ser tratado como um pacote
- `admin.py`: usado para registrar modelos para que possam ser gerenciados no site de administração integrado do Django
- `apps.py`: o arquivo de configuração do aplicativo
- `migrations`: armazena informações de migração relacionadas a modelos
- `models.py`: armazena os modelos de dados do aplicativo
- `tests.py`: contém testes para o aplicativo
- `views.py`: funções ou classes que tratam de solicitações HTTP e retornam respostas HTTP

2. Modifique o arquivo `views.py` do aplicativo.

```python
from django.http import HttpResponse


def show_index(request):
    return HttpResponse('<h1>Hello, Django!</h1>')
```

3. Modifique o arquivo `urls.py` do projeto Django para que a função de visualização seja mapeada para um caminho do navegador.

```python
from django.contrib import admin
from django.urls import path, include

from first.views import show_index

urlpatterns = [
    path('admin/', admin.site.urls),
    path('hello/', show_index),
]
```

4. Execute o projeto novamente e abra `http://127.0.0.1:8000/hello/` em seu navegador.

5. O conteúdo acima ainda é estático, embora seja gerado pelo código Python. Se quiser gerar conteúdo dinâmico, você pode alterar `views.py` conforme mostrado abaixo.

```python
from random import sample

from django.http import HttpResponse


def show_index(request):
    fruits = [
        'Apple', 'Orange', 'Pitaya', 'Durian', 'Waxberry', 'Blueberry',
        'Grape', 'Peach', 'Pear', 'Banana', 'Watermelon', 'Mango'
    ]
    selected_fruits = sample(fruits, 3)
    content = '<h3>Today\'s recommended fruits are:</h3>'
    content += '<hr>'
    content += '<ul>'
    for fruit in selected_fruits:
        content += f'<li>{fruit}</li>'
    content += '</ul>'
    return HttpResponse(content)
```

6. Atualize a página e observe o resultado. Cada atualização deve mostrar um conteúdo diferente.

### Usando modelos

Gerar HTML por concatenação de strings, conforme mostrado acima, não é aceitável no desenvolvimento real. As páginas front-end em projetos reais podem ser muito mais complexas e não há uma maneira prática de construí-las juntando manualmente strings. Para resolver este problema, preparamos uma página de modelo com antecedência. Em termos da MTV, este é o `T`. Uma página de modelo é uma página HTML com espaços reservados e instruções de modelo.

Django fornece uma função conveniente chamada `render` para renderizar modelos. Renderizar significa substituir espaços reservados e instruções de modelo por dados reais. Esta é a renderização do lado do servidor, o que significa que o servidor renderiza a página primeiro e depois a envia para o navegador. Em aplicativos com tráfego intenso, a renderização do lado do servidor pode sobrecarregar o servidor, e é por isso que muitos aplicativos da Web modernos preferem a renderização de front-end. Nesse modelo, o servidor fornece apenas os dados de que a página precisa, geralmente no formato JSON, e o JavaScript no navegador renderiza a página. Falaremos sobre renderização de frontend mais tarde. Por enquanto, usamos a abordagem de renderização do lado do servidor baseada em modelos.

As etapas para usar uma página de modelo são mostradas abaixo.

1. Crie uma pasta chamada `templates` no diretório do projeto.

![](res/pycharm-django-template.png)

2. Adicione a página de modelo `index.html`.

   > **Observação**: No desenvolvimento real do projeto, as páginas estáticas são fornecidas pelos desenvolvedores front-end. Os desenvolvedores de back-end precisam modificar as páginas estáticas em páginas de modelo, para que possam ser renderizadas por meio de programas Python. Esta é a renderização do lado do servidor mencionada acima.

   ```html
   <!DOCTYPE html>
   <html lang="en">
       <head>
           <meta charset="UTF-8">
           <title>Home</title>
           <style>
               #fruits {
                   font-size: 1.25em;
               }
           </style>
       </head>
       <body>
           <h1>Today's recommended fruits are:</h1>
           <hr>
           <ul id="fruits">
               {% for fruit in fruits %}
               <li>{{ fruit }}</li>
               {% endfor %}
           </ul>
       </body>
   </html>
   ```

Na página de modelo acima, usamos sintaxe de espaço reservado de modelo como `{{ fruit }}` e também usamos instruções de modelo como `{% for %}`. Tudo isso faz parte da Django Template Language, DTL. Sobre sintaxe e instruções do modelo, todos podem consultar a documentação oficial. Esses conteúdos ainda são fáceis de entender e não precisam de muitas explicações.

3. Modifique o arquivo `views.py` e chame a função `render` para renderizar a página do modelo.

   ```python
   from random import sample
   
   from django.shortcuts import render
   
   
   def show_index(request):
       fruits = [
           'Apple', 'Orange', 'Pitaya', 'Durian', 'Waxberry', 'Blueberry',
           'Grape', 'Peach', 'Pear', 'Banana', 'Watermelon', 'Mango'
       ]
       selected_fruits = sample(fruits, 3)
       return render(request, 'index.html', {'fruits': selected_fruits})
   ```

O primeiro parâmetro da função `render` é o objeto de solicitação `request`, o segundo parâmetro é o nome da página do modelo que queremos renderizar e o terceiro parâmetro são os dados a serem renderizados na página. Fornecemos os dados para a página do modelo por meio de um dicionário. As chaves no dicionário são os nomes das variáveis usadas nas instruções do modelo ou espaços reservados na página do modelo.

4. Até este ponto, `render` na função de visualização ainda não consegue encontrar o arquivo de modelo `index.html`. Precisamos modificar o arquivo `settings.py` e configurar o caminho onde o arquivo de modelo está localizado. Modifique o arquivo `settings.py`, encontre a configuração `TEMPLATES` e altere a configuração `DIRS` nele.

   ```python
   TEMPLATES = [
       {
           'BACKEND': 'django.template.backends.django.DjangoTemplates',
           'DIRS': [os.path.join(BASE_DIR, 'templates'), ],
           'APP_DIRS': True,
           'OPTIONS': {
               'context_processors': [
                   'django.template.context_processors.debug',
                   'django.template.context_processors.request',
                   'django.contrib.auth.context_processors.auth',
                   'django.contrib.messages.context_processors.messages',
               ],
           },
       },
   ]
   ```

5. Execute o projeto novamente ou atualize a página diretamente para visualizar o resultado.
