## Envio de e-mail e SMS com Python

Nas lições anteriores, já mostramos a todos como usar programas Python para gerar documentos Excel, Word e PDF automaticamente. Em seguida, podemos dar um passo adiante, enviando por e-mail os documentos gerados para destinatários específicos e, em seguida, usando SMS para notificá-los de que o e-mail foi enviado. Essas tarefas também podem ser realizadas facilmente com Python.

### Enviando e-mail

Mesmo numa época em que o software de mensagens instantâneas está tão difundido, o e-mail ainda é um dos aplicativos mais utilizados na Internet. Ofertas de emprego da empresa aos candidatos, links de ativação de contas de sites e mensagens promocionais de bancos são comumente entregues por e-mail, e tarefas como essas devem ser concluídas automaticamente pelos programas.

Podemos usar HTTP, o protocolo de transferência de hipertexto, para acessar os recursos do site. HTTP é um protocolo de camada de aplicação construído sobre TCP, o Transmission Control Protocol, que fornece serviços confiáveis ​​de transferência de dados para muitos protocolos de camada de aplicação. Para enviar e-mail, precisamos de SMTP, o Simple Mail Transfer Protocol, que também é um protocolo de camada de aplicação construído sobre TCP. Ele define como um remetente de email se comunica com um servidor de email. Python simplifica essas operações por meio do módulo `smtplib`. Usar um objeto `SMTP_SSL` desse módulo, juntamente com seus métodos `login` e `sendmail`, é suficiente para enviar mensagens.

Vamos primeiro tentar enviar um e-mail muito simples. Este e-mail não contém anexos, imagens ou outro conteúdo de hipertexto. Para enviar e-mail, primeiro precisamos nos conectar a um servidor de e-mail. Podemos configurar um servidor de e-mail sozinhos. Essa coisa não é muito amigável para iniciantes, mas podemos optar por usar um serviço de correio terceirizado. Por exemplo, se já registramos uma conta no `126.com`, após fazer login com sucesso, podemos ativar o serviço SMTP nas configurações. Então é igual a obter um servidor de email. Ao digitalizar o código QR acima com um telemóvel, podemos obter um código de autorização enviando um SMS. Depois que o SMS for enviado com sucesso, clique em "Enviei" e você poderá obter o código de autorização. O código de autorização deve ser guardado com cuidado, pois uma vez vazado, outras pessoas podem se passar por você para enviar e-mail. A seguir, podemos escrever o código para envio de email, conforme mostrado abaixo.

```python
import smtplib
from email.header import Header
from email.mime.multipart import MIMEMultipart
from email.mime.text import MIMEText

# Create the email object
email = MIMEMultipart()
# Set the sender, recipients, and subject
email['From'] = 'xxxxxxxxx@126.com'
email['To'] = 'yyyyyy@qq.com;zzzzzz@1000phone.com'
email['Subject'] = Header('Report on work in the first half of the year', 'utf-8')
# Add the email body
content = """According to German media reports, members of the German Union of Train Drivers voted on the 9th local time.
A nationwide strike is scheduled to start on the 10th local time, and a strike in freight transportation has started at 19:00 local time on the 10th.
Thereafter, there will be a 48-hour strike across Germany's passenger and rail infrastructure from 2 a.m. on the 11th to 2 a.m. on the 13th."""
email.attach(MIMEText(content, 'plain', 'utf-8'))

# Create an SMTP_SSL object and connect to the mail server
smtp_obj = smtplib.SMTP_SSL('smtp.126.com', 465)
# Log in with the username and authorization code
smtp_obj.login('xxxxxxxxx@126.com', 'mail-server-authorization-code')
# Send the email
smtp_obj.sendmail(
    'xxxxxxxxx@126.com',
    ['yyyyyy@qq.com', 'zzzzzz@1000phone.com'],
    email.as_string()
)
```

Se quisermos enviar e-mail com anexos, precisamos apenas processar o conteúdo do anexo na codificação Base64 e quase não será diferente do conteúdo de texto normal. Base64 é um método de representação baseado em 64 caracteres imprimíveis para representar dados binários. É frequentemente usado em situações em que dados binários precisam ser representados, transmitidos e armazenados, e o email é um deles. Se você não entende essa forma de codificação, recomendo a leitura de [Notas sobre Base64](http://www.ruanyifeng.com/blog/2008/06/base64.html). No conteúdo anterior, também mencionamos que o módulo `base64` na biblioteca padrão Python fornece suporte para codificação e decodificação Base64.

O código abaixo demonstra como enviar um email com anexo.

```python
import smtplib
from email.header import Header
from email.mime.multipart import MIMEMultipart
from email.mime.text import MIMEText
from urllib.parse import quote

# Create the email object
email = MIMEMultipart()
# Set the sender, recipient, and subject
email['From'] = 'xxxxxxxxx@126.com'
email['To'] = 'zzzzzzzz@1000phone.com'
email['Subject'] = Header('Please check your resignation documents', 'utf-8')
# Add HTML content as the email body
content = """<p>Dear Former Colleague,</p>
<p>The proof of separation you need is attached, please check it!</p>
<br>
<p>Best regards,</p>
<hr>
<p>Sun Meili Same Day</p>"""
email.attach(MIMEText(content, 'html', 'utf-8'))
# Read the file that will be sent as an attachment
with open('resources/Wang Dahuang Proof of Separation.docx', 'rb') as file:
    attachment = MIMEText(file.read(), 'base64', 'utf-8')
    # Set the content type
    attachment['content-type'] = 'application/octet-stream'
    # Percent-encode the filename
    filename = quote('Wang Dahuang Proof of Separation.docx')
    # Specify how the content should be handled
    attachment['content-disposition'] = f'attachment; filename="{filename}"'
    email.attach(attachment)

# Create an SMTP_SSL object and connect to the mail server
smtp_obj = smtplib.SMTP_SSL('smtp.126.com', 465)
# Log in with the username and authorization code
smtp_obj.login('xxxxxxxxx@126.com', 'mail-server-authorization-code')
# Send the email
smtp_obj.sendmail(
    'xxxxxxxxx@126.com',
    'zzzzzzzz@1000phone.com',
    email.as_string()
)
```

Para tornar conveniente para todos usarem Python para enviar e-mail, envolvi o código acima em uma função. Ao usá-lo, todos precisam apenas ajustar o nome de domínio, porta, nome de usuário e código de autorização do servidor de e-mail.

```python
import smtplib
from email.header import Header
from email.mime.multipart import MIMEMultipart
from email.mime.text import MIMEText
from urllib.parse import quote

# Mail server domain
EMAIL_HOST = 'smtp.126.com'
# Mail server port, usually 465
EMAIL_PORT = 465
# Account used to log in to the mail server
EMAIL_USER = 'xxxxxxxxx@126.com'
# SMTP authorization code
EMAIL_AUTH = 'Authorization code for the mail server'


def send_email(*, from_user, to_users, subject='', content='', filenames=[]):
    """Send an email

    :param from_user: sender
    :param to_users: recipients, separated by semicolons
    :param subject: email subject
    :param content: email body
    :param filenames: file paths for attachments
    """
    email = MIMEMultipart()
    email['From'] = from_user
    email['To'] = to_users
    email['Subject'] = subject

    message = MIMEText(content, 'plain', 'utf-8')
    email.attach(message)
    for filename in filenames:
        with open(filename, 'rb') as file:
            pos = filename.rfind('/')
            display_filename = filename[pos + 1:] if pos >= 0 else filename
            display_filename = quote(display_filename)
            attachment = MIMEText(file.read(), 'base64', 'utf-8')
            attachment['content-type'] = 'application/octet-stream'
            attachment['content-disposition'] = f'attachment; filename="{display_filename}"'
            email.attach(attachment)

    smtp = smtplib.SMTP_SSL(EMAIL_HOST, EMAIL_PORT)
    smtp.login(EMAIL_USER, EMAIL_AUTH)
    smtp.sendmail(from_user, to_users.split(';'), email.as_string())
```

### Enviando SMS

O envio de SMS também é um recurso comum em projetos. Códigos de registro de sites, códigos de verificação e mensagens de marketing geralmente são entregues por SMS. O envio de SMS requer suporte de uma plataforma de terceiros. Aqui usamos [Plataforma Luosimao](https://luosimao.com/) como exemplo para apresentar como enviar SMS com um programa Python. Não descreveremos os detalhes de registro de uma conta e compra de serviço de SMS aqui. Você pode solicitar o atendimento ao cliente da plataforma.

A seguir, podemos enviar uma solicitação HTTP ao gateway SMS fornecido pela plataforma através da biblioteca `requests`. Passando como parâmetros o número do celular do destinatário e o conteúdo da mensagem, podemos enviar a mensagem. O código é mostrado abaixo.

```python
import random

import requests


def send_message_by_luosimao(tel, message):
    """Send SMS through the Luosimao SMS gateway"""
    resp = requests.post(
        url='http://sms-api.luosimao.com/v1/send.json',
        auth=('api', 'key-your-platform-key'),
        data={
            'mobile': tel,
            'message': message
        },
        timeout=10,
        verify=False
    )
    return resp.json()


def gen_mobile_code(length=6):
    """Generate a mobile verification code of the given length"""
    return ''.join(random.choices('0123456789', k=length))


def main():
    code = gen_mobile_code()
    message = f'Your SMS verification code is{code}，You can't tell anyone even if you kill them! [Python Lesson]'
    print(send_message_by_luosimao('13500112233', message))


if __name__ == '__main__':
    main()
```

A solicitação ao gateway SMS de Luosimao, `http://sms-api.luosimao.com/v1/send.json`, retorna dados no formato JSON. Se retornar `{'error': 0, 'msg': 'OK'}`, significa que o SMS foi enviado com sucesso. Se o valor de `error` não for `0`, você pode verificar a [documentação oficial de desenvolvimento](https://luosimao.com/docs/api/) para saber qual parte apresentou problema.
