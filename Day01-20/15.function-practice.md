## Exemplos práticos de funções

### Exemplo 1: Código de verificação aleatório

Projete uma função para gerar um código de verificação aleatório. O código de verificação é composto por dígitos e letras inglesas maiúsculas e minúsculas, e o comprimento pode ser definido por meio de um parâmetro.

```python
import random
import string

ALL_CHARS = string.digits + string.ascii_letters


def generate_code(*, code_len=4):
    """
    Generate a verification code of the specified length.
    :param code_len: The length of the verification code (default is 4 characters)
    :return: A random verification code string made up of uppercase and lowercase English letters and digits
    """
    return ''.join(random.choices(ALL_CHARS, k=code_len))
```

> **Nota 1**: `string.digits` no módulo `string` significa a string `'0123456789'`, e `string.ascii_letters` no módulo `string` significa a string `'abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ'`.
>
> **Nota 2**: As funções `sample` e `choices` no módulo `random` podem fazer amostragem aleatória. `sample` faz amostragem sem reposição, o que significa que os elementos amostrados não são repetidos. `choices` faz amostragem com reposição, o que significa que alguns elementos podem ser escolhidos novamente. O primeiro parâmetro dessas duas funções é toda a população da qual amostrar, e o parâmetro `k` significa o tamanho da amostra. Deve-se observar que o parâmetro `k` da função `choices` é um parâmetro somente de palavra-chave, portanto você deve fornecer o nome do parâmetro ao passá-lo.

Podemos usar o código abaixo para gerar 5 códigos de verificação aleatórios para testar a função acima.

```python
for _ in range(5):
    print(generate_code())
```

Ou:

```python
for _ in range(5):
    print(generate_code(code_len=6))
```

> **Observação**: O parâmetro da função `generate_code` que projetamos é um parâmetro somente de palavra-chave. Por ter um valor padrão, podemos optar por não passar um valor para ele e usar o valor padrão `4`. Se quisermos passar um valor para a função, devemos usar o nome do parâmetro `code_len`.

### Exemplo 2: Verifique se há números primos

Projete uma função que verifique se um determinado número inteiro positivo maior que 1 é um número primo. Um número primo é um número inteiro positivo maior que 1 que só pode ser dividido exatamente por 1 e por ele mesmo. Se um número inteiro positivo $\small{N}$ maior que 1 for primo, isso significa que não há fatores dele entre `2` e $\small{N-1}$.

```python
def is_prime(num: int) -> bool:
    """
    Determine whether a positive integer is prime.
    :param num: A positive integer greater than 1
    :return: True if num is prime, otherwise False
    """
    for i in range(2, int(num ** 0.5) + 1):
        if num % i == 0:
            return False
    return True
```

> **Nota 1**: O `: int` após o parâmetro `num` acima é usado para marcar o tipo de parâmetro. Embora não afete o resultado da execução do código, melhora também a legibilidade do código. Da mesma forma, `-> bool` após a lista de parâmetros é usado para marcar o tipo de retorno da função. Também não afeta o resultado da execução do código, mas nos diz claramente que chamar esta função retorna um valor booleano, seja `True` ou `False`.
>
> **Nota 2**: O laço acima não precisa ir de `2` até $\small{N-1}$, porque se no momento em que o laço atingir $\small{\sqrt{N}}$ ainda não tivermos encontrado um fator de $\small{N}$, então nenhum fator de $\small{N}$ aparecerá depois de $\small{\sqrt{N}}$. Você pode pensar por si mesmo sobre o porquê disso.

### Exemplo 3: Máximo Divisor Comum e Mínimo Múltiplo Comum

Projete funções para calcular o máximo divisor comum e o mínimo múltiplo comum de dois inteiros positivos. O máximo divisor comum de $\small{x}$ e $\small{y}$ é o maior número inteiro que pode dividir $\small{x}$ e $\small{y}$ exatamente. Se $\small{x}$ e $\small{y}$ são coprimos, então seu máximo divisor comum é 1. O mínimo múltiplo comum de $\small{x}$ e $\small{y}$ é o menor inteiro positivo que pode ser dividido exatamente por $\small{x}$ e $\small{y}$. Se $\small{x}$ e $\small{y}$ são coprimos, então seu mínimo múltiplo comum é $\small{x \times y}$. Deve-se notar que calcular o máximo divisor comum e calcular o mínimo múltiplo comum são duas funções diferentes, portanto devemos projetá-las como duas funções, em vez de colocar ambas as funções em uma função.

```python
def lcm(x: int, y: int) -> int:
    """Compute the least common multiple."""
    return x * y // gcd(x, y)


def gcd(x: int, y: int) -> int:
    """Compute the greatest common divisor."""
    while y % x != 0:
        x, y = y % x, x
    return x
```

> **Nota**: As funções podem chamar umas às outras. A função `lcm` acima chama a função `gcd` e usa $\frac{x \times y}{gcd(x, y)}$ para calcular o mínimo múltiplo comum.

### Exemplo 4: Estatísticas de Dados

Suponha que os dados de amostra sejam salvos em uma lista. Funções de design para calcular as estatísticas descritivas dos dados amostrais. As estatísticas descritivas geralmente incluem média aritmética, mediana, intervalo, variância, desvio padrão, coeficiente de variação e assim por diante. As fórmulas são mostradas abaixo.

```python
def ptp(data):
    """Range."""
    return max(data) - min(data)


def mean(data):
    """Arithmetic mean."""
    return sum(data) / len(data)


def median(data):
    """Median."""
    temp, size = sorted(data), len(data)
    if size % 2 != 0:
        return temp[size // 2]
    else:
        return mean(temp[size // 2 - 1:size // 2 + 1])


def var(data, ddof=1):
    """Variance."""
    x_bar = mean(data)
    temp = [(num - x_bar) ** 2 for num in data]
    return sum(temp) / (len(temp) - ddof)


def std(data, ddof=1):
    """Standard deviation."""
    return var(data, ddof) ** 0.5


def cv(data, ddof=1):
    """Coefficient of variation."""
    return std(data, ddof) / mean(data)


def describe(data):
    """Print descriptive statistics."""
    print(f'Mean: {mean(data)}')
    print(f'Median: {median(data)}')
    print(f'Very Poor: {ptp(data)}')
    print(f'Variance: {var(data)}')
    print(f'Standard deviation: {std(data)}')
    print(f'Coefficient of Variation: {cv(data)}')
```

> **Nota 1**: A mediana é o número intermediário após a classificação dos dados em ordem crescente ou decrescente e descreve o nível intermediário dos dados. Existem dois casos no cálculo da mediana: quando a quantidade de dados $\small{n}$ é ímpar, a mediana é o elemento na posição $\frac{n + 1}{2}$; quando a quantidade de dados $\small{n}$ é par, a mediana é a média dos elementos nas posições $\frac{n}{2}$ e $\frac{n}{2} + 1`.
>
> **Nota 2**: Nas funções que calculam variância e desvio padrão, existe um parâmetro denominado `ddof`. Significa os graus de liberdade ajustáveis e seu valor padrão é `1`. Ao calcular a variância e o desvio padrão da amostra, precisamos fazer a correção do grau de liberdade. Se quisermos calcular a variância populacional e o desvio padrão populacional, podemos definir o parâmetro `ddof` como `0`, o que significa que nenhuma correção de grau de liberdade é necessária.
>
> **Nota 3**: A função `describe` reúne as funções estatísticas acima e é usada para gerar as estatísticas descritivas dos dados. Na verdade, a biblioteca padrão do Python possui um módulo chamado `statistics` e já empacotou funções para obter estatísticas descritivas. Os leitores interessados podem aprender sobre isso por si próprios.

### Exemplo 5: Gerador de números de loteria com bola de duas cores

Vamos usar funções para refatorar o exemplo Double Color Ball de uma lição anterior. Empacotamos a função que gera números aleatórios e a função que gera um grupo de números em duas funções e, em seguida, usamos chamadas de função para gerar tickets aleatórios `N`.

```python
"""
Random number generator for Double Color Ball

Author: Luo Hao
Version: 1.3
"""
import random

RED_BALLS = [i for i in range(1, 34)]
BLUE_BALLS = [i for i in range(1, 17)]


def choose():
    """
    Generate one group of random numbers.
    :return: A list that saves the random numbers
    """
    selected_balls = random.sample(RED_BALLS, 6)
    selected_balls.sort()
    selected_balls.append(random.choice(BLUE_BALLS))
    return selected_balls


def display(balls):
    """
    Print one group of numbers in a formatted way.
    :param balls: A list that saves the random numbers
    """
    for ball in balls[:-1]:
        print(f'\033[031m{ball:0>2d}\033[0m', end=' ')
    print(f'\033[034m{balls[-1]:0>2d}\033[0m')


n = int(input('Generate a few bets: '))
for _ in range(n):
    display(choose())
```

> **Nota**: Observe a linha `display(choose())`. Aqui, primeiro usamos a função `choose` para obter um grupo de números aleatórios, depois usamos o valor de retorno da função `choose` como parâmetro da função `display` e usamos a função `display` para mostrar os números aleatórios selecionados. Após a refatoração, a lógica do código torna-se muito clara e a legibilidade do código torna-se mais forte. Se alguém já empacotou essas duas funções para você e você é apenas o chamador das funções, não precisa se preocupar com a implementação interna da função `choose` e da função `display`. Você só precisa saber que chamar a função `choose` pode gerar um grupo de números aleatórios, e chamar a função `display` com uma lista pode gerar esse grupo de números. No futuro, quando usarmos muitas bibliotecas Python de terceiros, também não nos importaremos com sua implementação na camada inferior. O que precisamos saber é apenas qual função chamar para resolver o problema.

### Resumo

Ao escrever código, especialmente ao desenvolver projetos comerciais, devemos conscientemente **empacotar funções relativamente independentes que serão usadas repetidamente em funções**. Dessa forma, tanto você quanto outros membros da equipe podem usar essas funções chamando-as, e isso pode reduzir o trabalho repetitivo e enfadonho no desenvolvimento.
