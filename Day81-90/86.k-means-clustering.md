## K-Means Clustering

Clustering is an important technique in data mining and machine learning. It is used to divide the samples in a dataset into several similar groups or categories. This method is widely used in many fields. For example:

- **E-commerce**: According to user behavior data, companies can divide consumers into different groups, such as high-spending users, highly active users, users with churn risk, and price-sensitive users, so they can make targeted operation strategies and do more accurate advertising. Representative companies include Alibaba and Amazon.
- **Finance**: Banks can use clustering algorithms to analyze user credit records, income level, spending behavior, and other data, and divide users into different risk groups. High-risk groups need stricter credit review, while low-risk groups can enjoy better loan rates or credit limits.
- **Healthcare**: By analyzing patient health data such as medical history, gene data, and living habits, we can divide patients into different health-risk groups, improve the accuracy of disease prediction, and push forward precision medicine. Representative companies include Johnson & Johnson and Pfizer.
- **Social media**: By analyzing user friend relationships, interests, and social interactions, we can divide users into different social circles, and use that for friend recommendation, personalized content, and optimization of the information-flow recommendation system on the platform.

Clustering is a kind of **unsupervised learning**, because it does not need predefined labels. It only learns according to the features of the data, measures feature similarity or distance, and then divides the known dataset into several different classes. Different from classification, the goal of clustering is to discover the inner structure of the data. Broadly speaking, clustering algorithms can be divided into **distance-based clustering**, **density-based clustering**, **hierarchical clustering**, **spectral clustering**, and so on.

### How the Algorithm Works

In this lesson we focus on the K-Means clustering algorithm. K-Means is a prototype-based partitioning method. Its goal is to divide a dataset into `K` clusters and make the data points inside each cluster as similar as possible. The algorithm works like this:

1. **Initialize the cluster centers.** Randomly choose `K` samples as the initial cluster centers, which are also called centroids.
2. **Assign samples to the nearest centroid.** Compute the distance from each sample to every centroid and assign the sample to the closest cluster.
3. **Update the centroids.** Compute the mean value of all samples in each cluster and use it as the new centroid.
4. **Repeat steps 2 and 3** until the centroids converge or the maximum number of iterations is reached.

### Mathematical Description

We can describe the same idea more formally. Given a dataset, the goal of K-Means is to minimize the objective function, which is the total sum of squared errors:

$$
J = \sum_{i=1}^{K} \sum_{x \in C_{i}} {\lVert x - \mu_{i} \rVert}^{2}
$$

Here, `K` is the number of clusters, `C_i` is the set of samples in the `i`-th cluster, `\mu_i` is the center of the `i`-th cluster, and `x` is a data point. Since this is an NP-hard combinational optimization problem, in practice we solve it with an iterative procedure.

First, we randomly choose `K` points as the initial centroids `\mu_1, \mu_2, \cdots, \mu_K`. For each sample `x_j`, we compute its distance to each centroid and assign it to the nearest one:

$$
C_{i} = \lbrace {x_{j} \ \vert \ {\lVert x_{j} - \mu_{i} \rVert}^{2} \le {\lVert x_{j} - \mu_{k} \rVert}^{2} \ \text{for all} \ k \ne i} \rbrace
$$

Then we update each centroid with the mean of all points in the cluster:

$$
\mu_{i} = \frac{1}{\lvert C_{i} \rvert}\sum_{x \in C_{i}} x
$$

We repeat these two actions until the centroids stop changing or the change becomes smaller than a given threshold. That is what gives the algorithm its convergence behavior.

### A Plain Python Implementation

Below, we use Python code to implement K-Means clustering. For now, we do not use scikit-learn. The main purpose is to help everyone understand how the algorithm works.

```python
import numpy as np


def distance(u, v, p=2):
    """Calculate the distance between two vectors."""
    return np.sum(np.abs(u - v) ** p) ** (1 / p)


def init_centroids(X, k):
    """Randomly choose k centroids."""
    index = np.random.choice(np.arange(len(X)), k, replace=False)
    return X[index]


def closest_centroid(sample, centroids):
    """Find the centroid closest to a sample."""
    distances = [distance(sample, centroid) for i, centroid in enumerate(centroids)]
    return np.argmin(distances)


def build_clusters(X, centroids):
    """Divide the data into clusters according to the centroids."""
    clusters = [[] for _ in range(len(centroids))]
    for i, sample in enumerate(X):
        centroid_index = closest_centroid(sample, centroids)
        clusters[centroid_index].append(i)
    return clusters


def update_centroids(X, clusters):
    """Update centroid positions."""
    return np.array([np.mean(X[cluster], axis=0) for cluster in clusters])


def make_label(X, clusters):
    """Generate labels from the clusters."""
    labels = np.zeros(len(X))
    for i, cluster in enumerate(clusters):
        for j in cluster:
            labels[j] = i
    return labels


def kmeans(X, *, k, max_iter=1000, tol=1e-4):
    """K-Means clustering."""
    # randomly choose k centroids
    centroids = init_centroids(X, k)
    for _ in range(max_iter):
        # divide the data into different clusters by the centroids
        clusters = build_clusters(X, centroids)
        # calculate the new centroid positions again
        new_centroids = update_centroids(X, clusters)
        # stop early if the centroids almost do not change
        if np.allclose(new_centroids, centroids, rtol=tol):
            break
        # record the new centroid positions
        centroids = new_centroids
    # generate labels for the data
    return make_label(X, clusters), centroids
```

We can still use the iris dataset as an example and see whether our own `kmeans` function can divide the three species into three different groups. Since this is unsupervised learning, we feed the entire dataset into `kmeans` directly.

```python
from sklearn.datasets import load_iris

iris = load_iris()
X, y = iris.data, iris.target
labels, centers = kmeans(X, k=3)
```

Here, do not directly compare `labels` with `y`. We said before that a clustering algorithm does not know the labels of the data. It only divides the data into different classes according to features. So the output `0`, `1`, and `2` do not directly correspond to Iris setosa, Iris versicolor, and Iris virginica. Directly using the `KMeans` class in the `cluster` module of scikit-learn to implement K-Means clustering is a better choice, as shown below.

```python
from sklearn.cluster import KMeans

km_cluster = KMeans(
    n_clusters=3,       # value of k, number of clusters
    max_iter=30,        # maximum number of iterations
    n_init=10,          # number of tries for choosing initial centroids
    init='k-means++',   # algorithm for choosing initial centroids
    algorithm='elkan',  # whether to use triangle-inequality optimization
    tol=1e-4,           # tolerance of centroid change
    random_state=3      # random seed
)
km_cluster.fit(X)
print(km_cluster.labels_)           # cluster labels
print(km_cluster.cluster_centers_)  # centroid positions
print(km_cluster.inertia_)          # sum of squared distances from samples to centroids
```

Below, we explain several hyperparameters of the `KMeans` class:

1. `n_clusters`: the number of clusters, which is the value of `K`. The default is `8`.
2. `max_iter`: the maximum number of iterations. The default is `300`.
3. `init`: the method for initializing centroids. The default value is `'k-means++'`, which means choosing K centroids from the data by random selection many times, calculating the distances between the chosen center points each time, and then taking the group with the largest distance as the initial centroids. We recommend using this value. If you set it to `'random'`, it chooses the initial centroids randomly.
4. `n_init`: the number of initialization attempts used together with `init`.
5. `algorithm`: the implementation of K-Means. The default is `'lloyd'`. Another option is `'elkan'`, which uses triangle-inequality optimization and is often more efficient when `K` is larger.
6. `tol`: the convergence tolerance. If the dataset is large, raising this value a little can speed up convergence.

### Summary

K-Means is a classic clustering algorithm. Its advantages include simple implementation and fast convergence. Its disadvantages are that the result is not stable, because it depends on the initial setting, it cannot solve sample-imbalance problems, it easily converges to a local optimum, and it is strongly affected by noisy data.
