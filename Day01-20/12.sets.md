## Estruturas de dados comuns: conjuntos

Depois de aprender sobre listas e tuplas, vamos estudar outro tipo de dados contêiner, chamado **set**. A palavra “conjunto” não deveria parecer estranha, porque esse conceito também aparece nos livros didáticos de matemática. Se **tratarmos uma coleção de coisas dentro de um certo escopo, que são definidas e distinguíveis, como um todo**, então esse todo é um conjunto, e cada coisa no conjunto é chamada de **elemento** do conjunto.

Um conjunto geralmente tem estas propriedades:

1. **Não ordenado**: os elementos não possuem posições.
2. **Único**: elementos duplicados não são permitidos.
3. **Associação definitiva**: para qualquer elemento, ele pertence ao conjunto ou não.

Os conjuntos em Python não são essencialmente diferentes dos conjuntos em matemática. O que precisa ser especialmente enfatizado são as propriedades únicas e desordenadas mencionadas acima. Não ordenado significa que os elementos de um conjunto não possuem uma ordem como os elementos de uma lista, portanto não podemos acessar nenhum elemento por meio de indexação. **Os conjuntos não suportam operações de indexação**. Além disso, a singularidade dos conjuntos significa que **os conjuntos não podem ter elementos duplicados**, e este também é um lugar onde os conjuntos são diferentes das listas. Não podemos adicionar elementos duplicados a um conjunto. O tipo de conjunto de curso suporta as operações de pertinência `in` e `not in`, para que possamos determinar se um elemento pertence a um conjunto, que é a certeza mencionada acima. **As operações de associação em conjuntos têm melhor desempenho do que as operações de associação em listas**. Isto é decidido pelas características de armazenamento subjacentes dos conjuntos. Não discutiremos isso aqui por enquanto. Apenas lembre-se desta conclusão.

> **Observação**: A camada inferior de um conjunto usa armazenamento hash. Os leitores que não entendem de armazenamento de hash podem primeiro ler a explicação de [tabelas hash](https://www.hello-algo.com/chapter_hashing/) no site Hello Algo.

### Criando Conjuntos

Em Python, você pode criar um conjunto com a sintaxe literal `{}`, mas deve haver pelo menos um elemento dentro de `{}`. Isso ocorre porque `{}` sem nenhum elemento não é um conjunto vazio, mas um dicionário vazio, e apresentaremos os dicionários na próxima lição. Claro, você também pode usar a função `set` integrada do Python para criar um conjunto. Mais exatamente, `set` não é uma função comum, mas um construtor para criar objetos definidos. Apresentaremos esse ponto mais tarde, quando discutirmos a programação orientada a objetos. Podemos usar a função `set` para criar um conjunto vazio e também para transformar outras sequências em conjuntos. Por exemplo, `set('hello')` fornece um conjunto contendo caracteres `4`, porque o caractere repetido `l` aparece apenas uma vez no conjunto. Além dessas duas formas, você também pode usar a sintaxe de compreensão para criar um conjunto, assim como usamos anteriormente a sintaxe de compreensão para criar listas.

```python
set1 = {1, 2, 3, 3, 3, 2}
print(set1)

set2 = {'banana', 'pitaya', 'apple', 'apple', 'banana', 'grape'}
print(set2)

set3 = set('hello')
print(set3)

set4 = set([1, 2, 2, 3, 3, 3, 2, 1])
print(set4)

set5 = {num for num in range(1, 20) if num % 3 == 0 or num % 7 == 0}
print(set5)
```

Deve-se ressaltar que os elementos em um conjunto devem ser do tipo **hashable**. Um tipo hashável significa um tipo de dados que pode calcular um código hash. Geralmente, tipos imutáveis ​​são hasháveis, como números inteiros (`int`), números de ponto flutuante (`float`), valores booleanos (`bool`), strings (`str`) e tuplas (`tuple`). Os tipos mutáveis ​​não são hasháveis, porque os tipos mutáveis ​​não podem calcular um código hash estável e, portanto, não podem ser colocados em um conjunto. Por exemplo, não podemos usar uma lista como elemento de um conjunto. Da mesma forma, como um conjunto em si também é mutável, um conjunto não pode ser usado como elemento de outro conjunto. Podemos criar listas aninhadas, onde os elementos de uma lista também são listas, mas não podemos criar conjuntos aninhados. Este ponto deve ser lembrado ao usar conjuntos.

> **Dica**: Se você não entende os conceitos de código hash ou armazenamento de hash mencionados acima, você pode deixá-los por enquanto, porque eles não afetam seu aprendizado contínuo e uso do Python. Claro, se você é um estudante de ciência da computação, é difícil perdoar não entender o armazenamento de hash, então você deve preparar essa lição rapidamente.

### Iterando sobre conjuntos

Podemos usar a função `len` para obter quantos elementos há em um conjunto, mas não podemos usar a indexação para percorrer os elementos de um conjunto, porque os elementos do conjunto não têm uma ordem fixa. Claro, se quisermos percorrer os elementos de um conjunto, ainda podemos usar um loop `for-in`, conforme mostrado abaixo.

```python
set1 = {'Python', 'C++', 'Java', 'Kotlin', 'Swift'}
for elem in set1:
    print(elem)
```

> **Dica**: observe a saída do código acima e sinta a natureza desordenada dos conjuntos a partir da ordem em que as palavras são impressas.

### Definir operações

Python fornece operações muito ricas para o tipo de conjunto, incluindo principalmente operações de associação, interseção, união, diferença, operações de comparação como igualdade, subconjunto e superconjunto, e assim por diante.

#### Associação

Podemos usar as operações de associação `in` e `not in` para verificar se um elemento está em um conjunto, conforme mostrado abaixo.

```python
set1 = {11, 12, 13, 14, 15}
print(10 in set1)      # False
print(15 in set1)      # True
set2 = {'Python', 'Java', 'C++', 'Swift'}
print('Ruby' in set2)  # False
print('Java' in set2)  # True
```

#### Operações Binárias

```python
set1 = {1, 2, 3, 4, 5, 6, 7}
set2 = {2, 4, 6, 8, 10}

print(set1 & set2)                      # intersection
print(set1.intersection(set2))

print(set1 | set2)                      # union
print(set1.union(set2))

print(set1 - set2)                      # difference
print(set1.difference(set2))

print(set1 ^ set2)                      # symmetric difference
print(set1.symmetric_difference(set2))
```

Pelo código acima, podemos ver que ao encontrar a interseção de dois conjuntos, o operador `&` e o método `intersection` fazem exatamente a mesma coisa. Usar o operador é claramente mais direto e o código também é mais curto. Deve ser explicado que as operações binárias em conjuntos também podem ser combinadas com operações de atribuição para formar operações de atribuição compostas. Por exemplo, `set1 |= set2` é igual a `set1 = set1 | set2`, e o método com o mesmo efeito que `|=` é `update`; `set1 &= set2` é igual a `set1 = set1 & set2`, e o método com o mesmo efeito de `&=` é `intersection_update`, conforme mostrado abaixo.

```python
set1 = {1, 3, 5, 7}
set2 = {2, 4, 6}
set1 |= set2
print(set1)  # {1, 2, 3, 4, 5, 6, 7}
set3 = {3, 6, 9}
set1 &= set3
# set1.intersection_update(set3)
print(set1)  # {3, 6}
set2 -= set1
# set2.difference_update(set1)
print(set2)  # {2, 4}
```

#### Comparação

Dois conjuntos podem usar `==` e `!=` para julgar a igualdade. Se os elementos em dois conjuntos forem exatamente iguais, o resultado de `==` será `True`; caso contrário, é `False`. Se cada elemento do conjunto `A` também for um elemento do conjunto `B`, então o conjunto `A` é chamado de subconjunto do conjunto `B`; ao contrário, `B` pode ser chamado de superconjunto de `A`. Se `A` for um subconjunto de `B` e `A` não for igual a `B`, então `A` é um subconjunto adequado de `B`. Python fornece operadores para o tipo de conjunto para julgar relacionamentos de subconjunto e superconjunto. Estes são os operadores muito familiares `<`, `<=`, `>` e `>=`. Claro, também podemos usar os métodos de conjunto `issubset` e `issuperset` para julgar a relação entre os conjuntos.

```python
set1 = {1, 3, 5}
set2 = {1, 2, 3, 4, 5}
set3 = {5, 4, 3, 2, 1}

print(set1 < set2)   # True
print(set1 <= set2)  # True
print(set2 < set3)   # False
print(set2 <= set3)  # True
print(set2 > set1)   # True
print(set2 == set3)  # True

print(set1.issubset(set2))
print(set2.issuperset(set1))
```

> **Observação**: No código acima, `set1 < set2` verifica se `set1` é um subconjunto adequado de `set2`, `set1 <= set2` verifica se `set1` é um subconjunto de `set2` e `set2 > set1` verifica se `set2` é um superconjunto de `set1`. Claro, também podemos usar `set1.issubset(set2)` para verificar se `set1` é um subconjunto de `set2` e usar `set2.issuperset(set1)` para verificar se `set2` é um superconjunto de `set1`.

### Definir métodos

Como dissemos há pouco, conjuntos em Python são tipos mutáveis, portanto podemos adicionar elementos a um conjunto ou excluir elementos de um conjunto por meio de métodos de conjunto.

```python
set1 = {1, 10, 100}

set1.add(1000)
set1.add(10000)
print(set1)

set1.discard(10)
if 100 in set1:
    set1.remove(100)
print(set1)

set1.clear()
print(set1)  # set()
```

> **Nota**: O método `remove` para excluir um elemento gera `KeyError` quando o elemento não existe, portanto, no código acima, primeiro usamos uma operação de associação para verificar se o elemento está no conjunto. O tipo de conjunto também possui um método chamado `pop`, que pode excluir aleatoriamente um elemento do conjunto. Ao excluir o elemento, ele também retorna o elemento excluído. Os métodos `remove` e `discard` apenas excluem o elemento e não o retornam.

O tipo de conjunto também possui um método denominado `isdisjoint`, que pode verificar se dois conjuntos possuem elementos comuns. Caso não possuam elementos comuns, este método retorna `True`; caso contrário, este método retornará `False`, conforme mostrado abaixo.

```python
set1 = {'Java', 'Python', 'C++', 'Kotlin'}
set2 = {'Kotlin', 'Swift', 'Java', 'Dart'}
set3 = {'HTML', 'CSS', 'JavaScript'}
print(set1.isdisjoint(set2))  # False
print(set1.isdisjoint(set3))  # True
```

### Conjuntos Imutáveis

Python também possui um tipo de conjunto imutável chamado `frozenset`. A diferença entre `set` e `frozenset` é como a diferença entre `list` e `tuple`. Como `frozenset` é um tipo imutável, ele pode calcular um código hash, portanto pode ser usado como um elemento em um `set`. Além de não poder adicionar ou excluir elementos, `frozenset` é igual a `set` em outros aspectos. O código abaixo mostra simplesmente como usar `frozenset`.

```python
fset1 = frozenset({1, 3, 5, 7})
fset2 = frozenset(range(1, 6))
print(fset1)
print(fset2)
print(fset1 & fset2)
print(fset1 | fset2)
print(fset1 - fset2)
print(fset1 < fset2)
```

### Resumo

Em Python, o **tipo set é um contêiner não ordenado** e **não permite elementos duplicados**. Como a camada inferior usa armazenamento hash, os elementos em um conjunto devem ser do tipo hash. A maior diferença entre um conjunto e uma lista é que **os elementos de um conjunto não têm ordem**, portanto **não podemos acessar elementos através de operações de indexação**. Mas os conjuntos podem realizar operações binárias, como interseção, união e diferença, e também podem usar operadores relacionais para verificar se dois conjuntos têm relacionamentos de superconjunto ou subconjunto.
