## Estruturas de dados comuns: tuplas

Nas duas lições anteriores, falamos sobre listas em Python. Uma lista é um tipo de dados do tipo contêiner. Por meio de variáveis do tipo lista, podemos armazenar vários dados e usar loops para processar dados em lotes. Claro, Python também possui outros tipos de dados do tipo contêiner. A seguir falaremos de outra, cujo nome é **tupla**.

### Definindo Tuplas e Operações Básicas

Em Python, uma tupla também é uma sequência composta de vários elementos em uma determinada ordem. A diferença entre uma tupla e uma lista é que **uma tupla é um tipo imutável**. Isso significa que uma vez definida uma variável do tipo tupla, os elementos não podem ser adicionados ou excluídos, e os valores dos elementos também não podem ser alterados. Se você tentar modificar um elemento em uma tupla, isso gerará `TypeError` e fará com que o programa trave. As tuplas geralmente são definidas com sintaxe literal como `(x, y, z)`, e os operadores suportados pelas tuplas são os mesmos suportados pelas listas. Veja o código abaixo.

```python
# A tuple with three elements.
t1 = (35, 12, 98)
# A tuple with four elements.
t2 = ('Luo Hao', 45, True, 'Chengdu, Sichuan')

# Check the types of the variables.
print(type(t1))  # <class 'tuple'>
print(type(t2))  # <class 'tuple'>

# Check the number of elements in the tuple.
print(len(t1))  # 3
print(len(t2))  # 4

# Indexing
print(t1[0])    # 35
print(t1[2])    # 98
print(t2[-1])   # Chengdu, Sichuan

# Slicing
print(t2[:2])   # ('Luo Hao', 45)
print(t2[::3])  # ('Luo Hao', 'Chengdu, Sichuan')

# Loop through the elements in the tuple.
for elem in t1:
    print(elem)

# Membership operations
print(12 in t1)            # True
print(99 in t1)            # False
print('Hao' not in t2)     # True

# Concatenation
t3 = t1 + t2
print(t3)

# Comparison
print(t1 == t3)            # False
print(t1 >= t3)            # False
print(t1 <= (35, 11, 99))  # False
```

Se uma tupla tiver dois elementos, chamamos-lhe 2 tuplas. Se uma tupla tiver cinco elementos, chamamos-lhe 5 tuplas. Uma coisa que precisa de atenção especial é esta: `()` significa a tupla vazia, mas se uma tupla tiver apenas um elemento, precisamos adicionar uma vírgula. Caso contrário, `()` não significa sintaxe literal de tupla. Significa parênteses usados para alterar a precedência do operador. Portanto, `('hello', )` e `(100, )` são tuplas de 1, enquanto `('hello')` e `(100)` são apenas uma string e um número inteiro. Podemos verificar isso com o código abaixo.

```python
a = ()
print(type(a))  # <class 'tuple'>
b = ('hello')
print(type(b))  # <class 'str'>
c = (100)
print(type(c))  # <class 'int'>
d = ('hello', )
print(type(d))  # <class 'tuple'>
e = (100, )
print(type(e))  # <class 'tuple'>
```

### Embalagem e desembalagem

Quando atribuímos vários valores separados por vírgulas a uma variável, os vários valores são compactados em uma tupla. Quando atribuímos uma tupla a múltiplas variáveis, a tupla é descompactada em vários valores e então atribuída às variáveis correspondentes, conforme mostrado abaixo.

```python
# Packing
a = 1, 10, 100
print(type(a))  # <class 'tuple'>
print(a)        # (1, 10, 100)

# Unpacking
i, j, k = a
print(i, j, k)  # 1 10 100
```

Se o número de valores descompactados não corresponder ao número de variáveis, o Python gera `ValueError` e a mensagem de erro será `too many values to unpack` ou `not enough values to unpack`.

```python
a = 1, 10, 100, 1000
# i, j, k = a          # ValueError: too many values to unpack
# i, j, k, l, m, n = a # ValueError: not enough values to unpack
```

Existe uma maneira de resolver o problema quando o número de variáveis é menor que o número de elementos: use uma expressão com estrela. Com uma expressão com estrela, uma variável pode receber vários valores, conforme mostrado abaixo. Dois pontos precisam de atenção: primeiro, uma variável marcada com `*` torna-se uma lista, e essa lista pode conter 0 ou mais elementos; segundo, apenas uma expressão com estrela pode aparecer em uma instrução de descompactação.

```python
a = 1, 10, 100, 1000
i, j, *k = a
print(i, j, k)        # 1 10 [100, 1000]
i, *j, k = a
print(i, j, k)        # 1 [10, 100] 1000
*i, j, k = a
print(i, j, k)        # [1, 10] 100 1000
*i, j = a
print(i, j)           # [1, 10, 100] 1000
i, *j = a
print(i, j)           # 1 [10, 100, 1000]
i, j, k, *l = a
print(i, j, k, l)     # 1 10 100 [1000]
i, j, k, l, *m = a
print(i, j, k, l, m)  # 1 10 100 1000 []
```

Mais uma coisa deve ser explicada: a sintaxe de descompactação funciona para todas as sequências. Isso significa que as listas sobre as quais falamos antes, as sequências de intervalo construídas pela função `range` e até mesmo as strings podem usar a sintaxe de descompactação. Tente executar o código abaixo e veja o resultado obtido.

```python
a, b, *c = range(1, 10)
print(a, b, c)
a, b, c = [1, 10, 100]
print(a, b, c)
a, *b, c = 'hello'
print(a, b, c)
```

### Trocando Variáveis

Trocar os valores das variáveis é uma operação muito comum ao escrever código. Mas em muitas linguagens de programação, a troca dos valores de duas variáveis requer uma variável intermediária. Se você não usar uma variável intermediária, precisará usar operações bit a bit bastante obscuras. Em Python, a troca dos valores das variáveis `a` e `b` requer apenas o código abaixo.

```python
a, b = b, a
```

Da mesma forma, se você quiser girar os valores de três variáveis `a`, `b` e `c`, de modo que o valor de `b` seja atribuído a `a`, o valor de `c` será atribuído a `b`, e o valor de `a` é atribuído a `c`, isso pode ser feito da mesma maneira.

```python
a, b, c = b, c, a
```

Deve ser explicado que as operações acima não utilizam sintaxe de empacotamento e descompactação. O bytecode Python possui instruções como `ROT_TWO` e `ROT_THREE` que podem fazer essa operação diretamente, portanto a eficiência é muito alta. Mas se os valores de mais de três variáveis precisarem ser girados em ordem, então não haverá instrução de bytecode diretamente utilizável e teremos que completar a troca por meio de empacotamento e descompactação.

### Tuplas vs listas

Há outra questão aqui que vale a pena discutir. Python já possui o tipo lista, então por que ainda precisamos de tuplas? Esta questão pode parecer um pouco difícil para iniciantes, mas não importa. Vamos primeiro apresentar a ideia, e todos poderão continuar aprendendo e compreendendo-a gradativamente.

1. Tuplas são imutáveis e **tipos imutáveis são mais adequados para ambientes multithreaded**, porque reduzem o custo de sincronização do acesso simultâneo a variáveis. Discutiremos esse ponto mais tarde, quando falarmos sobre programação simultânea.
2. Tuplas são imutáveis e geralmente **tipos imutáveis são mais rápidos de criar** do que os tipos mutáveis correspondentes. Podemos usar a função `timeit` no módulo `timeit` para ver quanto tempo leva para criar uma tupla e uma lista que contenha os mesmos elementos. O parâmetro `number` da função `timeit` significa quantas vezes o código é executado. No código abaixo, criamos uma lista e uma tupla que armazenam os números inteiros de `1` a `9`, e cada operação executa `10000000` vezes.

```python
import timeit

print('%.3f seconds' % timeit.timeit('[1, 2, 3, 4, 5, 6, 7, 8, 9]', number=10000000))
print('%.3f seconds' % timeit.timeit('(1, 2, 3, 4, 5, 6, 7, 8, 9)', number=10000000))
```

Saída:

```text
0.635 seconds
0.078 seconds
```

> **Nota**: O resultado do código acima será diferente em diferentes sistemas de hardware e software. No computador que estou usando agora, a criação da lista de tempos `10000000` leva `0.635` segundos, enquanto a criação da tupla de tempos `10000000` leva segundos `0.078`. Claramente, criar uma tupla é mais rápido e a diferença de tempo é de cerca de uma ordem de magnitude. Você pode executar este código em seu próprio computador e comparar os resultados.

Claro, tuplas e listas em Python podem ser convertidas umas nas outras. Podemos fazer isso com o código abaixo.

```python
infos = ('Luo Hao', 45, True, 'Chengdu, Sichuan')
# Convert the tuple to a list.
print(list(infos))  # ['Luo Hao', 45, True, 'Chengdu, Sichuan']

frts = ['apple', 'banana', 'orange']
# Convert the list to a tuple.
print(tuple(frts))  # ('apple', 'banana', 'orange')
```

### Resumo

**Listas e tuplas são tipos de dados do tipo contêiner**, o que significa que uma variável pode armazenar vários dados e ambos são contêineres ordenados que organizam os elementos em uma determinada ordem. **Listas são tipos de dados mutáveis**, enquanto **tuplas são tipos de dados imutáveis**, portanto, as listas podem adicionar elementos, excluir elementos, limpar elementos, classificar e reverter, mas essas operações não funcionam para tuplas. Listas e tuplas suportam **concatenação**, **operações de associação**, **indexação** e **fatiamento**. O tipo string do qual falaremos mais tarde também suporta essas operações, pois uma string também é uma sequência composta de caracteres em uma determinada ordem. Nesse aspecto, não há diferença entre os três. **Recomendamos que todos usem sintaxe de compreensão de lista para criar listas**. Não é apenas fácil de usar, mas também muito eficiente, e é uma das peças de sintaxe mais distintas do Python.
