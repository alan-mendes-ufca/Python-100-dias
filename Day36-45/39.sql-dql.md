## DQL em detalhes

A seguir, usaremos o banco de dados de seleção de cursos escolares criado anteriormente para explicar as operações de consulta em DQL. A consulta é muito importante tanto para desenvolvedores quanto para analistas de dados, porque está relacionada à possibilidade de obtermos os dados que precisamos de um banco de dados relacional. É recomendado que você execute novamente o DDL da penúltima lição que criou o banco de dados e as tabelas, e o DML da lição anterior que inseriu os dados, para ter certeza de que não há problemas com as tabelas e dados antes de executar as operações abaixo.

```sql
USE school;

-- Consulta todas as informações de todos os estudantes
SELECT stu_id,
       stu_name,
       stu_sex,
       stu_birth,
       stu_addr,
       col_id
  FROM tb_student;

-- Consulta ID, nome e cidade natal dos estudantes (projeção e alias)
SELECT stu_id AS StudentID,
       stu_name AS Name,
       stu_addr AS Hometown
  FROM tb_student;

-- Consulta nomes e créditos dos cursos (projeção e alias)
SELECT cou_name AS CourseName,
       cou_credit AS Credits
  FROM tb_course;

-- Consulta nomes e datas de nascimento de todas as estudantes (filtragem de dados)
SELECT stu_name,
       stu_birth
  FROM tb_student
 WHERE stu_sex = 0;

-- Consulta estudantes mulheres de Chengdu, Sichuan (filtragem de dados)
SELECT stu_name,
       stu_birth
  FROM tb_student
 WHERE stu_sex = 0
       AND stu_addr = 'Chengdu, Sichuan';

-- Consulta estudantes mulheres ou de Chengdu, Sichuan (filtragem de dados)
SELECT stu_name,
       stu_birth
  FROM tb_student
 WHERE stu_sex = 0
       OR stu_addr = 'Chengdu, Sichuan';

-- Consulta todos os estudantes nascidos na década de 1980 (filtragem de dados)
SELECT stu_name,
       stu_sex,
       stu_birth
  FROM tb_student
 WHERE '1980-1-1' <= stu_birth
       AND stu_birth <= '1989-12-31';

SELECT stu_name,
       stu_sex,
       stu_birth
  FROM tb_student
 WHERE stu_birth BETWEEN '1980-1-1' AND '1989-12-31';

-- Consulta cursos com mais de 2 créditos (filtragem de dados)
SELECT cou_name,
       cou_credit
  FROM tb_course
 WHERE cou_credit > 2;

-- Consulta cursos cujo número de créditos é ímpar (filtragem de dados)
SELECT cou_name,
       cou_credit
  FROM tb_course
 WHERE cou_credit MOD 2 <> 0;

-- Consulta IDs dos estudantes que escolheram o curso 1111 e tiveram nota acima de 90 (filtragem de dados)
SELECT stu_id
  FROM tb_record
 WHERE cou_id = 1111
       AND score > 90;

-- Consulta o nome e o sexo do estudante chamado Yang Guo (filtragem de dados)
SELECT stu_name AS Name,
       CASE stu_sex WHEN 1 THEN 'Male' ELSE 'Female' END AS Gender
  FROM tb_student
 WHERE stu_name = 'Yang Guo';

SELECT stu_name AS Name,
       IF(stu_sex, 'Male', 'Female') AS Gender
  FROM tb_student
 WHERE stu_name = 'Yang Guo';

-- Consulta estudantes cujo sobrenome é Yang (correspondência aproximada)
-- O curinga % corresponde a zero ou mais caracteres
SELECT stu_name AS Name,
       CASE stu_sex WHEN 1 THEN 'Male' ELSE 'Female' END AS Gender
  FROM tb_student
 WHERE stu_name LIKE 'Yang%';

-- Consulta estudantes cujo sobrenome é Yang e cujos nomes têm dois caracteres chineses (correspondência aproximada)
-- O curinga _ corresponde a um caractere
SELECT stu_name AS Name,
       CASE stu_sex WHEN 1 THEN 'Male' ELSE 'Female' END AS Gender
  FROM tb_student
 WHERE stu_name LIKE 'Yang_';

-- Consulta estudantes cujo sobrenome é Yang e cujos nomes têm três caracteres chineses (correspondência aproximada)
SELECT stu_name AS Name,
       CASE stu_sex WHEN 1 THEN 'Male' ELSE 'Female' END AS Gender
  FROM tb_student
 WHERE stu_name LIKE 'Yang__';

-- Consulta ID e nome dos estudantes cujo ID termina com 3 (correspondência aproximada)
SELECT stu_id,
       stu_name
  FROM tb_student
 WHERE stu_id LIKE '%3';

-- Consulta ID e nome dos estudantes cujos nomes contêm Bu ou Yan (correspondência aproximada e união)
SELECT stu_id,
       stu_name
  FROM tb_student
 WHERE stu_name LIKE '%Bu%'
       OR stu_name LIKE '%Yan%';

SELECT stu_id,
       stu_name
  FROM tb_student
 WHERE stu_name LIKE '%Bu%'
 UNION
SELECT stu_id,
       stu_name
  FROM tb_student
 WHERE stu_name LIKE '%Yan%';

-- Consulta ID e nome dos estudantes cujo sobrenome é Yang ou Lin e cujos nomes têm três caracteres chineses no total (correspondência aproximada com regex)
SELECT stu_id,
       stu_name
  FROM tb_student
 WHERE stu_name REGEXP '[LinYang][\\u4e00-\\u9fa5]{2}';

-- Consulta nomes dos estudantes cuja cidade natal não foi informada (tratamento de nulos)
SELECT stu_name
  FROM tb_student
 WHERE TRIM(stu_addr) = ''
       OR stu_addr is null;

-- Consulta nomes dos estudantes cuja cidade natal foi informada (tratamento de nulos)
SELECT stu_name
  FROM tb_student
 WHERE TRIM(stu_addr) <> ''
       AND stu_addr is not null;

-- Consulta todas as datas de seleção de cursos (remove duplicatas)
SELECT DISTINCT sel_date
  FROM tb_record;

-- Consulta cidades natais dos estudantes (remove duplicatas)
SELECT DISTINCT stu_addr
  FROM tb_student
 WHERE TRIM(stu_addr) <> ''
       AND stu_addr is not null;

-- Consulta nomes e datas de nascimento dos estudantes homens, ordenados do mais velho para o mais novo (ordenação)
SELECT stu_name,
       stu_birth
  FROM tb_student
 WHERE stu_sex = 1
 ORDER BY stu_birth ASC;

-- Extra: converte as datas de nascimento acima em idades (função de data e função numérica)
SELECT stu_name AS Name,
       FLOOR(DATEDIFF(CURDATE(), stu_birth) / 365) AS Age
  FROM tb_student
 WHERE stu_sex = 1
 ORDER BY Age DESC;

-- Consulta a data de nascimento do estudante mais velho (função agregada)
SELECT MIN(stu_birth)
  FROM tb_student;

-- Consulta a data de nascimento do estudante mais novo (função agregada)
SELECT MAX(stu_birth)
  FROM tb_student;

-- Consulta a maior nota no curso 1111 (função agregada)
SELECT MAX(score)
  FROM tb_record
 WHERE cou_id = 1111;

-- Consulta nota mínima, nota máxima, média, desvio padrão e variância do estudante 1001 (função agregada)
SELECT MIN(score) AS MinScore,
       MAX(score) AS MaxScore,
       ROUND(AVG(score), 1) AS AvgScore,
       STDDEV(score) AS StdDev,
       VARIANCE(score) AS Variance
  FROM tb_record
 WHERE stu_id = 1001;

-- Consulta a média do estudante 1001; se houver valor nulo, conta como 0 (função agregada)
SELECT ROUND(SUM(score) / COUNT(*), 1) AS AvgScore
  FROM tb_record
 WHERE stu_id = 1001;

-- Consulta a quantidade de estudantes homens e mulheres (agrupamento e função agregada)
SELECT CASE stu_sex WHEN 1 THEN 'Male' ELSE 'Female' END AS Gender,
       COUNT(*) AS Total
  FROM tb_student
 GROUP BY stu_sex;

-- Consulta a quantidade de estudantes em cada faculdade (agrupamento e função agregada)
SELECT col_id AS CollegeID,
       COUNT(*) AS Total
  FROM tb_student
 GROUP BY col_id
  WITH ROLLUP;

-- Consulta a quantidade de estudantes homens e mulheres em cada faculdade (agrupamento e função agregada)
SELECT col_id AS CollegeID,
       CASE stu_sex WHEN 1 THEN 'Male' ELSE 'Female' END AS Gender,
       COUNT(*) AS Total
  FROM tb_student
 GROUP BY col_id, stu_sex;

-- Consulta o ID e a média de cada estudante (agrupamento e função agregada)
SELECT stu_id AS StudentID,
       ROUND(AVG(score), 1) AS AvgScore
  FROM tb_record
 GROUP BY stu_id;

-- Consulta ID e média dos estudantes cuja média é pelo menos 90 (filtragem após agrupamento)
SELECT stu_id AS StudentID,
       ROUND(AVG(score), 1) AS AvgScore
  FROM tb_record
 GROUP BY stu_id
HAVING AvgScore >= 90;

-- Consulta ID e média dos estudantes cuja média nos cursos 1111, 2222 e 3333 é pelo menos 90 (filtragem antes e depois do agrupamento)
SELECT stu_id AS StudentID,
       ROUND(AVG(score), 1) AS AvgScore
  FROM tb_record
 WHERE cou_id in (1111, 2222, 3333)
 GROUP BY stu_id
HAVING AvgScore >= 90
 ORDER BY AvgScore ASC;

-- Consulta o nome do estudante mais velho (subconsulta)
SELECT stu_name
  FROM tb_student
 WHERE stu_birth = (SELECT MIN(stu_birth)
                      FROM tb_student);

-- Consulta os nomes dos estudantes que selecionaram mais de dois cursos (subconsulta e operação de conjunto)
SELECT stu_name
  FROM tb_student
 WHERE stu_id in (SELECT stu_id
                    FROM tb_record
                   GROUP BY stu_id
                  HAVING COUNT(*) > 2);

-- Consulta nomes dos estudantes, datas de nascimento e nomes das faculdades (junção de tabelas)
SELECT stu_name,
       stu_birth,
       col_name
  FROM tb_student AS t1, tb_college AS t2
 WHERE t1.col_id = t2.col_id;

SELECT stu_name,
       stu_birth,
       col_name
  FROM tb_student INNER JOIN tb_college
       ON tb_student.col_id = tb_college.col_id;

SELECT stu_name,
       stu_birth,
       col_name
  FROM tb_student NATURAL JOIN tb_college;

SELECT stu_name,
       stu_birth,
       col_name
  FROM tb_student CROSS JOIN tb_college;

-- Consulta nomes dos estudantes, nomes dos cursos e notas
SELECT stu_name,
       cou_name,
       score
  FROM tb_student, tb_course, tb_record
 WHERE tb_student.stu_id = tb_record.stu_id
       AND tb_course.cou_id = tb_record.cou_id
       AND score is not null;

SELECT stu_name,
       cou_name,
       score
  FROM tb_student
       INNER JOIN tb_record
           ON tb_student.stu_id = tb_record.stu_id
       INNER JOIN tb_course
           ON tb_course.cou_id = tb_record.cou_id
 WHERE score is not null;

SELECT stu_name,
       cou_name,
       score
  FROM tb_student
       NATURAL JOIN tb_record
       NATURAL JOIN tb_course
 WHERE score is not null;

-- Complemento: obtém as primeiras 5 linhas
SELECT stu_name,
       cou_name,
       score
  FROM tb_student
       NATURAL JOIN tb_record
       NATURAL JOIN tb_course
 WHERE score is not null
 ORDER BY cou_id ASC, score DESC
 LIMIT 5;

-- Complemento: obtém as linhas 6 a 10
SELECT stu_name,
       cou_name,
       score
  FROM tb_student
       NATURAL JOIN tb_record
       NATURAL JOIN tb_course
 WHERE score is not null
 ORDER BY cou_id ASC, score DESC
 LIMIT 5
OFFSET 5;

-- Complemento: obtém as linhas 11 a 15
SELECT stu_name,
       cou_name,
       score
  FROM tb_student
       NATURAL JOIN tb_record
       NATURAL JOIN tb_course
 WHERE score is not null
 ORDER BY cou_id ASC, score DESC
 LIMIT 10, 5;

-- Consulta nomes dos estudantes e médias
-- Código de erro: 1248. Toda tabela derivada deve ter seu próprio alias
SELECT stu_name,
       avg_score
  FROM tb_student
       NATURAL JOIN (SELECT stu_id,
                            ROUND(AVG(score), 1) AS avg_score
                       FROM tb_record
                      GROUP BY stu_id) as tmp;

-- Consulta nomes dos estudantes e quantidade de cursos selecionados
SELECT stu_name,
       total
  FROM tb_student
       NATURAL JOIN (SELECT stu_id,
                            COUNT(*) AS total
                       FROM tb_record
                      GROUP BY stu_id) as tmp;

-- Consulta o nome de cada estudante e a quantidade de cursos selecionados
SELECT stu_name AS Name,
       COALESCE(total, 0) AS CourseCount
  FROM tb_student AS t1
       LEFT JOIN (SELECT stu_id,
                         COUNT(*) AS total
                    FROM tb_record
                   GROUP BY stu_id) AS t2
              ON t1.stu_id = t2.stu_id;
```

Existem vários lugares que precisam de explicação extra:

1. A versão atual do MySQL não suporta junção externa completa. Acima, usamos a operação `union` para obter a união dos resultados da junção externa esquerda e da junção externa direita, e isso dá o efeito de junção externa completa. Uma junção externa completa mantém as linhas sem correspondência dos dois lados; nas colunas do lado ausente, o resultado contém `NULL`. Assim, a junção interna contém somente as correspondências, enquanto as junções esquerda e direita acrescentam, respectivamente, as linhas sem correspondência de cada tabela.

2. MySQL suporta muitos tipos de operadores, incluindo operadores aritméticos (`+`, `-`, `*`, `/`, `%`), operadores de comparação (`=`, `<>`, `<=>`, `<`, `<=`, `>`, `>=`, `BETWEEN...AND..`, `IN`, `IS NULL`, `IS NOT NULL`, `LIKE`, `RLIKE`, `REGEXP`), operadores lógicos (`NOT`, `AND`, `OR`, `XOR`) e operadores bit a bit (`&`, `|`, `^`, `~`, `>>`, `<<`). Podemos usar esses operadores em DML para processar dados.

3. Ao consultar dados, funções podem ser usadas na instrução `SELECT` e suas cláusulas, como a cláusula `WHERE`, a cláusula `ORDER BY` e a cláusula `HAVING`. Essas funções incluem funções de string, funções numéricas, funções de data e hora, funções de controle de fluxo e assim por diante, conforme mostrado nas tabelas abaixo.

Funções de string comuns.

    | Função | Significado |
   | --------------------------- | ------- |
    | `CONCAT` | Junte várias strings em uma string |
    | `FORMAT` | Formate um número em uma string e especifique quantas casas decimais manter |
    | `FROM_BASE64` / `TO_BASE64` | Decodificação/codificação BASE64 |
    | `BIN` / `OCT` / `HEX` | Converter um número em uma string binária/octal/hexadecimal |
    | `LOCATE` | Encontre a posição de uma substring em uma string |
    | `LEFT` / `RIGHT` | Retorna os caracteres de um comprimento especificado no lado esquerdo/direito de uma string |
    | `LENGTH` / `CHAR_LENGTH` | Retorna o comprimento de uma string em bytes/caracteres |
    | `LOWER` / `UPPER` | Retornar a forma minúscula/maiúscula de uma string |
    | `LPAD` / `RPAD` | Se o comprimento da string não for suficiente, preencha os caracteres especificados no lado esquerdo/direito |
    | `LTRIM` / `RTRIM` | Remova os espaços na frente/atrás de uma string |
    | `ORD` / `CHAR` | Retornar o código de um caractere / retornar o caractere de um código |
    | `STRCMP` | Compare strings e retorne -1, 0, 1 para significar menor que, igual a, maior que |
    | `SUBSTRING` | Retorna a substring no intervalo especificado de uma string |

Funções numéricas comuns.

    | Função | Significado |
   | -------------------------------------------------------- | ------- |
    | `ABS` | Retornar o valor absoluto de um número |
    | `CEILING` / `FLOOR` | Retornar o resultado do arredondamento de um número para cima/para baixo |
    | `CONV` | Converter um número de uma base para outra base |
    | `CRC32` | Calcule o código de verificação de redundância cíclica |
    | `EXP` / `LOG` / `LOG2` / `LOG10` | Calcular expoente / logaritmo |
    | `POW` | Calcular uma potência |
    | `RAND` | Retorne um número aleatório no intervalo [0,1) |
    | `ROUND` | Retornar o resultado arredondado de um número |
    | `SQRT` | Retornar a raiz quadrada de um número |
    | `TRUNCATE` | Truncar um número com a precisão especificada |
    | `SIN` / `COS` / `TAN` / `COT` / `ASIN` / `ACOS` / `ATAN` | Funções trigonométricas |

Funções comuns de data e hora.

    | Função | Significado |
   | ----------------------------------------- | ------- |
    | `CURDATE` / `CURTIME` / `NOW` | Obtenha a data/hora/data e hora atuais |
    | `ADDDATE` / `SUBDATE` | Adicione/subtraia duas expressões de data e retorne o resultado |
    | `DATE` / `TIME` | Obtenha a data/hora de uma string |
    | `YEAR` / `MONTH` / `DAY` | Obtenha ano/mês/dia de uma data |
    | `HOUR` / `MINUTE` / `SECOND` | Obtenha hora/minuto/segundo de um horário |
    | `DATEDIFF` / `TIMEDIFF` / `TIMESTAMPDIFF` | Retorne quantos dias/horas duas expressões de data e hora diferem em |
    | `MAKEDATE` / `MAKETIME` | Marque uma data/hora |

Funções comuns de controle de fluxo.

    | Função | Significado |
   | -------- | ------- |
    | `IF` | Retorna valores diferentes dependendo se a condição é verdadeira |
    | `IFNULL` | Se for `NULL`, retorne o valor especificado, caso contrário, retorne a si mesmo |
    | `NULLIF` | Se duas expressões forem iguais, retorne `NULL`, caso contrário, retorne o valor da primeira expressão |

Outras funções comuns.

    | Função | Significado |
   | ------------------------------ | ------- |
    | `MD5` / `SHA1` / `SHA2` | Retornar o resumo hash de uma string |
    | `CHARSET` / `COLLATION` | Retornar o conjunto de caracteres/agrupamento |
    | `USER` / `CURRENT_USER` | Retornar o usuário atual |
    | `DATABASE` | Retorne o nome do banco de dados atual |
    | `VERSION` | Retornar a versão atual do banco de dados |
    | `FOUND_ROWS` / `ROW_COUNT` | Retornar o número de linhas consultadas/linhas afetadas |

    | `UUID` / `UUID_SHORT` | Retornar um identificador globalmente exclusivo |
