## Asynchronous Tasks and Scheduled Tasks

In a Web application, if a request runs a time-consuming operation, or the execution time of the request cannot be determined, and for the user it is enough only to know that the server has received the request and there is no need to get the result immediately, then this kind of operation should be handled asynchronously. If **using cache is the first key point of optimizing website performance**, then making time-consuming or time-uncertain tasks **asynchronous is the second key point of website performance optimization**. Simply speaking, **if something can be done later, do not do it right now**.

Take sending SMS messages and uploading files to cloud storage, which were mentioned in the previous lesson, as examples. The former is an operation whose time is uncertain, because as the caller, we cannot know the response time of the third-party platform. The latter is a time-consuming operation, because if the file is large or the third-party platform is unstable, the upload may take a long time. Clearly, both of these operations can be handled asynchronously.

In Python projects, we can use the third-party library Celery to complete asynchronous tasks and scheduled tasks. For content about Celery, please move to [Using Django for Commercial Projects](../Day91-100/95.django-for-commercial-projects.md).
