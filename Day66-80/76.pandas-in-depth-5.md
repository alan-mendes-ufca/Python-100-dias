### Calculating Month-over-Month and Year-over-Year Growth

We previously used an example of counting monthly sales. We could do grouped aggregation with `groupby`, or generate a pivot table with `pivot_table`, as shown below.

```python
sales_df = pd.read_excel('data/2020-sales-data.xlsx')
sales_df['Month'] = sales_df['Sales Date'].dt.month
sales_df['Sales Amount'] = sales_df['Price'] * sales_df['Sales Quantity']
result_df = sales_df.pivot_table(index='Month', values='Sales Amount', aggfunc='sum')
result_df.rename(columns={'Sales Amount': 'Current Month Sales'}, inplace=True)
result_df
```

After getting monthly sales, if we need to calculate month-over-month growth, there are two ways here. The first way is to use the `shift` method to move the data, align last month's data with this month's data, and then calculate month-over-month growth with `(current month sales - last month sales) / last month sales`.

```python
result_df['Previous Month Sales'] = result_df['Current Month Sales'].shift(1)

```

In the example above, the parameter of `shift` is `1`, which means moving the data down by one cell. Of course, we can use `-1` to move the data upward by one cell. You can probably also think of this: if we had data for more years, we could set the parameter to `12`, and then we could calculate the year-over-year comparison between each month this year and the same month last year.

```python
result_df['MoM'] = (
    result_df['Current Month Sales'] - result_df['Previous Month Sales']
) / result_df['Previous Month Sales']
result_df.style.format(

```

A simpler second way is to directly use the `pct_change` method to calculate the percentage change. First, let us remove the previous month sales and month-over-month columns.

```python
result_df.drop(columns=['Previous Month Sales', 'MoM'], inplace=True)
```

Next, we use the `pct_change` method of the `DataFrame` object to calculate the month-over-month value. It is worth mentioning that `pct_change` has a parameter named `periods`, and its default value is `1`, which calculates the percentage change between adjacent items. That is exactly the month-over-month result we want. If we had many years of data and changed this parameter to `12`, we could get the year-over-year change between adjacent years.

```python
result_df['MoM'] = result_df.pct_change()
result_df
```

### Window Calculations

The `rolling` method of a `DataFrame` lets us put data into a window, and then perform calculations on the data inside the window. For example, if we get recent stock data for a company and want to create a 5-day moving average and a 10-day moving average, we need to set a window first and then calculate inside it. Let us first read Baidu's 2022 stock data.

```python
baidu_df = pd.read_excel('data/2022-stock-data.xlsx', sheet_name='BIDU', index_col='Date')
baidu_df.sort_index(inplace=True)
baidu_df
```

Output:

<img src="res/baidu_stock.png" style="zoom:50%;">

The `DataFrame` above has five columns, `Open`, `High`, `Low`, `Close`, and `Volume`, which represent the opening price, highest price, lowest price, closing price, and trading volume of the stock. Next, we do window calculations on Baidu's stock data.

```python
baidu_df.rolling(5).mean()
```

Output:

<img src="res/baidu_stock_ma5.png" style="zoom:50%;">

We can also use `rolling` on a `Series` and do calculations inside the window. For example, we can calculate the 5-day moving average and 10-day moving average for the closing price (`Close` column), then use the `merge` function to assemble them into a `DataFrame`, and finally draw a double-moving-average chart.

```python
close_ma5 = baidu_df.Close.rolling(5).mean()
close_ma10 = baidu_df.Close.rolling(10).mean()
result_df = pd.merge(close_ma5, close_ma10, left_index=True, right_index=True)
result_df.rename(columns={'Close_x': 'MA5', 'Close_y': 'MA10'}, inplace=True)
result_df.plot(kind='line', figsize=(10, 6))
plt.show()
```

Output:

<img src="res/baidu_double_MA.png" style="zoom:50%;">

### Judging Correlation

In statistics, we usually use covariance to measure the degree of joint change between two random variables. If large values of variable $\small{X}$ mostly correspond to large values of another variable $\small{Y}$, and small values also correspond to small values, then the two variables tend to show similar behavior, and the covariance is positive. If large values of one variable mostly correspond to small values of the other variable, then the two variables tend to show opposite behavior, and the covariance is negative. Simply speaking, the sign of covariance shows the correlation between two variables. Variance is a special case of covariance, that is, the covariance of a variable with itself.

$$
cov(X,Y) = E((X - \mu)(Y - \upsilon)) = E(X \cdot Y) - \mu\upsilon
$$

If $\small{X}$ and $\small{Y}$ are statistically independent, then their covariance is `0`, because in that case:

$$
E(X \cdot Y) = E(X) \cdot E(Y) = \mu\upsilon
$$

The size of covariance depends on the size of the variables, so it is usually not easy to explain directly. But the normalized form of covariance can show the strength of the linear relationship between variables. In statistics, the Pearson product-moment correlation coefficient is the normalized form of covariance. It is used to measure the degree of linear correlation between two variables $\small{X}$ and $\small{Y}$, and its value is between `-1` and `1`.

$$
\frac {cov(X, Y)} {\sigma_{X}\sigma_{Y}}
$$

By estimating sample covariance and sample standard deviation, we get the sample Pearson coefficient, usually represented by the Greek letter $\small{\rho}$.

$$
\rho = \frac {\sum_{i=1}^{n}(X_i - \bar{X})(Y_i - \bar{Y})} {\sqrt{\sum_{i=1}^{n}(X_i - \bar{X})^2} \sqrt{\sum_{i=1}^{n}(Y_i - \bar{Y})^2}}
$$

When we use $\small{\rho}$ to judge the correlation of indicators, we usually follow two steps.

1. Judge whether the indicators are positively correlated, negatively correlated, or not correlated.
   - When $\small{\rho \gt 0}$, we think the variables are positively correlated, which means they have the same trend.
   - When $\small{\rho \lt 0}$, we think the variables are negatively correlated, which means their trends are opposite.
   - When $\small{\rho \approx 0}$, we think the variables are not correlated, but that does not mean they are statistically independent.
2. Judge the strength of the correlation.
   - When the absolute value of $\small{\rho}$ is in $\small{[0.6,1]}$, we think the variables are strongly correlated.
   - When the absolute value of $\small{\rho}$ is in $\small{[0.1,0.6)}$, we think the variables are weakly correlated.
   - When the absolute value of $\small{\rho}$ is in $\small{[0,0.1)}$, we think there is no correlation.

Pearson correlation is suitable when:

1. The two variables have a linear relationship and are continuous data.
2. The populations of the two variables are normally distributed, or close to a normal unimodal distribution.
3. The observations of the two variables are paired, and each pair is independent.

Here, let us also mention what to do if the two groups of variables are not continuous values from a normal population. For ordinal-scale data, we can use the Spearman rank correlation coefficient, whose formula is:

$$
r_{s}=1-{\frac {6\sum d_{i}^{2}}{n(n^{2}-1)}}
$$

where $\small{d_{i}=\operatorname {R} (X_{i})-\operatorname {R} (Y_{i})}$ is the difference between the ranks of the two variables in each observation group, and $\small{n}$ is the sample size.

For nominal-scale data, we can use the chi-square test to judge whether they are related. In fact, many times continuous values can also be binned into discrete ranks or categories, and then we can use the Spearman rank coefficient or chi-square test to judge correlation.

The `cov` method of a `DataFrame` is used to calculate covariance, and the `corr` method is used to calculate correlation coefficients. The `corr` method has a parameter named `method`, whose default value is `pearson`, meaning Pearson correlation. Besides that, we can also specify `kendall` or `spearman` to calculate Kendall's coefficient or Spearman's rank correlation coefficient.

The Spearman rank correlation coefficient has less strict requirements on data conditions than Pearson correlation. As long as the observations of the two variables are paired rank data, or data converted from continuous variables into rank data, we can use Spearman rank correlation regardless of the population distribution or sample size. We can preprocess part of the features in the following way and then calculate the Spearman rank correlation coefficient.

```python
boston_df['CRIM'] = boston_df.CRIM.apply(lambda x: x // 5 if x < 25 else 5).map(int)
boston_df['ZN'] = pd.qcut(boston_df.ZN, q=[0, 0.75, 0.8, 0.85, 0.9, 0.95, 1], labels=np.arange(6))
boston_df['AGE'] = (boston_df.AGE // 20).map(int)
boston_df['DIS'] = (boston_df.DIS // 2.05).map(int)
boston_df['B'] = (boston_df.B // 66).map(int)
boston_df['PRICE'] = pd.qcut(boston_df.PRICE, q=[0, 0.15, 0.3, 0.5, 0.7, 0.85, 1], labels=np.arange(6))
boston_df[['CRIM', 'ZN', 'AGE', 'DIS', 'B', 'PRICE']].corr(method='spearman')
```

Output:

<img src="res/boston_spearman_correlation.png" style="zoom:50%;">

We can see that housing price has a fairly obvious negative correlation with crime rate (`CRIM`) and housing age (`AGE`), and a weak positive correlation with housing lot size (`ZN`). Correlation can help us find business handles in actual work, that is, to find those related factors that can affect or change work results.
