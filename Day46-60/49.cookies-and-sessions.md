### Implementing User Tracking

Today, if a website cannot remember who you are and what you did on the website before by some method, it loses usability and convenience, and then it is very likely to lose users. So remembering a user, or more professionally, **user tracking**, is a necessary function for most Web applications.

On the server side, the simplest way to remember a user is to create an object. Through this object, we can save all user-related information. This object is what we often call a session. Then the problem comes: HTTP itself is a **connectionless** protocol, because after the server finishes responding to the client request, it disconnects; it is also a **stateless** protocol, because when the client sends another request to the server, the server cannot know any previous information about this client. Even if the server keeps user data through the session object, we still need some way to know which previously saved session is related to the current request. We can assign a globally unique identifier to each session object to identify it. Let us call it `sessionid`. Every time the client sends a request, as long as it carries this `sessionid`, we have a way to find the corresponding session object, so we can remember the user's information between two requests, which is the user tracking mentioned before.

To let the client remember the `sessionid` and carry it on every request, we have the following methods:

1. URL rewriting. URL rewriting means carrying the `sessionid` in the URL, for example: `http://www.example.com/index.html?sessionid=123456`. The server gets the value of the `sessionid` parameter and finds the corresponding session object.

2. Hidden fields. When submitting a form, we can send extra data to the server by setting hidden fields in the form. For example: `<input type="hidden" name="sessionid" value="123456">`.

3. Local storage. Modern browsers support many local storage solutions, including cookie, localStorage, sessionStorage, and IndexedDB. Among these solutions, cookie has the longest history and is also the one that has received the most criticism. It is also the first solution we will explain here. Simply speaking, a cookie is data saved in the browser's temporary files as key-value pairs. On every request, the request header carries the cookie of this site to the server. So if we write the `sessionid` into a cookie, then on the next request the server only needs to read the cookie from the request header to get this `sessionid`, as shown below.

   ![](./res/sessionid_from_cookie.png)

   In the HTML5 era, besides cookie, we can also use the new local storage APIs to save data, which are the localStorage, sessionStorage, and IndexedDB technologies just mentioned, as shown below.

   ![](./res/cookie_xstorage_indexeddb.png)

**To sum up**, to implement user tracking, the server can create a session object for each user session and write the ID of the session object into the browser's cookie. When the user requests the server next time, the browser carries the cookie information saved for the website in the HTTP request header. Then the server can find the ID of the session object from the cookie and get the session object that was created before according to this ID. Since the session object can save user data in key-value pairs, all the information saved in the session object before can be taken out. The server can also decide the user's identity and know the user's preferences according to this information, so it can provide better personalized service.

### Django Support for Session

When creating a Django project, a middleware named `SessionMiddleware` is already enabled by default in the `settings.py` file. We will explain middleware in detail in a later lesson. Here, you only need to know it exists. Because this middleware exists, we can directly use the `session` attribute of the request object to work with the session object. As we said before, the `session` attribute is a container object that can read and write data like a dictionary, so we can keep user data by using "key-value pairs". At the same time, `SessionMiddleware` also wraps cookie operations, and it saves the `sessionid` in the cookie. We already mentioned this above.

In the default case, Django serializes the session data and saves it in a relational database. In versions after Django 1.6, the default serialization method is JSON serialization. Before that, Pickle serialization was used all the time. The difference between JSON serialization and Pickle serialization is that the former serializes objects into strings, while the latter serializes objects into byte strings. Because of security reasons, JSON serialization became the default way for Django to serialize data. This also means the data we save in the session must be JSON-serializable, otherwise an exception will happen. One more thing should be explained: in most cases, using a relational database to save session data is not the best choice, because the database may take huge pressure and become the performance bottleneck of the system. In later lessons, we will tell you how to save session data into a cache service to improve system performance.

Write the class `Captcha` that generates the captcha image.

```python
"""
Image captcha
"""
import os
import random
from io import BytesIO

from PIL import Image
from PIL import ImageFilter
from PIL.ImageDraw import Draw
from PIL.ImageFont import truetype


class Bezier:
    """Bezier curve"""

    def __init__(self):
        self.tsequence = tuple([t / 20.0 for t in range(21)])
        self.beziers = {}

    def make_bezier(self, n):
        """Draw a Bezier curve"""
        try:
            return self.beziers[n]
        except KeyError:
            combinations = pascal_row(n - 1)
            result = []
            for t in self.tsequence:
                tpowers = (t ** i for i in range(n))
                upowers = ((1 - t) ** i for i in range(n - 1, -1, -1))
                coefs = [c * a * b for c, a, b in zip(combinations,
                                                      tpowers, upowers)]
                result.append(coefs)
            self.beziers[n] = result
            return result


class Captcha:
    """Captcha"""

    def __init__(self, width, height, fonts=None, color=None):
        self._image = None
        self._fonts = fonts if fonts else \
            [os.path.join(os.path.dirname(__file__), 'fonts', font)
             for font in ['Arial.ttf', 'Georgia.ttf', 'Action.ttf']]
        self._color = color if color else random_color(0, 200, random.randint(220, 255))
        self._width, self._height = width, height

    @classmethod
    def instance(cls, width=200, height=75):
        """Class method used to get a Captcha object"""
        prop_name = f'_instance_{width}_{height}'
        if not hasattr(cls, prop_name):
            setattr(cls, prop_name, cls(width, height))
        return getattr(cls, prop_name)

    def _background(self):
        """Draw the background"""
        Draw(self._image).rectangle([(0, 0), self._image.size],
                                    fill=random_color(230, 255))

    def _smooth(self):
        """Smooth the image"""
        return self._image.filter(ImageFilter.SMOOTH)

    def _curve(self, width=4, number=6, color=None):
        """Draw curves"""
        dx, height = self._image.size
        dx /= number
        path = [(dx * i, random.randint(0, height))
                for i in range(1, number)]
        bcoefs = Bezier().make_bezier(number - 1)
        points = []
        for coefs in bcoefs:
            points.append(tuple(sum([coef * p for coef, p in zip(coefs, ps)])
                                for ps in zip(*path)))
        Draw(self._image).line(points, fill=color if color else self._color, width=width)

    def _noise(self, number=50, level=2, color=None):
        """Draw interference code"""
        width, height = self._image.size
        dx, dy = width / 10, height / 10
        width, height = width - dx, height - dy
        draw = Draw(self._image)
        for i in range(number):
            x = int(random.uniform(dx, width))
            y = int(random.uniform(dy, height))
            draw.line(((x, y), (x + level, y)),
                      fill=color if color else self._color, width=level)

    def _text(self, captcha_text, fonts, font_sizes=None, drawings=None, squeeze_factor=0.75, color=None):
        """Draw text"""
        color = color if color else self._color
        fonts = tuple([truetype(name, size)
                       for name in fonts
                       for size in font_sizes or (65, 70, 75)])
        draw = Draw(self._image)
        char_images = []
        for c in captcha_text:
            font = random.choice(fonts)
            c_width, c_height = draw.textsize(c, font=font)
            char_image = Image.new('RGB', (c_width, c_height), (0, 0, 0))
            char_draw = Draw(char_image)
            char_draw.text((0, 0), c, font=font, fill=color)
            char_image = char_image.crop(char_image.getbbox())
            for drawing in drawings:
                d = getattr(self, drawing)
                char_image = d(char_image)
            char_images.append(char_image)
        width, height = self._image.size
        offset = int((width - sum(int(i.size[0] * squeeze_factor)
                                  for i in char_images[:-1]) -
                      char_images[-1].size[0]) / 2)
        for char_image in char_images:
            c_width, c_height = char_image.size
            mask = char_image.convert('L').point(lambda i: i * 1.97)
            self._image.paste(char_image,
                              (offset, int((height - c_height) / 2)),
                              mask)
            offset += int(c_width * squeeze_factor)

    @staticmethod
    def _warp(image, dx_factor=0.3, dy_factor=0.3):
        """Warp the image"""
        width, height = image.size
        dx = width * dx_factor
        dy = height * dy_factor
        x1 = int(random.uniform(-dx, dx))
        y1 = int(random.uniform(-dy, dy))
        x2 = int(random.uniform(-dx, dx))
        y2 = int(random.uniform(-dy, dy))
        warp_image = Image.new(
            'RGB',
            (width + abs(x1) + abs(x2), height + abs(y1) + abs(y2)))
        warp_image.paste(image, (abs(x1), abs(y1)))
        width2, height2 = warp_image.size
        return warp_image.transform(
            (width, height),
            Image.QUAD,
            (x1, y1, -x1, height2 - y2, width2 + x2, height2 + y2, width2 - x2, -y1))

    @staticmethod
    def _offset(image, dx_factor=0.1, dy_factor=0.2):
        """Offset the image"""
        width, height = image.size
        dx = int(random.random() * width * dx_factor)
        dy = int(random.random() * height * dy_factor)
        offset_image = Image.new('RGB', (width + dx, height + dy))
        offset_image.paste(image, (dx, dy))
        return offset_image

    @staticmethod
    def _rotate(image, angle=25):
        """Rotate the image"""
        return image.rotate(random.uniform(-angle, angle),
                            Image.BILINEAR, expand=1)

    def generate(self, captcha_text='', fmt='PNG'):
        """Generate the captcha (text and image)
        :param captcha_text: captcha text
        :param fmt: generated captcha image format
        :return: binary data of the captcha image
        """
        self._image = Image.new('RGB', (self._width, self._height), (255, 255, 255))
        self._background()
        self._text(captcha_text, self._fonts,
                   drawings=['_warp', '_rotate', '_offset'])
        self._curve()
        self._noise()
        self._smooth()
        image_bytes = BytesIO()
        self._image.save(image_bytes, format=fmt)
        return image_bytes.getvalue()


def pascal_row(n=0):
    """Generate Pascal's triangle row"""
    result = [1]
    x, numerator = 1, n
    for denominator in range(1, n // 2 + 1):
        x *= numerator
        x /= denominator
        result.append(x)
        numerator -= 1
    if n & 1 == 0:
        result.extend(reversed(result[:-1]))
    else:
        result.extend(reversed(result))
    return result


def random_color(start=0, end=255, opacity=255):
    """Get a random color"""
    red = random.randint(start, end)
    green = random.randint(start, end)
    blue = random.randint(start, end)
    if opacity is None:
        return red, green, blue
    return red, green, blue, opacity
```

### Reading and Writing Cookies in View Functions

Next, let us explain in more detail how to use cookies, so that it is easier for everyone to use this technology in Web projects. The `HttpRequest` and `HttpResponse` objects wrapped by Django provide operations for reading and writing cookies.

Properties and methods wrapped by `HttpRequest`:

1. The `COOKIES` property: this property contains all cookies carried in the HTTP request.
2. The `get_signed_cookie` method: get a signed cookie. If signature validation fails, a `BadSignature` exception is raised.

Methods wrapped by `HttpResponse`:

1. The `set_cookie` method: this method can set a group of key-value pairs, and they will finally be written to the browser.
2. The `set_signed_cookie` method: this method is similar to the one above, but it signs the cookie to prevent tampering. Because if the data in the cookie is changed, it is impossible to generate a valid signature without knowing the [key](https://en.wikipedia.org/wiki/Cryptographic_key) and [salt](https://en.wikipedia.org/wiki/Salt_(cryptography)). Then when the server reads the cookie, it will find the data and signature do not match, and a `BadSignature` exception will be raised. It should be explained that the key here is the `SECRET_KEY` specified in the Django project configuration file, and the salt is a string set in the program. You can set it to whatever you like, as long as it is a valid string.

If you are not clear about how to use the methods mentioned above, you can read the Django [official documentation](https://docs.djangoproject.com/en/2.1/ref/request-response/) by yourself. No material can tell you more clearly than the official documentation how these methods are used.

As we just said, after `SessionMiddleware` is enabled, every `HttpRequest` object is bound to a `session` attribute. It is a dict-like object. Besides saving user data, it also provides methods to test whether the browser supports cookies, including:

1. The `set_test_cookie` method: set a cookie for testing.
2. The `test_cookie_worked` method: test whether the test cookie works.
3. The `delete_test_cookie` method: delete the cookie used for testing.
4. The `set_expiry` method: set the expiration time of the session.
5. The `get_expire_age` / `get_expire_date` methods: get the expiration time of the session.
6. The `clear_expired` method: clear expired sessions.

Below is the code that checks whether the browser supports cookies before login. Usually, browsers enable cookie support by default, but the user may disable browser cookies for some reason. In this case, we can provide a checking function in the view. If the user's browser does not support cookies, we can give a matching prompt.

```python
def login(request):
    if request.method == 'POST':
        if request.session.test_cookie_worked():
            request.session.delete_test_cookie()
            # Add your code to perform login process here
        else:
            return HttpResponse("Please enable cookies and try again.")
    request.session.set_test_cookie()
    return render_to_response('login.html')
```

### Alternatives to Cookies

As we said before, cookies have never had a very good reputation. Of course, in real development, we do not save sensitive user information in cookies, such as user passwords or credit card account numbers. Also, the data saved in cookies is usually encoded and signed. For browsers that support HTML5, we can use localStorage and sessionStorage as alternatives to cookies. You can probably hear the difference from the names: data stored in `localStorage` can be kept for a long time; data stored in `sessionStorage` will be cleared when the browser is closed. For how to use these cookie alternatives, it is recommended that everyone read [MDN](https://developer.mozilla.org/docs/Web).
