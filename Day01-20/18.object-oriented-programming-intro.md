## Introdução à Programação Orientada a Objetos

A programação orientada a objetos é um **paradigma de programação** muito popular, ou seja, uma metodologia para design de programas. Simplificando, é como os programadores entendem os programas e escrevem códigos.

Anteriormente, dissemos que **um programa é uma coleção de instruções**. Quando um programa é executado, suas instruções são transformadas em instruções executadas pela CPU. Para simplificar o design do programa, introduzimos funções, que nos permitem **empacotar código relativamente independente e frequentemente reutilizado**. Se uma função ficar muito grande, podemos dividi-la em funções menores para reduzir a complexidade.

Se você pensar bem, programar é realmente a pessoa que escreve o programa controlando a máquina por meio de código de acordo com a forma como o computador funciona. No entanto, a forma como os computadores funcionam é diferente da forma normal de pensar dos seres humanos. Isso não quer dizer que não possamos escrever código de acordo com a forma como os computadores funcionam, mas quando precisarmos desenvolver um sistema complexo, esta forma tornará o código muito complexo e isso, por sua vez, tornará o desenvolvimento e a manutenção muito difíceis.

À medida que a complexidade do software aumenta, escrever código correto e confiável torna-se uma tarefa extremamente difícil. Esta é também uma das razões pelas quais muitas pessoas acreditam firmemente que “o desenvolvimento de software é a mais complexa de todas as atividades humanas para transformar o mundo”. Portanto, como usar programas para descrever sistemas complexos e resolver problemas complexos tornou-se algo que todo programador deve pensar e enfrentar diretamente. A linguagem Smalltalk, nascida na década de 1970, permitiu que os desenvolvedores de software vissem esperança, porque introduziu um novo paradigma de programação chamado programação orientada a objetos. No mundo da programação orientada a objetos, **os dados em um programa e as funções que operam nos dados são um todo lógico**, e chamamos isso de **objeto**. **Objetos podem receber mensagens**, e a maneira de resolver problemas é **criar objetos e enviar todos os tipos de mensagens para eles**. Através da passagem de mensagens, vários objetos em um programa podem trabalhar juntos e, então, construir sistemas complexos e resolver problemas reais.

> **Observação**: Muitas linguagens de programação avançadas que usamos hoje oferecem suporte à programação orientada a objetos, mas a programação orientada a objetos não é a solução mágica para resolver todos os problemas no desenvolvimento de software. Na verdade, ainda não existe a chamada solução mágica no desenvolvimento de software. Se você estiver interessado neste ponto, você pode ler o artigo de Frederick Brooks *No Silver Bullet* ou o clássico livro de engenharia de software *The Mythical Man-Month*.

### Classes e objetos

Uma descrição concisa da programação orientada a objetos é esta:

> **Programação orientada a objetos**: organize os dados e os métodos que operam neles em **objetos**, agrupe objetos com comportamento semelhante em **classes**, oculte a implementação interna por meio de **encapsulamento**, especialize-se e generalize por meio de **herança** e obtenha envio dinâmico com base no tipo de objeto por meio de **polimorfismo**.

As palavras-chave aqui são:

- **objeto**
- **aula**
- **encapsulamento**
- **herança**
- **polimorfismo**

Na programação orientada a objetos, **uma classe é um conceito abstrato**, enquanto **um objeto é um conceito concreto**. Extraímos as características comuns de um tipo de objeto e isso se torna uma classe. Resumindo, **uma classe é o modelo e modelo de um objeto, e um objeto é uma instância de uma classe, uma entidade concreta que pode receber mensagens**.

No mundo da programação orientada a objetos:

- tudo é um objeto
- todo objeto tem atributos e comportamento
- cada objeto é único
- todo objeto pertence a alguma classe

Os atributos são as propriedades estáticas de um objeto e o comportamento é a parte dinâmica.

### Definindo uma Classe

Em Python, definimos uma classe com a palavra-chave `class`. Dentro do corpo da classe, escrevemos funções que representam comportamentos comuns dos objetos daquela classe. Funções definidas dentro de uma classe são geralmente chamadas de **métodos**. O primeiro parâmetro de um método de instância geralmente é `self`, que representa o objeto que recebe a mensagem.

```python
class Student:

    def study(self, course_name):
        print(f'The student is studying {course_name}.')

    def play(self):
        print('The student is playing games.')
```

### Criando e usando objetos

Depois de definirmos uma classe, podemos usar a sintaxe do construtor para criar objetos, conforme mostrado abaixo.

```python
stu1 = Student()
stu2 = Student()
print(stu1)
print(stu2)
print(hex(id(stu1)), hex(id(stu2)))
```

Colocar parênteses após o nome da classe é a sintaxe do construtor. O código acima cria dois objetos de aluno, um atribuído a `stu1` e outro atribuído a `stu2`. Quando usamos `print` para gerar `stu1` e `stu2`, podemos ver o endereço do objeto na memória. Agora podemos dizer a todos que as variáveis ​​que definimos na verdade salvam o endereço lógico de um objeto na memória. Através deste endereço lógico podemos encontrar o objeto na memória. Portanto, uma atribuição como `stu3 = stu2` não cria um novo objeto. Ele usa apenas uma nova variável para salvar o endereço de um objeto existente.

Agora vamos enviar mensagens para objetos chamando métodos.

```python
Student.study(stu1, 'Python Programming')
stu1.study('Python Programming')

Student.play(stu2)
stu2.play()
```

A segunda forma, `object.method(...)`, é o estilo usual.

### Método de inicialização

Até agora, nossos objetos estudantis têm comportamento, mas nenhum atributo. Para definir atributos, podemos adicionar um método denominado `__init__`. Quando um objeto é criado, o Python aloca memória para ele e chama automaticamente `__init__` para inicializar seus atributos.

```python
class Student:
    """student"""

    def __init__(self, name, age):
        """Initialization method"""
        self.name = name
        self.age = age

    def study(self, course_name):
        """learn"""
        print(f'{self.name}Learning{course_name}.')

    def play(self):
        """play"""
        print(f'{self.name}Playing a game.')
```

Altere o código para criar objetos e enviar mensagens aos objetos, execute-o novamente e veja o que muda na saída do programa.

```python
stu1 = Student('Luo Hao', 44)
stu2 = Student('Wang Dahuang', 25)
stu1.study('PythonProgramming')    # Luo Hao is learning Python programming.
stu2.play()                    # Sledgehammer Wang is playing a game.
```

### Os Pilares da OOP

Os três pilares da programação orientada a objetos são:

- **encapsulamento**
- **herança**
- **polimorfismo**

Por enquanto, vamos nos concentrar no encapsulamento. Minha própria compreensão do encapsulamento é:

> **Oculte todos os detalhes de implementação que podem ser ocultados e exponha apenas uma interface de chamada simples para o mundo externo.**

Os métodos que definimos em uma classe são, na verdade, um tipo de encapsulamento. Esse tipo de encapsulamento nos permite criar um objeto e depois apenas enviar uma mensagem ao objeto para executar o código do método. Em outras palavras, podemos usar o método quando conhecemos apenas o nome e os parâmetros do método, que são a visão externa do método, e não conhecemos os detalhes internos da implementação do método, que são a visão interna.

Deixe-nos dar um exemplo. Suponha que eu queira controlar um robô para me servir um copo d’água. Se não usarmos programação orientada a objetos e não fizermos nenhum encapsulamento, então precisaremos enviar uma série de instruções ao robô, como levantar, virar à esquerda, caminhar 5 passos para frente, pegar o copo da frente, virar, caminhar 10 passos para frente, abaixar-se, pousar o copo, pressionar o botão de água, esperar 10 segundos, soltar o botão de água, pegar o copo, virar à direita, caminhar 5 passos para frente e pousar o copo, antes de terminar esta ação simples. Só de pensar nisso parece problemático. De acordo com a programação orientada a objetos, podemos empacotar a ação de despejar água em um método do robô. Quando precisamos que o robô despeje água, precisamos apenas enviar a mensagem de água derramada para o objeto do robô. Isso não é melhor?

Em muitas cenas, a programação orientada a objetos é na verdade um problema de três etapas:

1. definir classes
2. criar objetos
3. enviar mensagens para objetos

Claro que às vezes não precisamos do primeiro passo, pois a classe que queremos utilizar pode já existir. Como dissemos antes, `list`, `set` e `dict` integrados do Python são, na verdade, todas classes. Se quisermos criar objetos de lista, conjunto ou dicionário, não precisaremos definir essas classes nós mesmos. É claro que algumas classes não são fornecidas diretamente pela biblioteca padrão do Python. Eles podem vir de código de terceiros. Como instalar e usar código de terceiros será discutido mais adiante. Em algumas cenas especiais, usamos os chamados **objetos integrados**. Os chamados objetos integrados significam que nas três etapas acima, tanto a primeira quanto a segunda etapas não são necessárias, porque a classe já existe e o objeto já foi criado. Precisamos apenas enviar mensagens diretamente ao objeto. Isso é o que as pessoas costumam chamar de “fora da caixa”.

### Exemplos Orientados a Objetos

#### Exemplo 1: Relógio

> **Tarefa**: Definir uma classe que represente um relógio digital com métodos para avançar a hora e exibi-la.

```python
import time


class Clock:
    """Digital clock"""

    def __init__(self, hour=0, minute=0, second=0):
        self.hour = hour
        self.min = minute
        self.sec = second

    def run(self):
        self.sec += 1
        if self.sec == 60:
            self.sec = 0
            self.min += 1
            if self.min == 60:
                self.min = 0
                self.hour += 1
                if self.hour == 24:
                    self.hour = 0

    def show(self):
        return f'{self.hour:0>2d}:{self.min:0>2d}:{self.sec:0>2d}'


clock = Clock(23, 59, 58)
while True:
    print(clock.show())
    time.sleep(1)
    clock.run()
```

#### Exemplo 2: Um ponto em um plano

> **Tarefa**: Definir uma classe que represente um ponto em um plano e fornecer um método para calcular a distância até outro ponto.

```python
class Point:
    """A point on a plane"""

    def __init__(self, x=0, y=0):
        self.x, self.y = x, y

    def distance_to(self, other):
        dx = self.x - other.x
        dy = self.y - other.y
        return (dx * dx + dy * dy) ** 0.5

    def __str__(self):
        return f'({self.x}, {self.y})'


p1 = Point(3, 5)
p2 = Point(6, 9)
print(p1)
print(p2)
print(p1.distance_to(p2))
```

### Resumo

A programação orientada a objetos é um paradigma de programação muito popular. Além dela, existem também paradigmas como **programação imperativa** e **programação funcional**. Como o mundo real é composto de objetos, e os objetos são entidades que podem receber mensagens, **a programação orientada a objetos se adapta aos hábitos normais de pensamento humano com mais naturalidade**. As classes são abstratas e os objetos são concretos. Assim que tivermos classes, podemos criar objetos; assim que tivermos objetos, eles poderão receber mensagens. Esta é a base da programação orientada a objetos. O processo de definição de uma classe é um processo abstrato. Encontrar as propriedades comuns dos objetos pertence à abstração de dados, e encontrar os métodos comuns dos objetos pertence à abstração de comportamento. O processo de abstração é algo em que pessoas diferentes podem obter resultados diferentes para o mesmo tipo de objeto.
