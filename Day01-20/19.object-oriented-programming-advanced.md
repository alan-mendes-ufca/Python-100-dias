## Programação Avançada Orientada a Objetos

Na lição anterior, apresentamos os fundamentos da programação orientada a objetos em Python. Nesta lição, continuamos com tópicos OOP mais avançados.

### Visibilidade e Decoradores de Propriedades

Em muitas linguagens de programação orientadas a objetos, os atributos dos objetos são geralmente definidos como membros privados ou protegidos. Simplesmente falando, isso significa que esses atributos não podem ser acessados ​​diretamente. Os métodos de um objeto geralmente são públicos, porque os métodos públicos são as mensagens que um objeto pode receber e também são a interface de chamada exposta pelo objeto ao mundo exterior. Esta é a chamada visibilidade de acesso.

Em Python, podemos usar um prefixo de sublinhado no nome do atributo de um objeto para mostrar a visibilidade do acesso. Por exemplo:

- `__name` sugere um atributo privado
- `_name` sugere um atributo protegido

```python
class Student:

    def __init__(self, name, age):
        self.__name = name
        self.__age = age

    def study(self, course_name):
        print(f'{self.__name} is studying {course_name}.')


stu = Student('Wang Dachui', 20)
stu.study('PythonProgramming')
print(stu.__name)  # AttributeError: 'Student' object has no attribute '__name'
```

A última linha do código acima gera `AttributeError`. A partir disso podemos ver que o atributo `__name` começando com `__` é como um atributo privado e não pode ser acessado diretamente fora da classe. Mas o método `study` dentro da classe pode acessar este atributo através de `self.__name`. Deve ser explicado que a maioria das pessoas que usam Python geralmente não escolhem tornar os atributos dos objetos privados ou protegidos ao definir classes. Como diz um ditado famoso: “**Somos todos adultos aqui consentidos**”. Os adultos podem ser responsáveis ​​pelo seu próprio comportamento e não precisam do próprio Python para restringir a visibilidade do acesso. Na verdade, a maioria dos programadores acredita que **aberto é melhor que fechado**, e tornar os atributos do objeto privados não é absolutamente necessário. Portanto, o Python não faz a limitação mais estrita semanticamente. Em outras palavras, se você realmente quiser, ainda poderá acessar o atributo privado `__name` acima usando `stu._Student__name`.

### Atributos Dinâmicos

Python é uma linguagem dinâmica. Uma explicação comum na Wikipédia é que se trata de uma linguagem cuja estrutura pode ser alterada em tempo de execução. Por exemplo, novas funções, objetos e até mesmo código podem ser introduzidos, e funções existentes podem ser excluídas ou outras mudanças estruturais podem ocorrer. Em Python, podemos adicionar atributos dinamicamente aos objetos. Este é um privilégio do Python como linguagem dinâmica.

```python
class Student:

    def __init__(self, name, age):
        self.name = name
        self.age = age


stu = Student('Wang Dahuang', 20)
stu.sex = 'male'
```

Se você não deseja adicionar atributos dinamicamente a um objeto ao usá-lo, você pode usar a mágica `__slots__`.

```python
class Student:
    __slots__ = ('name', 'age')

    def __init__(self, name, age):
        self.name = name
        self.age = age


stu = Student('Wang Dahuang', 20)
# AttributeError: 'Student' object has no attribute 'sex'
stu.sex = 'male'
```

### Métodos estáticos e métodos de classe

Os métodos que definimos antes eram todos métodos de instância. Além dos métodos de instância, uma classe também pode ter **métodos estáticos** e **métodos de classe**. Esses dois tipos de métodos são mensagens enviadas para a classe. Não há diferença essencial entre eles. No mundo orientado a objetos, tudo é um objeto. Cada classe que definimos também é na verdade um objeto, e métodos estáticos e métodos de classe são mensagens enviadas para objetos de classe.

Por exemplo, defina uma classe de triângulo. Use três comprimentos laterais para construir um triângulo e forneça métodos para calcular perímetro e área. Calcular perímetro e área são claramente métodos de um objeto triangular. Mas ao criar um objeto triangular, os comprimentos dos três lados passados ​​podem não ser capazes de formar um triângulo. Portanto, podemos primeiro escrever um método para verificar se três comprimentos de lados determinados podem formar um triângulo. Esse tipo de método claramente não é um método de objeto, porque o objeto triângulo ainda não foi criado ao chamar esse método. Podemos projetar esse tipo de método como um método estático ou um método de classe.

```python
class Triangle(object):
    """Triangle"""

    def __init__(self, a, b, c):
        self.a = a
        self.b = b
        self.c = c

    @staticmethod
    def is_valid(a, b, c):
        """Check whether three sides can form a triangle."""
        return a + b > c and b + c > a and a + c > b

    def perimeter(self):
        return self.a + self.b + self.c

    def area(self):
        p = self.perimeter() / 2
        return (p * (p - self.a) * (p - self.b) * (p - self.c)) ** 0.5


if Triangle.is_valid(3, 4, 5):
    t = Triangle(3, 4, 5)
    print(f'Perimeter: {t.perimeter()}')
    print(f'Area: {t.area()}')
else:
    print('Invalid side length!!!')
```

O código acima usa o decorador `staticmethod` para declarar `is_valid` como um método estático da classe `Triangle`. Se quisermos declarar um método de classe, podemos usar o decorador `classmethod`. Podemos usar `ClassName.method_name` diretamente para chamar métodos estáticos e métodos de classe. A diferença entre eles é que o primeiro parâmetro de um método de classe é o próprio objeto de classe, enquanto um método estático não possui esse parâmetro. Para resumir brevemente, **métodos de instância, métodos de classe e métodos estáticos podem ser chamados no formato `ClassName.method_name(...)`; a diferença é se o primeiro parâmetro do método é um objeto comum, um objeto de classe ou se não há nenhum objeto recebendo a mensagem**. Métodos estáticos geralmente também podem ser escritos diretamente como funções independentes, porque não estão vinculados a um objeto específico.

Aqui, vamos adicionar mais uma explicação. Podemos adicionar o decorador `property` aos métodos que calculam o perímetro e a área do triângulo. Desta forma, `perimeter` e `area` da classe triângulo tornam-se dois atributos. Não os acessamos mais chamando métodos, mas os obtemos diretamente pelo acesso aos atributos do objeto. O código alterado é mostrado abaixo.

```python
class Triangle(object):
    """Triangle"""

    def __init__(self, a, b, c):
        self.a = a
        self.b = b
        self.c = c

    @staticmethod
    def is_valid(a, b, c):
        return a + b > c and b + c > a and a + c > b

    @property
    def perimeter(self):
        return self.a + self.b + self.c

    @property
    def area(self):
        p = self.perimeter / 2
        return (p * (p - self.a) * (p - self.b) * (p - self.c)) ** 0.5


if Triangle.is_valid(3, 4, 5):
    t = Triangle(3, 4, 5)
    print(f'Perimeter: {t.perimeter}')
    print(f'Area: {t.area}')
else:
    print('Invalid side length!!!')
```

### Herança e Polimorfismo

As linguagens de programação orientadas a objetos suportam a criação de novas classes baseadas em classes existentes, para que o código duplicado possa ser reduzido. A classe que fornece informações de herança é chamada de **classe pai** (ou classe base), e a classe que obtém informações de herança é chamada de **classe filha** (ou classe derivada).

Por exemplo, se definirmos uma classe de alunos e uma classe de professores, descobriremos que elas têm muito código duplicado, e esse código duplicado são propriedades e comportamentos comuns de professores e alunos como pessoas. Portanto, neste caso, devemos primeiro definir uma classe `Person` e, em seguida, derivar `Student` e `Teacher` da classe `Person`.

```python
class Person:
    """Person"""

    def __init__(self, name, age):
        self.name = name
        self.age = age

    def eat(self):
        print(f'{self.name} is eating.')

    def sleep(self):
        print(f'{self.name} is sleeping.')


class Student(Person):
    """Student"""

    def __init__(self, name, age):
        super().__init__(name, age)

    def study(self, course_name):
        print(f'{self.name} is studying {course_name}.')


class Teacher(Person):
    """Teacher"""

    def __init__(self, name, age, title):
        super().__init__(name, age)
        self.title = title

    def teach(self, course_name):
        print(f'{self.name} {self.title} is teaching {course_name}.')


stu1 = Student('Bai Yuanfang', 21)
stu2 = Student('Di Renjie', 22)
tea1 = Teacher('Wu Zetian', 35, 'Associate Professor')
stu1.eat()
stu2.sleep()
tea1.eat()
stu1.study('PythonProgramming')
tea1.teach('PythonProgramming')
stu2.study('Introduction to Data Science')
```

A sintaxe da herança consiste em especificar a classe pai da classe atual entre parênteses após o nome da classe ao definir a classe. Se uma classe não especificar sua classe pai, a classe pai padrão será `object`. A classe `object` é a classe de nível superior em Python, e isso significa que todas as classes são suas classes filhas, herdando dela direta ou indiretamente. Python também permite herança múltipla, o que significa que uma classe pode ter uma ou mais classes pai. Discutiremos a herança múltipla com mais detalhes posteriormente. No método de inicialização de uma classe filha, podemos chamar o método de inicialização da classe pai por meio de `super().__init__()`. A função `super` é uma função Python integrada especialmente projetada para obter o objeto da classe pai do objeto atual.

Uma classe filha também pode **substituir** métodos herdados da classe pai. Diferentes classes filhas podem fornecer diferentes versões de implementação do mesmo método na classe pai. Métodos como esse mostrarão comportamento polimórfico quando o programa for executado, o que significa que chamar o mesmo método faz coisas diferentes. O polimorfismo é a parte mais essencial da programação orientada a objetos e, claro, também é a parte mais difícil para os iniciantes realmente entenderem e usarem com flexibilidade.

### Resumo

Python é uma linguagem de tipo dinâmico e objetos em Python podem ter atributos adicionados dinamicamente. Os métodos de um objeto também são atributos em essência, exceto que o atributo corresponde a uma função que pode ser chamada. No mundo orientado a objetos, **tudo é um objeto**, e até mesmo **classes podem receber mensagens**. Através da herança, **podemos criar novas classes a partir de classes existentes** e então reutilizar o código das classes existentes.
