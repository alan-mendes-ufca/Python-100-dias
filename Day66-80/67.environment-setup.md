## Configuração do ambiente

Se você deseja começar a usar Python rapidamente para trabalhos relacionados à ciência de dados, é recomendável instalar diretamente o Anaconda e, em seguida, usar as ferramentas Notebook ou JupyterLab integradas ao Anaconda para escrever código. Isso ocorre porque, para iniciantes, instalar primeiro o interpretador Python oficial e depois instalar as bibliotecas de terceiros que serão usadas no trabalho, uma por uma, é relativamente problemático. Especialmente no ambiente Windows, a instalação geralmente falha devido à falta de ferramentas de construção ou arquivos DLL, e os iniciantes geralmente acham difícil tomar a ação correta com base na mensagem de erro, por isso é fácil desenvolver um forte sentimento de frustração. Se já houver um ambiente interpretador Python no computador, você também pode usar diretamente a ferramenta de gerenciamento de pacotes do Python `pip` para instalar o Jupyter e, em seguida, instalar bibliotecas de terceiros de acordo com as necessidades reais de trabalho. Esta forma é mais indicada para usuários que já possuem alguma experiência.

### Instalando e usando o Anaconda

Para usuários individuais, você pode baixar o instalador "Individual Edition" do [Site oficial da Anaconda](https://www.anaconda.com/). Após a conclusão da instalação, seu computador não terá apenas um ambiente Python e Spyder, que é uma ferramenta de desenvolvimento integrada semelhante ao PyCharm, mas também terá cerca de 200 pacotes de ferramentas relacionadas ao trabalho de ciência de dados, incluindo as três grandes ferramentas de análise de dados Python que mencionamos acima. Além disso, o Anaconda também fornece uma ferramenta de gerenciamento de pacotes chamada `conda`. Através desta ferramenta, podemos não apenas gerenciar pacotes Python, mas também usá-la para criar ambientes virtuais para execução de programas Python.

Durante a instalação, as configurações padrão geralmente funcionam bem. Uma vez que já tenhamos um ambiente Python, podemos instalar completamente as bibliotecas de terceiros que precisamos de acordo com nossos desejos.**

#### Comandos `conda`

Para usuários não iniciantes, se desejar usar a ferramenta `conda` para gerenciar dependências ou criar ambientes virtuais para projetos, você pode usar os comandos `conda` no terminal ou prompt de comando. Os usuários do Windows podem encontrar `Anaconda3` no menu Iniciar e clicar em `Anaconda Prompt` ou `Anaconda PowerShell` para iniciar um prompt de linha de comando que suporte `conda`. Se usuários iniciantes desejam criar novos ambientes virtuais ou gerenciar bibliotecas de terceiros, ou seja, dependências, é recomendado usar diretamente `Environments` em `Anaconda-Navigator` para gerenciar visualmente ambientes virtuais e dependências.

1. Versão e informações de ajuda.

   - Verifique a versão: `conda -V` ou `conda --version`
   - Obtenha ajuda: `conda -h` ou `conda --help`
   - Informações relacionadas: `conda list`

2. Comandos relacionados ao ambiente virtual.

   - Mostrar todos os ambientes virtuais: `conda env list`
   - Crie um ambiente virtual: `conda create --name venv`
   - Crie um ambiente virtual com uma versão específica do Python: `conda create --name venv python=3.7`
   - Crie um ambiente virtual com uma versão específica do Python e instale as dependências especificadas: `conda create --name venv python=3.7 numpy pandas`
   - Crie um ambiente virtual clonando um ambiente virtual existente: `conda create --name venv2 --clone venv`
   - Compartilhe um ambiente virtual e redirecione-o para um arquivo especificado: `conda env export > environment.yml`
   - Crie um ambiente virtual a partir de um arquivo de ambiente virtual compartilhado: `conda env create -f environment.yml`
   - Ative um ambiente virtual: `conda activate venv`
   - Deixe um ambiente virtual: `conda deactivate`
   - Excluir um ambiente virtual: `conda remove --name venv --all`

   > **Nota**: Nos comandos acima, `venv` e `venv2` são os nomes das pastas do ambiente virtual. Você pode substituí-los por qualquer nome que desejar, mas é **altamente recomendado** usar nomes em inglês e não usar caracteres especiais.

3. Pacote, significando biblioteca ou ferramenta de terceiros, gerenciamento.

   - Mostrar pacotes instalados: `conda list`
   - Procure um pacote especificado: `conda search matplotlib`
   - Instale um pacote especificado: `conda install matplotlib`
   - Atualizar um pacote especificado: `conda update matplotlib`
   - Remover um pacote especificado: `conda remove matplotlib`

   > **Nota**: Ao pesquisar, instalar e atualizar pacotes, ele se conecta ao site oficial por padrão. Se você acha que a velocidade não é forte o suficiente, você pode substituir o site oficial padrão por um site espelho doméstico. O site espelho de código aberto da Universidade Tsinghua é recomendado. Os comandos para alterar a fonte padrão para um espelho doméstico são `conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/` e `conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main`. Se precisar voltar para a fonte padrão, você pode usar o comando `conda config --remove-key channels`.

### Instalando e usando JupyterLab

#### Instalação e inicialização

Se o Anaconda já estiver instalado, você pode iniciar diretamente o Notebook ou JupyterLab em `Anaconda-Navigator` conforme mencionado acima. De acordo com o comunicado oficial, o JupyterLab é a próxima geração de Notebook e oferece uma interface mais amigável e funções mais poderosas, por isso também recomendamos que todos usem o JupyterLab. Os usuários do Windows também podem abrir `Anaconda Prompt` ou `Anaconda PowerShell` no menu Iniciar. Como o ambiente virtual Anaconda padrão já está ativado, você só precisa inserir o comando `jupyter lab` para iniciar o JupyterLab. No macOS, após a instalação do Anaconda, o ambiente virtual Anaconda padrão é ativado automaticamente toda vez que você abre o terminal, então você também pode iniciar o JupyterLab digitando o comando `jupyter lab`.

Para usuários que instalaram um ambiente Python, mas não o Anaconda, você pode usar a ferramenta de gerenciamento de pacotes do Python `pip` para instalar o JupyterLab. Após a instalação ser bem-sucedida, execute o comando `jupyter lab` no terminal ou prompt de comando para iniciar o JupyterLab, conforme mostrado abaixo.

Instale o JupyterLab:

```bash
pip install jupyterlab
```

Instale as três excelentes ferramentas de análise de dados Python:

```bash
pip install numpy pandas matplotlib
```

Inicie o JupyterLab:

```bash
jupyter lab
```

JupyterLab é um aplicativo baseado na web para computação interativa. Ele pode ser usado para desenvolvimento de código, escrita de documentos, execução de código e apresentação de resultados. Simplificando, você pode **escrever código** e **executar código** diretamente em uma página da web, e os resultados da execução do código também são mostrados diretamente abaixo da célula de código. Se precisar escrever documentos explicativos enquanto escreve o código, você pode escrevê-los na mesma página no formato Markdown e ver diretamente o efeito renderizado. Além disso, o objetivo original do design do Notebook era fornecer um ambiente de trabalho que pudesse suportar muitas linguagens de programação. Atualmente, pode suportar mais de 40 linguagens de programação, incluindo Python, R, Julia, Scala e assim por diante.

#### Dicas de uso

Se você usa Python para desenvolvimento de projetos no estilo engenharia, PyCharm é definitivamente a melhor escolha. Ele fornece todas as funções que um ambiente de desenvolvimento integrado deve ter. Em particular, funções como dicas inteligentes, conclusão de código e correção automática de erros fazem com que os desenvolvedores se sintam muito confortáveis. Se você usa Python para trabalhos relacionados à ciência de dados, o JupyterLab não é inferior ao PyCharm, e o JupyterLab é ainda melhor na apresentação de dados e gráficos. Por conta disso, a JetBrains também desenvolveu especialmente uma nova ferramenta, DataSpell, para competir com o JupyterLab. Os leitores interessados ​​​​podem aprender sobre isso por si próprios. Abaixo, apresentamos algumas dicas para usar o JupyterLab, esperando que possam ajudar todos a melhorar a eficiência do trabalho.

1. Preenchimento automático. Ao escrever código no JupyterLab, pressionar a tecla `Tab` fornece dicas e conclusão do código.

2. Obtendo ajuda. Se quiser saber as informações relacionadas ou o uso de um objeto, como uma variável, classe ou função, você pode colocar `?` após o objeto e executar o código. As informações correspondentes serão mostradas abaixo da janela para nos ajudar a entender o objeto, conforme mostrado abaixo.

   <img src="res/JupyterLab_3.png" style="zoom:100%;">

3. Procurando nomes. Se você lembrar apenas parte do nome de uma classe ou função, poderá usar o curinga `*` junto com `?` para pesquisar, conforme mostrado abaixo.

   <img src="res/JupyterLab_4.png" style="zoom:100%;">

4. Executando comandos. No JupyterLab, você pode executar comandos do sistema colocando `!` antes do comando do sistema.

5. Comandos mágicos. Existem muitos comandos mágicos muito interessantes e úteis no JupyterLab. Por exemplo, você pode usar `%timeit` para testar o tempo de execução de uma instrução e `%pwd` para ver o diretório de trabalho atual. Se quiser ver todos os comandos mágicos, você pode usar `%lsmagic`. Se você quiser entender como os comandos mágicos são usados, você pode usar `%magic` para ver, conforme mostrado abaixo.

   <img src="res/JupyterLab_5.png" style="zoom:100%;">

Comandos mágicos comuns são mostrados abaixo.

    | Comando Mágico | Descrição |
   | ------------- | ----------- |
    | `%pwd` | Mostrar o diretório de trabalho atual |
    | `%ls` | Liste o conteúdo da pasta atual ou especificada |
    | `%cat` | Mostrar o conteúdo do arquivo especificado |
    | `%hist` | Mostrar histórico de entrada |
    | `%matplotlib inline` | Defina gráficos de saída matplotlib para serem incorporados na página |
    | `%config Inlinebackend.figure_format='svg'` | Defina gráficos para usar o formato SVG |
    | `%run` | Execute o programa especificado |
    | `%load` | Carregue o arquivo especificado em uma célula |
    | `%quickref` | Mostre a referência rápida do IPython |
    | `%timeit` | Execute o código várias vezes e conte o tempo de execução |
    | `%prun` | Use `cProfile.run` para executar o código e mostrar a saída do criador de perfil |
    | `%who` / `%whos` | Mostrar as variáveis ​​no namespace |
    | `%xdel` | Exclua um objeto e limpe todas as referências a ele |

6. Atalhos. Muitas operações no JupyterLab podem ser realizadas por meio de atalhos, e o uso de atalhos pode melhorar a eficiência do trabalho. Os atalhos do JupyterLab podem ser divididos em atalhos no modo de comando e atalhos no modo de edição. O chamado modo de edição é o modo em que você insere código ou escreve documentos. No modo de edição, pressionar `Esc` pode voltar ao modo de comando. No modo de comando, pressionar `Enter` pode entrar no modo de edição.

Atalhos no modo de comando:

    | Atalho | Descrição |
   | -------- | ----------- |
    | `Alt` + `Enter` | Execute a célula atual e insira uma nova célula abaixo |
    | `Shift` + `Enter` | Execute a célula atual e selecione a célula abaixo |
    | `Ctrl` + `Enter` | Execute a célula atual |
    | `j` / `k`, `Shift` + `j` / `Shift` + `k` | Selecione a célula abaixo/acima, selecione continuamente as células abaixo/acima |
    | `a` / `b` | Insira uma nova célula abaixo/acima |
    | `c` / `x` | Copiar uma célula / recortar uma célula |
    | `v` / `Shift` + `v` | Cole uma célula abaixo/acima |
    | `dd` / `z` | Excluir uma célula / restaurar uma célula excluída |
    | `Shift` + `l` | Mostrar ou ocultar números de linha para as células atuais/todas |
    | `Space` / `Shift` + `Space` | Role a página para baixo/para cima |

Atalhos no modo de edição:

    | Atalho | Descrição |
   | -------- | ----------- |
    | `Shift` + `Tab` | Obtenha informações de dicas |
    | `Ctrl` + `]` / `Ctrl` + `[` | Aumentar/diminuir recuo |
    | `Alt` + `Enter` | Execute a célula atual e insira uma nova célula abaixo |
    | `Shift` + `Enter` | Execute a célula atual e selecione a célula abaixo |
    | `Ctrl` + `Enter` | Execute a célula atual |
    | `Ctrl` + `Left` / `Right` | Mova o cursor para o início/fim da linha |
    | `Ctrl` + `Up` / `Down` | Mova o cursor para o início/fim do código |
    | `Up` / `Down` | Mova o cursor para cima/baixo uma linha ou vá para a célula anterior/seguinte |

   > **Observação**: no macOS, você pode substituir a chave `Alt` pela chave `Option` e substituir a chave `Ctrl` pela chave `Command`.
