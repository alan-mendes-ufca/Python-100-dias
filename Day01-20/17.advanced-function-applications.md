## Aplicativos de funções avançadas

Na lição anterior, exploramos funções de ordem superior em Python e acredito que agora todos têm uma compreensão mais profunda da definição e do uso de funções. Nesta lição, continuaremos a falar sobre conhecimento relacionado a funções. Um são os decoradores, que são uma sintaxe distinta em Python, e o outro são as chamadas de função recursivas.

### Decoradores

Em Python, um decorador é um recurso de sintaxe que **usa uma função para decorar outra função e dar a ela habilidade extra**. O próprio decorador é uma função. Seu parâmetro é a função que está sendo decorada e seu valor de retorno é uma função com capacidade de decoração.

O conceito de decorador ainda é uma dor de cabeça para iniciantes, então vamos primeiro usar um exemplo simples para explicar o que um decorador faz. Suponha que existam duas funções denominadas `download` e `upload`, usadas para download e upload de arquivos, conforme mostrado abaixo.

```python
import random
import time


def download(filename):
    """Download a file."""
    print(f'Start download{filename}.')
    time.sleep(random.random() * 6)
    print(f'{filename}Download complete.')


def upload(filename):
    """Upload a file."""
    print(f'Start upload{filename}.')
    time.sleep(random.random() * 8)
    print(f'{filename}Upload complete.')


download('MySQLFrom deleting the library to running.avi')
upload('PythonFrom Beginner to Inpatient.pdf')
```

> **Observação**: O código acima usa suspensão por um período de tempo aleatório para simular que o download e o upload de arquivos demoram algum tempo. Na verdade, ele não faz upload ou download de arquivos pela rede. Na verdade, também é muito fácil usar Python para fazer upload e download de arquivos pela rede, e falaremos sobre isso mais tarde.

Agora há um novo requisito. Queremos saber exatamente quanto tempo leva para chamar `download` e `upload`. Como devemos fazer isso? Acredito que muitos leitores já tenham pensado nisso: podemos registrar um momento em que a função começa a ser executada, registrar outro momento após o término da chamada da função e subtrair os dois para calcular o tempo gasto no download ou upload, conforme mostrado abaixo.

```python
start = time.time()
download('MySQL_Full_Crash_Course.avi')
end = time.time()
print(f'Time spent: {end - start:.2f}Second')
start = time.time()
upload('PythonFrom Beginner to Inpatient.pdf')
end = time.time()
print(f'Time spent: {end - start:.2f}Second')
```

Através do código acima, podemos registrar o tempo gasto no download e upload de arquivos, mas você deve ter notado que o código para registrar o tempo, calcular o tempo e exibi-lo é todo um código duplicado. Pessoas com experiência em programação sabem que **código duplicado é a fonte de todo o mal**, então existe uma maneira simples e elegante de registrar o tempo de execução de funções sem escrever código duplicado? Em Python, os decoradores são a melhor solução para esse tipo de problema. Através da sintaxe do decorador, podemos empacotar o código de temporização, que não tem nada a ver com o negócio original de upload e download, em uma função. Se as funções `upload` e `download` precisarem de temporização, só precisamos aplicar o decorador a essas duas funções.

A estrutura básica é assim:

```python
def record_time(func):

    def wrapper(*args, **kwargs):

        result = func(*args, **kwargs)

        return result

    return wrapper
```

O parâmetro `func` na função `record_time` acima significa a função decorada. A função `wrapper` definida dentro da função é a função com capacidade de decoração. Ele executará a função decorada `func` e também precisará retornar o valor produzido ao final da chamada da função. Você deve ter notado que deixei dois espaços em branco no código acima. Isso significa que podemos adicionar código nesses locais para fornecer capacidade extra. A função `record_time` finalmente retorna esta função decorada `wrapper` e a usa para substituir a função original `func`. Depois que a função original `func` é decorada por `record_time`, quando a chamamos, o que realmente estamos chamando é a função `wrapper`, então obtemos a habilidade extra. Os parâmetros de `wrapper` são especiais. Como queremos usar `wrapper` para substituir a função original `func`, mas não sabemos quais parâmetros `func` receberá, usamos argumentos posicionais variáveis ​​​​e argumentos de palavras-chave para aceitar tudo e, em seguida, passamos todos eles para `func` inalterados. Mais uma coisa a enfatizar é que Python permite definições de funções aninhadas. Assim como acima, podemos definir `wrapper` dentro de `record_time`. Muitas linguagens de programação não suportam isso.

Agora vamos adicionar o comportamento de tempo:

```python
import time


def record_time(func):

    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f'{func.__name__}Execution time: {end - start:.2f}Second')
        return result

    return wrapper
```

Escrever um decorador exige algum esforço, mas, uma vez escrito, pode ser reutilizado continuamente. Se mais tarde precisarmos registrar novamente o tempo de execução da função, só precisaremos adicionar este decorador. Existem duas maneiras de usar a função decorador acima. A primeira maneira é chamar diretamente a função decoradora, passar a função decorada e obter o valor de retorno. Podemos usar esse valor de retorno para substituir diretamente a função original. Então, ao chamá-lo, já conseguimos a habilidade extra fornecida pelo decorador.

```python
download = record_time(download)
upload = record_time(upload)
download('MySQLFrom deleting the library to running.avi')
upload('PythonFrom Beginner to Inpatient.pdf')
```

Em Python, existe um **açúcar de sintaxe** ainda mais conveniente para usar decoradores. Podemos usar `@decorator_function` para colocar a função decoradora diretamente na função decorada, e o efeito é o mesmo do código acima.

```python
import random
import time


def record_time(func):

    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f'{func.__name__}Execution time: {end - start:.2f}Second')
        return result

    return wrapper


@record_time
def download(filename):
    print(f'Start download{filename}.')
    time.sleep(random.random() * 6)
    print(f'{filename}Download complete.')


@record_time
def upload(filename):
    print(f'Start upload{filename}.')
    time.sleep(random.random() * 8)
    print(f'{filename}Upload complete.')


download('MySQLFrom deleting the library to running.avi')
upload('PythonFrom Beginner to Inpatient.pdf')
```

No código acima, usamos açúcar de sintaxe do decorador para adicionar o decorador às funções `download` e `upload`. As funções `download` e `upload` decoradas são na verdade a função `wrapper` retornada no decorador. Chamá-los é realmente chamar `wrapper`, então eles ganham a capacidade de gravação do tempo de execução da função.

Se em alguns lugares do código quisermos remover o efeito do decorador e executar a função original, então, ao definir a função do decorador, precisaremos fazer um pouco mais de trabalho. A função `wraps` no módulo `functools` da biblioteca padrão do Python também é um decorador. Se colocarmos na função `wrapper`, este decorador pode nos ajudar a manter a função antes da decoração. Desta forma, quando precisarmos cancelar o decorador, podemos obter a função antes da decoração através do atributo `__wrapped__` da função decorada.

```python
import random
import time

from functools import wraps


def record_time(func):

    @wraps(func)
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f'{func.__name__}Execution time: {end - start:.2f}Second')
        return result

    return wrapper


@record_time
def download(filename):
    print(f'Start download{filename}.')
    time.sleep(random.random() * 6)
    print(f'{filename}Download complete.')


@record_time
def upload(filename):
    print(f'Start upload{filename}.')
    time.sleep(random.random() * 8)
    print(f'{filename}Upload complete.')


# Calling the decorated function records execution time.
download('MySQLFrom deleting the library to running.avi')
upload('PythonFrom Beginner to Inpatient.pdf')
# Cancel the decorator effect and do not record execution time.
download.__wrapped__('MySQLMust have.pdf')
upload.__wrapped__('PythonFrom novice to master.pdf')
```

**As próprias funções do decorador também podem ser parametrizadas**. Simplificando, os decoradores também podem ser personalizados por parâmetros passados ​​pelo chamador. Explicaremos esse ponto mais tarde, quando o usarmos.

### Recursão

Python permite definições de funções aninhadas e também permite que funções chamem umas às outras. Uma função pode até chamar a si mesma direta ou indiretamente. Quando uma função chama a si mesma, isso é chamado de **recursão**.

Um exemplo recursivo clássico é o fatorial. O fatorial de um número inteiro não negativo `N` satisfaz:

$$N! = N \vezes (N-1)! $$

Sendo esse o caso, podemos usar chamadas de função recursivas para escrever uma função fatorial, conforme mostrado abaixo.

```python
def fac(num):
    if num in (0, 1):
        return 1
    return num * fac(num - 1)
```

No código acima, a condição `if num in (0, 1)` é chamada de condição de convergência de recursão. Simplesmente falando, significa quando a chamada recursiva da função deve parar. A expressão `num * fac(num - 1)` é a fórmula recursiva, ou seja, a definição recursiva de fatorial.

Abaixo, vamos simplesmente analisar como seria todo o processo se usarmos `fac(5)` para calcular o fatorial de `5`.

```python
# 5 * fac(4)
# 5 * (4 * fac(3))
# 5 * (4 * (3 * fac(2)))
# 5 * (4 * (3 * (2 * fac(1))))
# 5 * (4 * (3 * (2 * 1)))
# 5 * (4 * (3 * 2))
# 5 * (4 * 6)
# 5 * 24
# 120
print(fac(5))  # 120
```

Observe que as chamadas de função salvam o contexto de execução atual e restauram o contexto de execução anterior por meio de uma estrutura de dados na memória chamada **pilha**. A pilha é uma estrutura de dados do tipo último a entrar, primeiro a sair. Isso significa que a função que entra na pilha primeiro retorna por último, enquanto a função que entra na pilha por último retorna primeiro. Por exemplo, se chamarmos uma função chamada `a`, e dentro da função `a` chamamos a função `b`, e dentro da função `b` chamamos a função `c`, então a primeira função a entrar na pilha é `a`, mas a primeira função a sair da pilha é `c`. Cada vez que inserimos uma chamada de função, a pilha adiciona um quadro de pilha. O stack frame é a estrutura que acabamos de mencionar para salvar o contexto de execução atual. Cada vez que uma chamada de função termina, a pilha remove um quadro de pilha. Normalmente, o espaço de pilha na memória é muito pequeno; portanto, se o número de chamadas recursivas for muito grande, causará estouro de pilha. Portanto, **chamadas recursivas devem sempre garantir que possam convergir rapidamente**. Podemos tentar executar `fac(5000)` e ver se ele gera um `RecursionError`. A mensagem de erro é `maximum recursion depth exceeded in comparison`, que é realmente um estouro de pilha.

Se usarmos o interpretador Python oficial, CPython, a profundidade máxima de pilha padrão para chamadas de função geralmente é definida como `1000`. Se essa profundidade for ultrapassada, ocorre o `RecursionError` mencionado acima. Claro, podemos usar a função `setrecursionlimit` do módulo `sys` para alterar a profundidade máxima de recursão, mas isso geralmente não é recomendado. Fazer a recursão convergir rapidamente é o que devemos fazer; caso contrário, é melhor considerar o loop iterativo em vez da recursão.

Aqui está outro exemplo do qual falamos antes: gerar a sequência de Fibonacci. Como os dois primeiros números na sequência de Fibonacci são `1` e o terceiro número de cada número é a soma dos dois números anteriores, ele pode ser escrito como `f(n) = f(n - 1) + f(n - 2)`. Claramente, esta é novamente uma definição recursiva, então podemos usar a função recursiva abaixo para calcular o número de Fibonacci `n`th.

```python
def fib1(n):
    if n in (1, 2):
        return 1
    return fib1(n - 1) + fib1(n - 2)


for i in range(1, 21):
    print(fib1(i))
```

O que precisa ser lembrado é que embora o código acima para calcular os números de Fibonacci pareça muito simples e claro, seu desempenho de execução é bastante ruim. Você pode tentar você mesmo: altere o segundo argumento da função `range` no loop `for` acima para `51`, o que significa gerar os primeiros 50 números de Fibonacci e ver quanto tempo leva. Quanto ao motivo de ser tão lento, pense você mesmo. Claramente, usar diretamente a recorrência do loop para obter a sequência de Fibonacci é uma escolha melhor, conforme mostrado abaixo.

```python
def fib2(n):
    a, b = 0, 1
    for _ in range(n):
        a, b = b, a + b
    return a
```

Além disso, também podemos usar a função `lru_cache` no módulo `functools` da biblioteca padrão do Python para otimizar o código recursivo acima. A função `lru_cache` é um decorador. Se o colocarmos acima da função `fib1`, ele poderá armazenar em cache os resultados da execução desta função e evitar uma grande quantidade de trabalho repetido durante a recursão. Desta forma, o desempenho da execução melhora dramaticamente. Você pode tentar gerar os primeiros 50 números de Fibonacci novamente e ver quanto tempo leva após adicionar o decorador.

```python
from functools import lru_cache


@lru_cache()
def fib1(n):
    if n in (1, 2):
        return 1
    return fib1(n - 1) + fib1(n - 2)


for i in range(1, 51):
    print(i, fib1(i))
```

> **Dica**: `lru_cache` é um decorador com parâmetros, por isso é escrito como `@lru_cache()` entre parênteses. Um de seus parâmetros mais importantes é o `maxsize`, que controla o tamanho do cache.

### Resumo

Decoradores são um recurso de sintaxe distinto em Python. **Podemos usar decoradores para aprimorar funções existentes**, e esta é uma habilidade de programação muito útil. Por outro lado, através de chamadas de função recursivas, podemos simplificar alguns problemas complexos no nível do código, mas **chamadas recursivas devem prestar atenção à condição de convergência e à fórmula recursiva**. Somente encontrando a fórmula recursiva teremos a chance de usar a recursão, e a condição de convergência garante que a recursão possa parar. As chamadas de função salvam e restauram o contexto de execução por meio do espaço de pilha na memória, e o espaço de pilha geralmente é muito pequeno, portanto **se a recursão não puder convergir rapidamente, é muito provável que cause um erro de estouro de pilha e trave o programa**.
