After data loading is completed, we may need to join fact tables and dimension tables, which is the basis for breaking down data from multiple dimensions. We may also load data with the same structure from different sources and need to splice them together. We collectively call these operations data reshaping. Of course, because the level of enterprise informatization and data-platform construction is different from company to company, the data we get may not always be of very good quality. ### Data Reshaping

Sometimes, the raw data needed for analysis does not come from a single place. In the previous chapter, we read three tables from a relational database and got three `DataFrame` objects, but in actual work, we may need to combine them. For example, `emp_df` and `emp2_df` both contain employee data and have exactly the same structure, so we can use the `concat` function provided by pandas to concatenate two or more `DataFrame` objects.

```python
all_emp_df = pd.concat([emp_df, emp2_df])
```

Output:

```
Concatenated employee table with 19 rows.
```

The code above concatenates the two `DataFrame` objects that represent employee data. Next, we use the `merge` function to combine the employee table and the department table into one table.

First, use `reset_index` to reset the index of `all_emp_df`, so that `eno` is no longer the index but a normal column. The `inplace` parameter of `reset_index` is set to `True`, meaning the operation is performed directly on `all_emp_df`.

```python
all_emp_df.reset_index(inplace=True)
```

Now merge the data with the `merge` function. Of course, we can also call the `merge` method on a `DataFrame` object to achieve the same effect.

```python
pd.merge(all_emp_df, dept_df, how='inner', on='dno')
```

Output:

```
Merged employee and department table with matched rows from both tables.
```

The first parameter of `merge` is the left table, and the second parameter is the right table. If you have SQL experience, these words should feel very familiar. As you may have guessed, merging `DataFrame` objects is very similar to table joins in a database. So the `how` parameter in the code above represents the way the two tables are merged, and it has four options: `left`, `right`, `inner`, and `outer`. The `on` parameter represents which column is used to merge the tables, which is like the join condition in SQL. If the corresponding column names in the left and right tables are different, we can use `left_on` and `right_on` instead of `on`.

If we make a small change and set `how='right'`, then the result is a right join. That means all rows from the right table `dept_df` will be returned. Since `all_emp_df` does not have any employee in department `40`, the corresponding positions will be filled with null values.

The extra row looks like this:

```
eno  ename  job  mgr  sal  comm  dno  dname       dloc
NaN  NaN    NaN  NaN  NaN  NaN    40  Operations  Shenzhen
```

### Data Cleaning

Usually, the data we get from Excel, CSV, or a database is not perfect. Because of system or human reasons, duplicate values or outliers may be mixed in. There may also be missing values in some fields. In addition, the data in a `DataFrame` may have inconsistent formats, inconsistent units, and various other problems. So before starting data analysis, data cleaning becomes especially important.

#### Missing values

We can use the `isnull` or `isna` method of a `DataFrame` object to find missing values in the table.

```python
emp_df.isnull()
```

or

```python
emp_df.isna()
```

Correspondingly, the `notnull` and `notna` methods can mark non-null values as `True`. If we want to delete rows or columns with missing values, we can use the `dropna` method. The `axis` parameter controls whether to delete along axis 0 or axis 1, which means whether rows or columns are deleted when null values are found. By default, deletion is along axis 0.

```python
emp_df.dropna()
```

Output:

```
Only rows without any null values are kept.
```

If we want to delete along axis 1, we can use:

```python
emp_df.dropna(axis=1)
```

Output:

```
Only columns without any null values are kept.
```

> **Notice**: Many methods of `DataFrame` objects have an `inplace` parameter. Its default value is `False`, meaning the operation does not modify the original `DataFrame` but returns a new one. If the value is set to `True`, the operation is performed directly on the original `DataFrame`, and the return value becomes `None`.

In some specific scenarios, we can fill missing values instead of deleting them. The corresponding method is `fillna`. When filling null values, we can use a specified value through the `value` parameter, or use the previous cell through `method='ffill'`, or the next cell through `method='bfill'`.

```python
emp_df.fillna(value=0)
```

> **Notice**: How to choose the filling value is also a topic worth discussing. In actual work, some statistical value such as the mean or mode may be used, or an interpolation method such as random interpolation or Lagrange interpolation may be used. It is even possible to use regression models or Bayesian models to fill missing data.

Output:

```
All null values are replaced with 0.
```

#### Duplicate values

Next, let us first add two rows to the department table we used earlier, so that there are now two departments named `R&D Department` and two departments named `Sales Department`.

```python
dept_df.loc[50] = {'dname': 'R&D Department', 'dloc': 'Shanghai'}
dept_df.loc[60] = {'dname': 'Sales Department', 'dloc': 'Changsha'}
dept_df
```

Output:

```
Department table now has duplicated department names for R&D and Sales.
```

Now there is duplicated data in our table. We can use the `duplicated` method of a `DataFrame` to judge whether duplicate values exist. If no parameter is given, it checks whether row indexes are duplicated. We can also specify that duplication should be judged according to the department name `dname`.

```python
dept_df.duplicated('dname')
```

Output:

```
Rows 50 and 60 are marked as duplicated by department name.
```

From the result, we can see that departments `50` and `60` are duplicates in terms of department name. If we want to delete duplicates, we can use the `drop_duplicates` method. The `keep` parameter controls whether the first one, the last one, or none of the duplicated rows are kept.

```python
dept_df.drop_duplicates('dname')
```

Output:

```
The first duplicate is kept.
```

Change the value of `keep` to `last`:

```python
dept_df.drop_duplicates('dname', keep='last')
```

Output:

```
The last duplicate is kept.
```

In the same way, we can clear duplicated data in `all_emp_df`. For example, if we decide that rows with the same `ename` and `job` are duplicated, we can remove them like this:

```python
all_emp_df.drop_duplicates(['ename', 'job'], inplace=True)
```

#### Outliers

The full statistical name of an outlier is a suspected abnormal value, also called an outlying point. Outlier analysis is also called outlier detection. An outlier is an "extreme value" appearing in a sample. It looks abnormally large or abnormally small, and its distribution is obviously different from the other observed values. In actual work, some outliers may be caused by systems or human mistakes, but some are not. They can appear repeatedly and steadily and are actually normal extreme values. For example, in many game products, the data of top players is often an extreme outlying value. So we should neither ignore outliers nor simply remove them from analysis. Paying attention to outliers and analyzing the reasons behind them often becomes a chance to discover problems and improve decisions.

Common methods for detecting outliers include the Z-score method, the IQR method, DBScan clustering, and isolation forest. Here we introduce the first two very simply.

<img src="res/3sigma.png" style="zoom:50%;">

If data follows a normal distribution, according to the 3σ rule, values whose deviation from the mean exceeds three standard deviations are defined as outliers. If the data does not follow a normal distribution, we can still describe the distance from the mean in terms of how many standard deviations away it is, and this multiple is the Z-score.

$$
z = \frac {X - \mu} {\sigma}
$$

$$
\lvert z \rvert > 3
$$

The threshold of Z-score should be decided according to experience and actual conditions. Usually, data points farther than 3 standard deviations are regarded as outliers. The code below shows how to detect outliers using the Z-score method.

```python
def detect_outliers_zscore(data, threshold=3):
    avg_value = np.mean(data)
    std_value = np.std(data)
    z_score = np.abs((data - avg_value) / std_value)
    return data[z_score > threshold]
```

In the IQR method, IQR stands for Inter-Quartile Range, which is the difference between the upper quartile Q3 and the lower quartile Q1. Usually, values smaller than $\small{Q1 - 1.5 \times IQR}$ or larger than $\small{Q3 + 1.5 \times IQR}$ are regarded as outliers. This is also the default method used by box plots.

```python
def detect_outliers_iqr(data, whis=1.5):
    q1, q3 = np.quantile(data, [0.25, 0.75])
    iqr = q3 - q1
    lower, upper = q1 - whis * iqr, q3 + whis * iqr
    return data[(data < lower) | (data > upper)]
```

If we want to delete outliers, we can use the `drop` method of a `DataFrame` to delete specified rows or columns according to row index or column index. For example, if we think salaries lower than `2000` or higher than `8000` are outliers in the employee table, we can remove those records like this:

```python
emp_df.drop(emp_df[(emp_df.sal > 8000) | (emp_df.sal < 2000)].index)
```

Output:

```
Rows with salary lower than 2000 or higher than 8000 are removed.
```

If we want to replace outliers, we can assign values directly to cells, or use the `replace` method. For example, if we want to replace salaries of `1800` and `9000` with the average salary, and replace a subsidy value of `800` with `1000`, we can write:

```python
avg_sal = np.mean(emp_df.sal).astype(int)
emp_df.replace({'sal': [1800, 9000], 'comm': 800}, {'sal': avg_sal, 'comm': 1000})
```

Output:

```
The specified outlier salary and subsidy values are replaced.
```

#### Preprocessing

Data preprocessing is also a big topic. It includes splitting, transforming, reducing, discretizing, and other operations. Let us first look at data splitting. If a column contains date-time data, we usually need to split it into dimensions such as year, quarter, month, day, weekday, hour, and minute. If the date-time is represented as a string, we can first use pandas' `to_datetime` function to convert it into a date-time type.

In the example below, we first read an Excel file and get a group of sales data. The first column is the sales date. We split it into `month`, `quarter`, and `weekday`.

```python
sales_df = pd.read_excel(
    'data/2020-sales-data.xlsx',
    usecols=['Sales Date', 'Sales Area', 'Sales Channel', 'Brand', 'Sales Amount']
)
sales_df.info()
```

```python
sales_df['Month'] = sales_df['Sales Date'].dt.month
sales_df['Quarter'] = sales_df['Sales Date'].dt.quarter
sales_df['Weekday'] = sales_df['Sales Date'].dt.weekday
sales_df
```

Through the `dt` attribute of a date-time `Series`, we get an object for accessing date-time parts. Then we can use its properties like `year`, `month`, `quarter`, and `hour` to get time information. The result is still a `Series`, so this `dt` attribute is often also called a date-time vector.

Now let us talk about processing string-type data. First, read job data from a specified CSV file.

```python
jobs_df = pd.read_csv(
    'data/job-site-data.csv',
    usecols=['city', 'companyFullName', 'positionName', 'salary']
)
jobs_df.info()
```

View the first 5 rows:

```python
jobs_df.head()
```

Not all jobs in the table are data-analysis positions. If we want to filter out the positions containing the keyword `data analysis`, we can check whether the `positionName` field contains that phrase. Here we need fuzzy matching. Since the `positionName` column is a string `Series`, we can use the `str` attribute to get the corresponding string vector, and then use familiar string methods on it.

```python
jobs_df = jobs_df[jobs_df.positionName.str.contains('data analysis')]
jobs_df.shape
```

Then we also need to process the `salary` field. If we want to count the average salary of all positions or of each city, we first need to turn the salary range into its midpoint.

```python
jobs_df.salary.str.extract(r'(\d+)[kK]?-(\d+)[kK]?')
```

> **Note**: The code above uses regex capture groups to extract two groups of digits from the string, corresponding to the lower and upper bounds of the salary.

The two extracted columns are still strings, so we need to convert them to `int` before calculating the average. The `applymap` method of a `DataFrame` can do this, because its argument is a function that is applied to each element in the `DataFrame`. After that, we can use the `apply` method to turn the two salary columns into midpoint values.

```python
temp_df = jobs_df.salary.str.extract(r'(\d+)[kK]?-(\d+)[kK]?').applymap(int)
temp_df.apply(np.mean, axis=1)
```

Now we can use the result above to replace the original `salary` column or add a new column representing the salary of the position.

```python
temp_df = jobs_df.salary.str.extract(r'(\d+)[kK]?-(\d+)[kK]?').applymap(int)
jobs_df['salary'] = temp_df.apply(np.mean, axis=1)
jobs_df.head()
```

The `applymap` and `apply` methods are often used in preprocessing. A `Series` object also has an `apply` method. A `DataFrame` additionally has a method named `transform`, which also performs transformations through a passed function, similar to the `map` method of a `Series`. One thing to stress is that `apply` can have a reduction effect, which means it can reduce more data into less data or into one result, while `transform` has no reduction effect. It only transforms the data, and the number of rows stays the same.

If we want to analyze or mine the data more deeply, non-numeric types such as strings and date-time values usually need to be converted to numeric values, because non-numeric types cannot be used to calculate correlation, cannot be used in $\small{\chi^{2}}$ tests, and so on. For string data, we can usually divide it into the following three kinds and process it accordingly:

1. Ordinal variables: the string data has an order relationship, so it can be turned into ordered numbers.
2. Categorical or nominal variables: the string data has no magnitude or level relationship, so we can use one-hot encoding to turn it into a dummy-variable matrix.
3. Scale variables: the string essentially corresponds to data with magnitude and supports arithmetic, so we only need to convert it into the corresponding numeric value.

For the first and third kinds, we can use the `apply` or `transform` methods mentioned above, or later use tools such as `OrdinalEncoder` from `scikit-learn`. For the second kind, we can use pandas' `get_dummies()` function to generate a dummy-variable matrix.

```python
persons_df = pd.DataFrame(
    data={
        'Name': ['Guan Yu', 'Zhang Fei', 'Zhao Yun', 'Ma Chao', 'Huang Zhong'],
        'Occupation': ['Doctor', 'Doctor', 'Programmer', 'Painter', 'Teacher'],
        'Education': ['Graduate', 'College', 'Graduate', 'High School', 'Bachelor']
    }
)
persons_df
```

Turn occupation into a dummy-variable matrix:

```python
pd.get_dummies(persons_df['Occupation'])
```

Turn education into values with different levels:

```python
def handle_education(x):
    edu_dict = {'High School': 1, 'College': 3, 'Bachelor': 5, 'Graduate': 10}
    return edu_dict.get(x, 0)


persons_df['Education'].apply(handle_education)
```

Finally, let us talk about discretization. Discretization is also called binning. If a variable takes continuous values, it may have countless possible values, which makes grouping inconvenient. At this time, discretizing the continuous variable becomes important. The reason it is called binning is that we can prepare some bins in advance, and each bin represents a range of values, so continuous values can be assigned into different bins.

The example below reads the 2018 Beijing residence points data. We can group the data according to residence points as follows.

```python
luohu_df = pd.read_csv('data/2018-beijing-residence-points-data.csv', index_col='id')
luohu_df.score.describe()
```

We can see that the maximum score is `122.59` and the minimum score is `90.75`. So we can build 7 bins from `90` to `125`, one bin every `5` points. The `cut` function in pandas can help us bin the data.

```python
bins = np.arange(90, 126, 5)
pd.cut(luohu_df.score, bins, right=False)
```

> **Note**: The default value of the `right` parameter in `cut` is `True`, meaning the bins are left-open and right-closed. Changing it to `False` makes the left boundary closed and the right boundary open.

We can group the data according to the binning result and then use aggregation functions to count each group. This is a very common operation in data analysis. Besides `cut`, pandas also provides a `qcut` function, which can bin data according to quantiles. Interested readers can explore it by themselves.
