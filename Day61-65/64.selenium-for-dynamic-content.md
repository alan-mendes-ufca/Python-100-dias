## Usando Selenium para capturar conteúdo dinâmico da Web

De acordo com o relatório global de auditoria de acessibilidade da Internet divulgado por organizações conceituadas, cerca de três quartos dos sites no mundo têm conteúdo ou parte de seu conteúdo gerado dinamicamente por meio de JavaScript. Isso significa que quando usamos "Ver código-fonte da página" na janela do navegador, não conseguimos encontrar esse conteúdo no código HTML, e a forma que usávamos antes para obter os dados não funciona mais normalmente. Existem basicamente duas soluções para este problema. Uma delas é obter a interface de dados que fornece o conteúdo dinâmico. Essa forma também é adequada para obter dados de aplicativos móveis. A outra é executar um navegador por meio da ferramenta de teste automatizado Selenium e obter o conteúdo dinâmico renderizado. Para a primeira solução, podemos usar as Ferramentas de Desenvolvimento do navegador ou ferramentas mais profissionais de captura de pacotes, como Charles, Fiddler ou Wireshark, para obter a interface de dados. As operações posteriores são iguais às mencionadas na lição anterior para obter os dados do site 360 Image, portanto não as repetiremos aqui. Neste capítulo, explicamos principalmente como usar a ferramenta de teste automatizado Selenium para obter o conteúdo dinâmico de sites.

### Introdução ao Selenium

Selenium é uma ferramenta de teste automatizada. Ao usá-lo, podemos direcionar o navegador para realizar ações específicas e, finalmente, ajudar os desenvolvedores de rastreadores a obter o conteúdo dinâmico das páginas da web. Simplificando, desde que possamos ver o conteúdo na janela do navegador, o Selenium pode obtê-lo. Para os sites que usam tecnologia de renderização dinâmica JavaScript, o Selenium é uma escolha importante. Abaixo, ainda utilizamos o navegador Chrome como exemplo para explicar o uso do Selenium. Todos precisam primeiro instalar o Chrome e baixar seu driver. O programa de driver do Chrome pode ser baixado em [Site oficial do ChromeDriver](https://chromedriver.chromium.org/downloads). A versão do driver precisa corresponder à versão do navegador. Se não houver uma versão exatamente correspondente, escolha aquela cujo número de versão seja mais próximo.

### Usando Selenium

Podemos primeiro instalar o Selenium através do `pip`, e o comando é mostrado abaixo.

```bash
pip install selenium
```

#### Carregando páginas

A seguir, usamos o código abaixo para fazer com que o Chrome abra o Baidu.

```python
from selenium import webdriver

# Create a Chrome browser object
browser = webdriver.Chrome()
# Load the specified page
browser.get('https://www.baidu.com/')
```

Caso não queira usar o Chrome, você também pode alterar o código acima para controlar outros navegadores. Você só precisa criar o objeto de navegador correspondente, como Firefox ou Safari. Se você executar o programa acima e vir a mensagem de erro mostrada abaixo, significa que ainda não adicionamos o driver do navegador Chrome à variável de ambiente PATH e também não especificamos a localização do driver do navegador Chrome no programa.

```text
selenium.common.exceptions.WebDriverException: Message: 'chromedriver' executable needs to be in PATH. Please see https://sites.google.com/a/chromium.org/chromedriver/home
```

Existem três maneiras de resolver este problema:

1. Coloque o ChromeDriver baixado em um caminho que já esteja na variável de ambiente PATH. Recomenda-se colocá-lo diretamente no mesmo diretório do interpretador Python, pois quando o Python foi instalado antes, já colocamos o caminho do interpretador Python na variável de ambiente PATH.
2. Coloque o ChromeDriver na pasta `bin` no ambiente virtual do projeto e o diretório correspondente no Windows é `Scripts`. Desta forma, o ChromeDriver está no mesmo lugar que o interpretador Python no ambiente virtual e pode definitivamente ser encontrado.
3. Modifique o código acima. Ao criar o objeto Chrome, configure um objeto `Service` através do parâmetro `service`, e especifique a localização do ChromeDriver através do parâmetro `executable_path` ao criar o objeto `Service`, conforme mostrado abaixo.

    ```python
    from selenium import webdriver
    from selenium.webdriver.chrome.service import Service

    browser = webdriver.Chrome(service=Service(executable_path='venv/bin/chromedriver'))
    browser.get('https://www.baidu.com/')
    ```

#### Encontrando Elementos e Simulando o Comportamento do Usuário

Em seguida, podemos tentar simular o usuário digitando uma palavra-chave de pesquisa na caixa de texto da página inicial do Baidu e clicando no botão "Pesquisa do Baidu". Após o carregamento da página, podemos obter os elementos da página por meio dos métodos `find_element` e `find_elements` do objeto `Chrome`. Selenium oferece suporte a muitas maneiras de obter elementos, incluindo seletores CSS, XPath, nome do elemento, nome da tag de significado, ID do elemento, nome da classe e assim por diante. O primeiro obtém um único elemento de página, um objeto `WebElement`, e o último obtém uma lista composta por vários elementos de página. Depois de obter um objeto `WebElement`, podemos simular o comportamento de entrada do usuário por meio de `send_keys` e simular operações de clique do usuário por meio de `click`. O código é mostrado abaixo.

```python
from selenium import webdriver
from selenium.webdriver.common.by import By

browser = webdriver.Chrome()
browser.get('https://www.baidu.com/')
# Get an element through its ID
kw_input = browser.find_element(By.ID, 'kw')
# Simulate user input behavior
kw_input.send_keys('Python')
# Get an element through a CSS selector
su_button = browser.find_element(By.CSS_SELECTOR, '#su')
# Simulate user click behavior
su_button.click()
```

Se precisar realizar uma série de ações, como simular arrastar e soltar, você pode criar um objeto `ActionChains`. Os leitores interessados podem estudá-lo por conta própria.

#### Espera Implícita e Espera Explícita

Há mais um detalhe que todos precisam saber aqui. Os elementos de uma página podem ser gerados dinamicamente. Quando usamos os métodos `find_element` ou `find_elements` para obtê-los, a renderização pode ainda não ter sido concluída e, então, um erro `NoSuchElementException` é gerado. Para resolver esse problema, podemos usar a espera implícita definindo um tempo de espera para que o navegador possa terminar de renderizar os elementos da página. Além disso, também podemos usar espera explícita. Ao criar um objeto `WebDriverWait` e definir o tempo e as condições de espera, se as condições não forem atendidas, podemos esperar primeiro e depois tentar operações posteriores. O código específico é mostrado abaixo.

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support import expected_conditions
from selenium.webdriver.support.wait import WebDriverWait

browser = webdriver.Chrome()
# Set browser window size
browser.set_window_size(1200, 800)
browser.get('https://www.baidu.com/')
# Set the implicit waiting time to 10 seconds
browser.implicitly_wait(10)
kw_input = browser.find_element(By.ID, 'kw')
kw_input.send_keys('Python')
su_button = browser.find_element(By.CSS_SELECTOR, '#su')
su_button.click()
# Create an explicit wait object
wait_obj = WebDriverWait(browser, 10)
# Set the waiting condition, wait for the search result div to appear
wait_obj.until(
    expected_conditions.presence_of_element_located(
        (By.CSS_SELECTOR, '#content_left')
    )
)
# Take a screenshot
browser.get_screenshot_as_file('python_result.png')
```

A condição de espera definida acima, `presence_of_element_located`, significa aguardar o aparecimento do elemento especificado. A tabela abaixo lista as condições de espera comuns e seus significados.

| Condição de espera | Significado |
| ----------------- | ------- |
| `title_is / title_contains` | O título é o conteúdo especificado/o título contém o conteúdo especificado |
| `visibility_of` | O elemento está visível |
| `presence_of_element_located` | O elemento localizado terminou de carregar |
| `visibility_of_element_located` | O elemento localizado torna-se visível |
| `invisibility_of_element_located` | O elemento localizado torna-se invisível |
| `presence_of_all_elements_located` | Todos os elementos localizados terminaram de carregar |
| `text_to_be_present_in_element` | O elemento contém o conteúdo especificado |
| `text_to_be_present_in_element_value` | O atributo `value` do elemento contém o conteúdo especificado |
| `frame_to_be_available_and_switch_to_it` | Carregue e mude para a janela interna especificada |
| `element_to_be_clickable` | O elemento pode ser clicado |
| `element_to_be_selected` | O elemento está selecionado |
| `element_located_to_be_selected` | O elemento localizado é selecionado |
| `alert_is_present` | Um pop-up de alerta aparece |

#### Executando código JavaScript

Para uma página que usa carregamento em cascata, se quiser carregar mais conteúdo na janela do navegador, você pode fazer isso executando o código JavaScript por meio do método `execute_scripts` do objeto do navegador. Operações semelhantes também podem ser necessárias em algumas ações de rastreamento mais avançadas. Se o código do seu rastreador precisar de suporte a JavaScript, é recomendável primeiro aprender JavaScript corretamente, especialmente as operações BOM e DOM em JavaScript. Se adicionarmos o código abaixo antes de fazer a captura de tela no código acima, o JavaScript poderá ser usado para rolar a página até o final.

```python
# Run JavaScript code
browser.execute_script('document.documentElement.scrollTop = document.documentElement.scrollHeight')
```

#### Quebrando verificações anti-rastreador de Selenium

Alguns sites definem medidas anti-crawler especificamente para Selenium, porque em um navegador controlado por Selenium, o valor da propriedade `webdriver` mostrada no console é `true`, conforme mostrado abaixo. Se quiser ignorar essa verificação, você pode primeiro alterá-la para `undefined` executando o código JavaScript antes de carregar a página.

Por outro lado, também podemos ocultar o texto “O Chrome está sendo controlado por software de teste automatizado” na janela do navegador. O código completo é mostrado abaixo.

```python
# Create a Chrome options object
options = webdriver.ChromeOptions()
# Add experimental parameters
options.add_experimental_option('excludeSwitches', ['enable-automation'])
options.add_experimental_option('useAutomationExtension', False)
# Create a Chrome browser object and pass in the parameters
browser = webdriver.Chrome(options=options)
# Run a Chrome DevTools Protocol command
# execute the specified JavaScript code when the page is loaded
browser.execute_cdp_cmd(
    'Page.addScriptToEvaluateOnNewDocument',
    {'source': 'Object.defineProperty(navigator, "webdriver", {get: () => undefined})'}
)
browser.set_window_size(1200, 800)
browser.get('https://www.baidu.com/')
```

#### Navegadores sem cabeça

Muitas vezes, quando estamos coletando dados, não precisamos ver a janela do navegador. Enquanto houver o Chrome e seu programa de driver correspondente, nosso rastreador poderá ser executado. Se você não quiser ver a janela do navegador, podemos configurar um navegador sem cabeça da maneira abaixo.

```python
options = webdriver.ChromeOptions()
options.add_argument('--headless')
browser = webdriver.Chrome(options=options)
```

### Exemplo Simples

O exemplo abaixo mostra como usar o Selenium para pesquisar e baixar imagens do site 360 Image.

```python
import os
import time
from concurrent.futures import ThreadPoolExecutor

import requests
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.common.keys import Keys

DOWNLOAD_PATH = 'images/'


def download_picture(picture_url: str):
    """
    Download and save the picture
    :param picture_url: URL of the picture
    """
    filename = picture_url[picture_url.rfind('/') + 1:]
    resp = requests.get(picture_url)
    with open(os.path.join(DOWNLOAD_PATH, filename), 'wb') as file:
        file.write(resp.content)


if not os.path.exists(DOWNLOAD_PATH):
    os.makedirs(DOWNLOAD_PATH)
browser = webdriver.Chrome()
browser.get('https://image.so.com/z?ch=beauty')
browser.implicitly_wait(10)
kw_input = browser.find_element(By.CSS_SELECTOR, 'input[name=q]')
kw_input.send_keys('Cang Laoshi')
kw_input.send_keys(Keys.ENTER)
for _ in range(10):
    browser.execute_script(
        'document.documentElement.scrollTop = document.documentElement.scrollHeight'
    )
    time.sleep(1)
imgs = browser.find_elements(By.CSS_SELECTOR, 'div.waterfall img')
with ThreadPoolExecutor(max_workers=32) as pool:
    for img in imgs:
        pic_url = img.get_attribute('src')
        pool.submit(download_picture, pic_url)
```

Execute o código acima e verifique se as imagens encontradas pela pesquisa por palavra-chave foram baixadas no diretório especificado.
