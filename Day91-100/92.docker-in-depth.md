## Docker Container Technology in Detail

Docker is an open-source application container engine developed in Go and released under the Apache License 2.0. It allows developers to package an application and its dependency packages into a portable container and then publish it on many Linux distributions.

### An Introduction to Docker

One of the most troublesome parts of software development is often environment configuration. Users work on many operating systems, and even if a language is cross-platform, that still does not guarantee that code will run normally in every environment. The software packages needed in different systems can also vary a lot.

This raises a natural question: can we package the runtime environment together with the software? Can we copy the original environment exactly?

Virtual machines are one solution. A virtual machine can run one operating system inside another one, such as Linux on Windows or Windows on macOS. Applications inside the virtual machine feel as if they are running on a real machine. But virtual machines consume a lot of system resources, and starting or stopping them is usually slow.

Docker is a packaging of Linux container technology, LXC. It uses Linux mechanisms such as namespaces and cgroups, but provides a simple and easy-to-use interface for container usage. Docker packages the application and its dependencies into one bundle. Running that bundle produces a virtual container. The program runs inside the virtual container as if it were running on a real physical machine.

Today, Docker is mainly used in several scenarios:

1. Providing disposable environments.
2. Providing elastic cloud services that can scale out or shrink back easily.
3. Practicing microservice architecture by running multiple isolated services in containers.

### Installing Docker

Using CentOS as an example, the installation process looks like this. Ubuntu, macOS, and Windows users can check the official documentation for the corresponding platform.

1. Confirm the kernel version.

   ```bash
   uname -r
   ```

2. Update the system libraries.

   ```bash
   yum update
   ```

3. Remove old Docker versions that may already exist.

   ```bash
   yum list installed | grep docker
   yum erase -y docker docker-common docker-engine
   ```

4. Install the required `yum` tools and dependencies.

   ```bash
   yum install -y yum-utils device-mapper-persistent-data lvm2
   ```

5. Add the Docker CE repository.

   ```bash
   yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
   ```

6. Install Docker CE and start it.

   ```bash
   yum -y install docker-ce
   systemctl start docker
   ```

7. Check the version and status.

   ```shell
   docker version
   docker info
   ```

After that, we can download an image and create a container to see whether Docker can run normally.

```shell
docker pull hello-world
docker images
docker container run --name mycontainer hello-world
docker container rm mycontainer
docker rmi hello-world
```

Output of `docker images`:

```shell
REPOSITORY               TAG        IMAGE ID            CREATED             SIZE
docker.io/hello-world    latest     fce289e99eb9        7 months ago        1.84 kB
```

Output of running the container:

```shell
Hello from Docker!
This message shows that your installation appears to be working correctly.
```

For many users, pulling images from Docker Hub can be slow, so it is often a good idea to configure registry mirrors in `/etc/docker/daemon.json`.

### Using Docker

The easiest way to learn Docker is to create useful containers directly.

#### Running Nginx

Nginx is a high-performance web server and also an excellent reverse proxy. With Docker, creating an Nginx container is very easy:

```shell
docker container run -d -p 80:80 --rm --name mynginx nginx
```

Here:

- `-d` runs the container in the background;
- `-p` maps a host port to a container port;
- `--rm` removes the container automatically after it stops;
- `--name` assigns a custom container name.

If you want to deploy your own web project into Nginx, you can either copy files into the container:

```shell
docker container cp /root/web/index.html mynginx:/usr/share/nginx/html
```

or mount a host directory as a volume:

```shell
docker container stop mynginx
docker container run -d -p 80:80 --rm --name mynginx --volume /root/docker/nginx/html:/usr/share/nginx/html nginx
```

Output of `docker ps` looks like this:

```shell
CONTAINER ID    IMAGE    COMMAND                  CREATED            STATUS             PORTS                 NAMES
3c38d2476384    nginx    "nginx -g 'daemon ..."   4 seconds ago      Up 4 seconds       0.0.0.0:80->80/tcp    mynginx
```

You can inspect running containers with:

```shell
docker ps
```

And start, stop, or remove them with:

```shell
docker start mynginx
docker stop mynginx
docker rm mynginx
docker rm -f mynginx
```

#### Running MySQL

We can also use Docker to install a MySQL server.

```shell
docker search mysql
docker pull mysql:5.7
docker images
docker run -d -p 3306:3306 --name mysql57 \
  -v /root/docker/mysql/conf:/etc/mysql/mysql.conf.d \
  -v /root/docker/mysql/data:/var/lib/mysql \
  -e MYSQL_ROOT_PASSWORD=123456 mysql:5.7
```

Example output of `docker search mysql`:

```shell
INDEX        NAME            DESCRIPTION        STARS        OFFICIAL        AUTOMATED
docker.io    docker.io/mysql MySQL is a ...     8486         [OK]
...
```

Example output of `docker images` after downloading MySQL:

```shell
REPOSITORY          TAG                 IMAGE ID            CREATED             SIZE
docker.io/nginx     latest              e445ab08b2be        2 weeks ago         126 MB
docker.io/mysql     5.7                 f6509bac4980        3 weeks ago         373 MB
```

The volume mappings are especially important here, because containers are disposable while database data needs to be kept.

If you use MySQL 8.x, some client tools may fail with `caching_sha2_password` authentication problems. One workaround is to enter the container:

```shell
docker exec -it mysql8x /bin/bash
mysql -u root -p
```

and then change the authentication mode:

```sql
alter user 'root'@'%' identified with mysql_native_password by '123456' password expire never;
```

#### Running Redis

The lesson also demonstrates running multiple containers and letting them talk to each other over the network. For example, four Redis containers can be used to build a one-master, three-replica setup:

```shell
docker run -d -p 6379:6379 --name redis-master redis
docker run -d -p 6380:6379 --name redis-slave-1 --link redis-master:redis-master redis redis-server --replicaof redis-master 6379
docker run -d -p 6381:6379 --name redis-slave-2 --link redis-master:redis-master redis redis-server --replicaof redis-master 6379
docker run -d -p 6382:6379 --name redis-slave-3 --link redis-master:redis-master redis redis-server --replicaof redis-master 6379
```

The `--link` option is used to create a network alias so the replica containers can reach the master without depending on a container IP that may change later.

#### Running GitLab

GitLab is a Git repository management tool that also provides features such as wiki, issue tracking, and continuous integration. The lesson shows how to prepare host directories for mounted volumes:

```shell
mkdir -p /root/gitlab/{config,logs,data}
```

and then run the GitLab CE container:

```shell
docker run -d -p 80:80 -p 22:22 --name gitlab \
  -v /root/gitlab/config:/etc/gitlab \
  -v /root/gitlab/logs:/var/log/gitlab \
  -v /root/gitlab/data:/var/opt/gitlab \
  gitlab/gitlab-ce
```

GitLab starts slowly, so it may take a while before the browser interface becomes available.

### Building Images

So far we have created containers from official images, but we can also build our own images. In simple terms, **a Docker image is built from stacked file systems**. The bottom layer is `bootfs`, above that is `rootfs`, and Docker uses union mounting to combine layers into the final image.

There are two main ways to build images:

1. Use `docker commit` on a configured container. This is generally not recommended.
2. Use `docker build` together with a `Dockerfile`. This is the recommended approach.

#### Building an Image with `commit`

For example, we can start an Ubuntu container:

```shell
docker run --name myubuntu -it ubuntu /bin/bash
```

install Apache inside it:

```shell
apt -y upgrade
apt -y install apache2
exit
```

then save that configured container as an image:

```shell
docker container ls -a
docker commit 014bdb321612 jackfrued/mywebserver
docker images
```

#### Building an Image with `Dockerfile`

The better way is to use a `Dockerfile`. A `Dockerfile` is a small domain-specific language for image construction. Once the file is written, `docker build` can create the image in a repeatable way. The lesson demonstrates this with a small Flask project containing:

- application code;
- a `requirements.txt` file;
- a startup shell script;
- and a top-level `Dockerfile`.

This approach is much more maintainable than using `commit`, because the entire build process is explicit and can be version-controlled.

### Summary

Docker makes application environments portable and repeatable. It is useful for temporary environments, service deployment, database and middleware setup, and microservice-style architecture. More importantly, it gives the team a way to reduce the classic “it works on my machine” problem by making local, testing, and production environments look much more alike.
