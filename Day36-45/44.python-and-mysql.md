## Conectando Python ao MySQL

No Python 3, podemos usar a biblioteca de terceiros `mysqlclient` ou `pymysql` para conectar-se ao banco de dados MySQL e realizar operações de persistência de dados. O uso dos dois é exatamente o mesmo, apenas o nome do módulo importado é diferente. Recomendamos que todos usem a biblioteca de terceiros Python pura `pymysql`, porque é mais fácil de instalar com sucesso.

Abaixo, ainda usamos o banco de dados `hrs` criado anteriormente como exemplo, para mostrar como usar um programa Python para operar o banco de dados MySQL e fazer operações de persistência de dados.

### Conectando-se ao MySQL

Primeiro, podemos instalar `pymysql` com o comando abaixo na linha de comando ou no terminal do PyCharm. Se precisar se conectar ao MySQL 8, você também precisará instalar uma biblioteca de terceiros chamada `cryptography` para oferecer suporte ao método de autenticação de senha do MySQL 8.

```Shell
pip install pymysql cryptography
```

As etapas para usar `pymysql` para operar o MySQL são mostradas abaixo:

1. Crie uma conexão. Depois que o servidor MySQL é iniciado, ele fornece um serviço de rede baseado em TCP, o Transmission Control Protocol. Podemos nos conectar ao servidor MySQL através da função `connect` do módulo `pymysql`. Ao chamar a função `connect`, precisamos especificar parâmetros como host (`host`), porta (`port`), nome de usuário (`user`), senha (`password`), banco de dados (`database`) e conjunto de caracteres (`charset`). Esta função retorna um objeto `Connection`.
2. Obtenha um cursor. Depois de conectar-se com sucesso ao servidor MySQL, a próxima coisa a fazer é enviar instruções SQL ao servidor de banco de dados. O MySQL executará o SQL recebido e retornará o resultado da execução através da rede. Para fazer isso, primeiro precisamos obter um cursor, ou seja, um objeto `Cursor`, através do método `cursor` do objeto de conexão.
3. Envie SQL. Através do método `execute` do objeto cursor, podemos enviar instruções SQL para o banco de dados.
4. Se uma operação `insert`, `delete` ou `update` for executada, precisaremos confirmar ou reverter a transação de acordo com a situação real. Porque quando a conexão é criada, o ambiente de transação é ativado por padrão. Após a conclusão da operação, precisamos usar o método `commit` ou `rollback` do objeto de conexão para confirmar ou reverter a transação. O método `rollback` geralmente é colocado no bloco de tratamento de exceções `except`. Se uma operação `select` for executada, precisamos buscar o resultado da consulta através do objeto cursor. Existem três métodos para isso: `fetchone`, `fetchmany` e `fetchall`. Entre eles, `fetchone` busca um registro e o retorna como tupla ou dicionário; `fetchmany` e `fetchall` buscam vários registros e os retornam como tuplas aninhadas ou uma lista de dicionários.
5. Feche a conexão. Após finalizar a operação de persistência, não esqueça de fechar a conexão e liberar recursos externos. Geralmente usamos o método `close` do objeto de conexão no bloco `finally` para fechar a conexão.

Abaixo, usaremos código real para demonstrar as cinco etapas acima.

### Exemplos de código

#### Inserir dados

```Python
import pymysql

no = int(input('Department ID: '))
name = input('Department name: ')
location = input('Department location: ')

# 1. Create a connection
conn = pymysql.connect(host='127.0.0.1', port=3306,
                       user='guest', password='Guest.618',
                       database='hrs', charset='utf8mb4')
try:
    # 2. Get a cursor
    with conn.cursor() as cursor:
        # 3. Execute SQL
        affected_rows = cursor.execute(
            'insert into `tb_dept` values (%s, %s, %s)',
            (no, name, location)
        )
        if affected_rows == 1:
            print('Department inserted successfully!')
    # 4. Commit the transaction
    conn.commit()
except pymysql.MySQLError as err:
    conn.rollback()
    print(type(err), err)
finally:
    # 5. Close the connection
    conn.close()
```

> **Observação**: `127.0.0.1` acima é chamado de endereço de loopback. Significa a máquina local. O `guest` abaixo é um usuário que criei antecipadamente. Este usuário tem permissões `insert`, `delete`, `update` e `select` no banco de dados `hrs`. Não recomendamos que todos usem diretamente a conta de superadministrador `root` para acessar o banco de dados em um projeto. Fazer isso é realmente muito perigoso.

Exemplo de SQL para criar esse usuário:

```SQL
create user 'guest'@'%' identified by 'Guest.618';
grant insert, delete, update, select on `hrs`.* to 'guest'@'%';
```

Caso seja necessário inserir uma grande quantidade de dados, é recomendável utilizar o método `executemany` do objeto cursor para fazer o processamento em lote, ou seja, uma operação `insert` seguida de vários grupos de dados. Todos podem tentar inserir 10.000 registros em uma tabela e ver a diferença entre inserir um por um sem processamento em lote e inserir com processamento em lote. O primeiro parâmetro do método `executemany` ainda é uma instrução SQL e o segundo parâmetro pode ser uma lista ou tupla contendo muitos grupos de dados.

#### Excluir dados

```Python
import pymysql

no = int(input('Department ID: '))

conn = pymysql.connect(host='127.0.0.1', port=3306,
                       user='guest', password='Guest.618',
                       database='hrs', charset='utf8mb4',
                       autocommit=True)
try:
    with conn.cursor() as cursor:
        affected_rows = cursor.execute(
            'delete from `tb_dept` where `dno`=%s',
            (no, )
        )
        if affected_rows == 1:
            print('Department deleted successfully!')
finally:
    conn.close()
```

> **Observação**: Se você não quiser confirmar ou reverter manualmente a transação após cada operação SQL, você pode adicionar um parâmetro chamado `autocommit` na função `connect` e defini-lo como `True`, o que significa que após cada SQL ser executado com sucesso, ele será confirmado automaticamente. Mas recomendamos que todos façam commit ou rollback manualmente, para que o ambiente de transação possa ser construído de acordo com as reais necessidades do negócio. Se não quiser capturar e tratar exceções, você pode colocar `finally` diretamente após o bloco `try`. Omitir `except` significa que quando ocorrer uma exceção, o código irá travar diretamente e mostrar a pilha de exceções no terminal.

#### Atualizar dados

```Python
import pymysql

no = int(input('Department ID: '))
name = input('Department name: ')
location = input('Department location: ')

conn = pymysql.connect(host='127.0.0.1', port=3306,
                       user='guest', password='Guest.618',
                       database='hrs', charset='utf8mb4')
try:
    with conn.cursor() as cursor:
        affected_rows = cursor.execute(
            'update `tb_dept` set `dname`=%s, `dloc`=%s where `dno`=%s',
            (name, location, no)
        )
        if affected_rows == 1:
            print('Department updated successfully!')
    conn.commit()
except pymysql.MySQLError as err:
    conn.rollback()
    print(type(err), err)
finally:
    conn.close()
```

#### Consultar dados

1. Consulte a tabela de departamentos:

```Python
import pymysql

conn = pymysql.connect(host='127.0.0.1', port=3306,
                       user='guest', password='Guest.618',
                       database='hrs', charset='utf8mb4')
try:
    with conn.cursor() as cursor:
        cursor.execute('select `dno`, `dname`, `dloc` from `tb_dept`')
        row = cursor.fetchone()
        while row:
            print(row)
            row = cursor.fetchone()
except pymysql.MySQLError as err:
    print(type(err), err)
finally:
    conn.close()
```

> **Observação**: No código acima, usamos um loop `while` para buscar o resultado da consulta linha por linha. Essa forma é especialmente adequada quando o resultado da consulta possui um número muito grande de linhas. Porque se `fetchall` for usado para buscar todos os registros em uma tupla aninhada de uma só vez, isso causará uma sobrecarga de memória muito grande e, em muitos cenários, isso não é uma boa ideia. Se não quiser usar um loop `while`, você também pode pensar em usar a função `iter` para construir um iterador e buscar dados linha por linha. Os leitores interessados ​​podem estudá-lo por conta própria.

2. Consulta paginada na tabela de funcionários:

```Python
import pymysql

page = int(input('Page: '))
size = int(input('Page size: '))

con = pymysql.connect(host='127.0.0.1', port=3306,
                      user='guest', password='Guest.618',
                      database='hrs', charset='utf8')
try:
    with con.cursor(pymysql.cursors.DictCursor) as cursor:
        cursor.execute(
            'select `eno`, `ename`, `job`, `sal` from `tb_emp` order by `sal` desc limit %s,%s',
            ((page - 1) * size, size)
        )
        for emp_dict in cursor.fetchall():
            print(emp_dict)
finally:
    con.close()
```

### Exemplo: exportar dados do banco de dados para Excel

Abaixo explicamos um exemplo de exportação de dados de tabelas de banco de dados para um arquivo Excel. Primeiro precisamos instalar a biblioteca de terceiros `openpyxl`. O comando é mostrado abaixo.

```Bash
pip install openpyxl
```

A seguir, através do código abaixo, exportamos todos os IDs dos funcionários, nome, cargo, salário mensal, subsídio e nome do departamento do banco de dados `hrs` para um arquivo Excel.

```Python
import openpyxl
import pymysql

workbook = openpyxl.Workbook()
sheet = workbook.active
sheet.title = 'Employee Basic Info'
sheet.append(('Employee ID', 'Name', 'Job', 'Salary', 'Allowance', 'Department'))

conn = pymysql.connect(host='127.0.0.1', port=3306,
                       user='guest', password='Guest.618',
                       database='hrs', charset='utf8mb4')
try:
    with conn.cursor() as cursor:
        cursor.execute(
            'select `eno`, `ename`, `job`, `sal`, coalesce(`comm`, 0), `dname` '
            'from `tb_emp` natural join `tb_dept`'
        )
        row = cursor.fetchone()
        while row:
            sheet.append(row)
            row = cursor.fetchone()
    workbook.save('hrs.xlsx')
except pymysql.MySQLError as err:
    print(err)
finally:
    conn.close()
```

Você pode consultar o exemplo acima e tentar importar os dados de um arquivo Excel para a tabela especificada do banco de dados especificado e ver se isso pode ser feito com êxito.
