## The k-Nearest Neighbors Algorithm

k-nearest neighbors, or kNN, is a non-parametric statistical method used for classification and regression. It was proposed in 1951 by the American statisticians Evelyn Fix and Joseph Hodges Jr. The idea of the kNN algorithm is to find the $\small{k}$ instances in historical data that are nearest to the new input instance, and then classify the new instance according to the class that most of those neighbors belong to, or output the target value of the new instance according to those neighbors. We already showed a simple example of this algorithm in the previous lesson.

Different from mainstream machine learning algorithms, kNN has no explicit learning or training process. It uses a simple and plain idea: "stay near vermilion, get stained red; stay near ink, get stained black." There are two key questions in the kNN algorithm. The first is how to choose the value of $\small{k}$, that is, how many nearest neighbors to use to judge the class of a new instance or decide its target value. The second is how to judge whether two instances are near or far, and this leads to the problem of measuring distance.

### Distance Measurement

We can use distance to measure the similarity between two instances in feature space. Common distance measures include Minkowski distance, Mahalanobis distance, cosine distance, edit distance, and so on. The full name of Minkowski distance is Minkowski Distance. For two $\small{n}$-dimensional vectors $\small{\mathbf{x}=(x_{1}, x_{2}, \cdots, x_{n})}$ and $\small{\mathbf{y}=(y_{1}, y_{2}, \cdots, y_{n})}$, their distance can be defined as:

$$
d(\mathbf{x}, \mathbf{y}) = (\sum_{i=1}^{n}{\vert x_{i} - y_{i} \rvert}^{p})^{\frac{1}{p}}
$$

Here, $\small{p \ge 1}$. Although we can still calculate the formula when $\small{p \lt 1}$, it no longer strictly satisfies the definition of distance, so it is usually not treated as a real distance.

When $\small{p = 1}$, Minkowski distance becomes **Manhattan distance**:

$$
d(\mathbf{x}, \mathbf{y}) = \sum_{i=1}^{n} \lvert x_{i} - y_{i} \rvert
$$

When $\small{p = 2}$, Minkowski distance becomes **Euclidean distance**:

$$
d(\mathbf{x}, \mathbf{y}) = \sqrt{\sum_{i=1}^{n}(x_{i} - y_{i})^{2}}
$$

When $\small{p \to \infty}$, Minkowski distance becomes **Chebyshev distance**:

$$
d(\mathbf{x}, \mathbf{y}) = \underset{i}{max}(\lvert x_{i} - y_{i} \rvert)
$$

We will introduce other distance measures when we actually need them. When using kNN to do classification, our dataset is usually numeric data, and in that case Euclidean distance is a good choice.

### Implementing kNN Classification

Below, we will first use basic data-science libraries NumPy and SciPy to implement the kNN algorithm instead of using scikit-learn directly. The purpose is to help everyone understand the principle of the algorithm better. On this basis, we can then feel the power of scikit-learn and understand why its classes, functions, and parameters are designed in that way.

#### NumPy-based implementation

kNN needs distance calculation. So first, let us write a function for Euclidean distance:

```python
import numpy as np


def euclidean_distance(u, v):
    """Calculate the Euclidean distance between two n-dimensional vectors."""
    return np.sqrt(np.sum(np.abs(u - v) ** 2))
```

Next, let us design a function that generates a label for new data according to neighbor labels.

```python
from scipy import stats


def make_label(X_train, y_train, X_one, k):
    """
    Generate a label for new data according to the k nearest neighbors.
    :param X_train: features in the training set
    :param y_train: labels in the training set
    :param X_one: features of the sample to predict
    :param k: number of neighbors
    :return: generated label for the sample to predict
    """
    # calculate the distance from x to each training sample
    distes = [euclidean_distance(X_one, X_i) for X_i in X_train]
    # use one partition to find the indexes of the k smallest distances
    labels = y_train[np.argpartition(distes, k - 1)[:k]]
    # get the mode of the labels
    return stats.mode(labels).mode
```

> **Note**: The `np.partition` function can partition an array once, putting the `k` smaller elements on the left side of the array and the `n - k` larger elements on the right side. This works like one partition step in quicksort. The `np.argpartition` function in the code above puts the indexes of the `k` smaller elements on the left side of the array. In this way, after the `[:k]` slice and the fancy indexing on `y_train`, we get the labels of the samples whose distance from the new data is smaller. Finally, we use the `mode` function in SciPy's `stats` module to get the mode of the labels and use it as the predicted class label for the new data.

Then the prediction function becomes straightforward:

```python
def predict_by_knn(X_train, y_train, X_new, k=5):
    """
    kNN algorithm.
    :param X_train: features in the training set
    :param y_train: labels in the training set
    :param X_new: array of samples to predict
    :param k: number of neighbors, default value is 5
    :return: array that saves predicted labels
    """
    return np.array([make_label(X_train, y_train, X, k) for X in X_new])
```

### Model Evaluation

Of course, to judge whether the prediction effect of a classifier is good, we cannot simply look at accuracy, because when classes are imbalanced, accuracy can easily mislead your judgment. For example, if the number of Iris virginica samples in our test data is very small, then even if our model cannot judge Iris virginica at all, the model may still show high accuracy. So we also need to look at precision, recall, F1 score, and other metrics. The **confusion matrix** is a tool used to show the performance of a classification model in detail, and it is suitable for both binary classification and multi-classification tasks.

For a binary classification problem, the confusion matrix is:

|      | Predicted Positive | Predicted Negative |
| ---- | ------------------ | ------------------ |
| Actual Positive | True Positive (TP) | False Negative (FN) |
| Actual Negative | False Positive (FP) | True Negative (TN) |

The common metrics are:

1. Accuracy
2. Precision
3. Recall
4. F1 Score
5. Specificity and False Positive Rate
6. ROC and AUC

Let us use an example to show how to use a confusion matrix and compute the evaluation metrics of a classification model. Suppose a medical test system predicts whether a person has a certain disease. The "positive class" means "has the disease", and the "negative class" means "does not have the disease". The test results of 1000 samples are shown below.

|      | Predicted Sick | Predicted Not Sick |
| ---- | -------------- | ------------------ |
| Actual Sick | 80 (TP) | 20 (FN) |
| Actual Not Sick | 30 (FP) | 870 (TN) |

1. **Accuracy**.

$$
\text{Accuracy} = \frac{\text{TP} + \text{TN}}{\text{TP} + \text{FP} + \text{FN} + \text{TN}}
$$

In the example above, the model accuracy is: $\frac{80 + 870}{80 + 30 + 20 + 870} = \frac{950}{1000} = 0.95$ .

2. **Precision**. Precision measures the proportion of samples that are actually positive among all samples predicted as positive. It is also often called positive predictive value.

$$
\text{Precesion} = \frac{\text{TP}}{\text{TP} + \text{FP}}
$$

In the example above, precision is: $\frac{80}{80 + 30} = \frac{80}{110} = 0.73$ .

3. **Recall**. Recall measures the proportion of samples correctly predicted as positive among all actually positive samples. It is also often called true positive rate.

$$
\text{Recall} = \frac{\text{TP}}{\text{TP} + \text{FN}}
$$

In the example above, recall is: $\frac{80}{80 + 20} = \frac{80}{100} = 0.8$ .

4. **F1 Score**. F1 score is the harmonic mean of precision and recall. It tries to find a balance between precision and recall, especially when there is a trade-off between them.

$$
\text{F1 Score} = \frac{2}{\frac{1}{\text{Precision}} + \frac{1}{\text{Recall}}} = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precesion} + \text{Recall}}
$$

In the example above, the F1 score is about `0.76`.

5. **Specificity** and **False Positive Rate**.

$$
\text{Specificity} = \frac{\text{TN}}{\text{TN} + \text{FP}} \\\\
\text{FPR} = 1 - \text{Specificity}
$$

In the example above, specificity is: $\frac{870}{870 + 30} = \frac{870}{900} = 0.97$ .

6. **ROC** and **AUC**.

ROC draws the relationship between recall and false positive rate, as shown below.

<img src="res/02_ROC_curve.png" style="zoom:38%;">

AUC is the area under the ROC curve. It measures the model's ability to distinguish positive and negative classes.

### Parameter Tuning

We said before that the kNN algorithm has two key problems. One is distance measurement, and the other is the choice of the value of `k`. When we use scikit-learn's `KNeighborsClassifier` to create a classifier model, we can set the hyperparameters of the model. Here are several important parameters:

1. `n_neighbors`: the number of neighbors, that is, the value of `k` in the kNN algorithm.
2. `weights`: can be `uniform` or `distance`. The first means all samples have the same weight, and the second means the nearer the distance, the higher the weight. The default value is `uniform`. Of course, we can also pass in a custom function to decide the weight of each sample.
3. `algorithm`: has four options, `auto`, `ball_tree`, `kd_tree`, and `brute`. The default value is `auto`. `ball_tree` is a tree structure that uses a sphere-based partition method to put data points into a hierarchical tree structure, and it performs well on high-dimensional data and sparse data. `kd_tree` is also a tree structure. It chooses one dimension to divide the space into several sub-regions and then searches, so it avoids comparing with every neighbor. It works well for low-dimensional data and data with even spatial distribution, but it meets the curse of dimensionality in high-dimensional space. The `auto` option automatically chooses `ball_tree` or `kd_tree` according to the dimension of the input data. The `brute` option uses the brute-force search algorithm. For small datasets, it is a simple and effective choice.
4. `leaf_size`: when using the `ball_tree` or `kd_tree` algorithm, this parameter is used to limit the largest number of samples in leaf nodes of the tree structure. The default value is `30`. This parameter affects the performance of tree construction and node lookup.
5. `p`: the `p` in the Minkowski distance formula. The default value is `2`, which means Euclidean distance.

We can use **grid search** and **cross validation** to tune model hyperparameters, evaluate the generalization ability of the model, and improve prediction performance. Grid search means using exhaustive search to go through the given hyperparameter space and find the best hyperparameter combination. Cross validation means splitting the training set into several subsets and then training and evaluating the model many times on different training sets and validation sets, so we can make a full judgment on prediction performance. K-Fold cross validation is the most common cross-validation method. It divides the dataset into `K` subsets. Each time, one subset is chosen as the validation set, and the other `K-1` subsets are used as the training set. This process is repeated `K` times, and the average value is used as the final performance evaluation of the model, as shown below.

![](res/02_k-fold_cross_validation.png)

The code below uses `GridSearchCV` in scikit-learn to do grid search and cross validation. In this way, we can find the best parameters for using the kNN algorithm on the iris dataset.

```python
from sklearn.model_selection import GridSearchCV

# grid search with cross validation
gs = GridSearchCV(
    estimator=KNeighborsClassifier(),
    param_grid={
        'n_neighbors': [1, 3, 5, 7, 9, 11, 13, 15],
        'weights': ['uniform', 'distance'],
        'p': [1, 2]
    },
    cv=5
)
gs.fit(X_train, y_train)
```

We can get the best parameters and the score with the following code.

```python

print('Score:', gs.best_score_)
```


### kNN Regression

kNN is usually used to solve classification problems, but of course it can also be used to solve regression problems. Its basic idea is still to find the `k` neighbors nearest to the new instance, and then predict the target value of the new instance according to the labels of those `k` neighbors. In regression problems, we usually call them target values instead of class labels. Because what we need to predict is not a class label but a number, we usually use average or weighted average to handle the target values of the `k` neighbors, and then get the predicted target value of the new instance. Let us go back to the example in the previous lesson that used monthly income to predict online-shopping expense, and then use `KNeighborsRegressor` in scikit-learn to build a regression model. ### Summary

kNN is a simple but still powerful machine learning algorithm. It is suitable for classification and regression tasks on small datasets. Its advantages are that it is simple to understand, has no explicit training process, does not depend on assumptions about the data distribution, and can adapt to complex data patterns. Of course, the disadvantages of kNN are also very clear. The biggest problem is computational efficiency, so when the dataset is large, it may not be the best choice. Besides that, kNN is very sensitive to noise, and the prediction result depends on the value of `k` and whether the samples are balanced. If the value of `k` is too small, it may cause overfitting. If the value of `k` is too large, it may cause underfitting. Unbalanced samples may also lead to class imbalance bias. In other words, if most training samples belong to class A and there are few class B samples, then during classification, kNN is more likely to predict the test sample as class A.
