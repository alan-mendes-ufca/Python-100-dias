## Visão geral de bancos de dados relacionais e MySQL

### Visão geral do banco de dados relacional

1. Persistência de dados significa salvar dados em um meio de armazenamento que possa mantê-los por um longo tempo, para que os dados não sejam perdidos mesmo se a energia for cortada.

2. A história do desenvolvimento de banco de dados inclui bancos de dados de rede, bancos de dados hierárquicos, bancos de dados relacionais, bancos de dados NoSQL e bancos de dados NewSQL.

   > Em 1970, o pesquisador da IBM E. F. Codd publicou o artigo *A Relational Model of Data for Large Shared Data Banks* em *Communications of the ACM*. Nesse artigo ele propôs o **modelo relacional**, estabelecendo a base teórica dos bancos de dados relacionais. Mais tarde, Codd publicou vários outros artigos sobre a teoria da normalização e as doze regras para avaliar sistemas relacionais, fundamentando ainda mais os bancos de dados relacionais na teoria matemática.

3. Características dos bancos de dados relacionais.

   - Fundamentação teórica: **álgebra relacional** (teoria dos conjuntos, predicados de primeira ordem, operações relacionais).
   - Forma concreta: use **tabelas bidimensionais** (com linhas e colunas) para organizar os dados.
   - Linguagem de programação: **Linguagem de consulta estruturada** (SQL).
     - DDL: Data Definition Language
     - DML: Data Manipulation Language
     - DCL: Data Control Language
     - TCL: Transaction Control Language

4. Modelo ER (Modelo Entidade-Relacionamento) e diagrama do modelo conceitual.

O **modelo ER**, cujo nome completo é **Modelo Entidade-Relacionamento**, foi proposto pelo cientista da computação sino-americano Peter Chen. É uma forma de alto nível de descrever um modelo de dados conceitual, conforme mostrado na imagem abaixo.

   <img src="res/er_diagram.png" width="75%">

   - entidade: retângulo
   - atributo: elipse
   - relacionamento: diamante
   - cardinalidade: 1:1 (um para um) / 1:N (um para muitos) / M:N (muitos para muitos)

No desenvolvimento de projetos reais, podemos usar ferramentas de modelagem de banco de dados, como o PowerDesigner, para desenhar um modelo de dados conceitual. Em seguida, definimos o sistema de banco de dados de destino, convertemos o modelo conceitual em um modelo físico e, finalmente, geramos o SQL para a criação de tabelas bidimensionais. Muitas ferramentas podem exportar SQL diretamente ou gerar tabelas de dados de acordo com o modelo físico que projetamos e o banco de dados de destino que definimos.

   <img src="res/conceptual_model.png" style="zoom:50%;">

5. Produtos de banco de dados relacional.

   - [Oráculo](https://www.oracle.com/index.html): atualmente o sistema de gerenciamento de banco de dados mais utilizado no mundo. Como um sistema de banco de dados de uso geral, possui recursos completos de gerenciamento de dados. Como banco de dados relacional, é um produto completo. Como um banco de dados distribuído, ele oferece suporte ao processamento distribuído. Nas versões mais recentes, a Oracle também introduziu uma arquitetura multitenant, que facilita a implantação e o gerenciamento de nuvens de banco de dados.
   - [DB2](https://www.ibm.com/analytics/us/en/db2/): um produto de banco de dados relacional desenvolvido pela IBM, executado principalmente em Unix, incluindo AIX, Linux e Windows Server da própria IBM. O DB2 tem uma longa história e é considerado um dos primeiros produtos de banco de dados baseados em SQL. Também possui fortes capacidades de inteligência de negócios.
   - [Servidor SQL](https://www.microsoft.com/en-us/sql-server/): um produto de banco de dados relacional desenvolvido e promovido pela Microsoft. Originalmente, era adequado principalmente para a gestão de dados em pequenas e médias empresas, mas nos últimos anos a sua utilização expandiu-se e algumas grandes empresas e até empresas multinacionais começaram a construir os seus próprios sistemas de gestão de dados com base nela.
   - [MySQL](https://www.mysql.com/): MySQL é de código aberto e qualquer pessoa pode baixá-lo sob a GPL, Licença Pública Geral, e modificá-lo para atender às suas próprias necessidades. O MySQL atraiu atenção por sua velocidade, confiabilidade e adaptabilidade.
   - [PostgreSQL](https://www.postgresql.org/): um produto de banco de dados relacional de código aberto distribuído sob a licença BSD.

### Introdução ao MySQL

O MySQL foi originalmente desenvolvido pela empresa sueca MySQL AB como um sistema de gerenciamento de banco de dados relacional de código aberto. Essa empresa foi adquirida pela Sun Microsystems em 2008. Em 2009, a Oracle adquiriu a Sun Microsystems, então o MySQL agora também é um produto da Oracle.

Como o MySQL tinha alto desempenho, baixo custo e boa confiabilidade, ele se tornou o banco de dados de código aberto mais popular do passado, por isso foi amplamente utilizado no desenvolvimento de sites de pequeno e médio porte. À medida que o MySQL foi se tornando mais maduro, ele foi gradualmente usado em sites e aplicativos de maior escala. Wikipedia, Google, Facebook, Baidu, Taobao, Tencent, Sina, Qunar e muitos outros usaram MySQL para fornecer serviços de persistência de dados.

Depois que a Oracle adquiriu a Sun Microsystems, o preço do MySQL Commercial Edition aumentou muito, e a Oracle não apoiou mais o desenvolvimento de outro projeto de software livre, o OpenSolaris. Por causa disso, a comunidade do software livre ficou preocupada se a Oracle continuaria a oferecer suporte ao MySQL Community Edition, a única versão gratuita entre os lançamentos do MySQL. Assim, o fundador do MySQL, Michael Widenius, criou o branch [Maria DB](https://en.wikipedia.org/wiki/MariaDB) baseado no MySQL e deu-lhe o nome de sua filha. Muitas empresas que originalmente usavam MySQL, como a Wikipedia, já terminaram de migrar do MySQL para o MariaDB.

#### Ambiente Linux

O exemplo abaixo usa CentOS 7.x para mostrar como instalar o MySQL 5.7.x. Se você precisar instalar outra versão do MySQL em outro sistema Linux, consulte você mesmo o tutorial de instalação correspondente.

1. Instale o MySQL.

Você pode baixar os arquivos de instalação do [site oficial do MySQL](https://www.mysql.com/). Primeiro escolha a plataforma e a versão na página de download, a seguir encontre o link de download correspondente, baixe diretamente o arquivo que contém todos os arquivos de instalação e, após extraí-lo, use a ferramenta de gerenciamento de pacotes para instalá-lo.

   ```bash
   wget https://dev.mysql.com/get/Downloads/MySQL-5.7/mysql-5.7.26-1.el7.x86_64.rpm-bundle.tar
   tar -xvf mysql-5.7.26-1.el7.x86_64.rpm-bundle.tar
   ```

Se houver arquivos relacionados ao MariaDB no sistema, você precisará removê-los primeiro.

   ```bash
   yum list installed | grep mariadb | awk '{print $1}' | xargs yum erase -y
   ```

Atualize e instale as bibliotecas de dependência de baixo nível que podem ser necessárias.

   ```bash
   yum update
   yum install -y libaio libaio-devel
   ```

A seguir, siga a ordem abaixo e utilize a ferramenta RPM (Red Hat Package Manager) para instalar o MySQL.

   ```bash
   rpm -ivh mysql-community-common-5.7.26-1.el7.x86_64.rpm
   rpm -ivh mysql-community-libs-5.7.26-1.el7.x86_64.rpm
   rpm -ivh mysql-community-libs-compat-5.7.26-1.el7.x86_64.rpm
   rpm -ivh mysql-community-devel-5.7.26-1.el7.x86_64.rpm
   rpm -ivh mysql-community-client-5.7.26-1.el7.x86_64.rpm
   rpm -ivh mysql-community-server-5.7.26-1.el7.x86_64.rpm
   ```

Você pode usar o comando abaixo para ver os pacotes relacionados ao MySQL instalados.

   ```bash
   rpm -qa | grep mysql
   ```

2. Configurar MySQL.

O arquivo de configuração do MySQL está localizado em `/etc` e é denominado `my.cnf`. Seu conteúdo padrão é mostrado abaixo.

   ```bash
   cat /etc/my.cnf
   ```

   ```ini
   # For advice on how to change settings please see
   # http://dev.mysql.com/doc/refman/5.7/en/server-configuration-defaults.html

   [mysqld]
   #
   # Remove leading # and set to the amount of RAM for the most important data
   # cache in MySQL. Start at 70% of total RAM for dedicated server, else 10%.
   # innodb_buffer_pool_size = 128M
   #
   # Remove leading # to turn on a very important data integrity option: logging
   # changes to the binary log between backups.
   # log_bin
   #
   # Remove leading # to set options mainly useful for reporting servers.
   # The server defaults are faster for transactions and fast SELECTs.
   # Adjust sizes as needed, experiment to find the optimal values.
   # join_buffer_size = 128M
   # sort_buffer_size = 2M
   # read_rnd_buffer_size = 2M
   datadir=/var/lib/mysql
   socket=/var/lib/mysql/mysql.sock

   # Disabling symbolic-links is recommended to prevent assorted security risks
   symbolic-links=0

   log-error=/var/log/mysqld.log
   pid-file=/var/run/mysqld/mysqld.pid
   ```

Através do arquivo de configuração, podemos modificar configurações como a porta usada pelo serviço MySQL, o conjunto de caracteres, o número máximo de conexões, o tamanho da fila do soquete, o tamanho máximo do pacote, a posição dos arquivos de log e o tempo de expiração do log. Claro, também podemos usar o arquivo de configuração para ajustar o desempenho do servidor MySQL e controlar a segurança.

3. Inicie o serviço MySQL.

Você pode usar o comando abaixo para iniciar o MySQL.

   ```bash
   service mysqld start
   ```

No CentOS 7, o comando abaixo é mais recomendado.

   ```bash
   systemctl start mysqld
   ```

Depois que o MySQL for iniciado com sucesso, você pode usar o seguinte comando para verificar o uso da porta de rede. MySQL usa a porta `3306` por padrão.

   ```bash
   netstat -ntlp | grep mysql
   ```

Você também pode usar o seguinte comando para verificar se existe um processo chamado `mysqld`.

   ```bash
   pgrep mysqld
   ```

4. Use a ferramenta cliente MySQL para conectar-se ao servidor.

Ferramenta de linha de comando:

   ```bash
   mysql -u root -p
   ```

   > **Nota**: Ao iniciar o cliente, a opção `-u` é usada para especificar o nome de usuário. A conta de superadministrador padrão do MySQL é `root`. `-p` significa que você digitará uma senha. Se você se conectar a outro host em vez da máquina local, poderá usar `-h` para especificar o nome do host ou endereço IP do host ao qual você se conecta.

Se esta for a primeira instalação do MySQL, você pode usar o comando abaixo para encontrar a senha inicial padrão.

   ```bash
   cat /var/log/mysqld.log | grep password
   ```

A parte após `root@localhost:` na linha de registro correspondente é a senha inicial.

Depois de entrar no cliente, você pode usar os comandos abaixo para alterar a senha de superusuário de `root` para `123456`.

   ```sql
   set global validate_password_policy=0;
   set global validate_password_length=6;
   alter user 'root'@'localhost' identified by '123456';
   ```

   > **Nota**: As versões mais recentes do MySQL não permitem senhas fracas por padrão, então os comandos acima primeiro alteram a política de validação de senha e o comprimento mínimo da senha. Na realidade, porém, não devemos usar senhas fracas, porque elas acarretam o risco de ataques de força bruta. Nos últimos anos, ataques que roubam dados de bancos de dados ou sequestram bancos de dados para resgate de Bitcoin tornaram-se cada vez mais comuns. A etapa mais importante para reduzir esses riscos é **não expor o servidor de banco de dados diretamente à Internet pública**. A melhor prática é colocar o banco de dados em uma rede interna. No mínimo, a porta do banco de dados não deve ser aberta ao público. Além disso, a senha da conta `root` deve ser protegida cuidadosamente. Os sistemas aplicativos normalmente não devem acessar o banco de dados usando `root`; em vez disso, **outras contas apenas com as permissões necessárias devem ser criadas**.

Ao usar a ferramenta cliente novamente para se conectar ao servidor MySQL, você poderá usar a nova senha. No desenvolvimento real, para facilitar as coisas, você também pode escolher uma ferramenta gráfica de cliente para se conectar ao servidor MySQL, incluindo:

### Comandos básicos do MySQL

#### Comandos para visualizar informações

1. Ver todos os bancos de dados

```sql
show databases;
```

2. Ver todos os conjuntos de caracteres

```sql
show character set;
```

3. Ver todos os agrupamentos

```sql
show collation;
```

4. Ver todos os motores

```sql
show engines;
```

5. Ver todos os logs binários

```sql
show binary logs;
```

6. Ver todas as tabelas no banco de dados atual

```sql
show tables;
```

#### Obtenha ajuda

Na ferramenta de linha de comando MySQL, você pode usar o comando `help` ou `?` para obter ajuda, conforme mostrado abaixo.

1. Veja a ajuda do comando `show`.

   ```sql
   ? show
   ```

2. Veja quais conteúdos de ajuda estão disponíveis.

   ```sql
   ? contents
   ```

3. Obtenha ajuda para funções.

   ```sql
   ? functions
   ```

4. Obtenha ajuda para tipos de dados.

   ```sql
   ? data types
   ```

#### Outros comandos

1. Crie uma nova conexão de servidor ou reconstrua a conexão atual: `connect`/`resetconnection`.

2. Limpe a entrada atual: `\c`. Se você digitar algo errado, poderá usar `\c` a tempo de limpar a entrada atual e começar novamente.

3. Altere o terminador, também chamado de delimitador: `delimiter`. O terminador padrão é `;`. Você pode usar este comando para alterá-lo para outro personagem. Por exemplo, para alterá-lo para `$`, você pode usar o comando `delimiter $`.

4. Abra o editor padrão do sistema: `edit`. Depois de terminar a edição, salve e feche-o, e a linha de comando executará automaticamente o que você editou.

5. Visualize o status do servidor: `status`.

6. Altere o prompt padrão: `prompt`.

7. Execute um comando do sistema: `system`. Você pode colocar um comando do sistema após `system` e executá-lo. O comando `system` também pode ser abreviado para `\!`.

8. Execute um arquivo SQL: `source`. Após o comando `source`, coloque o caminho do arquivo SQL.

9. Saída de redirecionamento: `tee` / `notee`. Você pode redirecionar a saída dos comandos para o arquivo especificado.

10. Banco de dados do switch: `use`.

11. Mostrar mensagens de aviso: `warnings`.

12. Saia da linha de comando: `quit` ou `exit`.
