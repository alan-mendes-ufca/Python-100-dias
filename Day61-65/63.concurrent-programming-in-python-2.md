## Programação Simultânea em Python, Parte 2

Na lição anterior, dissemos que devido à existência do GIL, o multithreading no CPython não pode aproveitar a vantagem multi-core da CPU. Se quiser ultrapassar o limite do GIL, você pode considerar o uso de multiprocessamento. Em um programa de multiprocessamento, cada processo possui seu próprio GIL, portanto o multiprocessamento não é afetado pelo GIL. Então, como devemos criar e usar múltiplos processos em programas Python?

### Criando Processos

Em Python, podemos criar processos baseados na classe `Process`. Embora os processos e threads sejam essencialmente diferentes, o uso da classe `Process` e da classe `Thread` é muito semelhante. Ao criar um objeto com o construtor da classe `Process`, também passamos uma função através do parâmetro `target` para especificar o código a ser executado pelo processo, enquanto os parâmetros `args` e `kwargs` podem especificar os valores dos argumentos utilizados por aquela função.

```python
from multiprocessing import Process, current_process
from time import sleep


def sub_task(content, nums):
    # Get the current process object through the current_process function
    # Get the process ID and name through the pid and name attributes of the process object
    print(f'PID: {current_process().pid}')
    print(f'Name: {current_process().name}')
    # Through the output below, it is not hard to find that each process has its own nums list
    # Processes do not share memory in the first place
    # When creating child processes, the data structure of the parent process is copied
    # so the value got by pop(0) from the list is 20 in all three processes
    counter, total = 0, nums.pop(0)
    print(f'Loop count: {total}')
    sleep(0.5)
    while counter < total:
        counter += 1
        print(f'{counter}: {content}')
        sleep(0.01)


def main():
    nums = [20, 30, 40]
    # Create and start processes to execute the specified function
    Process(target=sub_task, args=('Ping', nums)).start()
    Process(target=sub_task, args=('Pong', nums)).start()
    # Execute the sub_task function in the main process
    sub_task('Good', nums)


if __name__ == '__main__':
    main()
```

> **Observação**: O código acima obtém o objeto de processo atual por meio da função `current_process` e, em seguida, obtém o ID do processo por meio do atributo `pid` do objeto de processo. Em Python, usar a função `getpid` do módulo `os` também pode obter o mesmo efeito.

Se desejar, você também pode utilizar a função `fork` do módulo `os` para criar processos. Quando esta função é chamada, o sistema operacional copia automaticamente o processo atual, o processo pai, em um processo filho. A função `fork` no processo pai retorna o ID do processo filho, enquanto a função `fork` no processo filho retorna `0`. Em outras palavras, uma chamada para esta função obtém dois valores de retorno diferentes no processo pai e no processo filho. Deve-se observar que o sistema Windows não suporta a função `fork`. Se estiver usando Linux ou macOS, você pode tentar o código abaixo.

```python
import os

print(f'PID: {os.getpid()}')
pid = os.fork()
if pid == 0:
    print(f'Child process - PID: {os.getpid()}')
    print('Todo: code executed in the child process')
else:
    print(f'Parent process - PID: {os.getpid()}')
    print('Todo: code executed in the parent process')
```

Resumindo, ainda recomendamos que todos criem e utilizem o multiprocessamento de três maneiras: diretamente usando a classe `Process`, herdando a classe `Process` e usando um pool de processos, `ProcessPoolExecutor`. Diferente da função `fork` acima, essas três formas podem garantir compatibilidade e portabilidade de código. A forma específica é relativamente próxima da forma de criar e usar multithreading de que falamos antes, por isso não vamos repeti-la aqui.

### Comparação entre multiprocessamento e multithreading

Para tarefas com uso intensivo de E/S, como rastreadores, o multiprocessamento não oferece muitas vantagens. Mas para tarefas com uso intensivo de CPU, o multiprocessamento apresenta uma clara melhoria na eficiência em comparação com o multithreading, e podemos provar isso com o código abaixo. O código abaixo usa multithreading e multiprocessamento para julgar se um grupo de números inteiros grandes são números primos. Claramente, esta é uma tarefa que exige muita CPU. Colocamos a tarefa em vários threads e vários processos para acelerar a execução e então ver que diferença o código multithread e o código multiprocesso mostram.

Vamos primeiro implementar uma versão multithread. O código é mostrado abaixo.

```python
import concurrent.futures

PRIMES = [
    1116281,
    1297337,
    104395303,
    472882027,
    533000389,
    817504243,
    982451653,
    112272535095293,
    112582705942171,
    112272535095293,
    115280095190773,
    115797848077099,
    1099726899285419
] * 5


def is_prime(n):
    """Judge whether it is a prime"""
    for i in range(2, int(n ** 0.5) + 1):
        if n % i == 0:
            return False
    return n != 1


def main():
    """Main function"""
    with concurrent.futures.ThreadPoolExecutor(max_workers=16) as executor:
        for number, prime in zip(PRIMES, executor.map(is_prime, PRIMES)):
            print('%d is prime: %s' % (number, prime))


if __name__ == '__main__':
    main()
```

Suponha que o código acima seja salvo em um arquivo chamado `example.py`. No Linux ou macOS, podemos usar o comando `time python example.py` para executar o programa e obter as estatísticas do sistema operacional sobre o tempo de execução. No meu macOS, a última linha de saída de uma execução é mostrada abaixo.

```text

```

A partir do resultado da execução, podemos ver que o código multithread só pode fazer com que a utilização da CPU atinja cerca de 100%. Na verdade, isso já prova que o código multithread não pode usar o recurso multi-core da CPU para acelerar a execução. Vejamos agora a versão de multiprocessamento. Substituímos o pool de threads, `ThreadPoolExecutor`, no código acima pelo pool de processos, `ProcessPoolExecutor`.

Versão multiprocessamento.

Se essas tarefas com uso intensivo de CPU puderem ser paralelizadas, o uso de multiprocessamento deverá ser uma escolha melhor.

### Comunicação entre processos

Antes de falar sobre comunicação entre processos, vamos primeiro dar a todos uma tarefa: iniciar dois processos, um gera `Ping`, outro gera `Pong` e quando a contagem total de `Ping` e `Pong` produzidos pelos dois processos atinge 50, o programa termina. Parece muito simples, mas ao escrever o código, como vários processos não podem trocar dados diretamente compartilhando memória como vários threads fazem, o código abaixo não pode obter o resultado que desejamos.

```python
from multiprocessing import Process
from time import sleep

counter = 0


def sub_task(string):
    global counter
    while counter < 50:
        print(string, end='', flush=True)
        counter += 1
        sleep(0.01)


def main():
    Process(target=sub_task, args=('Ping', )).start()
    Process(target=sub_task, args=('Pong', )).start()


if __name__ == '__main__':
    main()
```

O código acima parece bom, mas no resultado final, `Ping` e `Pong` são gerados 50 vezes cada. Deixe-me lembrar a todos novamente que quando criamos processos em um programa, os processos filhos copiam o processo pai e todas as suas estruturas de dados. Cada processo filho possui seu próprio espaço de memória independente. Isso também significa que existe uma variável `counter` em cada processo filho, e ambas contarão de `0` a `50`, então o resultado pode ser imaginado. Uma maneira mais simples de resolver este problema é usar a classe `Queue` no módulo `multiprocessing`. É uma fila que pode ser compartilhada por vários processos. Na parte inferior, ele é implementado por meio dos pipes e semáforos do sistema operacional. O código é mostrado abaixo.

```python
import time
from multiprocessing import Process, Queue


def sub_task(content, queue):
    counter = queue.get()
    while counter < 50:
        print(content, end='', flush=True)
        counter += 1
        queue.put(counter)
        time.sleep(0.01)
        counter = queue.get()


def main():
    queue = Queue()
    queue.put(0)
    p1 = Process(target=sub_task, args=('Ping', queue))
    p1.start()
    p2 = Process(target=sub_task, args=('Pong', queue))
    p2.start()
    while p1.is_alive() and p2.is_alive():
        pass
    queue.put(50)


if __name__ == '__main__':
    main()
```

> **Dica**: por padrão, o método `get` de um objeto `multiprocessing.Queue` é bloqueado quando a fila está vazia e não retorna até obter dados. Se você não deseja que este método seja bloqueado ou precisa especificar o tempo limite de bloqueio, você pode configurá-lo através dos parâmetros `block` e `timeout`.

O código acima usa os métodos `get` e `put` da classe `Queue` para permitir que três processos, `p1`, `p2` e o processo principal, compartilhem dados. Esta é a chamada comunicação entre processos. Desta forma, quando o valor retirado de `Queue` já for maior ou igual a `50`, `p1` e `p2` saltarão do loop `while`, finalizando a execução dos processos. O loop na linha 22 é usado para esperar até que um dos `p1` e `p2` termine. Neste momento, o processo principal ainda precisa colocar um valor maior ou igual a `50` no `Queue`, para que o outro processo que ainda não finalizou também termine porque lê um valor maior ou igual a `50`.

Existem muitas outras formas de comunicação entre processos. Por exemplo, soquetes também podem ser usados ​​para comunicação entre dois processos, e esses dois processos nem precisam estar no mesmo host. Os leitores interessados ​​​​podem investigar isso por conta própria.

### Resumo

Em Python, também podemos usar a função `call` do módulo `subprocess` para executar outros comandos e criar processos filhos, o que equivale a chamar outros programas dentro do nosso programa. Não discutiremos esse conhecimento por enquanto. Os leitores interessados ​​podem estudá-lo por conta própria.

Para desenvolvedores Python, as seguintes situações precisam ser consideradas ao usar multithreading:

1. O programa precisa manter muitos estados compartilhados, especialmente estados mutáveis. Listas, dicionários e conjuntos em Python são thread-safe. Se vários threads operarem na mesma lista, dicionário ou conjunto ao mesmo tempo, eles não causarão erros e problemas de dados. Portanto, o custo de usar threads em vez de processos para manter o estado compartilhado é relativamente pequeno.
2. O programa gastará muito tempo em operações de E/S, não terá muita demanda por computação paralela e não precisará usar muita memória.

Então, ao atender as situações abaixo, você deve considerar o uso de multiprocessamento:

1. O programa executa tarefas que exigem uso intensivo da CPU, como codificação e decodificação de áudio e vídeo, compactação de dados, computação científica e assim por diante.
2. A entrada do programa pode ser dividida em blocos em paralelo e os resultados do cálculo podem ser mesclados.
3. O programa não tem restrição de uso de memória e não depende fortemente de operações de E/S, como leitura e gravação de arquivos ou soquetes.
