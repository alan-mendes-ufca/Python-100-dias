## Introdução ao Scrapy Framework

Depois de escrever muitos programas rastreadores, você descobrirá que toda vez que escrever um programa rastreador, será necessário implementar a obtenção de páginas, a análise de páginas, o agendamento do rastreador, o tratamento de exceções e a resposta anti-rastreador do início ao fim. Muito desse trabalho é, na verdade, um trabalho simples, chato e repetido. Então, há alguma maneira de melhorar a eficiência da escrita do código do rastreador? A resposta é sim. Essa forma é usar uma estrutura de rastreador e, entre todas as estruturas de rastreador, Scrapy deve ser a estrutura mais popular e poderosa.

### Visão geral do Scrapy

Scrapy é uma estrutura de rastreador da web muito popular baseada em Python. Ele pode ser usado para rastrear sites e extrair dados estruturados de páginas. A imagem abaixo mostra a arquitetura básica do Scrapy, incluindo seus principais componentes e o fluxo de processamento de dados do sistema, que são as setas vermelhas com números na imagem.

#### Componentes do Scrapy

Vamos primeiro falar sobre os componentes do Scrapy.

1. Scrapy Engine: usado para controlar o fluxo de processamento de dados de todo o sistema.
2. Agendador: o agendador aceita solicitações do mecanismo, classifica-as, coloca-as em uma fila e as devolve ao mecanismo após o mecanismo enviar as solicitações.
3. Downloader: a principal função do downloader é buscar páginas da web e retornar o conteúdo da página aos spiders.
4. Spiders: spiders são classes definidas pelo usuário usadas para analisar páginas da web e rastrear URLs específicos. Cada spider pode lidar com um domínio ou um grupo de domínios. Simplificando, são módulos utilizados para definir as regras de rastreamento e análise de sites específicos.
5. Pipeline de itens: a principal responsabilidade do pipeline é lidar com os itens de dados extraídos da página pelos spiders. Suas principais tarefas são limpar, validar e armazenar dados. Depois que a página é analisada pelo spider, ela é enviada ao pipeline de itens e os dados são tratados em vários pedidos específicos. Cada componente do pipeline de itens é uma classe Python. Eles obtêm o item de dados e executam métodos que processam o item de dados e, ao mesmo tempo, precisam decidir se a próxima etapa no pipeline do item deve continuar em execução ou se o item deve ser descartado diretamente sem processamento. As tarefas geralmente realizadas pelo pipeline de itens incluem limpar dados HTML, validar dados analisados, verificar se são dados duplicados e eliminá-los se forem, e armazenar os dados analisados em um banco de dados, seja um banco de dados relacional ou um banco de dados NoSQL.
6. Middlewares: middlewares são uma estrutura de gancho entre o mecanismo e outros componentes. Eles são usados principalmente para fornecer código personalizado para estender as funções do Scrapy, incluindo middleware de download e middleware spider.

#### Fluxo de processamento de dados

Todo o fluxo de processamento de dados do Scrapy é controlado pelo mecanismo. O fluxo normal de execução geralmente inclui as etapas abaixo:

1. O mecanismo pergunta ao spider qual site precisa ser gerenciado e permite que o spider entregue a primeira URL que precisa ser manipulada.
2. O mecanismo informa ao agendador para colocar a URL que precisa ser manipulada na fila.
3. O mecanismo obtém a próxima página para rastrear do agendador.
4. O agendador retorna o próximo URL para rastreamento ao mecanismo, e o mecanismo o envia ao downloader por meio do middleware do downloader.
5. Depois que a página é baixada pelo downloader, o conteúdo da resposta é enviado ao mecanismo por meio do middleware do downloader. Se o download falhar, o mecanismo informa ao agendador para gravar esta URL e baixá-la novamente mais tarde.
6. O mecanismo recebe a resposta do downloader e a envia ao spider por meio do middleware spider para tratamento.
7. O spider trata a resposta e retorna os itens de dados rastreados. Além disso, ele envia ao mecanismo as novas URLs que precisam ser seguidas.
8. O mecanismo envia os itens de dados rastreados para o pipeline de itens e envia os novos URLs para o agendador para serem colocados na fila.

As etapas 2 a 8 acima continuam se repetindo até que não haja nenhum URL no agendador que precise ser solicitado e, então, o rastreador pare de funcionar.

### Instalando e usando Scrapy

Podemos usar a ferramenta de gerenciamento de pacotes do Python `pip` para instalar o Scrapy.

```bash
pip install scrapy
```

Use o comando `scrapy` na linha de comando para criar um projeto denominado `demo`.

```bash
scrapy startproject demo
```

A estrutura de diretórios do projeto é mostrada abaixo.

```text
demo
|____ demo
|________ spiders
|____________ __init__.py
|________ __init__.py
|________ items.py
|________ middlewares.py
|________ pipelines.py
|________ settings.py
|____ scrapy.cfg
```

Mude para o diretório `demo` e use o comando abaixo para criar um programa spider chamado `douban`.

```bash
scrapy genspider douban movie.douban.com
```

#### Um exemplo simples

Em seguida, implementamos um rastreador que obtém os títulos, classificações e lemas dos filmes do Douban Movie Top250.

1. Defina campos na classe `Item` em `items.py`. Esses campos são usados para salvar dados e tornar as operações posteriores convenientes.

   ```python
   import scrapy


   class DoubanItem(scrapy.Item):
       title = scrapy.Field()
       score = scrapy.Field()
       motto = scrapy.Field()
   ```

2. Modifique o arquivo denominado `douban.py` na pasta `spiders`. Este é o núcleo do programa spider e precisamos adicionar código para analisar páginas. Aqui, podemos obter informações do filme analisando o objeto `Response`. O código é mostrado abaixo.

   ```python
   import scrapy
   from scrapy import Selector, Request
   from scrapy.http import HtmlResponse

   from demo.items import MovieItem


   class DoubanSpider(scrapy.Spider):
       name = 'douban'
       allowed_domains = ['movie.douban.com']
       start_urls = ['https://movie.douban.com/top250?start=0&filter=']

       def parse(self, response: HtmlResponse):
           sel = Selector(response)
           movie_items = sel.css('#content > div > div.article > ol > li')
           for movie_sel in movie_items:
               item = MovieItem()
               item['title'] = movie_sel.css('.title::text').extract_first()
               item['score'] = movie_sel.css('.rating_num::text').extract_first()
               item['motto'] = movie_sel.css('.inq::text').extract_first()
               yield item
   ```

Não é difícil ver no código acima que podemos usar seletores CSS para análise de páginas. Claro, se desejar, você também pode usar XPath ou expressões regulares para análise de página. Os métodos correspondentes são `xpath` e `re`.

Se você ainda precisar gerar solicitações de rastreamento de acompanhamento, podemos usar `yield` para produzir objetos `Request`. Um objeto `Request` possui duas propriedades muito importantes. Um é `url`, que significa o endereço a ser solicitado. O outro é `callback`, que significa a função de retorno de chamada a ser executada após a resposta ser obtida. Podemos alterar ligeiramente o código acima.

   ```python
   import scrapy
   from scrapy import Selector, Request
   from scrapy.http import HtmlResponse

   from demo.items import MovieItem


   class DoubanSpider(scrapy.Spider):
       name = 'douban'
       allowed_domains = ['movie.douban.com']
       start_urls = ['https://movie.douban.com/top250?start=0&filter=']

       def parse(self, response: HtmlResponse):
           sel = Selector(response)
           movie_items = sel.css('#content > div > div.article > ol > li')
           for movie_sel in movie_items:
               item = MovieItem()
               item['title'] = movie_sel.css('.title::text').extract_first()
               item['score'] = movie_sel.css('.rating_num::text').extract_first()
               item['motto'] = movie_sel.css('.inq::text').extract_first()
               yield item

           hrefs = sel.css('#content > div > div.article > div.paginator > a::attr("href")')
           for href in hrefs:
               full_url = response.urljoin(href.extract())
               yield Request(url=full_url)
   ```

Neste ponto já podemos executar o crawler através do comando abaixo.

   ```bash
   scrapy crawl movie
   ```

Podemos ver os dados rastreados no console. Se desejar salvar os dados em um arquivo, você pode usar o parâmetro `-o` para especificar o nome do arquivo. Scrapy oferece suporte à exportação de dados rastreados para JSON, CSV, XML e outros formatos.

   ```bash
   scrapy crawl moive -o result.json
   ```

Não sei se todos notaram que há itens `275` no arquivo JSON obtido ao executar o rastreador, e isso ocorre porque a página inicial foi rastreada repetidamente. Para resolver este problema, podemos ajustar ligeiramente o código acima. Não analisamos e obtemos os URLs de novas páginas no método `parse`. Em vez disso, preparamos os URLs das páginas para rastreamento antecipadamente por meio do método `start_requests`. O código ajustado é mostrado abaixo.

   ```python
   import scrapy
   from scrapy import Selector, Request
   from scrapy.http import HtmlResponse

   from demo.items import MovieItem


   class DoubanSpider(scrapy.Spider):
       name = 'douban'
       allowed_domains = ['movie.douban.com']

       def start_requests(self):
           for page in range(10):
               yield Request(url=f'https://movie.douban.com/top250?start={page * 25}')

       def parse(self, response: HtmlResponse):
           sel = Selector(response)
           movie_items = sel.css('#content > div > div.article > ol > li')
           for movie_sel in movie_items:
               item = MovieItem()
               item['title'] = movie_sel.css('.title::text').extract_first()
               item['score'] = movie_sel.css('.rating_num::text').extract_first()
               item['motto'] = movie_sel.css('.inq::text').extract_first()
               yield item
   ```

3. Se quisermos encerrar a persistência dos dados do rastreador, podemos manipular os objetos `Item` produzidos pelo programa spider no pipeline de itens. Por exemplo, podemos usar `openpyxl`, do qual falamos antes para trabalhar com arquivos Excel, e gravar os dados em um arquivo Excel. O código é mostrado abaixo.

   ```python
   import openpyxl

   from demo.items import MovieItem


   class MovieItemPipeline:

       def __init__(self):
           self.wb = openpyxl.Workbook()
           self.sheet = self.wb.active
           self.sheet.title = 'Top250'
           self.sheet.append(('Title', 'Score', 'Motto'))

       def process_item(self, item: MovieItem, spider):
           self.sheet.append((item['title'], item['score'], item['motto']))
           return item

       def close_spider(self, spider):
           self.wb.save('douban_movie_data.xlsx')
   ```

Tanto `process_item` quanto `close_spider` acima são métodos de retorno de chamada, ou seja, funções de gancho. Simplificando, esses são métodos que a estrutura Scrapy chama automaticamente. Quando o programa spider produz um objeto `Item` e o entrega ao mecanismo, o mecanismo entrega esse objeto `Item` ao pipeline de itens. Neste momento, o método `parse_item` do pipeline de itens configurado é executado, para que possamos obter os dados neste método e finalizar a persistência. O outro método, `close_spider`, é um método executado automaticamente antes do término do rastreador. No código acima, salvamos o arquivo Excel neste local. Acredito que todos podem ler esse código facilmente.

Resumindo, o pipeline de itens pode nos ajudar a concluir as seguintes operações:

   - Limpe os dados HTML e valide os dados rastreados.
   - Elimine conteúdo duplicado e desnecessário.
   - Persista os resultados do rastreamento.

4. Modifique o arquivo `settings.py` para configurar o projeto. As principais configurações que precisam ser alteradas são as seguintes.

   ```python
   # User browser
   USER_AGENT = 'Mozilla/5.0 (Macintosh; Intel Mac OS X 10_14_6) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/92.0.4515.159 Safari/537.36'

   # Number of concurrent requests
   CONCURRENT_REQUESTS = 4

   # Download delay
   DOWNLOAD_DELAY = 3
   # Randomize the download delay
   RANDOMIZE_DOWNLOAD_DELAY = True

   # Whether to obey the crawler protocol
   ROBOTSTXT_OBEY = True

   # Configure the item pipeline
   ITEM_PIPELINES = {
      'demo.pipelines.MovieItemPipeline': 300,
   }
   ```

   > **Nota**: A opção `ITEM_PIPELINES` no arquivo de configurações acima é um dicionário. Vários pipelines de processamento de dados podem ser configurados, e o número por trás deles representa a prioridade de execução. O número menor é executado primeiro.
