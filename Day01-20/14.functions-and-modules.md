## Funções e Módulos

Antes de começarmos esta lição, vamos primeiro estudar um problema matemático: quantos grupos de soluções inteiras positivas tem a equação abaixo?

$$ x_{1} + x_{2} + x_{3} + x_{4} = 8 $$

Você já deve ter pensado nisso: esse problema é na verdade o mesmo que dividir 8 maçãs em 4 grupos, com pelo menos uma maçã em cada grupo. Também é o mesmo que colocar 3 divisórias nos 7 espaços entre 8 maçãs para dividi-las em 4 grupos. Portanto, a resposta é $\small{C_{7}^{3} = 35}$, onde:

$$ C_m^n = \frac {m!} {n!(m-n)!} $$

Com base no que aprendemos antes, podemos usar loops e multiplicações repetidas para calcular $\small{m!}$, $\small{n!}$ e $\small{(m-n)!}$ separadamente e, em seguida, usar a divisão para obter o número da combinação $\small{C_{m}^{n}}$, conforme mostrado abaixo.

```python
"""
Read m and n, then compute C(m, n)

Version: 1.0
Author: Luo Hao
"""

m = int(input('m = '))
n = int(input('n = '))
# Calculate m!
fm = 1
for num in range(1, m + 1):
    fm *= num
# Calculate n!
fn = 1
for num in range(1, n + 1):
    fn *= num
# Calculate (m - n)!
fk = 1
for num in range(1, m - n + 1):
    fk *= num
# Calculate C(m, n)
print(fm // fn // fk)
```

Não sei se você percebeu, mas no código acima fizemos a operação fatorial três vezes. Embora os valores de $\small{m}$, $\small{n}$ e $\small{m - n}$ sejam diferentes, os três blocos de código não são essencialmente diferentes, portanto são códigos duplicados. O mestre de programação de classe mundial Martin Fowler disse uma vez: "**O código tem muitos maus cheiros e a duplicação é um dos piores!**" Para escrever código de alta qualidade, a primeira coisa é resolver o problema do código duplicado. Para o código acima, podemos empacotar a operação fatorial em um bloco de código chamado **função**. Então, sempre que precisarmos calcular um fatorial, só precisamos **chamar a função**.

### Definindo Funções

Em matemática, as funções são geralmente escritas em formas como $\small{y = f(x)}$ ou $\small{z = g(x, y)}$. Em $\small{y = f(x)}$, $\small{f}$ é o nome da função, $\small{x}$ é a variável independente e $\small{y}$ é a variável dependente. Em $\small{z = g(x, y)}$, $\small{g}$ é o nome da função, $\small{x}$ e $\small{y}$ são as variáveis ​​independentes e $\small{z}$ é a variável dependente. As funções Python seguem a mesma estrutura. Cada função tem seu próprio nome, variáveis ​​independentes e variável dependente. Geralmente chamamos as variáveis ​​independentes de uma função Python de **parâmetros**, e a variável dependente de **valor de retorno**.

Em Python, podemos usar a palavra-chave `def` para definir uma função. Assim como as variáveis, toda função também deve ter um bom nome, e as regras de nomenclatura são iguais às regras de nomenclatura para variáveis. Nos parênteses após o nome da função, podemos definir os parâmetros da função, que são as variáveis ​​independentes de que acabamos de falar. Após a execução da função, usamos a palavra-chave `return` para retornar o resultado da função, que é a variável dependente da qual acabamos de falar. Se não houver nenhuma instrução `return` em uma função, a função retornará `None`, o que significa um valor vazio. Além disso, uma função não pode ter variáveis ​​independentes, o que significa que não há parâmetros, mas os parênteses após o nome da função ainda são obrigatórios. As coisas que uma função precisa fazer são colocadas após a linha de definição da função por recuo, assim como os blocos de código nas estruturas de ramificação e loop, conforme mostrado abaixo.

Agora vamos colocar a operação fatorial do código anterior em uma função. Desta forma, refatoramos o código anterior. **Refatorar significa ajustar a estrutura do código sem alterar o resultado da execução.** O código refatorado é mostrado abaixo.

```python
"""
Read m and n, then compute C(m, n)

Version: 1.1
Author: Luo Hao
"""


def fac(num):
    """Return the factorial of a non-negative integer."""
    result = 1
    for n in range(2, num + 1):
        result *= n
    return result


m = int(input('m = '))
n = int(input('n = '))
# When calculating factorials, we do not need to write repeated code.
# We only need to call the function directly.
# The syntax for calling a function is to put parentheses after the function name and pass in parameters.
print(fac(m) // fac(n) // fac(m - n))
```

Você pode sentir que o código acima é mais simples e elegante do que a versão anterior. Mais importante ainda, a função fatorial `fac` que definimos também pode ser usada novamente em outro código que precise de fatoriais. Portanto, **usar funções pode nos ajudar a empacotar código que é relativamente independente em função e será usado repetidamente**. Quando precisamos desse código, em vez de escrever código duplicado novamente, **reutilizamos o código existente chamando a função**.

Na verdade, o módulo `math` na biblioteca padrão do Python já possui uma função chamada `factorial` que calcula fatoriais. Podemos importar diretamente o módulo `math` com `import math` e, em seguida, usar `math.factorial` para chamar a função fatorial. Também podemos importar diretamente a função `factorial` com `from math import factorial`, conforme mostrado abaixo.

```python
"""
Read m and n, then compute C(m, n)

Version: 1.2
Author: Luo Hao
"""
from math import factorial

m = int(input('m = '))
n = int(input('n = '))
print(factorial(m) // factorial(n) // factorial(m - n))
```

No futuro, as funções que usaremos serão funções que nós mesmos definimos ou funções fornecidas pela biblioteca padrão do Python ou por bibliotecas de terceiros. Se já existe uma função pronta para uso, não há necessidade de defini-la novamente. "**Reinventar a roda**" é uma coisa muito ruim. Para o código acima, se você acha que o nome `factorial` é muito longo e não muito conveniente ao escrever o código, também podemos usar a palavra-chave `as` para fornecer um alias ao importar a função. Ao chamar a função, podemos usar o alias em vez do nome original.

```python
"""
Read m and n, then compute C(m, n)

Version: 1.3
Author: Luo Hao
"""
from math import factorial as f

m = int(input('m = '))
n = int(input('n = '))
print(f(m) // f(n) // f(m - n))
```

### Parâmetros de Função

#### Argumentos posicionais e de palavras-chave

Vamos escrever outra função. De acordo com os comprimentos dos três lados, verifica se eles podem formar um triângulo. Se eles puderem formar um triângulo, retornará `True`; caso contrário, retornará `False`, conforme mostrado abaixo.

```python
def make_judgement(a, b, c):
    """Return whether three sides can form a triangle."""
    return a + b > c and b + c > a and a + c > b
```

Os três parâmetros na função `make_judgement` acima são chamados de parâmetros posicionais. Ao chamar a função, geralmente os passamos da esquerda para a direita, e o número de argumentos passados ​​​​deve ser igual ao número de parâmetros quando a função é definida, conforme mostrado abaixo.

```python
print(make_judgement(1, 2, 3))  # False
print(make_judgement(4, 5, 6))  # True
```

Se não quisermos fornecer os valores de `a`, `b` e `c` da esquerda para a direita, também podemos usar argumentos de palavras-chave e passar os parâmetros escrevendo `parameter_name=parameter_value`, conforme mostrado abaixo.

```python
print(make_judgement(b=2, c=3, a=1))  # False
print(make_judgement(c=6, b=4, a=5))  # True
```

Ao definir uma função, podemos usar `/` na lista de parâmetros para definir **parâmetros somente posicionais** e usar `*` para definir **parâmetros somente de palavras-chave**. Os chamados parâmetros somente posicionais são parâmetros que só podem receber valores por posição do parâmetro ao chamar a função. Parâmetros somente de palavra-chave só podem ser passados ​​e recebidos escrevendo `parameter_name=parameter_value`. Veja o exemplo abaixo.

```python
def make_judgement(a, b, c, /):
    return a + b > c and b + c > a and a + c > b
```

> **Observação**: Parâmetros somente posicionais são um novo recurso introduzido no Python 3.8. Se você usa uma versão inferior do Python, precisa prestar atenção a isso.

```python
def make_judgement(*, a, b, c):
    return a + b > c and b + c > a and a + c > b
```

#### Valores padrão

Python permite que parâmetros de função tenham valores padrão. Podemos empacotar a operação de lançar dados no exemplo do jogo CRAPS de que falamos antes em uma função, conforme mostrado abaixo.

```python
from random import randrange


def roll_dice(n=2):
    """Roll n dice and return the total."""
    total = 0
    for _ in range(n):
        total += randrange(1, 7)
    return total


print(roll_dice())
print(roll_dice(3))
```

Vejamos um exemplo mais simples.

```python
def add(a=0, b=0, c=0):
    """Add three numbers."""
    return a + b + c


print(add())         # 0
print(add(1))        # 1
print(add(1, 2))     # 3
print(add(1, 2, 3))  # 6
```

Deve-se observar que **parâmetros com valores padrão devem ser colocados após parâmetros sem valores padrão**. Caso contrário, aumentará `SyntaxError`. A mensagem de erro é `non-default argument follows default argument`, o que significa que um parâmetro sem valor padrão foi colocado após um parâmetro com valor padrão.

#### Argumentos de comprimento variável

Em Python, podemos usar a sintaxe de expressão em estrela para fazer uma função suportar argumentos de comprimento variável. Argumentos de comprimento variável significam que, ao chamar uma função, podemos passar `0` ou qualquer número de argumentos. No futuro, quando desenvolvermos projetos de negócios em equipes, poderemos projetar funções para outras pessoas usarem, mas às vezes não sabemos quantos argumentos o chamador passará para a função. Nesse momento, argumentos de comprimento variável são úteis.

```python
def add(*args):
    total = 0
    for val in args:
        if type(val) in (int, float):
            total += val
    return total


print(add())                      # 0
print(add(1))                     # 1
print(add(1, 2, 3))               # 6
print(add(1, 2, 'hello', 3.45, 6))  # 12.45
```

Se quisermos passar alguns parâmetros na forma de `parameter_name=parameter_value`, e também não tivermos certeza de quantos desses parâmetros haverá, também podemos adicionar argumentos de palavras-chave variáveis ​​à função e agrupar os argumentos de palavras-chave passados ​​em um dicionário, conforme mostrado abaixo.

```python
def foo(*args, **kwargs):
    print(args)
    print(kwargs)


foo(3, 2.1, True, name='Luo Hao', age=43, gpa=4.95)
```

Saída:

```python
(3, 2.1, True)
{'name': 'Luo Hao', 'age': 43, 'gpa': 4.95}
```

### Gerenciando funções com módulos

Não importa qual linguagem de programação usamos para escrever código, nomear variáveis ​​e funções pode levar à situação embaraçosa de **conflitos de nomes**. A cena mais simples é definir duas funções com o mesmo nome em um arquivo `.py`:

```python
def foo():
    print('hello, world!')


def foo():
    print('goodbye, world!')


foo()
```

É claro que a situação acima é fácil de evitar, mas em um projeto de equipe com muitos programadores, vários programadores podem definir uma função chamada `foo`. Neste caso, como resolvemos o conflito de nomes? A resposta é realmente muito simples. Em Python, cada arquivo representa um **módulo**. Podemos ter funções com o mesmo nome em módulos diferentes. Ao usar a função, importamos o módulo especificado com a palavra-chave `import` e, em seguida, usamos o **nome totalmente qualificado** `module_name.function_name`, e então podemos dizer qual função `foo` do módulo queremos usar.

`module1.py`

```python
def foo():
    print('hello, world!')
```

`module2.py`

```python
def foo():
    print('goodbye, world!')
```

`test.py`

```python
import module1
import module2

module1.foo()
module2.foo()
```

Ao importar um módulo, também podemos usar a palavra-chave `as` para fornecer um alias ao módulo. Dessa forma, podemos usar um nome totalmente qualificado mais curto.

```python
import module1 as m1
import module2 as m2

m1.foo()
m2.foo()
```

Nas duas seções de código acima, o que importamos foram os módulos que definem as funções. Também podemos importar diretamente as funções necessárias do módulo usando `from...import...`.

```python
from module1 import foo

foo()

from module2 import foo

foo()
```

Mas se importarmos diretamente duas funções com o mesmo nome de dois módulos diferentes, a importação posterior substituirá a importação anterior. Se quisermos usar as duas funções `foo` ao mesmo tempo no código acima, ainda há um caminho. Podemos usar a palavra-chave `as` para fornecer aliases às funções importadas.

```python
from module1 import foo as f1
from module2 import foo as f2

f1()
f2()
```

### Módulos de biblioteca padrão e funções integradas

A biblioteca padrão do Python contém muitos módulos e funções úteis. Já vimos:

- `random` para números aleatórios e amostragem
- `time` para operações relacionadas ao tempo
- `math` para funções matemáticas como seno, cosseno, exponenciais e logaritmos

Python também tem muitas **funções integradas** que não requerem importação. Os exemplos incluem:

| Função | Descrição |
| --- | --- |
| `abs` | valor absoluto |
| `bin` | converter um número inteiro em binário |
| `chr` | converter um ponto de código Unicode em um caractere |
| `hex` | converter um número inteiro em hexadecimal |
| `input` | leia uma linha de entrada |
| `len` | obter o comprimento de uma string, lista, etc. |
| `max` | retornar o valor máximo |
| `min` | retornar o valor mínimo |
| `oct` | converter um número inteiro em octal |
| `open` | abrir um arquivo |
| `ord` | converter um caractere em seu ponto de código Unicode |
| `pow` | exponenciação |
| `print` | saída de impressão |
| `range` | construir uma sequência de intervalo |
| `round` | arredondar um número |
| `sum` | somar valores em uma sequência |
| `type` | retornar o tipo de um objeto |

### Resumo

**Uma função é um pacote de código relativamente independente em função e que será usado repetidamente**. Depois de aprender como definir e usar funções, podemos escrever um código melhor. Claro, a biblioteca padrão do Python já forneceu muitos módulos e funções comuns para nós. Se usarmos bem esses módulos e funções, poderemos fazer mais coisas com menos código. Se esses módulos e funções ainda não atenderem às nossas necessidades, talvez seja necessário definir nossas próprias funções e, em seguida, usar módulos para gerenciar essas funções personalizadas.
