## pandas in Depth - 1

Pandas is a powerful toolkit for **analyzing structured data** developed by Wes McKinney in 2008. Built on top of NumPy for data storage and computation, pandas provides types, methods, and functions specially made for data analysis, and it gives very good support for data analysis and data mining. At the same time, pandas also works very well together with the visualization tool matplotlib, making it easy to present data visually.

The core data types in pandas are `Series` and `DataFrame`, which are used to process one-dimensional and two-dimensional data. Besides them, there is also a type named `Index` and its subtypes, which provide indexing ability for `Series` and `DataFrame`. In daily work, `DataFrame` is used most widely, because a two-dimensional data structure matches a table with rows and columns very well. Both `Series` and `DataFrame` provide many methods for processing data. On that basis, a data analyst can do filtering, merging, concatenation, cleaning, preprocessing, aggregation, pivoting, visualization, and many other operations.

### Creating `Series` Objects

A `Series` object in pandas can be used to represent a one-dimensional data structure, but it has indexes and some extra functions. Internally, a `Series` contains two arrays, one for storing data and the other for storing indexes. We can create a `Series` from a list or array, as shown below.

```python
import numpy as np
import pandas as pd

ser1 = pd.Series(data=[120, 380, 250, 360], index=['Q1', 'Q2', 'Q3', 'Q4'])
ser1
```

> **Note**: In the `Series` constructor, the `data` parameter represents the data, and the `index` parameter represents the indexes of the data, which are like labels corresponding to the data.

Output:

```
Q1    120
Q2    380
Q3    250
Q4    360
dtype: int64
```

We can also create a `Series` object from a dictionary.

```python
ser2 = pd.Series({'Q1': 320, 'Q2': 180, 'Q3': 300, 'Q4': 405})
ser2
```

> **Note**: When a `Series` is created from a dictionary, the keys of the dictionary become the data labels, and the values corresponding to those keys become the data.

Output:

```
Q1    320
Q2    180
Q3    300
Q4    405
dtype: int64
```

### Operations on `Series` Objects

#### Scalar operations

Let us try adding `10` to each quarter in `ser1`.

```python
ser1 += 10
ser1
```

Output:

```
Q1    130
Q2    390
Q3    260
Q4    370
dtype: int64
```

#### Vector operations

Let us try adding together the data from the corresponding quarters in `ser1` and `ser2`.

```python
ser1 + ser2
```

Output:

```
Q1    450
Q2    570
Q3    560
Q4    775
dtype: int64
```

#### Indexing operations

##### Ordinary indexing

Like arrays, `Series` objects can also be indexed and sliced. The difference is that a `Series` internally maintains an array of indexes, so besides using integer indexes to retrieve data, we can also use the indexes we set ourselves to get the corresponding data.

Use integer indexing:

```python
ser1[2]
```

Output:

```
260
```

Use custom indexing:

```python
ser1['Q3']
```

Output:

```
260
```

```python
ser1['Q1'] = 380
ser1
```

Output:

```
Q1    380
Q2    390
Q3    260
Q4    370
dtype: int64
```

##### Slice indexing

The slicing operation of a `Series` object is similar to that of a list or array. By giving a start and end index, we can take out or modify part of the data from the original `Series`. Here we can also use integer indexes and custom indexes.

```python
ser2[1:3]
```

Output:

```
Q2    180
Q3    300
dtype: int64
```

```python
ser2['Q2':'Q4']
```

Output:

```
Q2    180
Q3    300
Q4    405
dtype: int64
```

> **Tip**: When custom indexes are used for slicing, the element corresponding to the end index can also be included.

```python
ser2[1:3] = 400, 500
ser2
```

Output:

```
Q1    320
Q2    400
Q3    500
Q4    405
dtype: int64
```

##### Fancy indexing

```python
ser2[['Q2', 'Q4']]
```

Output:

```
Q2    400
Q4    405
dtype: int64
```

```python
ser2[['Q2', 'Q4']] = 600, 520
ser2
```

Output:

```
Q1    320
Q2    600
Q3    500
Q4    520
dtype: int64
```

##### Boolean indexing

```python
ser2[ser2 >= 500]
```

Output:

```
Q2    600
Q3    500
Q4    520
dtype: int64
```

### Properties and Methods of `Series` Objects

There are many properties and methods on a `Series` object. Let us focus on the important ones. The table below shows commonly used properties of a `Series`.

| Property                  | Description |
| ------------------------- | ----------- |
| `dtype` / `dtypes`        | Return the data type of the `Series` |
| `hasnans`                 | Check whether the `Series` contains null values |
| `at` / `iat`              | Access a single value in a `Series` by index |
| `loc` / `iloc`            | Access one or more values in a `Series` by index |
| `index`                   | Return the index of the `Series` (`Index` object) |
| `is_monotonic`            | Check whether the data in the `Series` is monotonic |
| `is_monotonic_increasing` | Check whether the data is monotonic increasing |
| `is_monotonic_decreasing` | Check whether the data is monotonic decreasing |
| `is_unique`               | Check whether each value is unique |
| `size`                    | Return the number of elements in the `Series` |
| `values`                  | Return the values of the `Series` as an `ndarray` |

We can understand these properties through the code below.

```python
print(ser2.dtype)                    # data type
print(ser2.hasnans)                  # whether there are null values
print(ser2.index)                    # index
print(ser2.values)                   # values
print(ser2.is_monotonic_increasing)  # whether it is monotonic increasing
print(ser2.is_unique)                # whether each value is unique
```

Output:

```
int64
False
Index(['Q1', 'Q2', 'Q3', 'Q4'], dtype='object')
[320 600 500 520]
False
True
```

There are many methods on `Series` objects. Below we use a few code snippets to introduce common methods.


Output:

```
4

600
320

13966.666666666666
```

`Series` objects also have a method named `describe()` that can obtain all the descriptive statistics mentioned above.

```python
ser2.describe()
```

Output:


> **Tip**: Because `describe()` also returns a `Series` object, we can use `ser2.describe()['mean']` to get the mean, and `ser2.describe()[['max', 'min']]` to get the maximum and minimum.

If a `Series` object has duplicated values, we can use the `unique()` method to get an array made of unique values. We can use the `nunique()` method to count how many non-duplicated values there are. If we want to count how many times each value is repeated, we can use the `value_counts()` method. This method returns a `Series` object. Its indexes are the values from the original `Series`, and the data are the counts of how many times each value appears. By default, it is sorted in descending order of count, as shown below.

```python
ser3 = pd.Series(data=['apple', 'banana', 'apple', 'pitaya', 'apple', 'pitaya', 'durian'])
ser3.value_counts()
```

Output:

```
apple     3
pitaya    2
durian    1
banana    1
dtype: int64
```

```python
ser3.nunique()
```

Output:

```
4
```

For `ser3`, we can also use the `mode()` method to find the mode of the data. Because the mode may not be unique, the return value of the `mode()` method is still a `Series` object.

```python
ser3.mode()
```

Output:

```
0    apple
dtype: object
```

#### Processing data

The `isna()` and `isnull()` methods of a `Series` object can be used to check null values. The `notna()` and `notnull()` methods can be used to check non-null values, as shown below.

```python
ser4 = pd.Series(data=[10, 20, np.nan, 30, np.nan])
ser4.isna()
```

> **Note**: `np.nan` is a floating-point value from the IEEE 754 standard, specially used to mean "not a number". In the code above, we use it to represent null values. Of course, we can also use Python's `None` to represent null values, and in pandas `None` will also be treated as `np.nan`.

Output:

```
0    False
1    False
2     True
3    False
4     True
dtype: bool
```

```python
ser4.notna()
```

Output:

```
0     True
1     True
2    False
3     True
4    False
dtype: bool
```

The `dropna()` and `fillna()` methods of a `Series` object are used to delete null values and fill null values. Their usage is shown below.

```python
ser4.dropna()
```

Output:

```
0    10.0
1    20.0
3    30.0
dtype: float64
```

```python
ser4.fillna(value=40)  # fill null values with 40
```

Output:

```
0    10.0
1    20.0
2    40.0
3    30.0
4    40.0
dtype: float64
```

```python
ser4.fillna(method='ffill')  # fill with the non-null value before the null
```

Output:

```
0    10.0
1    20.0
2    20.0
3    30.0
4    30.0
dtype: float64
```

It is worth reminding you that both `dropna()` and `fillna()` have a parameter named `inplace`. Its default value is `False`, which means deleting or filling null values does not modify the original `Series` object. Instead, it returns a new `Series` object. If the value of `inplace` is changed to `True`, then deleting or filling null values will happen in place and directly modify the original `Series` object. At that time, the return value of the method is `None`. Many methods that we will see later, including many methods of `DataFrame` objects, also have this parameter, and it means the same thing.

The `mask()` and `where()` methods of a `Series` object can replace values that satisfy or do not satisfy a condition, as shown below.


The `apply()` and `map()` methods of a `Series` object are very important. They can process data through a dictionary or a specified function, and map or convert the data into the form we want. These two methods are very important in the data-preparation stage. Let us first try the method named `map`.

```python
ser6 = pd.Series(['cat', 'dog', np.nan, 'rabbit'])
ser6
```

Output:

```
0       cat
1       dog
2       NaN
3    rabbit
dtype: object
```

```python
ser6.map({'cat': 'kitten', 'dog': 'puppy'})
```

> **Note**: This processes the data according to the mapping rules given by the dictionary.

Output:

```
0    kitten
1     puppy
2       NaN
3       NaN
dtype: object
```

```python
ser6.map('I am a {}'.format, na_action='ignore')
```

> **Note**: This applies the `format` method of the specified string to the data in the series, while ignoring all null values.

Output:

```
0       I am a cat
1       I am a dog
2              NaN
3    I am a rabbit
dtype: object
```

We create a new `Series` object:

```python
ser7 = pd.Series([20, 21, 12], index=['London', 'New York', 'Helsinki'])
ser7
```

Output:

```
London      20
New York    21
Helsinki    12
dtype: int64
```

```python
ser7.apply(np.square)
```

> **Note**: This applies the square function to the data in the series. We can also replace `np.square` with `lambda x: x ** 2`.

Output:

```
London      400

```

```python
ser7.apply(lambda x, value: x - value, args=(5, ))
```

> **Note**: The `lambda` function in the `apply` method above has two parameters. The first parameter is the data in the series, and the second parameter needs to be passed in by us, so we add the `args` parameter to the `apply` method to pass a value to the second parameter of the `lambda` function.

Output:

```
London      15
New York    16
Helsinki     7
dtype: int64
```

#### Getting head values and sorting

The `sort_index()` and `sort_values()` methods of a `Series` object can be used to sort indexes and data. The sorting methods have a boolean parameter named `ascending`, which controls whether the result is in ascending order or descending order. A parameter named `kind` is used to control the sorting algorithm. By default it uses `quicksort`, but `mergesort` or `heapsort` can also be chosen. If there are null values, the `na_position` parameter can be used to decide whether null values are placed at the front or at the end. The default is `last`, as shown below.

```python
ser8 = pd.Series(
    data=[35, 96, 12, 57, 25, 89],
    index=['grape', 'banana', 'pitaya', 'apple', 'peach', 'orange']
)
ser8.sort_values()  # sort by value from small to large
```

Output:

```
pitaya    12
peach     25
grape     35
apple     57
orange    89
banana    96
dtype: int64
```

```python
ser8.sort_index(ascending=False)  # sort by index from large to small
```

Output:

```
pitaya    12
peach     25
orange    89
grape     35
banana    96
apple     57
dtype: int64
```

If we want to find the "Top-N" largest or smallest elements in a `Series` object, we do not need to sort all values. We can use the `nlargest()` and `nsmallest()` methods, as shown below.

```python
ser8.nlargest(3)  # the 3 largest values
```

Output:

```
banana    96
orange    89
apple     57
dtype: int64
```

```python
ser8.nsmallest(2)  # the 2 smallest values
```

Output:

```
pitaya    12
peach     25
dtype: int64
```

#### Drawing charts

A `Series` object has a method named `plot` that can be used to generate charts. If we choose to generate a line chart, pie chart, bar chart, and so on, by default it will use the index of the `Series` object as the horizontal axis and the data of the `Series` object as the vertical axis. Below we create a `Series` object and draw a bar chart from it.

```python
import matplotlib.pyplot as plt

ser9 = pd.Series({'Q1': 400, 'Q2': 520, 'Q3': 180, 'Q4': 380})
# use the kind parameter of plot to specify a bar chart
ser9.plot(kind='bar')
# customize the range of the y-axis
plt.ylim(0, 600)
# customize the x-axis ticks
plt.xticks(rotation=0)
# add data labels to the bars
for i in range(ser9.size):
    plt.text(i, ser9[i] + 5, ser9[i], ha='center')
plt.show()
```
