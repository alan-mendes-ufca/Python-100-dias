## Ramificação

Até agora, nos programas Python, escrevemos todas as instruções de execução, uma por uma, de cima para baixo. Esse tipo de estrutura de código é chamada de **estrutura sequencial**. Contudo, a estrutura sequencial por si só não pode resolver todos os problemas. Por exemplo, suponha que projetamos um jogo. A condição para passar do primeiro nível é que o jogador obtenha 1000 pontos. Após o término do primeiro nível, precisamos decidir, com base na pontuação do jogador, se o jogador entra no segundo nível ou se informamos ao jogador `Game Over`. Nesse tipo de situação, nosso código terá duas ramificações, e apenas uma ramificação será executada. Existem muitas situações semelhantes. Chamamos esse tipo de estrutura de **estrutura ramificada** ou **estrutura de seleção**. Dê a si mesmo um minuto. Você deve ser capaz de pensar em pelo menos cinco exemplos semelhantes.

### Construindo Filiais com `if` e `else`

Em Python, as três palavras-chave usadas com mais frequência para construir estruturas de ramificação são `if`, `elif` e `else`. Uma **palavra-chave** é uma palavra com significado especial na linguagem de programação, então claramente você não pode usá-la como nome de variável. É claro que não usamos todas as três palavras-chave sempre que construímos uma estrutura ramificada. Vamos explicar com um exemplo. Suponha que queiramos escrever uma calculadora de IMC.

O IMC, ou Índice de Massa Corporal, é um indicador usado internacionalmente para medir a forma corporal e a saúde. Sua fórmula é:

$$ IMC = \frac{peso}{altura^{2}} $$

Geralmente é considerado normal quando $\small{18,5 \le IMC < 24}$. Se $\small{IMC < 18,5}$, a pessoa está abaixo do peso. Se $\small{IMC \ge 24}$, a pessoa está acima do peso. Se $\small{IMC \ge 27}$, a pessoa já está na faixa de obesidade.

> **Observação**: Na fórmula acima, o peso é medido em quilogramas e a altura em metros.

```python
"""
BMI calculator

Version: 1.0
Author: Luo Hao
"""
height = float(input('Height (cm): '))
weight = float(input('Weight (kg): '))
bmi = weight / (height / 100) ** 2
print(f'{bmi = :.1f}')
if 18.5 <= bmi < 24:
    print('Your body shape is great!')
```

> **Dica**: O `:` no final da linha `if` deve ser dois pontos ASCII normal digitado com entrada em inglês. O mesmo se aplica a `'`, `"`, `=`, `(` e `)`. Já lembramos a todos disso antes. Muitos iniciantes ignoram esse ponto e, quando executam o código, veem uma grande pilha de mensagens de erro. Claro, se você ler as mensagens de erro com atenção, ainda será fácil encontrar o problema, mas é **altamente recomendado** que você **mude para a entrada em inglês** ao escrever o código. Isso pode evitar muitos problemas desnecessários.

No código acima, após calcular e imprimir o IMC, adicionamos uma estrutura ramificada. Se $\small{18.5 \le BMI < 24}$ for verdadeiro, o programa gera `Your body shape is great!`, mas se a condição não for satisfeita, esta saída não aparece. Isso é o que mencionamos agora: o código pode ter diferentes caminhos de execução e alguns códigos podem nem ser executados. Após a palavra-chave `if`, escrevemos a expressão `18.5 <= bmi < 24`. Dissemos antes que as operações de comparação produzem valores booleanos. Se o valor booleano após `if` for `True`, a linha `print('Your body shape is great!')` abaixo da instrução `if`, que é recuada por quatro espaços, será executada.

Exemplos de entradas:

Primeira entrada:

```
Height (cm): 175
Weight (kg): 68
bmi = 22.2
Your body shape is great!
```

Segunda entrada:

```
Height (cm): 175
Weight (kg): 95
bmi = 31.0
```

Terceira entrada:

```
Height (cm): 175
Weight (kg): 50
bmi = 16.3
```

Apenas o primeiro grupo de entrada fornece um IMC na faixa de 18,5 a 24, então ele aciona a condição `if` e gera `Your body shape is great!`. Deve ser explicado que, diferentemente de C, C++, Java e outras linguagens de programação, Python não usa colchetes para construir blocos de código. Em vez disso, **usa indentação para mostrar a estrutura do código**. Se várias instruções precisarem ser executadas quando a condição `if` for verdadeira, você só precisará mantê-las no mesmo recuo. Em outras palavras, se várias linhas contínuas de instruções tiverem o mesmo recuo, elas pertencerão ao mesmo **bloco de código** e poderão ser tratadas como um todo quando o código for executado. O recuo pode usar qualquer número de espaços, mas **geralmente são usados ​​4 espaços**. É altamente recomendável que você **não use a tecla Tab para recuar o código**. Se você já está acostumado a fazer isso, pode configurar seu editor de código para transformar automaticamente uma guia em quatro espaços. Muitos editores de código suportam isso, e o PyCharm é definido dessa forma por padrão.

Há mais um ponto. Em linguagens de programação como C, C++ e Java, `18.5 <= bmi < 24` deve ser escrito como duas condições, `bmi >= 18.5` e `bmi < 24`, e então as duas condições são conectadas ao operador lógico `and`. Python também pode fazer isso, então a instrução `if` anterior também pode ser escrita como `if bmi >= 18.5 and bmi < 24:`. Mas não há necessidade. `if 18.5 <= bmi < 24:` não é mais legal? O código Java abaixo faz a mesma coisa. Não importa se você não consegue ler o código Java. Apenas sinta a diferença entre ele e a sintaxe do Python.

```java
import java.util.Scanner;

class Test {

    public static void main(String[] args) {
        try (Scanner sc = new Scanner(System.in)) {
            System.out.print("Height(cm): ");
            double height = sc.nextDouble();
            System.out.print("Weight(kg): ");
            double weight = sc.nextDouble();
            double bmi = weight / Math.pow(height / 100, 2);
            System.out.printf("bmi = %.1f\n", bmi);
            if (bmi >= 18.5 && bmi < 24) {
                System.out.println("Your body shape is great!");
            }
        }
    }
}
```

> **Nota**: O código acima é a versão Java da Calculadora IMC 1.0. Existem boas razões pelas quais muitas pessoas gostam de Python. Geralmente pode resolver o mesmo problema com menos código.

A seguir, vamos fazer uma pequena alteração no código acima. Quando o IMC não satisfaz $\small{18.5 \le IMC < 24}$, também enviamos uma mensagem de alerta correspondente. Podemos adicionar um bloco `else` após o bloco `if`. Será executado quando a condição dada pela instrução `if` não for satisfeita. Claramente, apenas um dos `print('Your body shape is great!')` sob `if` e `print('Your body shape is not standard enough.')` sob `else` será executado.

```python
"""
BMI calculator

Version: 1.1
Author: Luo Hao
"""
height = float(input('Height (cm): '))
weight = float(input('Weight (kg): '))
bmi = weight / (height / 100) ** 2
print(f'{bmi = :.1f}')
if 18.5 <= bmi < 24:
    print('Your body shape is great!')
else:
    print('Your body shape is not standard enough.')
```

Se quisermos fornecer mensagens de prompt mais exatas, podemos modificar o código acima novamente e adicionar mais ramificações por meio da palavra-chave `elif`, conforme mostrado abaixo.

```python
"""

    print('Your body shape is great!')

    print('You are overweight!')
elif bmi < 30:
    print('You are mildly obese!')
elif bmi < 35:
    print('You are moderately obese!')

```

### Construindo Filiais com `match` e `case`

Python 3.10 adicionou uma nova maneira de construir estruturas ramificadas. Usando as palavras-chave `match` e `case`, podemos construir facilmente estruturas com várias filiais. Quando a documentação oficial do Python introduziu essa nova sintaxe, ela usou um exemplo sobre códigos de status de resposta HTTP, o que é bastante interessante. Se você não sabe o que é um código de status de resposta HTTP, consulte [documentação no MDN](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Status). Abaixo fazemos uma pequena alteração no exemplo da documentação oficial para explicar esta sintaxe. Primeiro, observe o código implementado com a estrutura `if-else` abaixo.

```python
status_code = int(input('Status code: '))
if status_code == 400:
    description = 'Bad Request'
elif status_code == 401:
    description = 'Unauthorized'
elif status_code == 403:
    description = 'Forbidden'
elif status_code == 404:
    description = 'Not Found'
elif status_code == 405:
    description = 'Method Not Allowed'
elif status_code == 418:
    description = 'I am a teapot'
elif status_code == 429:
    description = 'Too many requests'
else:
    description = 'Unknown status code'
print('Description:', description)
```

Resultado da corrida:

```
Status code: 403
Description: Forbidden
```

Abaixo está o código implementado com `match-case`. Embora faça exatamente a mesma coisa, o código parece mais simples e elegante.

```python
status_code = int(input('Status code: '))
match status_code:
    case 400: description = 'Bad Request'
    case 401: description = 'Unauthorized'
    case 403: description = 'Forbidden'
    case 404: description = 'Not Found'
    case 405: description = 'Method Not Allowed'
    case 418: description = 'I am a teapot'
    case 429: description = 'Too many requests'
    case _: description = 'Unknown status code'
print('Description:', description)
```

> **Observação**: A instrução `case` com `_` funciona como um curinga. Se as ramificações anteriores não corresponderem, o código chegará a `case _`. `case _` é opcional. Nem toda estrutura ramificada precisa de uma opção curinga. Se `case _` aparecer em uma estrutura ramificada, ele só poderá ser colocado no final. Se houver outras ramificações depois dela, essas ramificações ficarão inacessíveis.

Claro, `match-case` tem muitas maneiras avançadas de usá-lo. Um deles são os padrões mesclados, e podemos aprendê-lo primeiro. Por exemplo, se quisermos colocar os códigos de status `401`, `403` e `404` em uma ramificação e colocar `400` e `405` em uma ramificação, mantendo os outros inalterados, o código também pode ser escrito assim.

```python
status_code = int(input('Status code: '))
match status_code:
    case 400 | 405: description = 'Invalid Request'
    case 401 | 403 | 404: description = 'Not Allowed'
    case 418: description = 'I am a teapot'
    case 429: description = 'Too many requests'
    case _: description = 'Unknown status code'
print('Description:', description)
```

Resultado da corrida:

```
Status code: 403
Description: Not Allowed
```

### Aplicações de ramificação

#### Exemplo 1: Avaliação de função por partes

Dada a função por partes abaixo, insira `x` e calcule `y`.

$$ y = \begin{casos} 3x - 5, & (x \gt 1) \\\\ x + 2, & (-1 \le x \le 1) \\\\ 5x + 3, & (x \lt -1) \end{casos} $$

```python
"""
Evaluate a piecewise function

Version: 1.0
Author: Luo Hao
"""
x = float(input('x = '))
if x > 1:
    y = 3 * x - 5
elif x >= -1:
    y = x + 2
else:
    y = 5 * x + 3
print(f'{y = }')
```

De acordo com as necessidades reais de desenvolvimento, as estruturas ramificadas podem ser aninhadas. Em outras palavras, dentro do bloco de código `if`, `elif` ou `else` de uma estrutura de ramificação, podemos introduzir outra estrutura de ramificação novamente. Por exemplo, se a condição `if` for verdadeira significa que o jogador passou de nível, mas depois de passar de nível ainda precisamos avaliar o desempenho do jogador de acordo com quantos tesouros ou itens ele obteve, como acender uma, duas ou três estrelas, então precisamos construir uma nova estrutura ramificada dentro do `if`. Da mesma forma, também podemos construir novas filiais em `elif` e `else`. Chamamos isso de estrutura ramificada aninhada. Seguindo essa ideia, o exemplo da função por partes acima também pode ser implementado com o código abaixo.

```python
"""
Evaluate a piecewise function

Version: 1.1
Author: Luo Hao
"""
x = float(input('x = '))
if x > 1:
    y = 3 * x - 5
else:
    if x >= -1:
        y = x + 2
    else:
        y = 5 * x + 3
print(f'{y = }')
```

> **Nota**: Você pode julgar por si mesmo qual dos dois estilos acima é melhor. Em [**O Zen do Python**](https://zhuanlan.zhihu.com/p/111843067), há uma frase: `Flat is better than nested.` O código simples é considerado melhor porque se o aninhamento se tornar muito profundo, a legibilidade será seriamente afetada. Então, eu pessoalmente recomendo o primeiro estilo.

#### Exemplo 2: Converter uma pontuação numérica em uma nota alfabética

Se a pontuação de entrada estiver acima de 90, incluindo 90, produza `A`; se a pontuação estiver entre 80 e 90, excluindo 90, produza `B`; se a pontuação estiver entre 70 e 80, excluindo 80, produza `C`; se a pontuação estiver entre 60 e 70, excluindo 70, produza `D`; se a pontuação estiver abaixo de 60, produza `E`.

```python
"""
Convert a numerical score to a letter grade

Version: 1.0
Author: Luo Hao
"""
score = float(input('Enter the score: '))
if score >= 90:
    grade = 'A'
elif score >= 80:
    grade = 'B'
elif score >= 70:
    grade = 'C'
elif score >= 60:
    grade = 'D'
else:
    grade = 'E'
print(f'{grade = }')
```

#### Exemplo 3: Calcule o perímetro e a área de um triângulo

Insira os comprimentos dos três lados. Se conseguirem formar um triângulo, calcule o perímetro e a área; caso contrário, imprima uma mensagem dizendo que um triângulo não pode ser formado.

```python
"""
Calculate the perimeter and area of a triangle

Version: 1.0
Author: Luo Hao
"""
a = float(input('a = '))
b = float(input('b = '))
c = float(input('c = '))
if a + b > c and a + c > b and b + c > a:
    perimeter = a + b + c
    print(f'Perimeter: {perimeter}')
    s = perimeter / 2
    area = (s * (s - a) * (s - b) * (s - c)) ** 0.5
    print(f'Area: {area}')
else:
    print('A triangle cannot be formed')
```

> **Observação**: A condição `if` acima significa que a soma de quaisquer dois lados é maior que o terceiro lado. Esta é a condição necessária para formar um triângulo. Quando esta condição for satisfeita, precisamos calcular e gerar o perímetro e a área, de modo que todas as cinco instruções em `if` mantenham o mesmo recuo. Eles são um todo e, desde que a condição `if` seja satisfeita, todos serão executados. Este é o conceito de bloco de código que mencionamos anteriormente. Além disso, a fórmula usada acima para calcular a área de um triângulo é chamada de fórmula de Heron. Suponha que haja um triângulo cujos comprimentos laterais são $\small{a}$, $\small{b}$ e $\small{c}$, então a área do triângulo $\small{A}$ pode ser obtida pela fórmula $\small{A = \sqrt{s(s-a)(s-b)(s-c)}}$, onde $\small{s=\frac{a + b + c}{2}}$ significa o semiperímetro.
