## Aplicando Programação Simultânea em Crawlers

Nas lições anteriores, já introduzimos multithreading, multiprocessamento e programação assíncrona em Python. Através destes três meios, podemos implementar programação simultânea ou paralela. Por um lado, isso pode acelerar a execução do código e, por outro lado, também pode trazer uma melhor experiência ao usuário. Os programas rastreadores são tarefas típicas de uso intensivo de E/S. Para tarefas com uso intensivo de E/S, tanto multithreading quanto E/S assíncrona são boas escolhas, porque quando uma parte do programa é bloqueada por uma operação de E/S, outras partes do programa ainda podem continuar em execução, portanto, não precisamos perder muito tempo esperando e bloqueando. Abaixo, usando imagens de rastreamento do site [Imagem 360](https://image.so.com/) e salvando-as localmente como exemplo, mostramos as diferenças entre um programa rastreador escrito com um thread, vários threads e E/S assíncrona, e também fazemos uma comparação simples de sua eficiência de execução.

A página do site 360 ​​Image utiliza tecnologia Ajax. Esta é uma tecnologia que muitos sites usam para carregar dados de forma assíncrona e atualizar páginas parcialmente. Simplificando, as imagens na página são obtidas de forma assíncrona pelo código JavaScript a partir de dados JSON e depois renderizadas dinamicamente, e a página inteira também usa carregamento em cascata, ou seja, conforme você rola para baixo, mais imagens são carregadas. Nas Ferramentas de Desenvolvimento do navegador, podemos encontrar a interface de dados que fornece o conteúdo dinâmico. Conforme mostrado na imagem abaixo, as informações da imagem que precisamos estão dentro dos dados JSON retornados pelo servidor.

Por exemplo, para obter imagens do canal “beleza”, podemos solicitar a URL mostrada abaixo. Nele, o parâmetro `ch` significa o canal solicitado, e o valor do parâmetro `beauty` após `=` significa o canal de beleza. O parâmetro `sn` é igual ao número da página. `0` significa a primeira página, com imagens `30` no total. `30` significa a segunda página, `60` significa a terceira página e assim por diante.

### Versão de thread único

Use o URL acima para baixar imagens `90` no total do canal de beleza.

```python
"""
example04.py - single-threaded crawler
"""
import os

import requests


def download_picture(url):
    filename = url[url.rfind('/') + 1:]
    resp = requests.get(url)
    if resp.status_code == 200:
        with open(f'images/beauty/{filename}', 'wb') as file:
            file.write(resp.content)


def main():
    if not os.path.exists('images/beauty'):
        os.makedirs('images/beauty')
    for page in range(3):
        resp = requests.get(f'https://image.so.com/zjl?ch=beauty&sn={page * 30}')
        if resp.status_code == 200:
            pic_dict_list = resp.json()['list']
            for pic_dict in pic_dict_list:
                download_picture(pic_dict['qhimg_url'])


if __name__ == '__main__':
    main()
```

No macOS ou Linux, podemos utilizar o comando `time` para saber o tempo de execução do código acima e a utilização da CPU, conforme mostrado abaixo.

```bash
time python3 example04.py
```

Abaixo está o resultado da execução do código do rastreador de thread único em meu computador.

```text
python3 example04.py  2.36s user 0.39s system 12% cpu 21.578 total
```

Aqui, só precisamos nos preocupar se o tempo total do código é `21.578` segundos e a utilização da CPU é `12%`.

### Versão multithread

Usamos a técnica de pool de threads mencionada anteriormente e alteramos o código acima para uma versão multithread.

```python
"""
example05.py - multithreaded crawler
"""
import os
from concurrent.futures import ThreadPoolExecutor

import requests


def download_picture(url):
    filename = url[url.rfind('/') + 1:]
    resp = requests.get(url)
    if resp.status_code == 200:
        with open(f'images/beauty/{filename}', 'wb') as file:
            file.write(resp.content)


def main():
    if not os.path.exists('images/beauty'):
        os.makedirs('images/beauty')
    with ThreadPoolExecutor(max_workers=16) as pool:
        for page in range(3):
            resp = requests.get(f'https://image.so.com/zjl?ch=beauty&sn={page * 30}')
            if resp.status_code == 200:
                pic_dict_list = resp.json()['list']
                for pic_dict in pic_dict_list:
                    pool.submit(download_picture, pic_dict['qhimg_url'])


if __name__ == '__main__':
    main()
```

Execute o comando mostrado abaixo.

```bash
time python3 example05.py
```

O resultado da execução do código é mostrado abaixo.

```text
python3 example05.py  2.65s user 0.40s system 95% cpu 3.193 total
```

### Versão de E/S assíncrona

Usamos `aiohttp` para alterar o código acima para uma versão de E/S assíncrona. Para implementar a obtenção de recursos de rede e a gravação de arquivos de forma assíncrona de E/S, primeiro precisamos instalar as bibliotecas de terceiros `aiohttp` e `aiofile`. O comando é mostrado abaixo.

```bash
pip install aiohttp aiofile
```

O uso de `aiohttp` foi brevemente apresentado na lição anterior. A função `async_open` no módulo `aiofile` é aproximadamente usada da mesma maneira que a função `open` integrada do Python, mas suporta operações assíncronas. Abaixo está o código do rastreador da versão de E/S assíncrona.

```python
"""
example06.py - asynchronous I/O crawler
"""
import asyncio
import json
import os

import aiofile
import aiohttp


async def download_picture(session, url):
    filename = url[url.rfind('/') + 1:]
    async with session.get(url, ssl=False) as resp:
        if resp.status == 200:
            data = await resp.read()
            async with aiofile.async_open(f'images/beauty/{filename}', 'wb') as file:
                await file.write(data)


async def fetch_json():
    async with aiohttp.ClientSession() as session:
        for page in range(3):
            async with session.get(
                url=f'https://image.so.com/zjl?ch=beauty&sn={page * 30}',
                ssl=False
            ) as resp:
                if resp.status == 200:
                    json_str = await resp.text()
                    result = json.loads(json_str)
                    for pic_dict in result['list']:
                        await download_picture(session, pic_dict['qhimg_url'])


def main():
    if not os.path.exists('images/beauty'):
        os.makedirs('images/beauty')
    loop = asyncio.get_event_loop()
    loop.run_until_complete(fetch_json())
    loop.close()


if __name__ == '__main__':
    main()
```

Execute o comando mostrado abaixo.

```bash
time python3 example06.py
```

O resultado da execução do código é mostrado abaixo.

```text
python3 example06.py  0.82s user 0.21s system 27% cpu 3.782 total
```

### Resumo

Comparando os resultados da execução dos três trechos de código acima, podemos chegar a uma conclusão. O uso de multithreading e E/S assíncrona pode melhorar o desempenho dos programas rastreadores, porque não precisamos perder tempo esperando e bloqueando causados ​​por operações de E/S. E o resultado da execução do comando `time` também nos diz que a utilização da CPU do código de thread único é apenas `12%`, enquanto a utilização da CPU da versão multithread é tão alta quanto `95%`. O rastreador de thread único leva cerca de `21` segundos para ser executado, enquanto as versões de E/S multithread e assíncrona levam apenas cerca de `3` segundos. Além disso, quando a diferença no tempo de execução não é grande, o código multithread usa mais recursos de CPU do que o código de E/S assíncrono, porque o agendamento e a comutação de multithreading também custam tempo de CPU. Neste ponto, os pontos fortes e fracos das três formas em tarefas com uso intensivo de E/S já estão muito claros. Claro, este é apenas o resultado executado no meu computador. Se a condição da rede não for ideal ou se o site de destino responder lentamente, a vantagem do multithreading e da E/S assíncrona será ainda mais óbvia. Os leitores interessados ​​​​podem experimentar eles mesmos.
