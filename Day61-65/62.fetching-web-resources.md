## Fetching Network Data with Python

Collecting network data is an area that Python is very good at. In the previous lesson, we talked about how programs that do network data collection are usually called web crawlers or spider programs. Even in the age of big data, data is still a weak point and a shortcoming for small and medium-sized companies. Some data can be got through open or paid data APIs, while other industry data and competitor data must be got through web data collection. No matter which way is used to get network data resources, Python is a very good choice, because Python's standard library and third-party libraries both provide good support for web data collection.

### The `requests` Library

To use Python to get network data, we recommend the third-party library named `requests`. We actually used this library in earlier lessons. According to its official website, `requests` is a wrapper built on Python's standard library, and it simplifies the operation of accessing network resources through HTTP or HTTPS. We mentioned in class that HTTP is a request-response protocol. When we enter the correct URL in a browser, usually also called a website address, and press Enter, we send an HTTP request to a Web server on the network. After the server receives the request, it gives us an HTTP response. In Chrome, open Developer Tools and switch to the Network tab, then you can see what HTTP requests and responses really look like, as shown below.

<img src="res/20210822093434.png" style="zoom:40%;">

Through the `requests` library, we can let a Python program send requests to a Web server just like a browser does, and receive the response returned by the server. From the response, we can extract the data we want. The web pages shown to us by the browser are written in HTML, and the browser is like an interpreter environment for HTML. The content we see on a web page is all contained in HTML tags. After getting the HTML code, we can extract content from the attributes of the tags or from the text inside the tags. The example below shows how to get the HTML code of a web page. We use the `get` function of the `requests` library to get the code of the Sohu home page.

```python
import requests

resp = requests.get('https://www.sohu.com/')
if resp.status_code == 200:
    print(resp.text)
```

> **Note**: The variable `resp` in the code above is a `Response` object, a type wrapped by the `requests` library. Through the `status_code` attribute of this object, we can get the response status code, and through the `text` attribute of this object, we can get the HTML code of the page.

Because the `text` of the `Response` object is a string, we can use the regular expression knowledge talked about before to extract news titles and links from the HTML code of the page. The code is shown below.

```python
import re

import requests

pattern = re.compile(r'<a.*?href="(.*?)".*?title="(.*?)".*?>')
resp = requests.get('https://www.sohu.com/')
if resp.status_code == 200:
    all_matches = pattern.findall(resp.text)
    for href, title in all_matches:
        print(href)
        print(title)
```

Besides text content, we can also use the `requests` library to get binary resources through a URL. The example below shows how to get the Baidu logo and save it into a local file named `baidu.png`. You can right-click the Baidu logo on the Baidu home page, and use the "Copy Image Address" menu item to get the image URL.

```python
import requests

resp = requests.get('https://www.baidu.com/img/PCtm_d9c8750bed0b3c7d089fa7d55720d6cf.png')
with open('baidu.png', 'wb') as file:
    file.write(resp.content)
```

> **Note**: The `content` attribute of the `Response` object can get the binary data returned by the server.

The `requests` library is very easy to use, and its functions are also strong and complete. We will slowly analyze more of it in use. If you want to unlock more knowledge about the `requests` library, you can read its [official documentation](https://docs.python-requests.org/zh_CN/latest/).

### Writing Crawler Code

Next, we use "Douban Movies" as an example to explain how to write crawler code. Following the way above, we first use `requests` to get the HTML code of the page, then treat the whole code as one long string, and in this way we can use capturing groups in regular expressions to extract the content we need from the string. The code below shows how to get the names of the Top 250 movies from [Douban Movies](https://movie.douban.com/). The page structure and matching code of [Douban Movie Top250](https://movie.douban.com/top250) are shown below. We can see that each page shows 25 movies, so if we want to get the Top250 data, we need to visit 10 pages in total. The matching address is `https://movie.douban.com/top250?start=xxx`. If `xxx` is `0`, it is the first page. If `xxx` is `100`, then we can visit the fifth page. To keep the code simple and easy to read, we only get the movie title and rating.

<img src="res/20210822093447.png" style="zoom:40%;">

```python
import random
import re
import time

import requests

for page in range(1, 11):
    resp = requests.get(
        url=f'https://movie.douban.com/top250?start={(page - 1) * 25}',
        # If we do not set User-Agent in the HTTP request header, Douban will detect that it is not a browser and block the request.
        # Through the headers parameter of the get function, we set the value of User-Agent.
        # The exact value can be seen in the browser's developer tools.
        # When visiting most websites with a crawler, disguising the crawler as a browser request is a very important step.
        headers={'User-Agent': 'Mozilla/5.0 (Macintosh; Intel Mac OS X 10_14_6) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/92.0.4515.159 Safari/537.36'}
    )
    # Use a regular expression to get span tags whose class attribute is title and whose text does not start with &
    # and use a capturing group to extract the tag content
    pattern1 = re.compile(r'<span class="title">([^&]*?)</span>')
    titles = pattern1.findall(resp.text)
    # Use a regular expression to get span tags whose class attribute is rating_num
    # and use a capturing group to extract the tag content
    pattern2 = re.compile(r'<span class="rating_num".*?>(.*?)</span>')
    ranks = pattern2.findall(resp.text)
    # Use zip to combine the two lists and loop through all movie titles and ratings
    for title, rank in zip(titles, ranks):
        print(title, rank)
    # Sleep randomly for 1 to 5 seconds to avoid crawling pages too often
    time.sleep(random.random() * 4 + 1)
```

> **Note**: By analyzing Douban's robots protocol, we find that Douban does not refuse Baidu's crawler from getting its data, so we can also disguise the crawler as Baidu's crawler, and change the `headers` parameter of the `get` function to `headers={'User-Agent': 'BaiduSpider'}`.
