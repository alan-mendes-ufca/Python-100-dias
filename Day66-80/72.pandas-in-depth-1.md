## pandas em profundidade - 1

Pandas é um kit de ferramentas poderoso para **análise de dados estruturados** desenvolvido por Wes McKinney em 2008. Construído sobre NumPy para armazenamento e computação de dados, o pandas fornece tipos, métodos e funções feitos especialmente para análise de dados e oferece um suporte muito bom para análise e mineração de dados. Ao mesmo tempo, o pandas também funciona muito bem em conjunto com a ferramenta de visualização matplotlib, facilitando a apresentação visual dos dados.

Os principais tipos de dados no pandas são `Series` e `DataFrame`, que são usados ​​para processar dados unidimensionais e bidimensionais. Além deles, existe também um tipo denominado `Index` e seus subtipos, que fornecem capacidade de indexação para `Series` e `DataFrame`. No trabalho diário, `DataFrame` é usado mais amplamente, porque uma estrutura de dados bidimensional combina muito bem com uma tabela com linhas e colunas. Tanto `Series` quanto `DataFrame` fornecem muitos métodos para processamento de dados. Com base nisso, um analista de dados pode realizar filtragem, fusão, concatenação, limpeza, pré-processamento, agregação, dinamização, visualização e muitas outras operações.

### Criando objetos `Series`

Um objeto `Series` em pandas pode ser usado para representar uma estrutura de dados unidimensional, mas possui índices e algumas funções extras. Internamente, um `Series` contém dois arrays, um para armazenar dados e outro para armazenar índices. Podemos criar um `Series` a partir de uma lista ou array, conforme mostrado abaixo.

```python
import numpy as np
import pandas as pd

ser1 = pd.Series(data=[120, 380, 250, 360], index=['Q1', 'Q2', 'Q3', 'Q4'])
ser1
```

> **Observação**: No construtor `Series`, o parâmetro `data` representa os dados, e o parâmetro `index` representa os índices dos dados, que são como rótulos correspondentes aos dados.

Saída:

```
Q1    120
Q2    380
Q3    250
Q4    360
dtype: int64
```

Também podemos criar um objeto `Series` a partir de um dicionário.

```python
ser2 = pd.Series({'Q1': 320, 'Q2': 180, 'Q3': 300, 'Q4': 405})
ser2
```

> **Observação**: Quando um `Series` é criado a partir de um dicionário, as chaves do dicionário tornam-se os rótulos de dados e os valores correspondentes a essas chaves tornam-se os dados.

Saída:

```
Q1    320
Q2    180
Q3    300
Q4    405
dtype: int64
```

### Operações em objetos `Series`

#### Operações escalares

Vamos tentar adicionar `10` a cada trimestre de `ser1`.

```python
ser1 += 10
ser1
```

Saída:

```
Q1    130
Q2    390
Q3    260
Q4    370
dtype: int64
```

#### Operações vetoriais

Vamos tentar somar os dados dos trimestres correspondentes em `ser1` e `ser2`.

```python
ser1 + ser2
```

Saída:

```
Q1    450
Q2    570
Q3    560
Q4    775
dtype: int64
```

#### Operações de indexação

##### Indexação comum

Assim como as matrizes, os objetos `Series` também podem ser indexados e fatiados. A diferença é que um `Series` mantém internamente um array de índices, portanto, além de usar índices inteiros para recuperar dados, também podemos usar os índices que definimos para obter os dados correspondentes.

Use indexação inteira:

```python
ser1[2]
```

Saída:

```
260
```

Use indexação personalizada:

```python
ser1['Q3']
```

Saída:

```
260
```

```python
ser1['Q1'] = 380
ser1
```

Saída:

```
Q1    380
Q2    390
Q3    260
Q4    370
dtype: int64
```

##### Indexação de fatia

A operação de fatiamento de um objeto `Series` é semelhante à de uma lista ou array. Ao fornecer um índice inicial e final, podemos retirar ou modificar parte dos dados do `Series` original. Aqui também podemos usar índices inteiros e índices personalizados.

```python
ser2[1:3]
```

Saída:

```
Q2    180
Q3    300
dtype: int64
```

```python
ser2['Q2':'Q4']
```

Saída:

```
Q2    180
Q3    300
Q4    405
dtype: int64
```

> **Dica**: quando índices personalizados são usados ​​para fatiar, o elemento correspondente ao índice final também pode ser incluído.

```python
ser2[1:3] = 400, 500
ser2
```

Saída:

```
Q1    320
Q2    400
Q3    500
Q4    405
dtype: int64
```

##### Indexação sofisticada

```python
ser2[['Q2', 'Q4']]
```

Saída:

```
Q2    400
Q4    405
dtype: int64
```

```python
ser2[['Q2', 'Q4']] = 600, 520
ser2
```

Saída:

```
Q1    320
Q2    600
Q3    500
Q4    520
dtype: int64
```

##### Indexação booleana

```python
ser2[ser2 >= 500]
```

Saída:

```
Q2    600
Q3    500
Q4    520
dtype: int64
```

### Propriedades e métodos de objetos `Series`

Existem muitas propriedades e métodos em um objeto `Series`. Vamos nos concentrar nos mais importantes. A tabela abaixo mostra as propriedades comumente usadas de um `Series`.

| Propriedade | Descrição |
| ------------------------- | ----------- |
| `dtype` / `dtypes` | Retorne o tipo de dados do `Series` |
| `hasnans` | Verifique se `Series` contém valores nulos |
| `at` / `iat` | Acesse um único valor em um `Series` por índice |
| `loc` / `iloc` | Acesse um ou mais valores em um `Series` por índice |
| `index` | Retornar o índice do `Series` (objeto `Index`) |
| `is_monotonic` | Verifique se os dados no `Series` são monotônicos |
| `is_monotonic_increasing` | Verifique se os dados estão aumentando monotônico |
| `is_monotonic_decreasing` | Verifique se os dados são decrescentes monotônicos |
| `is_unique` | Verifique se cada valor é único |
| `size` | Retorne o número de elementos no `Series` |
| `values` | Retorne os valores de `Series` como `ndarray` |

Podemos entender essas propriedades por meio do código abaixo.

```python
print(ser2.dtype)                    # data type
print(ser2.hasnans)                  # whether there are null values
print(ser2.index)                    # index
print(ser2.values)                   # values
print(ser2.is_monotonic_increasing)  # whether it is monotonic increasing
print(ser2.is_unique)                # whether each value is unique
```

Saída:

```
int64
False
Index(['Q1', 'Q2', 'Q3', 'Q4'], dtype='object')
[320 600 500 520]
False
True
```

Existem muitos métodos em objetos `Series`. Abaixo usamos alguns trechos de código para apresentar métodos comuns.

#### Métodos relacionados a estatísticas

```python
print(ser2.mean())    # mean
print(ser2.median())  # median
print(ser2.max())     # maximum
print(ser2.min())     # minimum
print(ser2.std())     # standard deviation
print(ser2.var())     # variance
```

Saída:

```
4
1940
485.0
510.0
600
320
118.18065267490557
13966.666666666666
```

Os objetos `Series` também possuem um método denominado `describe()` que pode obter todas as estatísticas descritivas mencionadas acima.

```python
ser2.describe()
```

Saída:

```
count      4.000000
mean     485.000000
std      118.180653
min      320.000000
25%      455.000000
50%      510.000000
75%      540.000000

dtype: float64
```

> **Dica**: como `describe()` também retorna um objeto `Series`, podemos usar `ser2.describe()['mean']` para obter a média e `ser2.describe()[['max', 'min']]` para obter o máximo e o mínimo.

Se um objeto `Series` tiver valores duplicados, podemos usar o método `unique()` para obter uma matriz composta de valores únicos. Podemos usar o método `nunique()` para contar quantos valores não duplicados existem. Se quisermos contar quantas vezes cada valor se repete, podemos usar o método `value_counts()`. Este método retorna um objeto `Series`. Seus índices são os valores do `Series` original, e os dados são as contagens de quantas vezes cada valor aparece. Por padrão, ele é classificado em ordem decrescente de contagem, conforme mostrado abaixo.

```python
ser3 = pd.Series(data=['apple', 'banana', 'apple', 'pitaya', 'apple', 'pitaya', 'durian'])
ser3.value_counts()
```

Saída:

```
apple     3
pitaya    2
durian    1
banana    1
dtype: int64
```

```python
ser3.nunique()
```

Saída:

```
4
```

Para `ser3`, também podemos usar o método `mode()` para encontrar a moda dos dados. Como o modo pode não ser exclusivo, o valor de retorno do método `mode()` ainda é um objeto `Series`.

```python
ser3.mode()
```

Saída:

```
0    apple
dtype: object
```

#### Processando dados

Os métodos `isna()` e `isnull()` de um objeto `Series` podem ser usados ​​para verificar valores nulos. Os métodos `notna()` e `notnull()` podem ser usados ​​para verificar valores não nulos, conforme mostrado abaixo.

```python
ser4 = pd.Series(data=[10, 20, np.nan, 30, np.nan])
ser4.isna()
```

> **Nota**: `np.nan` é um valor de ponto flutuante do padrão IEEE 754, usado especialmente para significar "não é um número". No código acima, usamos isso para representar valores nulos. Claro, também podemos usar `None` do Python para representar valores nulos, e em pandas `None` também será tratado como `np.nan`.

Saída:

```
0    False
1    False
2     True
3    False
4     True
dtype: bool
```

```python
ser4.notna()
```

Saída:

```
0     True
1     True
2    False
3     True
4    False
dtype: bool
```

Os métodos `dropna()` e `fillna()` de um objeto `Series` são usados ​​para excluir valores nulos e preencher valores nulos. Seu uso é mostrado abaixo.

```python
ser4.dropna()
```

Saída:

```
0    10.0
1    20.0
3    30.0
dtype: float64
```

```python
ser4.fillna(value=40)  # fill null values with 40
```

Saída:

```
0    10.0
1    20.0
2    40.0
3    30.0
4    40.0
dtype: float64
```

```python
ser4.fillna(method='ffill')  # fill with the non-null value before the null
```

Saída:

```
0    10.0
1    20.0
2    20.0
3    30.0
4    30.0
dtype: float64
```

Vale lembrar que tanto `dropna()` quanto `fillna()` possuem um parâmetro denominado `inplace`. Seu valor padrão é `False`, o que significa que excluir ou preencher valores nulos não modifica o objeto `Series` original. Em vez disso, ele retorna um novo objeto `Series`. Se o valor de `inplace` for alterado para `True`, a exclusão ou preenchimento de valores nulos ocorrerá no local e modificará diretamente o objeto `Series` original. Nesse momento, o valor de retorno do método é `None`. Muitos métodos que veremos mais tarde, incluindo muitos métodos de objetos `DataFrame`, também possuem esse parâmetro e significa a mesma coisa.

Os métodos `apply()` e `map()` de um objeto `Series` são muito importantes. Eles podem processar dados por meio de um dicionário ou de uma função específica e mapear ou converter os dados na forma desejada. Esses dois métodos são muito importantes na fase de preparação de dados. Vamos primeiro tentar o método denominado `map`.

```python
ser6 = pd.Series(['cat', 'dog', np.nan, 'rabbit'])
ser6
```

Saída:

```
0       cat
1       dog
2       NaN
3    rabbit
dtype: object
```

```python
ser6.map({'cat': 'kitten', 'dog': 'puppy'})
```

> **Nota**: Isso processa os dados de acordo com as regras de mapeamento fornecidas pelo dicionário.

Saída:

```
0    kitten
1     puppy
2       NaN
3       NaN
dtype: object
```

```python
ser6.map('I am a {}'.format, na_action='ignore')
```

> **Observação**: Isso aplica o método `format` da string especificada aos dados da série, ignorando todos os valores nulos.

Saída:

```
0       I am a cat
1       I am a dog
2              NaN
3    I am a rabbit
dtype: object
```

Criamos um novo objeto `Series`:

```python
ser7 = pd.Series([20, 21, 12], index=['London', 'New York', 'Helsinki'])
ser7
```

Saída:

```
London      20
New York    21
Helsinki    12
dtype: int64
```

```python
ser7.apply(np.square)
```

> **Observação**: Isso aplica a função quadrada aos dados da série. Também podemos substituir `np.square` por `lambda x: x ** 2`.

Saída:

```
London      400
New York    441
Helsinki    144
dtype: int64
```

```python
ser7.apply(lambda x, value: x - value, args=(5, ))
```

> **Observação**: A função `lambda` no método `apply` acima possui dois parâmetros. O primeiro parâmetro são os dados da série, e o segundo parâmetro precisa ser passado por nós, então adicionamos o parâmetro `args` ao método `apply` para passar um valor para o segundo parâmetro da função `lambda`.

Saída:

```
London      15
New York    16
Helsinki     7
dtype: int64
```

#### Obtendo valores principais e classificação

Os métodos `sort_index()` e `sort_values()` de um objeto `Series` podem ser usados ​​para classificar índices e dados. Os métodos de ordenação possuem um parâmetro booleano denominado `ascending`, que controla se o resultado está em ordem crescente ou decrescente. Um parâmetro denominado `kind` é usado para controlar o algoritmo de classificação. Por padrão ele usa `quicksort`, mas `mergesort` ou `heapsort` também podem ser escolhidos. Se houver valores nulos, o parâmetro `na_position` pode ser usado para decidir se os valores nulos serão colocados na frente ou no final. O padrão é `last`, conforme mostrado abaixo.

```python
ser8 = pd.Series(
    data=[35, 96, 12, 57, 25, 89],
    index=['grape', 'banana', 'pitaya', 'apple', 'peach', 'orange']
)
ser8.sort_values()  # sort by value from small to large
```

Saída:

```
pitaya    12
peach     25
grape     35
apple     57
orange    89
banana    96
dtype: int64
```

```python
ser8.sort_index(ascending=False)  # sort by index from large to small
```

Saída:

```
pitaya    12
peach     25
orange    89
grape     35
banana    96
apple     57
dtype: int64
```

Se quisermos encontrar os elementos maiores ou menores "Top-N" em um objeto `Series`, não precisamos classificar todos os valores. Podemos usar os métodos `nlargest()` e `nsmallest()`, conforme mostrado abaixo.

```python
ser8.nlargest(3)  # the 3 largest values
```

Saída:

```
banana    96
orange    89
apple     57
dtype: int64
```

```python
ser8.nsmallest(2)  # the 2 smallest values
```

Saída:

```
pitaya    12
peach     25
dtype: int64
```

#### Desenhar gráficos

Um objeto `Series` possui um método denominado `plot` que pode ser usado para gerar gráficos. Se optarmos por gerar um gráfico de linhas, gráfico de pizza, gráfico de barras e assim por diante, por padrão ele usará o índice do objeto `Series` como eixo horizontal e os dados do objeto `Series` como eixo vertical. Abaixo criamos um objeto `Series` e desenhamos um gráfico de barras a partir dele.

```python
import matplotlib.pyplot as plt

ser9 = pd.Series({'Q1': 400, 'Q2': 520, 'Q3': 180, 'Q4': 380})
# use the kind parameter of plot to specify a bar chart
ser9.plot(kind='bar')
# customize the range of the y-axis
plt.ylim(0, 600)
# customize the x-axis ticks
plt.xticks(rotation=0)
# add data labels to the bars
for i in range(ser9.size):
    plt.text(i, ser9[i] + 5, ser9[i], ha='center')
plt.show()
```
