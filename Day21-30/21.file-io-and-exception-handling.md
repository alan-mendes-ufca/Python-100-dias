## Leitura/gravação de arquivos e tratamento de exceções

No desenvolvimento real, frequentemente encontramos situações em que os dados precisam ser persistidos. A chamada persistência significa mover dados de um meio de armazenamento que não pode mantê-los por muito tempo, geralmente a memória, para um meio de armazenamento que pode mantê-los por um longo tempo, geralmente um disco rígido. A maneira mais direta e simples de tornar os dados persistentes é salvá-los em **arquivos** por meio do **sistema de arquivos**.

Um **sistema de arquivos** de computador é uma forma de armazenar e organizar dados de computador. Facilita o acesso e a localização de dados. O sistema de arquivos usa os conceitos lógicos abstratos de **arquivos** e **diretórios em forma de árvore** para substituir o conceito de bloco de dados de dispositivos físicos, como discos rígidos, discos ópticos e memória flash. Quando os usuários usam o sistema de arquivos para salvar dados, eles não precisam se preocupar em qual bloco de dados do disco rígido os dados são realmente salvos. Eles só precisam lembrar o caminho e o nome do arquivo. Antes de gravar novos dados, os usuários também não precisam se preocupar com quais blocos de dados no disco rígido não foram usados. O gerenciamento do espaço de armazenamento do disco rígido, incluindo alocação e liberação, é concluído automaticamente pelo sistema de arquivos. Os usuários só precisam lembrar em qual arquivo os dados foram gravados.

### Abrindo e fechando arquivos

Com um sistema de arquivos, podemos ler e gravar dados em arquivos de maneira muito conveniente. Em Python, é muito fácil realizar operações com arquivos. Podemos usar a função `open` integrada do Python para abrir um arquivo. Ao usar a função `open`, podemos especificar informações como **nome do arquivo**, **modo de operação** e **codificação de caracteres** por meio dos parâmetros da função. Então podemos ler ou escrever o arquivo. O modo de operação aqui significa que tipo de arquivo abrir, arquivo de texto ou arquivo binário, e que tipo de operação fazer, ler, escrever ou anexar. Os detalhes são mostrados na tabela abaixo.

| Modo | Significado |
| -------- | -------------------------------- |
| `'r'` | Ler (padrão) |
| `'w'` | Escreva, truncando primeiro o conteúdo existente |
| `'x'` | Escreva, mas gere uma exceção se o arquivo já existir |
| `'a'` | Anexar, escrevendo conteúdo no final de um arquivo existente |
| `'b'` | Modo binário |
| `'t'` | Modo de texto (padrão) |
| `'+'` | Modo de atualização, permitindo leitura e escrita |

Ao usar a função `open`, se o arquivo aberto for um arquivo de caracteres, um arquivo de texto, podemos usar o parâmetro `encoding` para especificar a codificação de caracteres usada para ler e gravar o arquivo. Se você não entende conceitos como codificação e conjunto de caracteres, pode ler [Conjuntos de caracteres e codificações de caracteres](https://www.cnblogs.com/skynet/archive/2011/05/03/2035105.html). Não vamos repetir isso aqui.

Se a função `open` abrir o arquivo com sucesso, ela retornará um objeto de arquivo. Através deste objeto podemos ler e escrever o arquivo. Se a abertura do arquivo falhar, a função `open` gerará uma exceção. Falaremos sobre isso mais tarde. Se quisermos fechar o arquivo aberto, podemos usar o método `close` do objeto arquivo. Desta forma, o arquivo pode ser liberado quando a operação do arquivo terminar.

### Lendo e escrevendo arquivos de texto

Quando usamos `open` para abrir um arquivo de texto, precisamos especificar o nome do arquivo e definir o modo como `'r'`; se não o especificarmos, o padrão também será `'r'`. Se precisarmos especificar uma codificação de caracteres, podemos passar o argumento `encoding`. Se não o especificarmos, o valor padrão será `None`, o que significa que a codificação padrão do sistema operacional será usada na leitura do arquivo. Uma coisa a ter em mente é que se não pudermos garantir que a codificação usada para salvar o arquivo corresponda à codificação especificada pelo argumento `encoding`, a leitura do arquivo poderá falhar porque os caracteres não podem ser decodificados.

O exemplo abaixo mostra como ler um arquivo de texto simples. Um arquivo de texto simples geralmente contém apenas caracteres brutos codificados e, diferentemente do rich text, não contém informações de controle de formatação, portanto pode ser lido diretamente pelos editores de texto mais simples.

```python
file = open('to oak.txt', 'r', encoding='utf-8')
print(file.read())
file.close()
```

Além de usar o método `read` do objeto de arquivo, também podemos ler um arquivo de texto linha por linha com um loop `for-in` ou ler todas as linhas em uma lista com `readlines`, conforme mostrado abaixo.

```python
file = open('to oak.txt', 'r', encoding='utf-8')
for line in file:
    print(line, end='')
file.close()

file = open('to oak.txt', 'r', encoding='utf-8')
lines = file.readlines()
for line in lines:
    print(line, end='')
file.close()
```

Se quisermos gravar conteúdo em um arquivo, podemos abri-lo no modo `'w'` ou `'a'`. O primeiro trunca o texto existente e escreve novo conteúdo, enquanto o último acrescenta novo conteúdo ao final do que já existe.

```python
file = open('to oak.txt', 'a', encoding='utf-8')
file.write('\nTitle: To the Oak Tree')
file.write('\nAuthor: Shu Ting')
file.write('\nTime: March 1977')
file.close()
```

### Tratamento de exceções

Observe que no código acima, se o arquivo especificado por `open` não existir ou não puder ser aberto, uma exceção será gerada e o programa travará. Para tornar nosso código mais robusto e tolerante a falhas, podemos **usar o mecanismo de exceção do Python para lidar adequadamente com código que pode falhar em tempo de execução**. Python tem cinco palavras-chave relacionadas a exceções: `try`, `except`, `else`, `finally` e `raise`. Vejamos primeiro o exemplo a seguir e depois explicaremos como eles são usados.

```python
file = None
try:
    file = open('to oak.txt', 'r', encoding='utf-8')
    print(file.read())
except FileNotFoundError:
    print('Unable to open the specified file!')
except LookupError:
    print('Unknown encoding specified!')
except UnicodeDecodeError:
    print('Decoding error while reading file!')
finally:
    if file:
        file.close()
```

Em Python, podemos colocar códigos que podem apresentar problemas em tempo de execução no bloco `try`. Após `try`, podemos seguir um ou mais blocos `except` para capturar exceções e tratá-las. Por exemplo, no código acima, se o arquivo não puder ser encontrado, ele gera `FileNotFoundError`; se uma codificação desconhecida for especificada, ela gerará `LookupError`; e se o arquivo não puder ser decodificado com a codificação especificada durante a leitura, ele gerará `UnicodeDecodeError`. Portanto, depois de `try`, adicionamos três blocos `except` para lidar com essas três situações de exceção diferentes.

Após `except`, também podemos adicionar um bloco `else`. Este é o código executado quando nenhuma exceção ocorre no código em `try`. Além disso, o código em `else` não fará captura de exceção novamente. Isso significa que se ocorrer uma exceção ali, o programa terminará por causa da exceção e reportará as informações da exceção. Por fim, utilizamos o bloco `finally` para fechar o arquivo aberto e liberar o recurso externo obtido no programa. Como o código no bloco `finally` é executado independentemente de o programa ser normal ou ter uma exceção, mesmo que a função `exit` do módulo `sys` seja chamada para encerrar o programa Python, o código em `finally` ainda será executado. Como a essência da função `exit` é gerar uma exceção `SystemExit`, chamamos o bloco `finally` de "bloco de código sempre executado". É mais adequado para liberar recursos externos.

Pela estrutura de herança acima, podemos ver que todas as exceções em Python são subtipos de `BaseException`. Possui quatro subclasses diretas: `SystemExit`, `KeyboardInterrupt`, `GeneratorExit` e `Exception`. Entre eles, `SystemExit` indica que o intérprete está sendo solicitado a sair, `KeyboardInterrupt` indica que o usuário interrompeu a execução do programa pressionando `Ctrl+C` e `GeneratorExit` indica que um gerador está saindo após uma notificação de exceção. Tudo bem se você ainda não entendeu isso completamente. O importante aqui é `Exception`, que é o tipo pai para exceções comuns, e muitas classes de exceção herdam dele direta ou indiretamente. Se as exceções integradas do Python não puderem atender às necessidades de um aplicativo, podemos definir nossos próprios tipos de exceção, e as exceções personalizadas também devem herdar direta ou indiretamente de `Exception`. Também podemos substituir ou adicionar métodos quando necessário.

Em Python, podemos usar a palavra-chave `raise` para gerar uma exceção, e o chamador pode capturar e tratar a exceção por meio de uma estrutura `try...except...`. Por exemplo, em uma função, quando a condição de execução da função não é satisfeita, podemos usar o lançamento de uma exceção para informar ao chamador onde está o problema. O chamador pode recuperar o código da exceção capturando e tratando a exceção. O código para definir e gerar uma exceção é mostrado abaixo.

```python
class InputError(ValueError):
    """Custom exception type"""
    pass


def fac(num):
    """Calculate factorial"""
    if num < 0:
        raise InputError('Factorial can only be calculated for non-negative integers')
    if num in (0, 1):
        return 1
    return num * fac(num - 1)
```

Podemos chamar a função `fac` abaixo, capturar o erro de entrada com `try...except...` e imprimir o objeto de exceção. Se a entrada estiver correta, o programa calcula o fatorial e termina.

```python
flag = True
while flag:
    num = int(input('n = '))
    try:
        print(f'{num}! = {fac(num)}')
        flag = False
    except InputError as err:
        print(err)
```

### Sintaxe do Gerenciador de Contexto

Para o objeto de arquivo retornado pela função `open`, também podemos usar a sintaxe do gerenciador de contexto `with`, para que após a conclusão da operação do arquivo, o método `close` do objeto de arquivo seja executado automaticamente. Isso pode tornar o código mais simples e limpo, pois não precisamos escrever o bloco `finally` para fechar o arquivo e liberar recursos. Uma coisa para lembrar a todos é que nem todo objeto pode ser colocado na sintaxe de contexto `with`. Somente objetos que correspondem ao **protocolo gerenciador de contexto**, que possuem os métodos mágicos `__enter__` e `__exit__`, podem usar esta sintaxe. O módulo `contextlib` na biblioteca padrão Python também fornece suporte para sintaxe de contexto `with`. Explicaremos isso mais tarde, quando o usarmos.

O código reescrito com `with` é mostrado abaixo.

```python
try:
    with open('to oak.txt', 'r', encoding='utf-8') as file:
        print(file.read())
except FileNotFoundError:
    print('Unable to open the specified file!')
except LookupError:
    print('Unknown encoding specified!')
except UnicodeDecodeError:
    print('Decoding error while reading file!')
```

### Lendo e gravando arquivos binários

Ler e gravar arquivos binários é semelhante a ler e gravar arquivos de texto, mas há duas coisas a serem observadas. Primeiro, ao usar `open`, o modo é `'rb'` para leitura e `'wb'` para gravação. Em segundo lugar, ao ler e gravar arquivos de texto, o valor de retorno de `read` e o argumento de `write` são objetos `str`, mas ao ler e gravar arquivos binários, o valor de retorno de `read` e o argumento de `write` são objetos semelhantes a bytes. O código a seguir copia o arquivo de imagem `guido.jpg` no caminho atual em um arquivo denominado `guido-copy.jpg`.

```python
try:
    with open('guido.jpg', 'rb') as file1:
        data = file1.read()
    with open('guido-copy.jpg', 'wb') as file2:
        file2.write(data)
except FileNotFoundError:
    print('Cannot open the specified file.')
except IOError:
    print('An error occurred while reading or writing the file.')
print('Program execution finished.')
```

Se você quiser copiar um arquivo binário grande, ler todos os dados na memória de uma só vez pode causar uso excessivo de memória. Para reduzir a sobrecarga de memória, podemos ler e gravar o arquivo em partes, conforme mostrado abaixo.

```python
try:
    with open('guido.jpg', 'rb') as file1, open('guido-copy.jpg', 'wb') as file2:
        data = file1.read(512)
        while data:
            file2.write(data)
            data = file1.read()
except FileNotFoundError:
    print('Cannot open the specified file.')
except IOError:
    print('An error occurred while reading or writing the file.')
print('Program execution finished.')
```

### Resumo

Ao ler e gravar arquivos, podemos persistir os dados. Em Python, a função `open` nos fornece um objeto de arquivo e, então, podemos usar os métodos `read` e `write` do objeto de arquivo para realizar operações de arquivo. Problemas que ocorrem em tempo de execução podem ser tratados através do mecanismo de exceção do Python. As palavras-chave principais do mecanismo de exceção do Python são `try`, `except`, `else`, `finally` e `raise`. A instrução `except` após `try` não é obrigatória e a instrução `finally` também não é obrigatória, mas pelo menos uma delas deve existir. Pode haver uma ou mais instruções `except`. Muitas instruções `except` são correspondidas na ordem em que são escritas. Se a exceção já tiver sido tratada, as instruções `except` posteriores não serão inseridas novamente. Em uma instrução `except`, também podemos usar uma tupla para capturar muitos tipos de exceção ao mesmo tempo. Se nenhum tipo de exceção for gravado após `except`, ele capturará todas as exceções por padrão. Depois de capturar uma exceção, podemos usar `raise` para lançá-la novamente, mas não é recomendado capturar e lançar a mesma exceção novamente. Também não é recomendado capturar todas as exceções quando você não entende claramente a lógica, pois isso pode esconder sérios problemas no programa. Um último ponto: **não use o mecanismo de exceção para lidar com a lógica de negócios normal ou controlar o fluxo normal do programa**. Em palavras simples, não abuse do mecanismo de exceção. Este é um erro que os iniciantes costumam cometer.
