## Aprendizado de máquina na prática

Nesta lição, usamos o projeto clássico de previsão de sobrevivência do Titanic para explicar um aplicativo completo de aprendizado de máquina. Este projeto vem da [Kaggle](https://www.kaggle.com/), uma plataforma de competição de ciência de dados mundialmente conhecida. Esta plataforma é uma das comunidades importantes em ciência de dados, aprendizado de máquina e inteligência artificial. Ele fornece um local para cientistas de dados e engenheiros de algoritmos praticarem, compartilharem e competirem. Quer você seja um iniciante ou um especialista experiente, o Kaggle pode fornecer recursos e desafios valiosos. Por meio de conjuntos de dados, competições, notebooks e cursos oferecidos na plataforma, os usuários podem aprimorar as habilidades relacionadas à ciência de dados de acordo com suas próprias necessidades e interagir com cientistas de dados em todo o mundo.

O projeto de previsão de sobrevivência do Titanic é um dos mais famosos projetos de aprendizado de máquina para iniciantes no Kaggle. A tarefa é prever se os passageiros sobreviveram ao desastre do Titanic a partir de informações básicas como idade, sexo, classe da passagem e porto de embarque. Este é um problema de classificação padrão. No Kaggle, a competição se chama [*Titanic - Aprendizado de Máquina após Desastres*](https://www.kaggle.com/competitions/titanic/). Dentro dele você pode baixar os arquivos de dados `train.csv` e `test.csv`. O primeiro é o conjunto de treinamento, que contém recursos e rótulos, enquanto o segundo é o conjunto de testes, que contém apenas recursos.

### Engenharia de recursos

A engenharia de recursos é uma parte importante do aprendizado de máquina. Através do processamento, transformação, seleção e construção dos dados originais, podemos extrair recursos que ajudam o modelo a compreender melhor os dados e melhorar o efeito de previsão do modelo.

As etapas principais geralmente incluem:

1. **Limpeza de dados**: lide com valores ausentes, valores duplicados e valores discrepantes.
2. **Transformação de recursos**: torne os dados brutos mais adequados para entrada de modelo, como padronização, normalização, codificação categórica e transformações de log.
3. **Seleção de recursos**: mantenha recursos úteis e remova os irrelevantes ou redundantes.
4. **Redução de dimensionalidade**: mapeia dados de um espaço de alta dimensão para um espaço de menor dimensão enquanto tenta preservar as informações principais.
5. **Construção de recursos**: derivar novos recursos dos originais.

Para o conjunto de dados do Titanic, podemos:

- preencha os valores `Age` ausentes com a mediana;
- preencha os valores `Embarked` ausentes com o modo;
- transforme `Cabin` em um recurso binário que indica se existem informações de cabine.

```python
df['Age'] = df.Age.fillna(df.Age.median())
df['Embarked'] = df.Embarked.fillna(df.Embarked.mode()[0])
df['Cabin'] = df.Cabin.replace(r'.+', '1', regex=True).replace(np.nan, 0).astype('i8')
```

Em seguida, dimensionamos `Fare` e `Age` com `StandardScaler`:

```python
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
df[['Fare', 'Age']] = scaler.fit_transform(df[['Fare', 'Age']])
```

Em seguida, codificamos one-hot as colunas categóricas `Sex` e `Embarked`:

```python
df = pd.get_dummies(df, columns=['Sex', 'Embarked'], drop_first=True)
```

A lição também deriva novos recursos de nomes de passageiros e informações familiares:

```python
title_mapping = {
    'Mr': 0, 'Miss': 1, 'Mrs': 2, 'Master': 3, 'Dr': 4, 'Rev': 5, 'Col': 6, 'Major': 7,
    'Mlle': 8, 'Ms': 9, 'Lady': 10, 'Sir': 11, 'Jonkheer': 12, 'Don': 13, 'Dona': 14, 'Countess': 15
}
df['Title'] = df['Name'].map(
    lambda x: x.split(',')[1].split('.')[0].strip()
).map(title_mapping).fillna(-1)
df['FamilySize'] = df['SibSp'] + df['Parch'] + 1
```

> **Observação**: Mapeamos cada título para um número. Não há necessidade de se preocupar muito com o mapeamento exato entre títulos e números, e alguns títulos nem aparecem no conjunto de treinamento. Lidamos com títulos desconhecidos como `-1`.

Depois disso, podemos descartar campos que não são mais necessários:

```python
df.drop(columns=['Name', 'SibSp', 'Parch', 'Ticket'], inplace=True)
```

### Treinamento de modelo

Agora dividimos o conjunto de dados em um conjunto de treinamento e um conjunto de validação. Não estamos fazendo a divisão final de trem/teste aqui, porque o Kaggle já nos fornece o conjunto de teste real em `test.csv`. Portanto, treinamos 90% do `train.csv` e validamos os 10% restantes.

```python
from sklearn.model_selection import train_test_split

X, y = df.drop(columns='Survived'), df.Survived
X_train, X_valid, y_train, y_valid = train_test_split(X, y, train_size=0.9, random_state=3)
```

Podemos primeiro tentar a regressão logística:

```python
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import classification_report

model = LogisticRegression(penalty='l1', tol=1e-6, solver='liblinear')
model.fit(X_train, y_train)
y_pred = model.predict(X_valid)
print(classification_report(y_valid, y_pred))
```

### Avaliação do modelo

Em seguida, carregamos os dados de teste reais de `test.csv`, processamos da mesma forma e usamos o modelo treinado para fazer previsões. O resultado pode ser salvo como um arquivo CSV com duas colunas, `PassengerId` e `Survived`, e depois enviado ao Kaggle para pontuação.

```python
test = pd.read_csv('data/test.csv', index_col='PassengerId')
test['Age'] = test.Age.fillna(test.Age.median())
test['Fare'] = test.Fare.fillna(test.Fare.median())
test['Embarked'] = test.Embarked.fillna(test.Embarked.mode()[0])
test['Cabin'] = test.Cabin.replace(r'.+', '1', regex=True).replace(np.nan, 0).astype('i8')
test[['Fare', 'Age']] = scaler.fit_transform(test[['Fare', 'Age']])
test = pd.get_dummies(test, columns=['Sex', 'Embarked'], drop_first=True)
test['Title'] = test['Name'].apply(lambda x: x.split(',')[1].split('.')[0].strip()).map(title_mapping).fillna(-1)
test['FamilySize'] = test['SibSp'] + test['Parch'] + 1
test.drop(columns=['Name', 'Ticket', 'SibSp', 'Parch'], inplace=True)

passenger_id, X_test = test.index, test
y_test_pred = model.predict(X_test)

result = pd.DataFrame({
    'PassengerId': passenger_id,
    'Survived': y_test_pred
})
result.to_csv('submission.csv', index=False)
```

A lição também sugere várias maneiras de continuar melhorando a engenharia de recursos, como:

1. discretizar `Age` em compartimentos;
2. manusear `Cabin` com mais cuidado em vez de simplesmente binarizá-lo;
3. combinando `Pclass` e `Sex` em recursos de interação;
4. derivar indicadores especiais de títulos como `Mrs`;
5. tentando se remover `Embarked` ajuda.

### Implantação de modelo

Depois que o modelo for treinado, ele poderá ser serializado para implantação:

```python
import joblib

joblib.dump(model, 'model.pkl')
```

Mais tarde, ele pode ser carregado novamente para previsão:

```python
import joblib

model = joblib.load('model.pkl')
model.predict(X_test)
```

Se quisermos implantá-lo como um serviço web, podemos construir uma API simples com Flask:

```python
from flask import Flask
from flask import jsonify
from flask import request

import joblib
import pandas as pd
import xgboost as xgb

app = Flask(__name__)


@app.route('/predict', methods=['POST'])
def predict():
    query_df = pd.DataFrame(request.json)
    model = joblib.load('model.pkl')
    y_pred = (model.predict(xgb.DMatrix(query_df)) > 0.5).tolist()
    return jsonify({'message': 'OK', 'result': y_pred})


if __name__ == '__main__':
    app.run(debug=True)
```

É claro que, em um projeto de engenharia real, você não esperaria até que chegasse uma solicitação para carregar o modelo, pois isso prejudicaria seriamente o desempenho do serviço web. O modelo deve ser carregado antecipadamente, em momento apropriado após o início do serviço, e seu uso de recursos também deve ser monitorado de acordo com as necessidades reais.

Se você executar o código acima, um servidor web será executado na porta local `5000` por padrão. Podemos então usar uma ferramenta de teste de API para enviar uma solicitação HTTP ao servidor e ver se o modelo pode fornecer um resultado de previsão.
