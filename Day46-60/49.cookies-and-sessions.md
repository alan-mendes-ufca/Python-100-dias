Antes disso, vamos primeiro falar sobre as maneiras como uma aplicação Web pode rastrear usuários e o suporte que o Django oferece para rastreamento de usuários. Para um aplicativo da Web, depois que um usuário efetua login com êxito, o servidor deve ser capaz de lembrar que o usuário já efetuou login. ### Implementando rastreamento de usuário

Hoje, se um site não consegue lembrar quem você é e o que você fez no site antes por algum método, ele perde usabilidade e conveniência, e então é muito provável que perca usuários. Portanto, lembrar de um usuário, ou mais profissionalmente, **rastrear usuários**, é uma função necessária para a maioria dos aplicativos da Web.

No lado do servidor, a maneira mais simples de lembrar um usuário é criar um objeto. Através deste objeto podemos salvar todas as informações relacionadas ao usuário. Este objeto é o que costumamos chamar de sessão. Aí surge o problema: o próprio HTTP é um protocolo **sem conexão**, porque depois que o servidor termina de responder à solicitação do cliente, ele se desconecta; também é um protocolo **stateless**, pois quando o cliente envia outra solicitação ao servidor, o servidor não pode saber nenhuma informação anterior sobre este cliente. Mesmo que o servidor mantenha os dados do usuário por meio do objeto de sessão, ainda precisamos saber qual sessão salva anteriormente está relacionada à solicitação atual. Podemos atribuir um identificador globalmente exclusivo a cada objeto de sessão para identificá-lo. Vamos chamá-lo de `sessionid`. Cada vez que o cliente envia uma solicitação, desde que carregue este `sessionid`, temos uma forma de encontrar o objeto de sessão correspondente, para que possamos lembrar as informações do usuário entre duas solicitações, que é o rastreamento do usuário mencionado anteriormente.

Para que o cliente lembre do `sessionid` e o carregue em todas as solicitações, temos os seguintes métodos:

1. Reescrita de URL. Reescrever URL significa carregar `sessionid` na URL, por exemplo: `http://www.example.com/index.html?sessionid=123456`. O servidor obtém o valor do parâmetro `sessionid` e encontra o objeto de sessão correspondente.

2. Campos ocultos. Ao enviar um formulário, podemos enviar dados extras ao servidor definindo campos ocultos no formulário. Por exemplo: `<input type="hidden" name="sessionid" value="123456">`.

3. Armazenamento local. Os navegadores modernos oferecem suporte a muitas soluções de armazenamento local, incluindo cookie, localStorage, sessionStorage e IndexedDB. Dentre essas soluções, o cookie tem o histórico mais longo e também é o que mais tem recebido críticas. É também a primeira solução que explicaremos aqui. Simplificando, um cookie são dados salvos nos arquivos temporários do navegador como pares de valores-chave. Em cada solicitação, o cabeçalho da solicitação carrega o cookie deste site para o servidor. Portanto, se escrevermos `sessionid` em um cookie, na próxima solicitação o servidor só precisará ler o cookie do cabeçalho da solicitação para obter este `sessionid`, conforme mostrado abaixo.

![](./res/sessionid_from_cookie.png)

Na era HTML5, além do cookie, também podemos usar as novas APIs de armazenamento local para salvar dados, que são as tecnologias localStorage, sessionStorage e IndexedDB mencionadas, conforme mostrado abaixo.

![](./res/cookie_xstorage_indexeddb.png)

**Resumindo**, para implementar o rastreamento de usuários, o servidor pode criar um objeto de sessão para cada sessão de usuário e gravar o ID do objeto de sessão no cookie do navegador. Na próxima vez que o usuário solicitar o servidor, o navegador carregará as informações do cookie salvas para o site no cabeçalho da solicitação HTTP. Em seguida, o servidor pode encontrar o ID do objeto de sessão no cookie e obter o objeto de sessão que foi criado anteriormente de acordo com esse ID. Como o objeto de sessão pode salvar dados do usuário em pares de valores-chave, todas as informações salvas anteriormente no objeto de sessão podem ser retiradas. O servidor também pode decidir a identidade do usuário e conhecer as preferências do usuário de acordo com essas informações, para poder prestar um serviço melhor personalizado.

### Suporte Django para Sessão

Ao criar um projeto Django, um middleware chamado `SessionMiddleware` já está habilitado por padrão no arquivo `settings.py`. Explicaremos o middleware em detalhes em uma lição posterior. Aqui, você só precisa saber que ele existe. Como esse middleware existe, podemos usar diretamente o atributo `session` do objeto de solicitação para trabalhar com o objeto de sessão. Como dissemos antes, o atributo `session` é um objeto contêiner que pode ler e gravar dados como um dicionário, para que possamos manter os dados do usuário usando "pares chave-valor". Ao mesmo tempo, `SessionMiddleware` também agrupa operações de cookie e salva `sessionid` no cookie. Já mencionamos isso acima.

No caso padrão, o Django serializa os dados da sessão e os salva em um banco de dados relacional. Nas versões posteriores ao Django 1.6, o método de serialização padrão é a serialização JSON. Antes disso, a serialização Pickle era usada o tempo todo. A diferença entre a serialização JSON e a serialização Pickle é que a primeira serializa objetos em strings, enquanto a última serializa objetos em strings de bytes. Por motivos de segurança, a serialização JSON se tornou a forma padrão para o Django serializar dados. Isso também significa que os dados que salvamos na sessão devem ser serializáveis em JSON, caso contrário, ocorrerá uma exceção. Mais uma coisa deve ser explicada: na maioria dos casos, usar um banco de dados relacional para salvar dados de sessão não é a melhor escolha, pois o banco de dados pode sofrer uma pressão enorme e se tornar o gargalo de desempenho do sistema. Nas lições posteriores, explicaremos como salvar os dados da sessão em um serviço de cache para melhorar o desempenho do sistema.

def gen_random_code(length=4):

Escreva a classe `Captcha` que gera a imagem captcha.

```python
"""
Image captcha
"""
import os
import random
from io import BytesIO

from PIL import Image
from PIL import ImageFilter
from PIL.ImageDraw import Draw
from PIL.ImageFont import truetype


class Bezier:
    """Bezier curve"""

    def __init__(self):
        self.tsequence = tuple([t / 20.0 for t in range(21)])
        self.beziers = {}

    def make_bezier(self, n):
        """Draw a Bezier curve"""
        try:
            return self.beziers[n]
        except KeyError:
            combinations = pascal_row(n - 1)
            result = []
            for t in self.tsequence:
                tpowers = (t ** i for i in range(n))
                upowers = ((1 - t) ** i for i in range(n - 1, -1, -1))
                coefs = [c * a * b for c, a, b in zip(combinations,
                                                      tpowers, upowers)]
                result.append(coefs)
            self.beziers[n] = result
            return result


class Captcha:
    """Captcha"""

    def __init__(self, width, height, fonts=None, color=None):
        self._image = None
        self._fonts = fonts if fonts else \
            [os.path.join(os.path.dirname(__file__), 'fonts', font)
             for font in ['Arial.ttf', 'Georgia.ttf', 'Action.ttf']]
        self._color = color if color else random_color(0, 200, random.randint(220, 255))
        self._width, self._height = width, height

    @classmethod
    def instance(cls, width=200, height=75):
        """Class method used to get a Captcha object"""
        prop_name = f'_instance_{width}_{height}'
        if not hasattr(cls, prop_name):
            setattr(cls, prop_name, cls(width, height))
        return getattr(cls, prop_name)

    def _background(self):
        """Draw the background"""
        Draw(self._image).rectangle([(0, 0), self._image.size],
                                    fill=random_color(230, 255))

    def _smooth(self):
        """Smooth the image"""
        return self._image.filter(ImageFilter.SMOOTH)

    def _curve(self, width=4, number=6, color=None):
        """Draw curves"""
        dx, height = self._image.size
        dx /= number
        path = [(dx * i, random.randint(0, height))
                for i in range(1, number)]
        bcoefs = Bezier().make_bezier(number - 1)
        points = []
        for coefs in bcoefs:
            points.append(tuple(sum([coef * p for coef, p in zip(coefs, ps)])
                                for ps in zip(*path)))
        Draw(self._image).line(points, fill=color if color else self._color, width=width)

    def _noise(self, number=50, level=2, color=None):
        """Draw interference code"""
        width, height = self._image.size
        dx, dy = width / 10, height / 10
        width, height = width - dx, height - dy
        draw = Draw(self._image)
        for i in range(number):
            x = int(random.uniform(dx, width))
            y = int(random.uniform(dy, height))
            draw.line(((x, y), (x + level, y)),
                      fill=color if color else self._color, width=level)

    def _text(self, captcha_text, fonts, font_sizes=None, drawings=None, squeeze_factor=0.75, color=None):
        """Draw text"""
        color = color if color else self._color
        fonts = tuple([truetype(name, size)
                       for name in fonts
                       for size in font_sizes or (65, 70, 75)])
        draw = Draw(self._image)
        char_images = []
        for c in captcha_text:
            font = random.choice(fonts)
            c_width, c_height = draw.textsize(c, font=font)
            char_image = Image.new('RGB', (c_width, c_height), (0, 0, 0))
            char_draw = Draw(char_image)
            char_draw.text((0, 0), c, font=font, fill=color)
            char_image = char_image.crop(char_image.getbbox())
            for drawing in drawings:
                d = getattr(self, drawing)
                char_image = d(char_image)
            char_images.append(char_image)
        width, height = self._image.size
        offset = int((width - sum(int(i.size[0] * squeeze_factor)
                                  for i in char_images[:-1]) -
                      char_images[-1].size[0]) / 2)
        for char_image in char_images:
            c_width, c_height = char_image.size
            mask = char_image.convert('L').point(lambda i: i * 1.97)
            self._image.paste(char_image,
                              (offset, int((height - c_height) / 2)),
                              mask)
            offset += int(c_width * squeeze_factor)

    @staticmethod
    def _warp(image, dx_factor=0.3, dy_factor=0.3):
        """Warp the image"""
        width, height = image.size
        dx = width * dx_factor
        dy = height * dy_factor
        x1 = int(random.uniform(-dx, dx))
        y1 = int(random.uniform(-dy, dy))
        x2 = int(random.uniform(-dx, dx))
        y2 = int(random.uniform(-dy, dy))
        warp_image = Image.new(
            'RGB',
            (width + abs(x1) + abs(x2), height + abs(y1) + abs(y2)))
        warp_image.paste(image, (abs(x1), abs(y1)))
        width2, height2 = warp_image.size
        return warp_image.transform(
            (width, height),
            Image.QUAD,
            (x1, y1, -x1, height2 - y2, width2 + x2, height2 + y2, width2 - x2, -y1))

    @staticmethod
    def _offset(image, dx_factor=0.1, dy_factor=0.2):
        """Offset the image"""
        width, height = image.size
        dx = int(random.random() * width * dx_factor)
        dy = int(random.random() * height * dy_factor)
        offset_image = Image.new('RGB', (width + dx, height + dy))
        offset_image.paste(image, (dx, dy))
        return offset_image

    @staticmethod
    def _rotate(image, angle=25):
        """Rotate the image"""
        return image.rotate(random.uniform(-angle, angle),
                            Image.BILINEAR, expand=1)

    def generate(self, captcha_text='', fmt='PNG'):
        """Generate the captcha (text and image)
        :param captcha_text: captcha text
        :param fmt: generated captcha image format
        :return: binary data of the captcha image
        """
        self._image = Image.new('RGB', (self._width, self._height), (255, 255, 255))
        self._background()
        self._text(captcha_text, self._fonts,
                   drawings=['_warp', '_rotate', '_offset'])
        self._curve()
        self._noise()
        self._smooth()
        image_bytes = BytesIO()
        self._image.save(image_bytes, format=fmt)
        return image_bytes.getvalue()


def pascal_row(n=0):
    """Generate Pascal's triangle row"""
    result = [1]
    x, numerator = 1, n
    for denominator in range(1, n // 2 + 1):
        x *= numerator
        x /= denominator
        result.append(x)
        numerator -= 1
    if n & 1 == 0:
        result.extend(reversed(result[:-1]))
    else:
        result.extend(reversed(result))
    return result


def random_color(start=0, end=255, opacity=255):
    """Get a random color"""
    red = random.randint(start, end)
    green = random.randint(start, end)
    blue = random.randint(start, end)
    if opacity is None:
        return red, green, blue
    return red, green, blue, opacity
```

def login(request: HttpRequest) -> HttpResponse:
    hint = ''
    if request.method == 'POST':
        username = request.POST.get('username')
        password = request.POST.get('password')
        if username and password:
            password = gen_md5_digest(password)
            user = User.objects.filter(username=username, password=password).first()

                request.session['username'] = user.username

```shell

```

O tempo de expiração da sessão padrão da estrutura Django é de duas semanas (1209600 segundos). Caso queira alterar esse horário, você pode adicionar o código abaixo ao arquivo de configuração do projeto.

```python
# Set the session timeout to 1 day (86400 seconds)
SESSION_COOKIE_AGE = 86400
```

Muitas aplicações com altos requisitos de segurança devem fazer com que a sessão expire quando a janela do navegador é fechada, e então nenhuma informação do usuário é mantida. Caso queira que a sessão expire quando a janela do navegador for fechada, ou seja, o `sessionid` no cookie se torna inválido, você pode adicionar a configuração abaixo.

```python
# If set to True, the session expires when the browser window is closed
SESSION_EXPIRE_AT_BROWSER_CLOSE = True
```

Se não quiser salvar os dados da sessão no banco de dados, você pode colocá-los no cache. A configuração relacionada é mostrada abaixo. Explicaremos a configuração e o uso do cache posteriormente.

```python
# Configure the session object to be stored in the cache
SESSION_ENGINE = 'django.contrib.sessions.backends.cache'
# Configure which cache group is used to save the session
SESSION_CACHE_ALIAS = 'default'
```

Se desejar alterar o método de serialização padrão dos dados da sessão, você poderá alterar o `JSONSerializer` padrão para `PickleSerializer`.

```python
SESSION_SERIALIZER = 'django.contrib.sessions.serializers.PickleSerializer'
```

Em seguida, podemos limitar a votação para professores para que apenas usuários logados possam fazê-lo. A função `praise_or_criticize` modificada é mostrada abaixo. Decidimos se o usuário efetuou login obtendo `userid` de `request.session`.

```python
def praise_or_criticize(request: HttpRequest) -> HttpResponse:
    if request.session.get('userid'):
        try:
            tno = int(request.GET.get('tno'))
            teacher = Teacher.objects.get(no=tno)
            if request.path.startswith('/praise/'):
                teacher.good_count += 1
                count = teacher.good_count
            else:
                teacher.bad_count += 1
                count = teacher.bad_count
            teacher.save()
            data = {'code': 20000, 'mesg': 'Vote succeeded', 'count': count}
        except (ValueError, Teacher.DoesNotExist):
            data = {'code': 20001, 'mesg': 'Vote failed'}
    else:
        data = {'code': 20002, 'mesg': 'Please log in first'}
    return JsonResponse(data)
```

### Lendo e gravando cookies em funções de visualização

A seguir, explicaremos com mais detalhes como utilizar cookies, para que seja mais fácil para todos utilizar esta tecnologia em projetos Web. Os objetos `HttpRequest` e `HttpResponse` agrupados pelo Django fornecem operações para leitura e gravação de cookies.

Propriedades e métodos agrupados por `HttpRequest`:

1. A propriedade `COOKIES`: esta propriedade contém todos os cookies transportados na solicitação HTTP.
2. O método `get_signed_cookie`: obtenha um cookie assinado. Se a validação da assinatura falhar, uma exceção `BadSignature` será gerada.

Métodos encapsulados por `HttpResponse`:

1. O método `set_cookie`: este método pode definir um grupo de pares de valores-chave e eles serão finalmente gravados no navegador.
2. O método `set_signed_cookie`: este método é semelhante ao anterior, mas assina o cookie para evitar adulteração. Porque se os dados do cookie forem alterados, é impossível gerar uma assinatura válida sem conhecer o [chave](https://en.wikipedia.org/wiki/Cryptographic_key) e [sal](https://en.wikipedia.org/wiki/Salt_(cryptography)). Então, quando o servidor ler o cookie, ele descobrirá que os dados e a assinatura não correspondem e uma exceção `BadSignature` será gerada. Deve ser explicado que a chave aqui é `SECRET_KEY` especificada no arquivo de configuração do projeto Django, e o salt é uma string definida no programa. Você pode configurá-lo como quiser, desde que seja uma string válida.

Se você não tiver certeza sobre como usar os métodos mencionados acima, você mesmo pode ler a [documentação oficial do Django](https://docs.djangoproject.com/en/2.1/ref/request-response/). Nenhum material pode explicar melhor do que a documentação oficial como esses métodos são usados.

Como acabamos de dizer, após `SessionMiddleware` ser habilitado, cada objeto `HttpRequest` é vinculado a um atributo `session`. É um objeto semelhante a um ditado. Além de salvar os dados do usuário, também fornece métodos para testar se o navegador suporta cookies, incluindo:

1. O método `set_test_cookie`: defina um cookie para teste.
2. O método `test_cookie_worked`: teste se o cookie de teste funciona.
3. O método `delete_test_cookie`: exclua o cookie usado para teste.
4. O método `set_expiry`: defina o tempo de expiração da sessão.
5. Os métodos `get_expire_age` / `get_expire_date`: obtêm o tempo de expiração da sessão.
6. O método `clear_expired`: limpar sessões expiradas.

Abaixo está o código que verifica se o navegador suporta cookies antes do login. Normalmente, os navegadores habilitam o suporte a cookies por padrão, mas o usuário pode desabilitar os cookies do navegador por algum motivo. Neste caso, podemos fornecer uma função de verificação na visualização. Se o navegador do usuário não suportar cookies, podemos fornecer uma solicitação correspondente.

```python
def login(request):
    if request.method == 'POST':
        if request.session.test_cookie_worked():
            request.session.delete_test_cookie()
            # Add your code to perform login process here
        else:
            return HttpResponse("Please enable cookies and try again.")
    request.session.set_test_cookie()
    return render_to_response('login.html')
```

### Alternativas aos biscoitos

Como dissemos antes, os cookies nunca tiveram uma reputação muito boa. É claro que, no desenvolvimento real, não salvamos informações confidenciais do usuário em cookies, como senhas de usuário ou números de contas de cartão de crédito. Além disso, os dados guardados nos cookies são normalmente codificados e assinados. Para navegadores que suportam HTML5, podemos usar localStorage e sessionStorage como alternativas aos cookies. Você provavelmente pode ouvir a diferença pelos nomes: os dados armazenados em `localStorage` podem ser mantidos por muito tempo; os dados armazenados em `sessionStorage` serão apagados quando o navegador for fechado. Para saber como usar essas alternativas de cookies, é recomendável que todos leiam [MDN](https://developer.mozilla.org/docs/Web).
