## Introduction to the Scrapy Framework

After you have written many crawler programs, you will find that every time you write a crawler program, you need to implement page getting, page parsing, crawler scheduling, exception handling, and anti-crawler response from beginning to end. A lot of this work is actually simple, boring, repeated work. Then is there any way to improve the efficiency of writing crawler code? The answer is yes. That way is to use a crawler framework, and among all crawler frameworks, Scrapy should be the most popular and the most powerful framework.

### Overview of Scrapy

Scrapy is a very popular web crawler framework based on Python. It can be used to crawl Web sites and extract structured data from pages. The picture below shows the basic architecture of Scrapy, including its main components and the system's data processing flow, which are the red arrows with numbers in the picture.

#### Components of Scrapy

Let us first talk about the components in Scrapy.

1. Scrapy Engine: used to control the data processing flow of the whole system.
2. Scheduler: the scheduler accepts requests from the engine, sorts them, puts them into a queue, and returns them to the engine after the engine sends requests.
3. Downloader: the main duty of the downloader is to fetch web pages and return the page content to spiders.
4. Spiders: spiders are user-defined classes used to parse web pages and crawl specific URLs. Each spider can handle one domain or one group of domains. Simply speaking, they are modules used to define the crawling and parsing rules of specific websites.
5. Item Pipeline: the main responsibility of the pipeline is to handle the data items extracted from the page by spiders. Its main tasks are cleaning, validating, and storing data. After the page is parsed by the spider, it is sent to the item pipeline, and the data is handled in several specific orders. Each item pipeline component is a Python class. They get the data item and execute methods that process the data item, and at the same time they need to decide whether the next step in the item pipeline should continue to run, or the item should be directly dropped without processing. Tasks usually done by the item pipeline include cleaning HTML data, validating parsed data, checking whether it is duplicate data and dropping it if it is, and storing the parsed data into a database, whether a relational database or a NoSQL database.
6. Middlewares: middlewares are a hook framework between the engine and other components. They are mainly used to provide custom code to extend the functions of Scrapy, including downloader middleware and spider middleware.

#### Data Processing Flow

The whole data processing flow of Scrapy is controlled by the engine. The normal running flow usually includes the steps below:

1. The engine asks the spider which website needs to be handled, and lets the spider hand over the first URL that needs to be handled.
2. The engine tells the scheduler to put the URL that needs to be handled into the queue.
3. The engine gets the next page to crawl from the scheduler.
4. The scheduler returns the next URL to crawl to the engine, and the engine sends it to the downloader through the downloader middleware.
5. After the page is downloaded by the downloader, the response content is sent to the engine through the downloader middleware. If the download fails, the engine tells the scheduler to record this URL and download it again later.
6. The engine receives the downloader's response and sends it to the spider through the spider middleware for handling.
7. The spider handles the response and returns the crawled data items. Also, it sends the new URLs that need to be followed to the engine.
8. The engine sends the crawled data items to the item pipeline, and sends the new URLs to the scheduler to be placed into the queue.

Steps 2 through 8 above keep repeating until there is no URL in the scheduler that needs to be requested, and then the crawler stops working.

### Installing and Using Scrapy

We can use Python's package management tool `pip` to install Scrapy.

```bash
pip install scrapy
```

Use the `scrapy` command on the command line to create a project named `demo`.

```bash
scrapy startproject demo
```

The directory structure of the project is shown below.

```text
demo
|____ demo
|________ spiders
|____________ __init__.py
|________ __init__.py
|________ items.py
|________ middlewares.py
|________ pipelines.py
|________ settings.py
|____ scrapy.cfg
```

Switch to the `demo` directory, and use the command below to create a spider program named `douban`.

```bash
scrapy genspider douban movie.douban.com
```

#### A Simple Example

Next, we implement a crawler that gets the movie titles, ratings, and mottos of Douban Movie Top250.

1. Define fields in the `Item` class in `items.py`. These fields are used to save data and make later operations convenient.

   ```python
   import scrapy


   class DoubanItem(scrapy.Item):
       title = scrapy.Field()
       score = scrapy.Field()
       motto = scrapy.Field()
   ```

2. Modify the file named `douban.py` in the `spiders` folder. This is the core of the spider program, and we need to add code for parsing pages. Here, we can get movie information through parsing the `Response` object. The code is shown below.

   ```python
   import scrapy
   from scrapy import Selector, Request
   from scrapy.http import HtmlResponse

   from demo.items import MovieItem


   class DoubanSpider(scrapy.Spider):
       name = 'douban'
       allowed_domains = ['movie.douban.com']
       start_urls = ['https://movie.douban.com/top250?start=0&filter=']

       def parse(self, response: HtmlResponse):
           sel = Selector(response)
           movie_items = sel.css('#content > div > div.article > ol > li')
           for movie_sel in movie_items:
               item = MovieItem()
               item['title'] = movie_sel.css('.title::text').extract_first()
               item['score'] = movie_sel.css('.rating_num::text').extract_first()
               item['motto'] = movie_sel.css('.inq::text').extract_first()
               yield item
   ```

   It is not hard to see from the code above that we can use CSS selectors for page parsing. Of course, if you want, you can also use XPath or regular expressions for page parsing. The matching methods are `xpath` and `re`.

   If you still need to generate follow-up crawling requests, we can use `yield` to produce `Request` objects. A `Request` object has two very important properties. One is `url`, which stands for the address to request. The other is `callback`, which stands for the callback function to execute after the response is got. We can slightly change the code above.

   ```python
   import scrapy
   from scrapy import Selector, Request
   from scrapy.http import HtmlResponse

   from demo.items import MovieItem


   class DoubanSpider(scrapy.Spider):
       name = 'douban'
       allowed_domains = ['movie.douban.com']
       start_urls = ['https://movie.douban.com/top250?start=0&filter=']

       def parse(self, response: HtmlResponse):
           sel = Selector(response)
           movie_items = sel.css('#content > div > div.article > ol > li')
           for movie_sel in movie_items:
               item = MovieItem()
               item['title'] = movie_sel.css('.title::text').extract_first()
               item['score'] = movie_sel.css('.rating_num::text').extract_first()
               item['motto'] = movie_sel.css('.inq::text').extract_first()
               yield item

           hrefs = sel.css('#content > div > div.article > div.paginator > a::attr("href")')
           for href in hrefs:
               full_url = response.urljoin(href.extract())
               yield Request(url=full_url)
   ```

   At this point, we can already run the crawler through the command below.

   ```bash
   scrapy crawl movie
   ```

   We can see the crawled data in the console. If you want to save the data into a file, you can use the `-o` parameter to specify the file name. Scrapy supports exporting crawled data into JSON, CSV, XML, and other formats.

   ```bash
   scrapy crawl moive -o result.json
   ```

   I do not know whether everyone noticed that there are `275` items in the JSON file got by running the crawler, and that is because the home page was crawled repeatedly. To solve this problem, we can slightly adjust the code above. We do not parse and get the URLs of new pages in the `parse` method. Instead, we prepare the URLs of the pages to crawl in advance through the `start_requests` method. The adjusted code is shown below.

   ```python
   import scrapy
   from scrapy import Selector, Request
   from scrapy.http import HtmlResponse

   from demo.items import MovieItem


   class DoubanSpider(scrapy.Spider):
       name = 'douban'
       allowed_domains = ['movie.douban.com']

       def start_requests(self):
           for page in range(10):
               yield Request(url=f'https://movie.douban.com/top250?start={page * 25}')

       def parse(self, response: HtmlResponse):
           sel = Selector(response)
           movie_items = sel.css('#content > div > div.article > ol > li')
           for movie_sel in movie_items:
               item = MovieItem()
               item['title'] = movie_sel.css('.title::text').extract_first()
               item['score'] = movie_sel.css('.rating_num::text').extract_first()
               item['motto'] = movie_sel.css('.inq::text').extract_first()
               yield item
   ```

3. If we want to finish persistence of crawler data, we can handle the `Item` objects produced by the spider program in the item pipeline. For example, we can use `openpyxl`, which we talked about before for working with Excel files, and write the data into an Excel file. The code is shown below.

   ```python
   import openpyxl

   from demo.items import MovieItem


   class MovieItemPipeline:

       def __init__(self):
           self.wb = openpyxl.Workbook()
           self.sheet = self.wb.active
           self.sheet.title = 'Top250'
           self.sheet.append(('Title', 'Score', 'Motto'))

       def process_item(self, item: MovieItem, spider):
           self.sheet.append((item['title'], item['score'], item['motto']))
           return item

       def close_spider(self, spider):
           self.wb.save('douban_movie_data.xlsx')
   ```

   Both `process_item` and `close_spider` above are callback methods, meaning hook functions. Simply speaking, these are methods that the Scrapy framework automatically calls. When the spider program produces one `Item` object and hands it to the engine, the engine hands this `Item` object to the item pipeline. At this time, the `parse_item` method of the configured item pipeline is executed, so we can get the data in this method and finish persistence. The other method, `close_spider`, is a method automatically executed before the crawler ends. In the code above, we save the Excel file at this place. I believe everyone can read this code easily.

   In short, the item pipeline can help us complete the following operations:

   - Clean HTML data and validate crawled data.
   - Drop duplicated and unnecessary content.
   - Persist the crawling results.

4. Modify the `settings.py` file to configure the project. The main configurations that need to be changed are the following.

   ```python
   # User browser
   USER_AGENT = 'Mozilla/5.0 (Macintosh; Intel Mac OS X 10_14_6) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/92.0.4515.159 Safari/537.36'

   # Number of concurrent requests
   CONCURRENT_REQUESTS = 4

   # Download delay
   DOWNLOAD_DELAY = 3
   # Randomize the download delay
   RANDOMIZE_DOWNLOAD_DELAY = True

   # Whether to obey the crawler protocol
   ROBOTSTXT_OBEY = True

   # Configure the item pipeline
   ITEM_PIPELINES = {
      'demo.pipelines.MovieItemPipeline': 300,
   }
   ```

   > **Note**: The `ITEM_PIPELINES` option in the settings file above is a dictionary. Multiple data-processing pipelines can be configured, and the number behind them stands for execution priority. The smaller number is executed first.
