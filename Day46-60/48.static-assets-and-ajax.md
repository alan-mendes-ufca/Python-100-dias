## Ativos estáticos e solicitações Ajax

### Carregando ativos estáticos

Se você quiser usar recursos estáticos em um projeto Django, você pode primeiro criar um diretório para salvar recursos estáticos. No projeto `vote`, colocamos recursos estáticos em uma pasta chamada `static`, e esta pasta possui subpastas como:

- `css`
- `js`
- `images`

Em seguida, atualize `settings.py`:

```python
STATICFILES_DIRS = [os.path.join(BASE_DIR, 'static'), ]
STATIC_URL = '/static/'
```

Após configurar os recursos estáticos, todos podem executar o projeto e verificar se as imagens das páginas escritas anteriormente podem ser carregadas normalmente. Deve ser explicado que depois que o projeto é implantado on-line, geralmente entregamos recursos estáticos a um servidor especial de recursos estáticos, como Nginx ou Apache, em vez de permitir que o servidor de código Python gerencie recursos estáticos. Portanto a configuração acima não é adequada para o ambiente de produção. É apenas para teste durante o desenvolvimento do projeto.

### O que é Ajax

A seguir podemos implementar as funções "upvote" e "downvote". Obviamente, se essas duas funções puderem ser implementadas sem atualizar a página, a experiência do usuário será melhor, por isso consideramos usar Ajax para implementá-las. Ajax é a forma abreviada de JavaScript e XML assíncronos. Simplificando, o Ajax pode atualizar parcialmente a página sem recarregar a página inteira.

Para um aplicativo Web tradicional, sempre que um novo conteúdo é necessário na página, ele precisa solicitar novamente o servidor e atualizar a página inteira. Para um aplicativo Web usando Ajax, o navegador pode enviar uma solicitação assíncrona ao servidor para obter dados. Depois que o servidor retornar os novos dados, podemos usar JavaScript para operar o DOM e atualizar parcialmente a página, o que equivale a atualizar o conteúdo da página sem atualizar a página inteira.

![](res/synchronous-web-request.png)

![](res/asynchronous-web-request.png)

Quando o Ajax é usado, o navegador e o servidor geralmente trocam dados no formato XML ou JSON. O XML foi muito usado no passado, mas nos últimos anos foi quase totalmente substituído pelo JSON. Através da comparação abaixo, os dados JSON são obviamente muito mais compactos, então a eficiência de transferência é maior, e o próprio JSON também é uma sintaxe de expressão de objeto em JavaScript, por isso é mais conveniente processar em código JavaScript.

Exemplo XML:

```XML
<?xml version="1.0" encoding="utf-8"?>
<message>
	<from>Alice</from>
    <to>Bob</to>
    <content>Dinner is on me!</content>
</message>
```

Exemplo JSON:

```JSON
{
    "from": "Alice",
    "to": "Bob",
    "content": "Dinner is on me!"
}
```

### Implementar votação com Ajax

Atualização `urls.py`:

```python
from django.contrib import admin
from django.urls import path

from polls import views

urlpatterns = [
    path('', views.show_subjects),
    path('teachers/', views.show_teachers),
    path('praise/', views.praise_or_criticize),
    path('criticize/', views.praise_or_criticize),
    path('admin/', admin.site.urls),
]
```

Projete a função de visualização `praise_or_criticize` para suportar as funções "voto positivo" e "voto negativo". Esta função de visualização usa a classe `JsonResponse` do Django para serializar um dicionário em uma string JSON e retorná-lo ao navegador como o conteúdo da resposta.

```python
def praise_or_criticize(request):
    """Handle upvote/downvote."""
    try:
        tno = int(request.GET.get('tno'))
        teacher = Teacher.objects.get(no=tno)
        if request.path.startswith('/praise'):
            teacher.good_count += 1
            count = teacher.good_count
        else:
            teacher.bad_count += 1
            count = teacher.bad_count
        teacher.save()
        data = {'code': 20000, 'mesg': 'Operation succeeded', 'count': count}
    except (ValueError, Teacher.DoesNotExist):
        data = {'code': 20001, 'mesg': 'Operation failed'}
    return JsonResponse(data)
```

```html
            $('.comment>a').on('click', (evt) => {
                evt.preventDefault()
                let url = $(evt.target).attr('href')
                $.getJSON(url, (json) => {
                    if (json.code == 20000) {
                        $(evt.target).next().text(json.count)
                    } else {
                        alert(json.mesg)
                    }
                })
            })
        })
    </script>
</body>
</html>
```

No código frontend acima, o método `getJSON` empacotado por jQuery é usado para enviar uma solicitação assíncrona ao servidor. Se você não está familiarizado com jQuery, pode consultar o manual da API jQuery.
