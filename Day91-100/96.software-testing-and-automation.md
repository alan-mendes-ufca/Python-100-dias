## Software Testing and Automated Testing

### Overview of Software Testing

Software testing is a process used to help identify the correctness, completeness, security, and quality of software. In other words, it is the process of running a program under specified conditions to find errors in the program, measure the quality of the software, and judge whether it can meet the design requirements.

#### Testing Methods

Black-box testing: test the functions of an application, not its internal structure or operation. The tester does not need special knowledge of the application's code, internal structure, or programming language. The tester only needs to know what the system should do. That is, when a specific input is entered, a certain output should be produced. Test cases are designed according to the functions that the application system should provide, based on standards, specifications, or requirements. The tester chooses valid inputs and invalid inputs to check whether the output is correct. This testing method is suitable for most kinds of software testing, such as integration testing and system testing.

White-box testing: test the internal structure or operation of an application, not the functions of the application. In white-box testing, test cases are designed from the point of view of the programming language. The tester inputs data to verify the path of data flow inside the program and determines whether the output is proper, which is similar to testing nodes in a circuit.

Because of the limits of time and cost, one of the most critical questions in software testing is: "**Among all possible test cases, which subset can find the most errors?**" So when test cases are designed, white-box testing pays attention to the degree of program logic coverage, such as statement coverage, condition coverage, and branch coverage. Black-box testing can use methods such as equivalence partitioning, boundary value analysis, cause-effect graph analysis, and error guessing to design test cases.

#### Types of Testing (Stages)

Unit testing: test software units. Its purpose is to check the correctness of the basic software units. The object being tested is the smallest unit in software design, which is usually a function.

Integration testing: assemble program modules with a proper integration strategy, then test whether the system interfaces and the integrated functions are correct. Its main purpose is to check whether the interfaces between software units are correct. The objects tested in integration testing are modules that have already passed unit testing.

System testing: system testing mainly includes functional testing, interface testing, reliability testing, usability testing, and performance testing.

Regression testing: testing activities carried out to detect errors introduced by code changes. Regression testing is an important task during the software maintenance stage. Some studies show that the cost brought by regression testing can take more than one third of the total cost of the software life cycle.

#### Test-Driven Development (Agile Testing)

Test-driven development includes the following three steps:

1. Write automated tests for new features or improvements that have not been implemented yet.
2. Provide the minimum amount of code that can pass all defined tests.
3. Refactor the code so that it meets the required quality standard.

The benefit of test-driven development is that it can effectively prevent software regression and produce better-quality code. Also, acceptance testing should be done by the customer. The customer designs acceptance tests according to usage scenarios and objectively confirms whether the application can meet their requirements. A feature that can pass unit tests, or even system tests, may still fail the customer's acceptance test.

#### Testing Internet Applications and Mobile Applications

Testing strategies for internet applications:

1. Presentation-layer testing: content testing, site structure testing, and user-environment testing such as browsers and operating systems.
2. Business-layer testing: performance, data validation, transactions, and external services.
3. Persistence-layer testing: response time, data integrity, and fault tolerance.

Testing strategies for mobile applications:

1. Real-device testing.
2. Emulator-based testing.

### Unit (Module) Testing

Python's standard library has the `unittest` module for writing unit tests. When running tests, it is recommended to use [pytest](https://docs.pytest.org/en/latest/) or `nose2`. `pytest` is a test runner that can automatically search for and run tests, and it also outputs detailed error reports.

You can install the [testfixtures](https://pypi.org/project/testfixtures/) library to help with unit testing. It combines many typical fixtures and provides function modules for creating directories, changing the system date, and creating mock objects. These modules can help us separate unit tests from the environments they depend on. [mock](https://pypi.org/project/mock/) is a library that replaces the objects depended on by the test target with virtual objects. During testing, we can specify the return value of a virtual object when it is called, and whether an exception should happen.

`tox` can conveniently prepare the environment needed to run tests. `tox` builds test environments in multiple `virtualenv` environments, then runs the tests in those environments and shows the results. It can unify the options and environment variables of testing tools, so we only need to run the `tox` command to complete the required tests easily.

##### Web - [Selenium](https://docs.seleniumhq.org/)

Selenium is a group of browser-driven testing tools used to automate functional testing and integration testing of web applications. Like a user using a browser, Selenium can do mouse operations in the browser, type text into forms, verify form values, and so on. With this ability, manual operations can be turned into automated operations.

1. Advantages of Selenium

   - Making automated test cases is simple. Selenium provides the Selenium IDE tool. It can capture mouse and keyboard operations, and then replay them, so test cases can be made easily.
   - It supports many browsers and operating systems.

2. Components of Selenium

   - [Selenium IDE](https://www.seleniumhq.org/projects/ide/)
   - [Selenium Remote Control](https://www.seleniumhq.org/projects/remote-control/)
   - [Selenium WebDriver](https://www.seleniumhq.org/projects/webdriver/)

3. Working together with continuous integration tools

   Continuous integration means integrating code into the main branch frequently. It has two main benefits:

   - Errors can be found quickly. Every time a small update is finished, it is integrated into the main branch. This makes it easy to find errors quickly and also easier to locate them.
   - It prevents branches from drifting too far away from the main branch. If code is not integrated often, while the main branch keeps changing, later integration becomes harder and may even fail.

   The purpose of continuous integration is to let the product iterate quickly while still keeping high quality. Its core measure is that before code is integrated into the main branch, it must pass automated tests. As long as one test case fails, it cannot be integrated.

   In Jenkins, you can install the "Seleniumhq Plugin". Then the test cases made by Selenium IDE can be saved in HTML format and used by Jenkins. The basic steps are:

   - On the machine that runs the tests, download the test suite and test cases from the version control system.
   - Download Selenium Server on the machine that runs the tests.
   - In Jenkins, choose "Manage Jenkins" and then "Manage Plugins" to install "Seleniumhq Plugin".
   - In Jenkins, choose "Manage Jenkins" and then "Configure System", and configure "HTMLSuite Runner" under "Selenium Remote Control".
   - Create a new Jenkins job for testing and configure the browser, the start URL, the test suite, and the output file for test results.

   After the configuration is finished, you can click "Build Now" in Jenkins.

Besides Selenium, [WebTest](https://pypi.org/project/WebTest/), [Splinter](https://splinter.readthedocs.io/en/latest/), and [RobotFramework](https://robotframework.org/) are also choices for web-side testing. Among them, WebTest can send simulated requests to WSGI applications and get the results, so almost all WSGI applications can be tested with it. Splinter is a second-level wrapper around Selenium, so it is more convenient and simpler to use.
