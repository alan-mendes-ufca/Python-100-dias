## DDL em detalhes

Geralmente podemos dividir o SQL em quatro tipos: DDL (Linguagem de Definição de Dados), DML (Linguagem de Manipulação de Dados), DCL (Linguagem de Controle de Dados) e TCL (Linguagem de Controle de Transação). DDL é usado principalmente para criar, excluir e modificar objetos no banco de dados, por exemplo, criar, excluir e modificar tabelas bidimensionais. Suas palavras-chave principais incluem `create`, `drop` e `alter`. DML é o principal responsável por inserir, excluir, atualizar e consultar dados. Suas palavras-chave incluem `insert`, `delete`, `update` e `select`. DCL é usado para conceder e recuperar permissões. Suas palavras-chave principais são `grant` e `revoke`. O TCL geralmente é usado para controle de transações.

> **Observação**: SQL é uma linguagem que não distingue maiúsculas e minúsculas. Em geral, sugerimos escrever as palavras-chave em maiúsculas e as demais partes em minúsculas. Se a empresa tiver regras de codificação SQL obrigatórias, siga as regras da empresa. A preferência pessoal não deve estar acima das regras de codificação da empresa. Isso deve ser senso comum para um profissional.

### Crie bancos de dados e tabelas

A seguir, vamos construir um banco de dados muito simples para um sistema de seleção de cursos escolares. Chamamos o banco de dados de `school`. As quatro entidades principais são faculdades, professores, alunos e cursos. Entre eles, estudantes e faculdades mantêm uma relação de subordinação. Em termos de quantidade, trata-se de uma relação muitos para um, porque uma faculdade pode ter muitos alunos, e um aluno geralmente pertence a apenas uma faculdade. Da mesma forma, a relação subordinada entre professores e faculdades também é de muitos para um. Um professor pode ministrar muitos cursos. Se um curso tiver apenas um professor, então a relação entre cursos e professores também será de muitos para um. Se vários professores puderem ministrar um curso juntos, então os cursos e os professores serão muitos para muitos. Para simplificar, criamos cursos e professores como muitos para um. Alunos e cursos são uma relação típica de muitos para muitos, porque um aluno pode escolher vários cursos e um curso também pode ser escolhido por muitos alunos. Em um banco de dados relacional, uma tabela intermediária é necessária para manter o relacionamento muitos-para-muitos entre duas entidades. No final, nosso sistema de seleção de cursos escolares possui cinco tabelas no total: tabela de faculdade (`tb_college`), tabela de alunos (`tb_student`), tabela de professores (`tb_teacher`), tabela de cursos (`tb_course`) e tabela de registros de seleção de cursos (`tb_record`). A tabela de registros de seleção de cursos é a tabela intermediária que mantém o relacionamento muitos-para-muitos entre alunos e cursos.

```sql
-- If there is a database named school, drop it
DROP DATABASE IF EXISTS `school`;

-- Create a database named school and set the default character set and collation
CREATE DATABASE `school` DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

-- Switch to the school database context
USE `school`;

-- Create the college table
CREATE TABLE `tb_college`
(
`col_id`    int unsigned AUTO_INCREMENT      COMMENT 'ID',
`col_name`  varchar(50)  NOT NULL            COMMENT 'Name',
`col_intro` varchar(500) NOT NULL DEFAULT '' COMMENT 'Introduction',
PRIMARY KEY (`col_id`)
);

-- Create the student table
CREATE TABLE `tb_student`
(
`stu_id`    int unsigned NOT NULL           COMMENT 'Student ID',
`stu_name`  varchar(20)  NOT NULL           COMMENT 'Name',
`stu_sex`   boolean      NOT NULL DEFAULT 1 COMMENT 'Sex',
`stu_birth` date         NOT NULL           COMMENT 'Date of birth',
`stu_addr`  varchar(255) DEFAULT ''         COMMENT 'Native place',
`col_id`    int unsigned NOT NULL           COMMENT 'College',
PRIMARY KEY (`stu_id`),
CONSTRAINT `fk_student_col_id` FOREIGN KEY (`col_id`) REFERENCES `tb_college` (`col_id`)
);

-- Create the teacher table
CREATE TABLE `tb_teacher`
(
`tea_id`    int unsigned NOT NULL                COMMENT 'Employee ID',
`tea_name`  varchar(20)  NOT NULL                COMMENT 'Name',
`tea_title` varchar(10)  NOT NULL DEFAULT 'Teaching Assistant' COMMENT 'Title',
`col_id`    int unsigned NOT NULL                COMMENT 'College',
PRIMARY KEY (`tea_id`),
CONSTRAINT `fk_teacher_col_id` FOREIGN KEY (`col_id`) REFERENCES `tb_college` (`col_id`)
);

-- Create the course table
CREATE TABLE `tb_course`
(
`cou_id`     int unsigned NOT NULL COMMENT 'ID',
`cou_name`   varchar(50)  NOT NULL COMMENT 'Name',
`cou_credit` int          NOT NULL COMMENT 'Credits',
`tea_id`     int unsigned NOT NULL COMMENT 'Teacher',
PRIMARY KEY (`cou_id`),
CONSTRAINT `fk_course_tea_id` FOREIGN KEY (`tea_id`) REFERENCES `tb_teacher` (`tea_id`)
);

-- Create the course selection record table
CREATE TABLE `tb_record`
(
`rec_id`   bigint unsigned AUTO_INCREMENT COMMENT 'Course selection record ID',
`stu_id`   int unsigned    NOT NULL       COMMENT 'Student ID',
`cou_id`   int unsigned    NOT NULL       COMMENT 'Course ID',
`sel_date` date            NOT NULL       COMMENT 'Course selection date',
`score`    decimal(4,1)                   COMMENT 'Exam score',
PRIMARY KEY (`rec_id`),
CONSTRAINT `fk_record_stu_id` FOREIGN KEY (`stu_id`) REFERENCES `tb_student` (`stu_id`),
CONSTRAINT `fk_record_cou_id` FOREIGN KEY (`cou_id`) REFERENCES `tb_course` (`cou_id`),
CONSTRAINT `uk_record_stu_cou` UNIQUE (`stu_id`, `cou_id`)
);
```

- Ao criar e excluir bancos de dados, a palavra-chave `database` também pode ser substituída por `schema`. Os dois fazem a mesma coisa.

- Na instrução de criação de tabela, `not null` é uma restrição não nula, que limita o campo para que não possa ficar vazio. `default` é usado para especificar um valor padrão para um campo e chamamos isso de restrição de valor padrão. `primary key` é a restrição de chave primária. Ele define a coluna que pode determinar exclusivamente um registro e também garante que cada registro seja único, porque a chave primária não permite duplicatas. `foreign key` é a restrição de chave estrangeira. Mantém a integridade referencial de duas tabelas. Por exemplo, como o campo `col_id` na tabela student tem uma restrição de chave estrangeira, ele deve fazer referência a (`references`) `col_id` na tabela college. Portanto, o ID da faculdade na tabela do aluno deve vir da tabela da faculdade e não pode ser atribuído livremente a este campo. Se precisar nomear restrições de chave primária, restrições de chave estrangeira e assim por diante, você pode usar a palavra-chave `constraint` e colocar o nome da restrição depois dela.

- A palavra-chave `comment` na instrução de criação de tabela é usada para adicionar comentários a colunas e tabelas, o que melhora a legibilidade e a manutenção do código.

- Ao criar uma tabela, você mesmo pode escolher o mecanismo de armazenamento subjacente. MySQL oferece suporte a muitos mecanismos de armazenamento e você pode vê-los com o comando `show engines`. Ao escolher os tipos de dados, para salvar dados de string, geralmente usamos os tipos `VARCHAR` e `CHAR`. A primeira é geralmente chamada de string de comprimento variável e a última é geralmente chamada de string de comprimento fixo. Para o mecanismo de armazenamento InnoDB, o formato de armazenamento de linha não distingue entre colunas de comprimento fixo e de comprimento variável, portanto, não há diferença essencial entre `VARCHAR` e `CHAR`, e o último não é necessariamente mais rápido que o primeiro. Se quiser salvar uma string muito grande, você pode usar o tipo `TEXT`. Se quiser salvar uma sequência de bytes muito grande, você pode usar o tipo `BLOB` (Binary Large Object). No MySQL, `TEXT` e `BLOB` incluem ainda `TEXT`, `MEDIUMTEXT`, `LONGTEXT` e `BLOB`, `MEDIUMBLOB`, `LONGBLOB`. A principal diferença é o tamanho máximo dos dados que podem armazenar. Para salvar números de ponto flutuante, você pode usar `FLOAT` ou `DOUBLE`. `FLOAT` não é mais recomendado e pode ser removido em versões posteriores do MySQL. Para salvar números de ponto fixo, você deve usar o tipo `DECIMAL`. Pode especificar o número de dígitos válidos antes e depois da vírgula decimal. Se você deseja salvar data e hora, o tipo `DATETIME` é melhor que o tipo `TIMESTAMP`, pois o primeiro pode representar um intervalo maior de data e hora. O último é na verdade um número inteiro abaixo, que registra quantos milissegundos a data e hora especificadas diferem de `1970-01-01 00:00:00`. Este tipo irá estourar após `2038-01-19 03:14:07`.

Para o campo de incremento automático `AUTO_INCREMENT`, se você usar o MySQL 5.x, deverá prestar atenção ao problema de reversão dos campos de incremento automático. Claro, este problema foi bem resolvido no MySQL 8.x. Além disso, o MySQL 8.x tem muitas outras vantagens. Existem muitas otimizações e ajustes tanto nas funções quanto no desempenho, por isso recomendamos fortemente que todos usem o MySQL 8.x. Em cenários com acesso de alta simultaneidade ao banco de dados, `AUTO_INCREMENT` não apenas apresenta problemas de desempenho, mas também pode produzir valores de ID duplicados em uma estrutura de várias máquinas. Nesse tipo de cenário, usar um algoritmo de geração de ID distribuído, como SnowFlake ou TinyID, é a melhor escolha. Os leitores interessados podem estudá-lo por conta própria.

### Eliminar tabelas e modificar tabelas

A seguir, usaremos a tabela do aluno como exemplo para explicar como eliminar e modificar tabelas. Para eliminar uma tabela, você pode usar `drop table`, conforme mostrado abaixo.

```sql
DROP TABLE `tb_student`;
```

ou

```sql
DROP TABLE IF EXISTS `tb_student`;
```

Deve-se observar que se os dados já foram inseridos na tabela do aluno e esses dados são referenciados por outras tabelas, a tabela do aluno não pode ser eliminada, caso contrário a operação acima reportará um erro. Na próxima lição falaremos sobre como inserir dados em tabelas. Nesse momento, você pode tentar e ver se consegue excluir a tabela do aluno sem problemas.

Se quiser modificar a tabela de alunos, você pode usar `alter table`. Especificamente, pode ser dividido nos seguintes casos:

Modifique a tabela e adicione uma nova coluna. Por exemplo, adicione uma coluna de número de telefone de contato à tabela de alunos.

```sql
ALTER TABLE `tb_student` ADD COLUMN `stu_tel` varchar(20) NOT NULL COMMENT 'Contact phone number';
```

> **Observação**: Se você especificar uma restrição não nula (`not null`) ao adicionar uma nova coluna, a tabela de alunos já não poderá ter dados. Caso contrário, depois que os dados originais obtiverem a coluna `stu_tel`, não haverá dados nessa coluna e isso violará a restrição de não nulo. Claro, quando adicionamos uma coluna, também podemos usar uma restrição de valor padrão para resolver este problema.

Modifique a tabela e exclua a coluna especificada. Por exemplo, exclua a coluna do número de telefone do contato adicionada acima.

```sql
ALTER TABLE `tb_student` DROP COLUMN `stu_tel`;
```

Modifique a tabela e altere o tipo de dados de uma coluna. Por exemplo, altere `stu_sex` na tabela de alunos para um caractere.

```sql
ALTER TABLE `tb_student` MODIFY COLUMN `stu_sex` char(1) NOT NULL DEFAULT 'M' COMMENT 'Sex';
```

Modifique a tabela e altere o nome da coluna. Por exemplo, altere `stu_sex` na tabela de alunos para `stu_gender`.

```sql
ALTER TABLE `tb_student` CHANGE COLUMN `stu_sex` `stu_gender` boolean DEFAULT 1 COMMENT 'Sex';
```

Modifique a tabela e exclua uma restrição. Por exemplo, exclua a restrição de chave estrangeira na coluna `col_id` da tabela de alunos.

```sql
ALTER TABLE `tb_student` DROP FOREIGN KEY `fk_student_col_id`;
```

Modifique a tabela e adicione uma restrição. Por exemplo, adicione uma restrição de chave estrangeira à coluna `col_id` da tabela de alunos.

```sql
ALTER TABLE `tb_student` ADD FOREIGN KEY (`col_id`) REFERENCES `tb_college` (`col_id`);
```

ou

```sql
ALTER TABLE `tb_student` ADD CONSTRAINT `fk_student_col_id` FOREIGN KEY (`col_id`) REFERENCES `tb_college` (`col_id`);
```

> **Observação**: Ao adicionar uma restrição de chave estrangeira, você também pode usar `on update` e `on delete` para especificar o que deve ser feito quando a tabela referenciada for atualizada ou excluída. O valor padrão de ambos é `restrict`, o que significa que se houver uma restrição de chave estrangeira, os dados referenciados não poderão ser atualizados ou excluídos. Além de `restrict`, os valores possíveis aqui são `cascade` e `set null`. Os leitores interessados podem estudá-lo por conta própria.

Modifique o nome da tabela. Por exemplo, altere o nome da tabela de alunos para `tb_stu_info`.

```sql
ALTER TABLE `tb_student` RENAME TO `tb_stu_info`;
```

> **Dica**: Em geral, não altere facilmente o nome de um banco de dados ou tabela.
