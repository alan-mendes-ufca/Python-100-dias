## Static Assets and Ajax Requests

### Loading Static Assets

If you want to use static resources in a Django project, you can first create a directory for saving static resources. In the `vote` project, we put static resources in a folder named `static`, and this folder has subfolders such as:

- `css`
- `js`
- `images`

Then update `settings.py`:

```python
STATICFILES_DIRS = [os.path.join(BASE_DIR, 'static'), ]
STATIC_URL = '/static/'
```

After configuring static resources, everyone can run the project and check whether the images on the pages written before can be loaded normally. It should be explained that after the project is deployed online, we usually hand static resources to a special static-resource server such as Nginx or Apache, instead of letting the Python-code server manage static resources. So the configuration above is not suitable for the production environment. It is only for testing during project development.

### What Ajax Is

Next we can implement the "upvote" and "downvote" functions. Obviously, if these two functions can be implemented without refreshing the page, the user experience will be better, so we consider using Ajax to implement them. Ajax is the short form of Asynchronous JavaScript And XML. Simply speaking, Ajax can partially refresh the page without reloading the whole page.

For a traditional Web application, every time new content is needed on the page, it needs to request the server again and refresh the whole page. For a Web application using Ajax, the browser can send an asynchronous request to the server to get data. After the server returns the new data, we can use JavaScript to operate the DOM and partially refresh the page, which is equal to updating the content of the page without refreshing the whole page.

![](res/synchronous-web-request.png)

![](res/asynchronous-web-request.png)

When Ajax is used, the browser and the server usually exchange data in XML or JSON format. XML was used a lot in the past, but in recent years it has almost been completely replaced by JSON. Through the comparison below, JSON data is obviously much more compact, so transfer efficiency is higher, and JSON itself is also an object expression syntax in JavaScript, so it is more convenient to process in JavaScript code.

XML example:

```XML
<?xml version="1.0" encoding="utf-8"?>
<message>
	<from>Alice</from>
    <to>Bob</to>
    <content>Dinner is on me!</content>
</message>
```

JSON example:

```JSON
{
    "from": "Alice",
    "to": "Bob",
    "content": "Dinner is on me!"
}
```

### Implement Voting with Ajax

Update `urls.py`:

```python
from django.contrib import admin
from django.urls import path

from polls import views

urlpatterns = [
    path('', views.show_subjects),
    path('teachers/', views.show_teachers),
    path('praise/', views.praise_or_criticize),
    path('criticize/', views.praise_or_criticize),
    path('admin/', admin.site.urls),
]
```

Design the view function `praise_or_criticize` to support the "upvote" and "downvote" functions. This view function uses Django's `JsonResponse` class to serialize a dictionary into a JSON string and return it to the browser as the response content.

```python
def praise_or_criticize(request):
    """Handle upvote/downvote."""
    try:
        tno = int(request.GET.get('tno'))
        teacher = Teacher.objects.get(no=tno)
        if request.path.startswith('/praise'):
            teacher.good_count += 1
            count = teacher.good_count
        else:
            teacher.bad_count += 1
            count = teacher.bad_count
        teacher.save()
        data = {'code': 20000, 'mesg': 'Operation succeeded', 'count': count}
    except (ValueError, Teacher.DoesNotExist):
        data = {'code': 20001, 'mesg': 'Operation failed'}
    return JsonResponse(data)
```
